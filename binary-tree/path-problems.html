Binary Tree Path Problems

A huge family of binary tree interview questions boils down to reasoning about paths: sequences of connected nodes. They look different on the surface, but they share one DFS skeleton. Learn the skeleton once and the variations become bookkeeping.

The Family of Path Problems

Path problems differ along two axes: where a path may start/end and what we compute over it.

  • Root-to-leaf paths — start at the root, end at a leaf. Listing them, checking a target sum, or building numbers all fall here.
  • Downward paths — start at any node, go strictly downward, end at any descendant. Path Sum III counts these.
  • Any-to-any paths — start at any node, bend through at most one "peak", end at any node. Maximum Path Sum and the diameter live here.

All of them ride the same recursive shape: visit a node, recurse into children, combine results. The two recurring tricks you will see below are:

The shared DFS skeleton
-----------------------
dfs(node):
    if node is null: return base-case
    L = dfs(node.left)
    R = dfs(node.right)
    update global answer using node + L + R     (the "through-node" view)
    return best single-branch value to parent    (the "ending-here" view)

Two distinct quantities at each node:
  * best path that BENDS at this node  -> only updates a global, never returned
  * best path that ENDS at this node   -> returned upward so parent can extend it

1. All Root-to-Leaf Paths

Carry the current path down. When you reach a leaf (no children), snapshot the path into the answer. On the way back up, pop the node so sibling branches start clean — this is backtracking.

          1
        /   \
       2     3
        \      \
         5      4

Root-to-leaf paths:
  1 -> 2 -> 5
  1 -> 3 -> 4

struct TreeNode { int val; TreeNode *left, *right; };

void dfs(TreeNode* node, vector<int>& path,
         vector<vector<int>>& out) {
    if (!node) return;
    path.push_back(node->val);
    if (!node->left && !node->right) {
        out.push_back(path);              // leaf: snapshot
    } else {
        dfs(node->left,  path, out);
        dfs(node->right, path, out);
    }
    path.pop_back();                      // backtrack
}

vector<vector<int>> allPaths(TreeNode* root) {
    vector<vector<int>> out; vector<int> path;
    dfs(root, path, out);
    return out;
}

class TreeNode { int val; TreeNode left, right; }

void dfs(TreeNode node, List<Integer> path,
         List<List<Integer>> out) {
    if (node == null) return;
    path.add(node.val);
    if (node.left == null && node.right == null) {
        out.add(new ArrayList<>(path));   // leaf: copy
    } else {
        dfs(node.left,  path, out);
        dfs(node.right, path, out);
    }
    path.remove(path.size() - 1);          // backtrack
}

List<List<Integer>> allPaths(TreeNode root) {
    List<List<Integer>> out = new ArrayList<>();
    dfs(root, new ArrayList<>(), out);
    return out;
}

def all_paths(root):
    out, path = [], []
    def dfs(node):
        if not node:
            return
        path.append(node.val)
        if not node.left and not node.right:
            out.append(path[:])           # leaf: snapshot
        else:
            dfs(node.left)
            dfs(node.right)
        path.pop()                        # backtrack
    dfs(root)
    return out

2. Path Sum I — Does a Root-to-Leaf Path Equal Target?

We do not need to carry the running sum; instead subtract each node's value from the remaining target as we descend. At a leaf, the path works iff the remaining target equals the leaf's value. This returns a single boolean — the moment one branch succeeds, short-circuit.

bool hasPathSum(TreeNode* node, int target) {
    if (!node) return false;
    target -= node->val;
    if (!node->left && !node->right)        // leaf
        return target == 0;
    return hasPathSum(node->left,  target)
        || hasPathSum(node->right, target);
}

boolean hasPathSum(TreeNode node, int target) {
    if (node == null) return false;
    target -= node.val;
    if (node.left == null && node.right == null)
        return target == 0;
    return hasPathSum(node.left,  target)
        || hasPathSum(node.right, target);
}

def has_path_sum(node, target):
    if not node:
        return False
    target -= node.val
    if not node.left and not node.right:
        return target == 0
    return (has_path_sum(node.left,  target)
            or has_path_sum(node.right, target))

3. Path Sum II — All Root-to-Leaf Paths Equal to Target

Combine the path-tracking of problem 1 with the subtract-target trick of problem 2. Track the current path, and whenever a leaf brings the remaining target to zero, snapshot the path. Backtrack on the way up.

void dfs(TreeNode* node, int target,
         vector<int>& path, vector<vector<int>>& out) {
    if (!node) return;
    path.push_back(node->val);
    target -= node->val;
    if (!node->left && !node->right && target == 0)
        out.push_back(path);
    dfs(node->left,  target, path, out);
    dfs(node->right, target, path, out);
    path.pop_back();                      // backtrack
}

vector<vector<int>> pathSum(TreeNode* root, int target) {
    vector<vector<int>> out; vector<int> path;
    dfs(root, target, path, out);
    return out;
}

void dfs(TreeNode node, int target,
         List<Integer> path, List<List<Integer>> out) {
    if (node == null) return;
    path.add(node.val);
    target -= node.val;
    if (node.left == null && node.right == null && target == 0)
        out.add(new ArrayList<>(path));
    dfs(node.left,  target, path, out);
    dfs(node.right, target, path, out);
    path.remove(path.size() - 1);          // backtrack
}

List<List<Integer>> pathSum(TreeNode root, int target) {
    List<List<Integer>> out = new ArrayList<>();
    dfs(root, target, new ArrayList<>(), out);
    return out;
}

def path_sum(root, target):
    out, path = [], []
    def dfs(node, remaining):
        if not node:
            return
        path.append(node.val)
        remaining -= node.val
        if not node.left and not node.right and remaining == 0:
            out.append(path[:])
        dfs(node.left,  remaining)
        dfs(node.right, remaining)
        path.pop()                        # backtrack
    dfs(root, target)
    return out

4. Path Sum III — Count Downward Paths Summing to Target

Now the path can start and end anywhere, as long as it goes strictly downward (parent to child). A brute-force "start a fresh sum at every node" is O(n²). The elegant O(n) solution borrows the classic prefix-sum + hashmap trick from the subarray-sum problem.

Let prefix be the running sum from the root down to the current node. A downward path ending at the current node with sum target exists for every ancestor where prefixAtAncestor == prefix - target. So we keep a hashmap count of prefix sums seen on the current root-to-node path and, at each node, add map[prefix - target] to the answer.

target = 8

            10
           /  \
          5   (-3)
         / \     \
        3   2    11
       / \   \
      3 (-2)  1

Walk to node "2" (right child of 5): prefix sums on this path
  root path: 10, 5, 2   ->  prefixes seen: {0:1, 10:1, 15:1, 17:1}
  at node "2": prefix = 17, want prefix - target = 9 ... none -> +0

Walk to node "3" (left child of 5), then "3" (its left child):
  prefixes: 0,10,15,18,21
  at the lower "3": prefix = 21, want 21 - 8 = 13 ... none

Paths that DO sum to 8:
  5 -> 3                  (5 + 3 = 8)
  5 -> 2 -> 1             (5 + 2 + 1 = 8)
  (-3) -> 11              (-3 + 11 = 8)
Answer = 3

The map[0] = 1 seed lets a path that starts at the root itself count
(prefix - target == 0 means the whole root-to-node path equals target).

Critical detail: the map only reflects ancestors on the current path. After recursing into both children, decrement map[prefix] so sibling subtrees do not see this node's prefix. That decrement is the backtracking step.

// long long prefix to avoid overflow on deep sums
int dfs(TreeNode* node, long long prefix, int target,
        unordered_map<long long, int>& seen) {
    if (!node) return 0;
    prefix += node->val;
    int count = 0;
    auto it = seen.find(prefix - target);
    if (it != seen.end()) count += it->second;
    seen[prefix]++;                       // record this node
    count += dfs(node->left,  prefix, target, seen);
    count += dfs(node->right, prefix, target, seen);
    seen[prefix]--;                       // backtrack
    return count;
}

int pathSumIII(TreeNode* root, int target) {
    unordered_map<long long, int> seen;
    seen[0] = 1;                          // empty prefix
    return dfs(root, 0, target, seen);
}

int dfs(TreeNode node, long prefix, int target,
        Map<Long, Integer> seen) {
    if (node == null) return 0;
    prefix += node.val;
    int count = seen.getOrDefault(prefix - target, 0);
    seen.merge(prefix, 1, Integer::sum);   // record
    count += dfs(node.left,  prefix, target, seen);
    count += dfs(node.right, prefix, target, seen);
    seen.merge(prefix, -1, Integer::sum);  // backtrack
    return count;
}

int pathSumIII(TreeNode root, int target) {
    Map<Long, Integer> seen = new HashMap<>();
    seen.put(0L, 1);                       // empty prefix
    return dfs(root, 0L, target, seen);
}

from collections import defaultdict

def path_sum_iii(root, target):
    seen = defaultdict(int)
    seen[0] = 1                          # empty prefix
    def dfs(node, prefix):
        if not node:
            return 0
        prefix += node.val
        count = seen[prefix - target]
        seen[prefix] += 1                  # record
        count += dfs(node.left,  prefix)
        count += dfs(node.right, prefix)
        seen[prefix] -= 1                  # backtrack
        return count
    return dfs(root, 0)

5. Maximum Path Sum — Any Node to Any Node

This is the gold-standard tree-DP problem. A path may start and end at any node and may bend through one node (its "peak"). We want the maximum sum over all such paths.

At each node we juggle the two quantities from the skeleton:

  • Gain ending here (returned upward): node.val + max(leftGain, rightGain, 0). A parent can only extend through one child, so we pick the better single branch. We clamp at 0 because a negative branch is better left off the path entirely.
  • Path bending here (updates the global best): node.val + max(0, leftGain) + max(0, rightGain). This uses both branches, forming a path whose highest point is this node.

Why clamp negatives? If a child's best downward gain is negative, attaching it can only shrink our sum, so we treat it as 0 (don't take it). Why is the bending path not propagated upward? A path that already uses both children of a node cannot be extended by that node's parent without revisiting the node — that would no longer be a simple path. So the two-branch value only updates the global answer; the parent receives the single-branch value.

            -10
           /    \
          9      20
                /  \
               15   7

Best single branches:
  gain(9)  = 9
  gain(15) = 15
  gain(7)  = 7
  gain(20) = 20 + max(15,7,0) = 35   (returned upward)
  gain(-10)= -10 + max(9,35,0) = 25

Bending paths considered for the GLOBAL best:
  at 20: 20 + 15 + 7 = 42   <-- maximum, does NOT pass through root
  at -10: -10 + max(0,9) + max(0,35) = 34

Answer = 42  (path 15 -> 20 -> 7, peak at 20)

int best = INT_MIN;

// returns max gain of a path STARTING at node going down
int gain(TreeNode* node) {
    if (!node) return 0;
    int L = max(0, gain(node->left));   // clamp negatives
    int R = max(0, gain(node->right));
    best = max(best, node->val + L + R);  // path bending here
    return node->val + max(L, R);          // extend one branch
}

int maxPathSum(TreeNode* root) {
    best = INT_MIN;
    gain(root);
    return best;
}

int best = Integer.MIN_VALUE;

// returns max gain of a path STARTING at node going down
int gain(TreeNode node) {
    if (node == null) return 0;
    int L = Math.max(0, gain(node.left));   // clamp
    int R = Math.max(0, gain(node.right));
    best = Math.max(best, node.val + L + R);   // bend
    return node.val + Math.max(L, R);        // extend
}

int maxPathSum(TreeNode root) {
    best = Integer.MIN_VALUE;
    gain(root);
    return best;
}

def max_path_sum(root):
    best = float('-inf')
    def gain(node):
        nonlocal best
        if not node:
            return 0
        L = max(0, gain(node.left))      # clamp negatives
        R = max(0, gain(node.right))
        best = max(best, node.val + L + R) # path bending here
        return node.val + max(L, R)         # extend one branch
    gain(root)
    return best

6. Diameter as a Path Problem

The diameter is the longest path between any two nodes, measured in edges. It is exactly Maximum Path Sum with every node's value replaced by 0 and we count edges instead of summing values: at each node the "bending path" length is leftHeight + rightHeight, and we return 1 + max(leftHeight, rightHeight) upward. Same skeleton, different accumulator.

For the full diameter walkthrough and height-based DFS, see the Tree Traversals & Views page, which covers heights and depths in detail.

7. Sum Root-to-Leaf Numbers

Each root-to-leaf path spells out a number (e.g. 1 -> 2 -> 3 means 123). Carry the number built so far as cur * 10 + node.val; at a leaf, add cur to the total. A one-pass DFS, O(n).

def sum_numbers(root):
    def dfs(node, cur):
        if not node:
            return 0
        cur = cur * 10 + node.val
        if not node.left and not node.right:
            return cur                       # leaf: number complete
        return dfs(node.left, cur) + dfs(node.right, cur)
    return dfs(root, 0)

The recurring patterns to remember:
  • Two views per node. Return the best-ending-here value upward (so the parent can extend a single branch) while updating the best-through-here answer globally (a bending path that the parent cannot reuse). This single idea solves Maximum Path Sum, diameter, and most "any-to-any" tree problems.
  • Subtract the target. For root-to-leaf sum checks, decrement the target as you descend and test against 0 at the leaf — no running sum needed.
  • Prefix-sum map for downward path counting. Keep a hashmap of prefix sums seen on the current root-to-node path; map[prefix - target] gives the number of downward paths ending here. Always seed map[0] = 1 and decrement on the way back up so siblings stay isolated.
