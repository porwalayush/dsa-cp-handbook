Binary Tree Traversals

A traversal just means a plan for visiting every node in a tree exactly once. There are two very different styles, and a simple picture tells them apart.

Depth-First Search (DFS) is like exploring a cave: you pick a tunnel and follow it all the way to the dead end before you ever back up to try another. You go deep first. The three DFS flavours — preorder, inorder, and postorder — only differ in when you jot down the current room: before exploring, between the two tunnels, or after both.

Breadth-First Search (BFS), also called level-order, is like water flooding the cave: it fills everything one step away from the entrance, then everything two steps away, and so on. You visit the tree level by level, top to bottom, left to right — never diving deep until the current level is fully covered.

The Running Example

Throughout this page we use one fixed binary tree. It happens to be a Binary Search Tree (BST), which lets us highlight a useful property of inorder traversal later.

              4
            /   \
           2     6
          / \   / \
         1   3 5   7

DFS results for this tree:
  Preorder   (Root, Left, Right):  4 2 1 3 6 5 7
  Inorder    (Left, Root, Right):  1 2 3 4 5 6 7   <- sorted (BST!)
  Postorder  (Left, Right, Root):  1 3 2 5 7 6 4

BFS result:
  Level-order (top to bottom):     4 2 6 1 3 5 7

DFS Family

  • Explores one branch fully, then backtracks
  • Naturally recursive; iterative form uses an explicit stack
  • Variants: preorder, inorder, postorder
  • Space O(h) where h is tree height

BFS Family

  • Explores all nodes at depth d before depth d+1
  • Uses a queue (FIFO)
  • Variant: level-by-level grouping, zigzag
  • Space O(w) where w is max width (up to n/2)

1. Preorder Traversal (Root, Left, Right)

Visit the current node first, then recurse left, then recurse right. Preorder is useful for copying/serializing a tree (the root comes first, so reconstruction is natural) and for prefix-expression evaluation.

For our example tree the output is: 4 2 1 3 6 5 7

Recursive

void preorder(TreeNode* node, vector<int>& out) {
    if (!node) return;
    out.push_back(node->val);   // Root
    preorder(node->left, out);  // Left
    preorder(node->right, out); // Right
}

void preorder(TreeNode node, List<Integer> out) {
    if (node == null) return;
    out.add(node.val);          // Root
    preorder(node.left, out);   // Left
    preorder(node.right, out);  // Right
}

def preorder(node, out):
    if not node:
        return
    out.append(node.val)   # Root
    preorder(node.left, out)   # Left
    preorder(node.right, out)  # Right

Iterative (explicit stack)

Push the root, then repeatedly pop a node, record it, and push its right child first then left child so the left is processed next (LIFO order).

vector<int> preorderIterative(TreeNode* root) {
    vector<int> out;
    if (!root) return out;
    stack<TreeNode*> st;
    st.push(root);
    while (!st.empty()) {
        TreeNode* node = st.top(); st.pop();
        out.push_back(node->val);
        // push right first so left is popped next
        if (node->right) st.push(node->right);
        if (node->left)  st.push(node->left);
    }
    return out;
}

List<Integer> preorderIterative(TreeNode root) {
    List<Integer> out = new ArrayList<>();
    if (root == null) return out;
    Deque<TreeNode> st = new ArrayDeque<>();
    st.push(root);
    while (!st.isEmpty()) {
        TreeNode node = st.pop();
        out.add(node.val);
        // push right first so left is popped next
        if (node.right != null) st.push(node.right);
        if (node.left != null)  st.push(node.left);
    }
    return out;
}

def preorder_iterative(root):
    out = []
    if not root:
        return out
    stack = [root]
    while stack:
        node = stack.pop()
        out.append(node.val)
        # push right first so left is popped next
        if node.right:
            stack.append(node.right)
        if node.left:
            stack.append(node.left)
    return out

2. Inorder Traversal (Left, Root, Right)

Recurse left first, then visit the node, then recurse right.

For our example tree the output is: 1 2 3 4 5 6 7

Key property: An inorder traversal of a Binary Search Tree visits keys in sorted ascending order. This is why our example yields 1 2 3 4 5 6 7. It is a common interview test for BST validity: check that the inorder sequence is strictly increasing.

Recursive

void inorder(TreeNode* node, vector<int>& out) {
    if (!node) return;
    inorder(node->left, out);   // Left
    out.push_back(node->val);   // Root
    inorder(node->right, out);  // Right
}

void inorder(TreeNode node, List<Integer> out) {
    if (node == null) return;
    inorder(node.left, out);    // Left
    out.add(node.val);          // Root
    inorder(node.right, out);   // Right
}

def inorder(node, out):
    if not node:
        return
    inorder(node.left, out)    # Left
    out.append(node.val)   # Root
    inorder(node.right, out)   # Right

Iterative (explicit stack)

Walk left as far as possible, pushing nodes. When you can go no further, pop a node, record it, then move to its right child and repeat.

vector<int> inorderIterative(TreeNode* root) {
    vector<int> out;
    stack<TreeNode*> st;
    TreeNode* cur = root;
    while (cur || !st.empty()) {
        while (cur) {            // go as left as possible
            st.push(cur);
            cur = cur->left;
        }
        cur = st.top(); st.pop();
        out.push_back(cur->val); // visit node
        cur = cur->right;        // then go right
    }
    return out;
}

List<Integer> inorderIterative(TreeNode root) {
    List<Integer> out = new ArrayList<>();
    Deque<TreeNode> st = new ArrayDeque<>();
    TreeNode cur = root;
    while (cur != null || !st.isEmpty()) {
        while (cur != null) {     // go as left as possible
            st.push(cur);
            cur = cur.left;
        }
        cur = st.pop();
        out.add(cur.val);        // visit node
        cur = cur.right;         // then go right
    }
    return out;
}

def inorder_iterative(root):
    out, stack, cur = [], [], root
    while cur or stack:
        while cur:              # go as left as possible
            stack.append(cur)
            cur = cur.left
        cur = stack.pop()
        out.append(cur.val)    # visit node
        cur = cur.right        # then go right
    return out

3. Postorder Traversal (Left, Right, Root)

Recurse left, recurse right, then visit the node last. Postorder is the order you would use to delete/free a tree (children before parent) or to evaluate a postfix expression and compute values bottom-up.

For our example tree the output is: 1 3 2 5 7 6 4

Recursive

void postorder(TreeNode* node, vector<int>& out) {
    if (!node) return;
    postorder(node->left, out);  // Left
    postorder(node->right, out); // Right
    out.push_back(node->val);    // Root
}

void postorder(TreeNode node, List<Integer> out) {
    if (node == null) return;
    postorder(node.left, out);   // Left
    postorder(node.right, out);  // Right
    out.add(node.val);           // Root
}

def postorder(node, out):
    if not node:
        return
    postorder(node.left, out)  # Left
    postorder(node.right, out) # Right
    out.append(node.val)   # Root

Iterative — Two-Stack Method

Note that reverse postorder = "Root, Right, Left", which is a mirrored preorder. So run a preorder-like traversal pushing onto a second stack, then reverse it.

vector<int> postorderTwoStacks(TreeNode* root) {
    vector<int> out;
    if (!root) return out;
    stack<TreeNode*> s1, s2;
    s1.push(root);
    while (!s1.empty()) {
        TreeNode* node = s1.top(); s1.pop();
        s2.push(node);              // collect in Root,Right,Left order
        if (node->left)  s1.push(node->left);
        if (node->right) s1.push(node->right);
    }
    while (!s2.empty()) {            // reverse -> Left,Right,Root
        out.push_back(s2.top()->val);
        s2.pop();
    }
    return out;
}

List<Integer> postorderTwoStacks(TreeNode root) {
    LinkedList<Integer> out = new LinkedList<>();
    if (root == null) return out;
    Deque<TreeNode> st = new ArrayDeque<>();
    st.push(root);
    while (!st.isEmpty()) {
        TreeNode node = st.pop();
        out.addFirst(node.val);     // prepend -> reverses Root,Right,Left
        if (node.left != null)  st.push(node.left);
        if (node.right != null) st.push(node.right);
    }
    return out;
}

def postorder_two_stacks(root):
    if not root:
        return []
    stack, out = [root], []
    while stack:
        node = stack.pop()
        out.append(node.val)       # Root,Right,Left order
        if node.left:
            stack.append(node.left)
        if node.right:
            stack.append(node.right)
    return out[::-1]            # reverse -> Left,Right,Root

Iterative — Single Stack with Last-Visited Pointer

A true single-stack postorder. We only visit a node after its right subtree has been fully processed, tracked by a lastVisited pointer.

vector<int> postorderSingleStack(TreeNode* root) {
    vector<int> out;
    stack<TreeNode*> st;
    TreeNode* cur = root;
    TreeNode* lastVisited = nullptr;
    while (cur || !st.empty()) {
        while (cur) { st.push(cur); cur = cur->left; }
        TreeNode* peek = st.top();
        // if right exists and not yet processed, go right
        if (peek->right && lastVisited != peek->right) {
            cur = peek->right;
        } else {
            out.push_back(peek->val);
            lastVisited = peek;
            st.pop();
        }
    }
    return out;
}

List<Integer> postorderSingleStack(TreeNode root) {
    List<Integer> out = new ArrayList<>();
    Deque<TreeNode> st = new ArrayDeque<>();
    TreeNode cur = root, lastVisited = null;
    while (cur != null || !st.isEmpty()) {
        while (cur != null) { st.push(cur); cur = cur.left; }
        TreeNode peek = st.peek();
        if (peek.right != null && lastVisited != peek.right) {
            cur = peek.right;
        } else {
            out.add(peek.val);
            lastVisited = st.pop();
        }
    }
    return out;
}

def postorder_single_stack(root):
    out, stack = [], []
    cur, last_visited = root, None
    while cur or stack:
        while cur:
            stack.append(cur)
            cur = cur.left
        peek = stack[-1]
        if peek.right and last_visited is not peek.right:
            cur = peek.right
        else:
            out.append(peek.val)
            last_visited = stack.pop()
    return out

4. Level-Order Traversal (BFS)

Use a FIFO queue: enqueue the root, then repeatedly dequeue a node, record it, and enqueue its children. This visits nodes top-to-bottom, left-to-right.

For our example tree the output is: 4 2 6 1 3 5 7

BFS queue walk for the example tree:

  queue: [4]            -> pop 4, push 2,6     out: 4
  queue: [2, 6]         -> pop 2, push 1,3     out: 4 2
  queue: [6, 1, 3]      -> pop 6, push 5,7     out: 4 2 6
  queue: [1, 3, 5, 7]   -> pop 1               out: 4 2 6 1
  queue: [3, 5, 7]      -> pop 3               out: 4 2 6 1 3
  queue: [5, 7]         -> pop 5               out: 4 2 6 1 3 5
  queue: [7]            -> pop 7               out: 4 2 6 1 3 5 7
  queue: []  done.

Flat Level-Order

vector<int> levelOrder(TreeNode* root) {
    vector<int> out;
    if (!root) return out;
    queue<TreeNode*> q;
    q.push(root);
    while (!q.empty()) {
        TreeNode* node = q.front(); q.pop();
        out.push_back(node->val);
        if (node->left)  q.push(node->left);
        if (node->right) q.push(node->right);
    }
    return out;
}

List<Integer> levelOrder(TreeNode root) {
    List<Integer> out = new ArrayList<>();
    if (root == null) return out;
    Queue<TreeNode> q = new LinkedList<>();
    q.offer(root);
    while (!q.isEmpty()) {
        TreeNode node = q.poll();
        out.add(node.val);
        if (node.left != null)  q.offer(node.left);
        if (node.right != null) q.offer(node.right);
    }
    return out;
}

from collections import deque

def level_order(root):
    out = []
    if not root:
        return out
    q = deque([root])
    while q:
        node = q.popleft()
        out.append(node.val)
        if node.left:
            q.append(node.left)
        if node.right:
            q.append(node.right)
    return out

Level-by-Level (List of Lists)

The trick: at the start of each outer iteration, the queue holds exactly one full level. Snapshot its size and process that many nodes to group them. For our tree this returns [[4], [2, 6], [1, 3, 5, 7]].

vector<vector<int>> levelOrderByLevel(TreeNode* root) {
    vector<vector<int>> out;
    if (!root) return out;
    queue<TreeNode*> q;
    q.push(root);
    while (!q.empty()) {
        int sz = q.size();      // nodes in this level
        vector<int> level;
        for (int i = 0; i < sz; i++) {
            TreeNode* node = q.front(); q.pop();
            level.push_back(node->val);
            if (node->left)  q.push(node->left);
            if (node->right) q.push(node->right);
        }
        out.push_back(level);
    }
    return out;
}

List<List<Integer>> levelOrderByLevel(TreeNode root) {
    List<List<Integer>> out = new ArrayList<>();
    if (root == null) return out;
    Queue<TreeNode> q = new LinkedList<>();
    q.offer(root);
    while (!q.isEmpty()) {
        int sz = q.size();      // nodes in this level
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < sz; i++) {
            TreeNode node = q.poll();
            level.add(node.val);
            if (node.left != null)  q.offer(node.left);
            if (node.right != null) q.offer(node.right);
        }
        out.add(level);
    }
    return out;
}

from collections import deque

def level_order_by_level(root):
    out = []
    if not root:
        return out
    q = deque([root])
    while q:
        level = []
        for _ in range(len(q)):   # nodes in this level
            node = q.popleft()
            level.append(node.val)
            if node.left:
                q.append(node.left)
            if node.right:
                q.append(node.right)
        out.append(level)
    return out

Variant: Zigzag / Spiral Order

Same level grouping, but reverse the order of every other level (left-to-right, then right-to-left, alternating). For our tree this yields [[4], [6, 2], [1, 3, 5, 7]]. Just track a boolean and reverse the level list (or insert at the front) on odd levels.

vector<vector<int>> zigzag(TreeNode* root) {
    vector<vector<int>> out;
    if (!root) return out;
    queue<TreeNode*> q;
    q.push(root);
    bool leftToRight = true;
    while (!q.empty()) {
        int sz = q.size();
        vector<int> level(sz);
        for (int i = 0; i < sz; i++) {
            TreeNode* node = q.front(); q.pop();
            int idx = leftToRight ? i : sz - 1 - i;
            level[idx] = node->val;
            if (node->left)  q.push(node->left);
            if (node->right) q.push(node->right);
        }
        out.push_back(level);
        leftToRight = !leftToRight;
    }
    return out;
}

List<List<Integer>> zigzag(TreeNode root) {
    List<List<Integer>> out = new ArrayList<>();
    if (root == null) return out;
    Queue<TreeNode> q = new LinkedList<>();
    q.offer(root);
    boolean leftToRight = true;
    while (!q.isEmpty()) {
        int sz = q.size();
        Integer[] level = new Integer[sz];
        for (int i = 0; i < sz; i++) {
            TreeNode node = q.poll();
            int idx = leftToRight ? i : sz - 1 - i;
            level[idx] = node.val;
            if (node.left != null)  q.offer(node.left);
            if (node.right != null) q.offer(node.right);
        }
        out.add(Arrays.asList(level));
        leftToRight = !leftToRight;
    }
    return out;
}

from collections import deque

def zigzag(root):
    out = []
    if not root:
        return out
    q = deque([root])
    left_to_right = True
    while q:
        level = []
        for _ in range(len(q)):
            node = q.popleft()
            level.append(node.val)
            if node.left:
                q.append(node.left)
            if node.right:
                q.append(node.right)
        out.append(level if left_to_right else level[::-1])
        left_to_right = not left_to_right
    return out

5. Morris Traversal (O(1) Space Inorder)

Recursive and stack-based inorder both use O(h) auxiliary space. Morris traversal achieves inorder in O(1) extra space by temporarily creating threads: for each node with a left subtree, it rewires the left subtree's rightmost node (the inorder predecessor) to point back to the current node, then removes the thread once it has been used. It mutates the tree during traversal but restores it fully.

The rule for the current node cur:

  • If cur has no left child: visit cur, then move right.
  • Otherwise find pred = rightmost node of cur's left subtree:
    • If pred.right is null: set pred.right = cur (create thread), move left.
    • If pred.right == cur: thread already exists, so left subtree is done. Remove thread, visit cur, move right.
Morris walk on the example tree (visiting in inorder):

           4
         /   \
        2     6
       / \   / \
      1   3 5   7

cur=4: has left(2). pred = rightmost of {2,1,3} = 3.
       3.right is null  -> thread 3.right=4, move left to 2.
cur=2: has left(1). pred = 1. 1.right null -> thread 1.right=2, left to 1.
cur=1: no left -> VISIT 1, follow right thread to 2.
cur=2: pred = 1, 1.right == 2 -> remove thread, VISIT 2, move right to 3.
cur=3: no left -> VISIT 3, follow right thread to 4.
cur=4: pred = 3, 3.right == 4 -> remove thread, VISIT 4, move right to 6.
cur=6: has left(5). pred = 5. 5.right null -> thread 5.right=6, left to 5.
cur=5: no left -> VISIT 5, follow right thread to 6.
cur=6: pred = 5, 5.right == 6 -> remove thread, VISIT 6, move right to 7.
cur=7: no left -> VISIT 7, move right to null. Done.

Output: 1 2 3 4 5 6 7   (tree fully restored)

vector<int> morrisInorder(TreeNode* root) {
    vector<int> out;
    TreeNode* cur = root;
    while (cur) {
        if (!cur->left) {
            out.push_back(cur->val);     // visit
            cur = cur->right;
        } else {
            TreeNode* pred = cur->left;
            while (pred->right && pred->right != cur)
                pred = pred->right;      // find predecessor
            if (!pred->right) {
                pred->right = cur;        // create thread
                cur = cur->left;
            } else {
                pred->right = nullptr;     // remove thread
                out.push_back(cur->val);  // visit
                cur = cur->right;
            }
        }
    }
    return out;
}

List<Integer> morrisInorder(TreeNode root) {
    List<Integer> out = new ArrayList<>();
    TreeNode cur = root;
    while (cur != null) {
        if (cur.left == null) {
            out.add(cur.val);            // visit
            cur = cur.right;
        } else {
            TreeNode pred = cur.left;
            while (pred.right != null && pred.right != cur)
                pred = pred.right;       // find predecessor
            if (pred.right == null) {
                pred.right = cur;         // create thread
                cur = cur.left;
            } else {
                pred.right = null;       // remove thread
                out.add(cur.val);         // visit
                cur = cur.right;
            }
        }
    }
    return out;
}

def morris_inorder(root):
    out = []
    cur = root
    while cur:
        if not cur.left:
            out.append(cur.val)        # visit
            cur = cur.right
        else:
            pred = cur.left
            while pred.right and pred.right is not cur:
                pred = pred.right      # find predecessor
            if pred.right is None:
                pred.right = cur       # create thread
                cur = cur.left
            else:
                pred.right = None      # remove thread
                out.append(cur.val)    # visit
                cur = cur.right
    return out
Morris preorder (brief): The structure is identical, but you visit the node at the moment you create the thread (in the pred.right == null branch) instead of when you remove it. That shifts the visit order from Left-Root-Right to Root-Left-Right. Each node is still visited once and the tree is restored. There is no simple O(1) Morris postorder; the standard approach reverses paths and is rarely needed.
Caveat: Morris traversal mutates the tree while running. It is not safe to run two Morris traversals concurrently on the same tree or to interrupt one mid-way, and it cannot be used on immutable trees. Total work is still O(n): although the predecessor search looks like extra work, each edge is traversed at most a constant number of times.

6. Complexity Summary

Here n = number of nodes, h = tree height. For a balanced tree h = O(log n); for a degenerate (linked-list-like) tree h = O(n).

TraversalTimeRecursive SpaceIterative SpaceMorris Space
PreorderO(n)O(h)O(h)O(1)
InorderO(n)O(h)O(h)O(1)
PostorderO(n)O(h)O(h)O(1)*
Level-order (BFS)O(n)—O(w) ≤ O(n)—

* Morris postorder exists but is intricate (path reversal); inorder and preorder are the common O(1) cases. Recursive space is the implicit call-stack depth.

How to pick a traversal:
  • Preorder — serialize / clone a tree, write prefix notation, or process a parent before its children.
  • Inorder — read a BST's keys in sorted order, validate a BST, or find the kth smallest element.
  • Postorder — free/delete a tree, evaluate expression trees, or compute results that depend on both subtrees (e.g. subtree sums, height, diameter).
  • Level-order (BFS) — shortest path in unweighted trees, level grouping, right-side view, or "process nearest first" problems.
  • Morris — only when O(1) extra space is a hard requirement and the tree may be mutated; otherwise prefer the simpler recursive form.
