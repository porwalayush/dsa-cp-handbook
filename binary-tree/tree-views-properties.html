Binary Tree Views & Properties

Imagine the tree as a real object hanging in front of you. Some questions are about its shape — how tall is it, what's the longest stick you could lay across it? Those are its properties (height, diameter, balance, width). Other questions are about what you'd see if you walked around it and only noticed the nearest node at each level — those are its views (left, right, top, bottom, and so on).

The good news: almost all of these are answered by one walk over the tree, either top-down level by level (BFS) or all the way down each branch and back (DFS), in O(n) time. 1. Height & Depth

Think of the tree as a building with the root on the ground floor. Depth answers "how far down from the entrance am I?" — you count floors going down from the root. Height answers "how many floors are still below me before I hit the basement (a leaf)?" — you count floors going up from the lowest leaf. The two are mirror images: depth is measured from the top, height from the bottom.

With that picture in mind, here are the precise definitions:

  • Depth of a node = number of edges on the path from the root down to that node. The root has depth 0.
  • Height of a node = number of edges on the longest path from that node down to a leaf. A leaf has height 0.
  • Height of the tree = height of its root.

An empty tree is conventionally given height -1 (so that a single node has height 0). Some texts measure height/depth in nodes instead of edges, which shifts every value by 1 — pick one convention and stay consistent. Here we count edges.

                A          depth 0   height 3
              /   \
            B       C      depth 1   B.height 2,  C.height 0
          /   \
        D       E          depth 2   D.height 1,  E.height 0
       /
      F                    depth 3   height 0  (leaf)

depth grows DOWNWARD from the root (top-down).
height grows UPWARD from the leaves (bottom-up).
Tree height = A.height = longest root->leaf path = A-B-D-F = 3 edges.

Height is naturally recursive: a node's height is 1 + max(height(left), height(right)), with empty subtrees contributing -1.

// height in edges; empty tree returns -1
int height(Node* root) {
    if (!root) return -1;
    return 1 + max(height(root->left), height(root->right));
}

// height in edges; empty tree returns -1
int height(Node root) {
    if (root == null) return -1;
    return 1 + Math.max(height(root.left), height(root.right));
}

# height in edges; empty tree returns -1
def height(root):
    if root is None:
        return -1
    return 1 + max(height(root.left), height(root.right))

2. Counting Nodes, Leaves, and Internal/Full Nodes

Each count is a one-line postorder recursion. A leaf has no children; an internal node has at least one child; a full node has exactly two children.

int countNodes(Node* r) {
    if (!r) return 0;
    return 1 + countNodes(r->left) + countNodes(r->right);
}

int countLeaves(Node* r) {
    if (!r) return 0;
    if (!r->left && !r->right) return 1;
    return countLeaves(r->left) + countLeaves(r->right);
}

int countInternal(Node* r) {        // >= 1 child
    if (!r || (!r->left && !r->right)) return 0;
    return 1 + countInternal(r->left) + countInternal(r->right);
}

int countFull(Node* r) {            // exactly 2 children
    if (!r) return 0;
    int self = (r->left && r->right) ? 1 : 0;
    return self + countFull(r->left) + countFull(r->right);
}

int countNodes(Node r) {
    if (r == null) return 0;
    return 1 + countNodes(r.left) + countNodes(r.right);
}

int countLeaves(Node r) {
    if (r == null) return 0;
    if (r.left == null && r.right == null) return 1;
    return countLeaves(r.left) + countLeaves(r.right);
}

int countInternal(Node r) {        // >= 1 child
    if (r == null || (r.left == null && r.right == null)) return 0;
    return 1 + countInternal(r.left) + countInternal(r.right);
}

int countFull(Node r) {            // exactly 2 children
    if (r == null) return 0;
    int self = (r.left != null && r.right != null) ? 1 : 0;
    return self + countFull(r.left) + countFull(r.right);
}

def count_nodes(r):
    if r is None: return 0
    return 1 + count_nodes(r.left) + count_nodes(r.right)

def count_leaves(r):
    if r is None: return 0
    if r.left is None and r.right is None: return 1
    return count_leaves(r.left) + count_leaves(r.right)

def count_internal(r):              # >= 1 child
    if r is None or (r.left is None and r.right is None): return 0
    return 1 + count_internal(r.left) + count_internal(r.right)

def count_full(r):                  # exactly 2 children
    if r is None: return 0
    self = 1 if (r.left and r.right) else 0
    return self + count_full(r.left) + count_full(r.right)
Handy identity: in any binary tree, (# leaves) = (# full nodes) + 1. So a tree's leaf count exceeds its two-children-node count by exactly one.

3. Diameter of a Binary Tree

Picture stretching a piece of string between the two most distant nodes in the tree, pulled tight. The diameter is the length of that string, counted in edges. The catch beginners miss: that longest string often does not pass through the root — it can bend at any node, going down one branch and back up another.

Formally, the diameter is the length of the longest path between any two nodes, measured in edges. That path need not pass through the root. A naive solution computes height at every node (O(n²)). The clean approach is a single postorder pass: each call returns its node's height, and while returning, it updates a global maximum with the longest path that bends at the current node, namely leftHeight + rightHeight + 2 edges.

                1
              /   \
            2       3
          /   \
        4       5
       /         \
      6             7
     /               \
    8                   9

Longest path = 8-6-4-2-5-7-9  (6 edges).
It passes through node 2, NOT through the root (node 1).
At node 2:  leftHeight(subtree at 4) = 2,
            rightHeight(subtree at 5) = 2,
            path through 2 = 2 + 2 + 2 = 6 edges. <- diameter

int best = 0;  // global max diameter (in edges)

// returns height (edges) of subtree at r
int dfs(Node* r) {
    if (!r) return -1;
    int lh = dfs(r->left);
    int rh = dfs(r->right);
    best = max(best, lh + rh + 2);   // path bending at r
    return 1 + max(lh, rh);
}

int diameter(Node* root) {
    best = 0;
    dfs(root);
    return best;
}

int best = 0;  // global max diameter (in edges)

// returns height (edges) of subtree at r
int dfs(Node r) {
    if (r == null) return -1;
    int lh = dfs(r.left);
    int rh = dfs(r.right);
    best = Math.max(best, lh + rh + 2);  // path bending at r
    return 1 + Math.max(lh, rh);
}

int diameter(Node root) {
    best = 0;
    dfs(root);
    return best;
}

def diameter(root):
    best = 0
    def dfs(r):                  # returns height in edges
        nonlocal best
        if r is None:
            return -1
        lh = dfs(r.left)
        rh = dfs(r.right)
        best = max(best, lh + rh + 2)  # path bending at r
        return 1 + max(lh, rh)
    dfs(root)
    return best

If your height convention returns 0 for an empty subtree (counting nodes), the bend term becomes lh + rh instead of lh + rh + 2. Match the constant to your base case.

4. Balanced Binary Tree Check

A tree is height-balanced if, for every node, the heights of its left and right subtrees differ by at most 1. A top-down check that recomputes height per node is O(n²). The efficient O(n) version is bottom-up: each call returns the subtree height, or a sentinel -2 the moment imbalance is detected, which then short-circuits all the way up. (We use -2 as the sentinel because the legitimate height of an empty subtree is already -1.)

const int UNBAL = -2;

// returns height, or UNBAL if any subtree is unbalanced
int check(Node* r) {
    if (!r) return -1;
    int lh = check(r->left);
    if (lh == UNBAL) return UNBAL;
    int rh = check(r->right);
    if (rh == UNBAL) return UNBAL;
    if (abs(lh - rh) > 1) return UNBAL;
    return 1 + max(lh, rh);
}

bool isBalanced(Node* root) {
    return check(root) != UNBAL;
}

static final int UNBAL = -2;

// returns height, or UNBAL if any subtree is unbalanced
int check(Node r) {
    if (r == null) return -1;
    int lh = check(r.left);
    if (lh == UNBAL) return UNBAL;
    int rh = check(r.right);
    if (rh == UNBAL) return UNBAL;
    if (Math.abs(lh - rh) > 1) return UNBAL;
    return 1 + Math.max(lh, rh);
}

boolean isBalanced(Node root) {
    return check(root) != UNBAL;
}

UNBAL = -2

def is_balanced(root):
    def check(r):                # returns height, or UNBAL
        if r is None:
            return -1
        lh = check(r.left)
        if lh == UNBAL: return UNBAL
        rh = check(r.right)
        if rh == UNBAL: return UNBAL
        if abs(lh - rh) > 1: return UNBAL
        return 1 + max(lh, rh)
    return check(root) != UNBAL

5. Maximum Width of a Binary Tree

The maximum width is the largest number of nodes present at any single level, where the width of a level is measured between its leftmost and rightmost non-null nodes (counting the gaps for missing nodes in between). We BFS level by level, assigning each node an index as if the tree were a complete array: a node at index i has children at 2i (left) and 2i + 1 (right). The width of a level is lastIndex - firstIndex + 1.

            1            i = 1
          /   \
        2       3        i = 2,        i = 3
       / \       \
      4   5         7    i = 4, i = 5,        i = 7

Level 3 indices: 4, 5, 7  ->  width = 7 - 4 + 1 = 4
(the gap at index 6 is counted, matching array-style spacing)
Maximum width over all levels = 4.

To keep indices small (and avoid overflow), subtract the first index of each level from every index in that level so numbering restarts near 0.

// index-numbered BFS: child of i are 2i and 2i+1
long long maxWidth(Node* root) {
    if (!root) return 0;
    long long best = 0;
    queue<pair<Node*, long long>> q;
    q.push({root, 0});
    while (!q.empty()) {
        int sz = q.size();
        long long first = q.front().second, last = first;
        for (int k = 0; k < sz; k++) {
            auto [node, idx] = q.front(); q.pop();
            idx -= first;                // rebase to avoid overflow
            last = idx;
            if (node->left)  q.push({node->left,  2 * idx});
            if (node->right) q.push({node->right, 2 * idx + 1});
        }
        best = max(best, last + 1);
    }
    return best;
}

// index-numbered BFS: children of i are 2i and 2i+1
long maxWidth(Node root) {
    if (root == null) return 0;
    long best = 0;
    Deque<Object[]> q = new ArrayDeque<>();
    q.add(new Object[]{root, 0L});
    while (!q.isEmpty()) {
        int sz = q.size();
        long first = (Long) q.peek()[1], last = first;
        for (int k = 0; k < sz; k++) {
            Object[] cur = q.poll();
            Node node = (Node) cur[0];
            long idx = (Long) cur[1] - first;   // rebase
            last = idx;
            if (node.left  != null) q.add(new Object[]{node.left,  2 * idx});
            if (node.right != null) q.add(new Object[]{node.right, 2 * idx + 1});
        }
        best = Math.max(best, last + 1);
    }
    return best;
}

from collections import deque

# index-numbered BFS: children of i are 2i and 2i+1
def max_width(root):
    if root is None:
        return 0
    best = 0
    q = deque([(root, 0)])
    while q:
        sz = len(q)
        first = q[0][1]
        last = first
        for _ in range(sz):
            node, idx = q.popleft()
            idx -= first              # rebase (Python ints are unbounded)
            last = idx
            if node.left:  q.append((node.left,  2 * idx))
            if node.right: q.append((node.right, 2 * idx + 1))
        best = max(best, last - 0 + 1)
    return best
Overflow warning: indices double every level, so for a deep, sparse tree they grow exponentially (roughly 2^depth). Use a 64-bit type (long long / long) and rebase each level's indices against its first index. Without rebasing, a tree of depth ~63 overflows a 64-bit integer. Python integers are arbitrary-precision so they won't overflow, but rebasing still keeps the numbers cheap to handle.

6. Tree Views via Level Traversal

A view is what you see when you stand on one side of the tree and squint: at each level, only the nearest node to you is visible — the ones behind it are blocked. Walk to a different side and a different node blocks your view at each level. That single idea — "keep the closest node per level (or per column)" — gives every view below.

Left View and Right View

Stand to the left of the tree and look across each level: the closest node you see is the first one at that level — that's the left view. Walk around to the right side and the closest node per level is the last one — the right view. With BFS, that is simply the first / last node dequeued per level.

                1
              /   \
            2       3
              \       \
                5       7

Left view  (first per level): 1, 2, 5
Right view (last  per level): 1, 3, 7

vector<int> sideView(Node* root, bool rightSide) {
    vector<int> res;
    if (!root) return res;
    queue<Node*> q;
    q.push(root);
    while (!q.empty()) {
        int sz = q.size();
        for (int i = 0; i < sz; i++) {
            Node* node = q.front(); q.pop();
            // first node (i==0) for left, last (i==sz-1) for right
            if (rightSide ? (i == sz - 1) : (i == 0))
                res.push_back(node->val);
            if (node->left)  q.push(node->left);
            if (node->right) q.push(node->right);
        }
    }
    return res;
}

List<Integer> sideView(Node root, boolean rightSide) {
    List<Integer> res = new ArrayList<>();
    if (root == null) return res;
    Queue<Node> q = new LinkedList<>();
    q.add(root);
    while (!q.isEmpty()) {
        int sz = q.size();
        for (int i = 0; i < sz; i++) {
            Node node = q.poll();
            if (rightSide ? (i == sz - 1) : (i == 0))
                res.add(node.val);
            if (node.left  != null) q.add(node.left);
            if (node.right != null) q.add(node.right);
        }
    }
    return res;
}

from collections import deque

def side_view(root, right_side):
    res = []
    if root is None:
        return res
    q = deque([root])
    while q:
        sz = len(q)
        for i in range(sz):
            node = q.popleft()
            if (i == sz - 1) if right_side else (i == 0):
                res.append(node.val)
            if node.left:  q.append(node.left)
            if node.right: q.append(node.right)
    return res

Top View and Bottom View (Horizontal Distance)

Now look straight down from above the tree (top view) or straight up from underneath it (bottom view). Imagine each node drops a vertical line; nodes that line up in the same column overlap. From the top you see whichever node in a column is highest; from below, whichever is lowest. We label each column with a horizontal distance (HD): the root is HD 0, a left child is HD - 1, a right child is HD + 1. Do a BFS and keep a map keyed by HD.

  • Top view: for each HD, keep the first node encountered (the topmost). With BFS this is the first time an HD appears.
  • Bottom view: for each HD, keep the last node encountered (the lowest), so always overwrite the map entry.

BFS (not DFS) is important here so that ties on HD are resolved by the shallower / level-order node correctly. Finally, read the map in ascending HD order.

HD:    -2    -1     0     1     2
                    1
                  /   \
              2(-1)     3(+1)
             /   \     /   \
        4(-2) 5(0) 6(0) 7(+2)

Top view    (first per HD): 4, 2, 1, 3, 7   (HD -2,-1,0,+1,+2)
Bottom view (last  per HD): 4, 2, 6, 3, 7
  (at HD 0 the root 1 is seen first -> top; node 6 is seen last -> bottom)

// TOP VIEW: keep first node seen at each HD
vector<int> topView(Node* root) {
    vector<int> res;
    if (!root) return res;
    map<int, int> hdMap;          // HD -> node value (sorted)
    queue<pair<Node*, int>> q;     // (node, HD)
    q.push({root, 0});
    while (!q.empty()) {
        auto [node, hd] = q.front(); q.pop();
        if (!hdMap.count(hd))            // FIRST wins for top view
            hdMap[hd] = node->val;
        // for BOTTOM VIEW, drop the count() check and
        // always assign: hdMap[hd] = node->val; (LAST wins)
        if (node->left)  q.push({node->left,  hd - 1});
        if (node->right) q.push({node->right, hd + 1});
    }
    for (auto& [hd, val] : hdMap) res.push_back(val);
    return res;
}

// TOP VIEW: keep first node seen at each HD
List<Integer> topView(Node root) {
    List<Integer> res = new ArrayList<>();
    if (root == null) return res;
    TreeMap<Integer, Integer> hdMap = new TreeMap<>();
    Queue<Object[]> q = new LinkedList<>();
    q.add(new Object[]{root, 0});
    while (!q.isEmpty()) {
        Object[] cur = q.poll();
        Node node = (Node) cur[0];
        int hd = (Integer) cur[1];
        if (!hdMap.containsKey(hd))      // FIRST wins for top view
            hdMap.put(hd, node.val);
        // BOTTOM VIEW: always hdMap.put(hd, node.val); (LAST wins)
        if (node.left  != null) q.add(new Object[]{node.left,  hd - 1});
        if (node.right != null) q.add(new Object[]{node.right, hd + 1});
    }
    res.addAll(hdMap.values());      // TreeMap iterates by ascending HD
    return res;
}

from collections import deque

# TOP VIEW: keep first node seen at each HD
def top_view(root):
    if root is None:
        return []
    hd_map = {}                      # HD -> value
    q = deque([(root, 0)])
    while q:
        node, hd = q.popleft()
        if hd not in hd_map:           # FIRST wins for top view
            hd_map[hd] = node.val
        # BOTTOM VIEW: always hd_map[hd] = node.val  (LAST wins)
        if node.left:  q.append((node.left,  hd - 1))
        if node.right: q.append((node.right, hd + 1))
    return [hd_map[hd] for hd in sorted(hd_map)]

The only difference between top and bottom view is whether you keep the first or the last node at each HD: top view writes the map only when the HD is unseen; bottom view overwrites it every time.

Vertical Order Traversal

Vertical order groups all nodes by HD into columns, read left to right. When two nodes share the same HD, the strict ordering breaks ties by level (higher node first) and, if still tied, by value (smaller first). Implement it by collecting triples (hd, level, val) during a traversal, sorting by (hd, level, val), and emitting values grouped by HD. This is O(n log n) due to the sort.

7. Boundary Traversal

The boundary traversal (anti-clockwise) prints the outline of the tree in three parts, taking care not to print any node twice:

  1. Left boundary (top → down), excluding leaves.
  2. All leaves, left to right.
  3. Right boundary (down → top, i.e. collected top-down then reversed), excluding leaves.

The root is added once at the front. A node belongs to the left boundary if you reach it by preferring left children (or right when no left exists); the right boundary mirrors this preferring right children.

                1
              /   \
            2       3
          /   \    /  \
         4     5  6     7
              / \
             8   9

Left boundary (no leaves): 1, 2
Leaves (L to R):           4, 8, 9, 6, 7
Right boundary (rev):      3            (collected 3, then reversed)
Boundary = 1, 2, 4, 8, 9, 6, 7, 3

def boundary(root):
    if root is None:
        return []

    def is_leaf(n):
        return n.left is None and n.right is None

    res = [root.val] if not is_leaf(root) else []

    # Part 1: left boundary, top-down, excluding leaves
    cur = root.left
    while cur:
        if not is_leaf(cur):
            res.append(cur.val)
        cur = cur.left if cur.left else cur.right

    # Part 2: all leaves, left to right (DFS)
    def leaves(n):
        if n is None:
            return
        if is_leaf(n):
            res.append(n.val)
            return
        leaves(n.left)
        leaves(n.right)
    leaves(root)

    # Part 3: right boundary, collected top-down then reversed
    tmp, cur = [], root.right
    while cur:
        if not is_leaf(cur):
            tmp.append(cur.val)
        cur = cur.right if cur.right else cur.left
    res.extend(reversed(tmp))

    return res
Edge cases for boundary traversal:
  • If the root itself is a leaf, add it once and stop (the loops add nothing).
  • Do not double-count: skip leaves in parts 1 and 3 since part 2 prints all leaves.
  • If root.left or root.right is missing, the corresponding boundary loop simply contributes nothing.

8. Complexity Summary

Property / ViewApproachTimeSpace
Height / DepthDFS postorderO(n)O(h)
Count nodes / leaves / internal / fullDFS postorderO(n)O(h)
DiameterDFS, height + global maxO(n)O(h)
Balanced checkDFS bottom-up, -2 sentinelO(n)O(h)

Here h is the tree height (recursion stack depth). The map-based views are O(n log n) with an ordered map (std::map / TreeMap) or with a final sort; they drop to O(n) only if you sort by tracking the min/max HD range and bucketing.

The recurring "return-a-tuple, build bottom-up" pattern:
  • Diameter, balance, and height all share one shape: a postorder DFS where each call returns information about its subtree (typically the height) while updating a global / outer answer as it unwinds.
  • When a node needs more than one fact from its children — e.g. height and a balance flag, or height and diameter, or subtree sum and node count — return a tuple/struct instead of a single int. This keeps the whole computation in one O(n) pass and avoids the O(n²) trap of recomputing height at every node.
  • Mentally: "child returns its summary → I combine the two summaries → I update the answer → I return my own summary upward." Almost every tree-property problem fits this template.
