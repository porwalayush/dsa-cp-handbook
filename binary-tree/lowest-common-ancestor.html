Lowest Common Ancestor (LCA)

The Lowest Common Ancestor of two nodes p and q in a rooted tree is the deepest node that has both p and q as descendants. By convention a node is considered a descendant of itself, so if p is an ancestor of q, then p itself is the LCA.

Definition & Example

Consider the tree below. The LCA of nodes 4 and 5 is 2, because 2 is the deepest node whose subtree contains both. The LCA of 4 and 6 is the root 1.

                  (1)
                 /   \
              (2)      (3)
             /   \        \
          [4]    [5]      (6)
                 /
              (7)

LCA(4, 5) = 2      both sit under 2, 2 is deepest such node
LCA(4, 7) = 2      7 is a descendant of 2 as well
LCA(4, 6) = 1      paths diverge only at the root
LCA(2, 7) = 2      2 is an ancestor of 7  ->  LCA is 2 itself

Approach 1 — Recursive (General Binary Tree)

This is the classic, most elegant solution and needs no preprocessing. Do a postorder DFS. The recursion returns a non-null node if the subtree contains p or q:

  • If the current node is null, or equals p or q, return it.
  • Recurse into the left and right children.
  • If both sides return non-null, the current node is the split point — it is the LCA.
  • Otherwise propagate up whichever side is non-null.
struct TreeNode {
    int val;
    TreeNode *left, *right;
    TreeNode(int v) : val(v), left(nullptr), right(nullptr) {}
};

// Returns LCA of p and q, assuming both exist in the tree.
TreeNode* lca(TreeNode* root, TreeNode* p, TreeNode* q) {
    if (root == nullptr || root == p || root == q)
        return root;

    TreeNode* left  = lca(root->left,  p, q);
    TreeNode* right = lca(root->right, p, q);

    if (left && right) return root;  // p and q split here
    return left ? left : right;       // both (or none) on one side
}

class TreeNode {
    int val;
    TreeNode left, right;
    TreeNode(int v) { val = v; }
}

// Returns LCA of p and q, assuming both exist in the tree.
TreeNode lca(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q)
        return root;

    TreeNode left  = lca(root.left,  p, q);
    TreeNode right = lca(root.right, p, q);

    if (left != null && right != null) return root;
    return left != null ? left : right;
}

class TreeNode:
    def __init__(self, val):
        self.val = val
        self.left = None
        self.right = None

# Returns LCA of p and q, assuming both exist in the tree.
def lca(root, p, q):
    if root is None or root is p or root is q:
        return root

    left  = lca(root.left,  p, q)
    right = lca(root.right, p, q)

    if left and right:        # p and q split here
        return root
    return left if left else right

Walk-through on the example tree

Computing LCA(4, 5) on the tree from the definition:

lca(1)
 |- lca(2)
 |   |- lca(4) -> matches p (4), returns 4      [left of 2 = 4]
 |   |- lca(5) -> matches q (5), returns 5      [right of 2 = 5]
 |   |- left(4) and right(5) both non-null  =>  return 2
 |
 |- lca(3)
 |   |- lca(null) -> null
 |   |- lca(6) subtree -> null (no p, no q)
 |   |- returns null                            [right of 1 = null]
 |
 |- left(2) non-null, right(null)  =>  propagate up 2

Answer: LCA(4, 5) = 2

Runs in O(n) time and O(h) recursion stack, where h is the tree height.

Approach 2 — Root-to-Node Paths

Find the path from the root to p and from the root to q. Walk both paths from the start; the last node they share before they diverge is the LCA. This is intuitive and also returns the path itself as a by-product.

// Fill `path` with root..target; returns true if target found.
bool findPath(TreeNode* root, TreeNode* target, vector<TreeNode*>& path) {
    if (!root) return false;
    path.push_back(root);
    if (root == target) return true;
    if (findPath(root->left, target, path) ||
        findPath(root->right, target, path)) return true;
    path.pop_back();  // backtrack
    return false;
}

TreeNode* lca(TreeNode* root, TreeNode* p, TreeNode* q) {
    vector<TreeNode*> pathP, pathQ;
    if (!findPath(root, p, pathP) || !findPath(root, q, pathQ))
        return nullptr;
    TreeNode* ans = nullptr;
    for (int i = 0; i < (int)min(pathP.size(), pathQ.size()); i++) {
        if (pathP[i] == pathQ[i]) ans = pathP[i];
        else break;
    }
    return ans;
}

def find_path(root, target, path):
    if root is None:
        return False
    path.append(root)
    if root is target:
        return True
    if find_path(root.left, target, path) or \
       find_path(root.right, target, path):
        return True
    path.pop()  # backtrack
    return False

def lca(root, p, q):
    path_p, path_q = [], []
    if not find_path(root, p, path_p) or \
       not find_path(root, q, path_q):
        return None
    ans = None
    for a, b in zip(path_p, path_q):
        if a is b:
            ans = a
        else:
            break
    return ans

Time O(n) to build the two paths, plus O(h) to compare them. Extra space O(h) for the stored paths.

Approach 3 — Parent Pointers + Ancestor Set

This mirrors the intersection-of-two-linked-lists trick. First record each node's parent (via DFS/BFS). Then walk up from p, collecting all its ancestors into a hash set. Finally walk up from q; the first node already present in the set is the LCA.

TreeNode lca(TreeNode root, TreeNode p, TreeNode q) {
    Map<TreeNode, TreeNode> parent = new HashMap<>();
    Deque<TreeNode> stack = new ArrayDeque<>();
    parent.put(root, null);
    stack.push(root);

    // Record parent of every node until both p and q are seen.
    while (!parent.containsKey(p) || !parent.containsKey(q)) {
        TreeNode node = stack.pop();
        if (node.left != null)  { parent.put(node.left, node);  stack.push(node.left); }
        if (node.right != null) { parent.put(node.right, node); stack.push(node.right); }
    }

    Set<TreeNode> ancestors = new HashSet<>();
    for (TreeNode n = p; n != null; n = parent.get(n))
        ancestors.add(n);
    for (TreeNode n = q; n != null; n = parent.get(n))
        if (ancestors.contains(n)) return n;
    return null;
}

def lca(root, p, q):
    parent = {root: None}
    stack = [root]

    # Record parent of every node until both p and q are seen.
    while p not in parent or q not in parent:
        node = stack.pop()
        if node.left:
            parent[node.left] = node
            stack.append(node.left)
        if node.right:
            parent[node.right] = node
            stack.append(node.right)

    ancestors = set()
    n = p
    while n:
        ancestors.add(n)
        n = parent[n]
    n = q
    while n:
        if n in ancestors:
            return n
        n = parent[n]
    return None

Time O(n), extra space O(n) for the parent map and ancestor set. Handy when the node has a built-in parent pointer (no map needed).

LCA in a Binary Search Tree

A BST gives us extra structure: for any node, everything in the left subtree is smaller and everything in the right subtree is larger. So we can find the LCA by a single walk from the root — no recursion over both children needed:

  • If both p and q are greater than the current node, go right.
  • If both are smaller, go left.
  • Otherwise they fall on different sides (or one equals the node) — this is the split point, the LCA.
BST:               (6)
                  /   \
               (2)     (8)
              /   \    /  \
            (0)  (4) (7)  (9)
                /  \
              (3)  (5)

LCA(2, 8): 2 < 6 and 8 > 6  -> split at 6   => 6
LCA(2, 4): both < 6 -> go left to 2;
           2 == p -> split here            => 2
LCA(3, 5): both < 6 -> left to 2; both > 2 -> right to 4;
           3 < 4 and 5 > 4 -> split at 4    => 4

TreeNode* lcaBST(TreeNode* root, TreeNode* p, TreeNode* q) {
    while (root) {
        if (p->val > root->val && q->val > root->val)
            root = root->right;        // both larger -> go right
        else if (p->val < root->val && q->val < root->val)
            root = root->left;         // both smaller -> go left
        else
            return root;               // split point = LCA
    }
    return nullptr;
}

TreeNode lcaBST(TreeNode root, TreeNode p, TreeNode q) {
    while (root != null) {
        if (p.val > root.val && q.val > root.val)
            root = root.right;         // both larger -> go right
        else if (p.val < root.val && q.val < root.val)
            root = root.left;          // both smaller -> go left
        else
            return root;               // split point = LCA
    }
    return null;
}

def lca_bst(root, p, q):
    while root:
        if p.val > root.val and q.val > root.val:
            root = root.right          # both larger -> go right
        elif p.val < root.val and q.val < root.val:
            root = root.left           # both smaller -> go left
        else:
            return root               # split point = LCA
    return None

Time O(h) (height of the BST) and O(1) extra space — the best you can do for a single query.

Approach 4 — Binary Lifting (Many Queries)

When you must answer many LCA queries on a fixed tree, preprocessing pays off. Binary lifting precomputes, for every node, its 2^k-th ancestor. With this table you can jump up the tree in powers of two and resolve any query in O(log n).

The up-table

Define up[v][k] = the ancestor of v that is 2^k levels above it. The recurrence is the key idea: the 2^k-th ancestor is the 2^(k-1)-th ancestor of the 2^(k-1)-th ancestor.

up[v][0] = parent(v)                          (direct parent)
up[v][k] = up[ up[v][k-1] ][k-1]   for k >= 1

Example chain (LOG = 3, so k = 0,1,2):
  up[v][0] -> 1 step up
  up[v][1] -> 2 steps up   = up[ up[v][0] ][0]
  up[v][2] -> 4 steps up   = up[ up[v][1] ][1]

We also store depth[v] from a DFS.
A "missing" ancestor (above the root) is stored as 0 / a sentinel.

Answering a query

To find LCA(u, v):

  1. Equalize depths: lift the deeper node up until both are at the same depth, jumping by powers of two.
  2. If they are now the same node, that node is the answer (one was an ancestor of the other).
  3. Jump together: from the highest power down, jump both nodes up by 2^k whenever their 2^k-th ancestors differ. After the loop both stop just below the LCA, so the answer is up[u][0].
Step 1 (equalize): bring deeper node to depth of shallower.
Step 3 (jump together):

   k = 2:  up[u][2] != up[v][2] ?  yes -> u,v = up[u][2], up[v][2]
   k = 1:  up[u][1] != up[v][1] ?  no  -> skip
   k = 0:  up[u][0] != up[v][0] ?  yes -> u,v = up[u][0], up[v][0]

   Now u and v are children of the LCA  ->  answer = up[u][0]

const int LOG = 20;          // 2^20 > 1e6 nodes
vector<int> adj[MAXN];
int up[MAXN][LOG];
int depth[MAXN];

// Build up-table and depths via DFS. up[v][0] = parent (0 = sentinel root).
void dfs(int v, int parent) {
    up[v][0] = parent;
    for (int k = 1; k < LOG; k++)
        up[v][k] = up[ up[v][k - 1] ][k - 1];
    for (int child : adj[v]) {
        if (child != parent) {
            depth[child] = depth[v] + 1;
            dfs(child, v);
        }
    }
}

int lca(int u, int v) {
    if (depth[u] < depth[v]) swap(u, v);
    int diff = depth[u] - depth[v];
    // Step 1: lift u up to v's depth
    for (int k = 0; k < LOG; k++)
        if (diff & (1 << k)) u = up[u][k];
    if (u == v) return u;     // one was ancestor of the other
    // Step 3: jump both up while ancestors differ
    for (int k = LOG - 1; k >= 0; k--)
        if (up[u][k] != up[v][k]) {
            u = up[u][k];
            v = up[v][k];
        }
    return up[u][0];
}

// Setup: depth[root] = 0; dfs(root, root);  // root is its own parent sentinel

import sys
sys.setrecursionlimit(300000)

LOG = 20
def build(n, adj, root):
    up = [[0] * LOG for _ in range(n + 1)]
    depth = [0] * (n + 1)

    def dfs(v, parent):
        up[v][0] = parent
        for k in range(1, LOG):
            up[v][k] = up[up[v][k - 1]][k - 1]
        for child in adj[v]:
            if child != parent:
                depth[child] = depth[v] + 1
                dfs(child, v)

    dfs(root, root)        # root is its own parent sentinel
    return up, depth

def lca(u, v, up, depth):
    if depth[u] < depth[v]:
        u, v = v, u
    diff = depth[u] - depth[v]
    for k in range(LOG):          # lift u to v's depth
        if diff >> k & 1:
            u = up[u][k]
    if u == v:
        return u
    for k in range(LOG - 1, -1, -1):  # jump together
        if up[u][k] != up[v][k]:
            u, v = up[u][k], up[v][k]
    return up[u][0]

Complexity: O(n log n) preprocessing time and space for the up-table, then O(log n) per query.

Even faster: Euler tour + sparse table.
  • Record an Euler tour (the sequence of nodes visited during DFS) along with each node's depth, and the first index at which each node appears.
  • LCA of u and v becomes the node of minimum depth in the Euler array between their first occurrences — a Range Minimum Query (RMQ).
  • Build a sparse table over the depths in O(n log n), after which each RMQ — and thus each LCA — is answered in O(1).

Which one should I use?
  • One-off query on a general binary tree — use the recursive DFS (Approach 1). It is the shortest correct code and needs no setup.
  • Repeated queries on a fixed tree — preprocess with binary lifting for O(log n) per query, or Euler tour + sparse table for O(1) per query.
  • The tree is a BST — use the walk-down trick: O(h) time, O(1) space, and trivially simple.
