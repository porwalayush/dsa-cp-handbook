More Tree Operations

This page collects a few tree operations that reward a little cleverness. Some let you do better than the obvious O(n) by exploiting structure; others turn a tree into a tiny calculator or wire up shortcuts between nodes. Each one is still, at heart, a recursion that visits nodes and stitches answers together — we just pick smarter about which nodes we bother visiting.

All examples assume a standard node definition:

struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

class TreeNode {
    int val;
    TreeNode left, right;
    TreeNode(int x) { val = x; }
}

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

1. Count Nodes in a Complete Binary Tree — O(log² n)

Imagine you have a perfectly packed auditorium where every row fills up completely before the next one begins, and the last row fills strictly from the left. That's a complete binary tree. You could count heads one by one (O(n)), but because the packing is so regular, you can shortcut huge chunks: any time a section is full, you already know its head count from a formula — no need to walk it.

The key fact: a perfect subtree of height h has exactly 2^(h+1) - 1 nodes (counting the root row as height 0 gives a row count of h+1). So at each node we measure two cheap things:

  • The left-spine height — walk left children until you hit null, counting steps.
  • The right-spine height — walk right children until you hit null, counting steps.

If the two spine heights are equal, the subtree is perfect: return 2^h - 1 (where h is the number of levels) instantly, no recursion. If they differ, the bottom row is only partially filled, so we recurse into both children — but only one side will ever be the "ragged" one, so the recursion has depth O(log n), and each level does an O(log n) spine measurement. That multiplies out to O(log² n).

At each node, measure left spine vs right spine:

            1
          /   \
        2       3
       / \     / \
      4   5   6   7      left spine of root = 3 levels
     / \                 right spine of root = 3 levels?
    8   9                 NO -> ragged bottom row -> recurse

    For node 2: left spine (2,4,8)=3, right spine (2,5)=2  -> recurse
    For node 3: left spine (3,6)=2, right spine (3,7)=2    -> PERFECT!
                so subtree(3) = 2^2 - 1 = 3 nodes, skipped entirely.

Contrast with the naive count. The obvious recursion — 1 + count(left) + count(right) — touches every node, so it is O(n). For a complete tree of a million nodes that's a million visits; the O(log² n) method does roughly 20 × 20 = 400. The savings come entirely from recognizing perfect subtrees by their matching spines and never descending into them.

The widget tracks two numbers: nodes counted by formula (perfect subtrees we skipped into with a single arithmetic step) and nodes we had to recurse through. On a tree this small the difference is modest, but the same logic on a tree of height 20 turns ~1,000,000 visits into a few hundred.

int leftHeight(TreeNode* n) {
    int h = 0;
    while (n) { h++; n = n->left; }   // walk down left spine
    return h;
}
int rightHeight(TreeNode* n) {
    int h = 0;
    while (n) { h++; n = n->right; }  // walk down right spine
    return h;
}

int countNodes(TreeNode* root) {
    if (!root) return 0;
    int lh = leftHeight(root), rh = rightHeight(root);
    if (lh == rh) return (1 << lh) - 1;        // perfect: 2^lh - 1
    return 1 + countNodes(root->left)
             + countNodes(root->right);          // ragged: recurse
}

int leftHeight(TreeNode n) {
    int h = 0;
    while (n != null) { h++; n = n.left; }   // left spine
    return h;
}
int rightHeight(TreeNode n) {
    int h = 0;
    while (n != null) { h++; n = n.right; }  // right spine
    return h;
}

int countNodes(TreeNode root) {
    if (root == null) return 0;
    int lh = leftHeight(root), rh = rightHeight(root);
    if (lh == rh) return (1 << lh) - 1;        // perfect: 2^lh - 1
    return 1 + countNodes(root.left)
             + countNodes(root.right);            // ragged: recurse
}

def leftHeight(n):
    h = 0
    while n:                # walk down left spine
        h += 1; n = n.left
    return h

def rightHeight(n):
    h = 0
    while n:                # walk down right spine
        h += 1; n = n.right
    return h

def countNodes(root):
    if not root:
        return 0
    lh, rh = leftHeight(root), rightHeight(root)
    if lh == rh:
        return (1 << lh) - 1           # perfect: 2^lh - 1
    return 1 + countNodes(root.left) + countNodes(root.right)
Why O(log² n)?
  • The recursion only descends along the single "ragged" path from root to the last leaf — that's O(log n) levels deep.
  • At each of those levels we measure a spine, costing O(log n).
  • Every node we don't descend into is part of a perfect subtree, counted by the 2^h - 1 formula in O(1). Multiplying the two log factors gives O(log² n).
Only valid for complete trees. The formula shortcut relies on the completeness guarantee (rows fill top-to-bottom, left-to-right). On an arbitrary binary tree, matching spine heights do not imply a perfect subtree, so you must fall back to the plain O(n) count.

2. Expression Trees — Build and Evaluate

An expression tree turns arithmetic into a tree you can compute by walking. The rule is simple: operators (+ - * /) sit at the internal nodes, and operands (the numbers) are the leaves. To get the answer you evaluate bottom-up — a leaf evaluates to its own number, and an operator node evaluates by combining the already-computed values of its two children.

Infix:    (3 + 4) * 5
Postfix:  3 4 + 5 *

            (*)            evaluate bottom-up:
           /   \             (+) -> 3 + 4 = 7
         (+)    5            (*) -> 7 * 5 = 35
         / \
        3   4              result = 35

Building from postfix

Postfix (Reverse Polish) is the easiest to build from because it needs no precedence rules — just a stack. Scan left to right: push a leaf node for each operand; when you hit an operator, pop the top two nodes as its right and left children, then push the new operator node back. The last node left on the stack is the root.

Evaluating

Evaluation is a textbook postorder recursion: compute the left child's value, compute the right child's value, then apply this node's operator. Leaves (no children) just return their number.

// Node holds a token: an operator char or a number string.
struct ExprNode {
    std::string tok;
    ExprNode *left = nullptr, *right = nullptr;
    ExprNode(std::string t) : tok(t) {}
};

bool isOp(const std::string& t) {
    return t == "+" || t == "-" || t == "*" || t == "/";
}

// Build from postfix tokens using a stack.
ExprNode* buildFromPostfix(const std::vector<std::string>& toks) {
    std::vector<ExprNode*> st;
    for (auto& t : toks) {
        ExprNode* n = new ExprNode(t);
        if (isOp(t)) {
            n->right = st.back(); st.pop_back();  // right popped first
            n->left  = st.back(); st.pop_back();
        }
        st.push_back(n);
    }
    return st.back();
}

// Evaluate bottom-up (postorder).
double evaluate(ExprNode* n) {
    if (!n->left && !n->right) return std::stod(n->tok);  // leaf = number
    double a = evaluate(n->left), b = evaluate(n->right);
    if (n->tok == "+") return a + b;
    if (n->tok == "-") return a - b;
    if (n->tok == "*") return a * b;
    return a / b;
}

// Node holds a token: an operator char or a number string.
class ExprNode {
    String tok;
    ExprNode left, right;
    ExprNode(String t) { tok = t; }
}

boolean isOp(String t) {
    return t.equals("+") || t.equals("-")
        || t.equals("*") || t.equals("/");
}

// Build from postfix tokens using a stack.
ExprNode buildFromPostfix(String[] toks) {
    Deque<ExprNode> st = new ArrayDeque<>();
    for (String t : toks) {
        ExprNode n = new ExprNode(t);
        if (isOp(t)) {
            n.right = st.pop();   // right popped first
            n.left  = st.pop();
        }
        st.push(n);
    }
    return st.pop();
}

// Evaluate bottom-up (postorder).
double evaluate(ExprNode n) {
    if (n.left == null && n.right == null)
        return Double.parseDouble(n.tok);   // leaf = number
    double a = evaluate(n.left), b = evaluate(n.right);
    switch (n.tok) {
        case "+": return a + b;
        case "-": return a - b;
        case "*": return a * b;
        default: return a / b;
    }
}

class ExprNode:
    def __init__(self, tok):
        self.tok = tok
        self.left = self.right = None

OPS = {"+", "-", "*", "/"}

# Build from postfix tokens using a stack.
def build_from_postfix(toks):
    st = []
    for t in toks:
        n = ExprNode(t)
        if t in OPS:
            n.right = st.pop()   # right popped first
            n.left  = st.pop()
        st.append(n)
    return st[-1]

# Evaluate bottom-up (postorder).
def evaluate(n):
    if not n.left and not n.right:
        return float(n.tok)          # leaf = number
    a, b = evaluate(n.left), evaluate(n.right)
    if n.tok == "+": return a + b
    if n.tok == "-": return a - b
    if n.tok == "*": return a * b
    return a / b
Prefix builds the same way, scanned in reverse. To build from prefix (Polish) notation, scan the tokens right-to-left and, on an operator, pop the left child first then the right. The evaluation step is identical — the tree shape is what carries the precedence, so once it's built the notation it came from no longer matters.

This is the expression-tree evaluation promised in the basics page. Both building and evaluating are O(n) in the number of tokens, since each token is touched a constant number of times.

3. Populate Next-Right Pointers

Picture each level of the tree as a row of people, and you want every person to know who is standing immediately to their right in the same row (and the rightmost person knows there's no one — null). That's the next-right pointer problem: connect each node to its neighbor on the same level using an extra next field.

        1 -> null
       / \
      2 -> 3 -> null
     / \   / \
    4 ->5->6->7 -> null

  Each node's "next" points to the node on its right
  in the same level; last node in a level points to null.

The elegant trick: once a level is fully linked, you can traverse that level left-to-right using the next pointers you just set, and use it to wire up the level below — like using a finished row as scaffolding to build the next. For each node on the current level you connect its own two children, then bridge to the next node's children. This needs O(1) extra space (no queue), because the established level acts as the "queue" for the next.

// Node has an extra `next` pointer, default nullptr.
struct Node {
    int val;
    Node *left = nullptr, *right = nullptr, *next = nullptr;
};

// Works for any binary tree, O(1) extra space.
Node* connect(Node* root) {
    Node* leftmost = root;
    while (leftmost) {
        Node* curr = leftmost;       // walk the current level via next
        Node* dummy = new Node();   // head of the level below
        Node* tail = dummy;
        while (curr) {
            if (curr->left)  { tail->next = curr->left;  tail = tail->next; }
            if (curr->right) { tail->next = curr->right; tail = tail->next; }
            curr = curr->next;
        }
        leftmost = dummy->next;       // descend to next level
        delete dummy;
    }
    return root;
}

// Node has an extra `next` pointer, default null.
class Node {
    int val;
    Node left, right, next;
}

// Works for any binary tree, O(1) extra space.
Node connect(Node root) {
    Node leftmost = root;
    while (leftmost != null) {
        Node curr = leftmost;        // walk current level via next
        Node dummy = new Node();    // head of the level below
        Node tail = dummy;
        while (curr != null) {
            if (curr.left  != null) { tail.next = curr.left;  tail = tail.next; }
            if (curr.right != null) { tail.next = curr.right; tail = tail.next; }
            curr = curr.next;
        }
        leftmost = dummy.next;        // descend to next level
    }
    return root;
}

class Node:
    def __init__(self, val=0):
        self.val = val
        self.left = self.right = self.next = None

# Works for any binary tree, O(1) extra space.
def connect(root):
    leftmost = root
    while leftmost:
        curr = leftmost              # walk current level via next
        dummy = Node()               # head of the level below
        tail = dummy
        while curr:
            if curr.left:
                tail.next = curr.left;  tail = tail.next
            if curr.right:
                tail.next = curr.right; tail = tail.next
            curr = curr.next
        leftmost = dummy.next         # descend to next level
    return root
The simpler BFS alternative: a plain level-order traversal with a queue connects each node to the one dequeued after it (within the same level). That's just as correct and easier to remember, but it uses O(width) extra space for the queue. The scaffolding method above is the prized O(1)-space version interviewers like to see.

This is distinct from mirroring a tree (covered as Invert / Mirror in Structural Operations) — here we're adding sideways links, not rearranging children.

4. Bonus: Sum and Maximum of All Nodes

The friendliest recursion on the page: to total every value, a node's answer is simply its own value plus the totals of its two subtrees. Swap + for max and the same skeleton finds the largest value instead. Both are plain O(n) postorder walks — a gentle palate cleanser after the trickier ones above.

int sumNodes(TreeNode* root) {
    if (!root) return 0;
    return root->val + sumNodes(root->left) + sumNodes(root->right);
}

int maxNode(TreeNode* root) {
    if (!root) return INT_MIN;
    return std::max({root->val, maxNode(root->left), maxNode(root->right)});
}

int sumNodes(TreeNode root) {
    if (root == null) return 0;
    return root.val + sumNodes(root.left) + sumNodes(root.right);
}

int maxNode(TreeNode root) {
    if (root == null) return Integer.MIN_VALUE;
    return Math.max(root.val,
        Math.max(maxNode(root.left), maxNode(root.right)));
}

def sum_nodes(root):
    if not root:
        return 0
    return root.val + sum_nodes(root.left) + sum_nodes(root.right)

def max_node(root):
    if not root:
        return float("-inf")
    return max(root.val, max_node(root.left), max_node(root.right))

Here n is the number of nodes and h is the tree height (O(log n) when balanced, O(n) when skewed).

The takeaways:
  • Structure buys speed. The complete-tree count beats O(n) only because completeness lets matching spine heights certify a perfect subtree — recognize the guarantee, then exploit it.
  • Trees can compute. An expression tree makes operator precedence structural; a postorder walk turns it into a calculator.
  • A finished level is free scaffolding. Next-right pointers let one level build the next with no extra queue, the classic route to O(1) space.
  • When in doubt, reach for the plain node-once recursion — sum and max show the baseline pattern the cleverer tricks are optimizing away from.
