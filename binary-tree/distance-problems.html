Distance Problems in a Tree

Here is the one mental shift that unlocks this entire family of problems: a binary tree is normally a one-way street — you can walk from a parent down to its children, but never back up. Most "distance" questions, though, need to travel in any direction. The fix is to stop thinking of the tree as parent-points-to-child and start thinking of it as a plain undirected graph where every edge can be walked both ways.

Once a node can reach its left child, its right child, and its parent, distance becomes simple: it is just the number of edges on the shortest walk between two nodes. And the cleanest tool for "shortest number of steps" in an unweighted graph is breadth-first search (BFS) — it explores outward in expanding rings, one ring per unit of distance.

This page builds up that idea step by step: first the shared prerequisite (a parent map), then four classic problems that all reuse it — nodes at distance K, burning the tree, cousins, and the distance between two nodes.

1. The Shared Prerequisite: a Parent Map

Think of every node as knowing only who its kids are — like a manager who knows their direct reports but, oddly, has forgotten who their own boss is. To walk "upward" we first have to hand each node a sticky note saying "your parent is X". That collection of sticky notes is the parent map.

We build it with one pass of BFS or DFS from the root: whenever we step from a node into a child, we record parent[child] = node. After this single pass, any node can move in three directions — left, right, and parent — which is exactly what turns the tree into an undirected graph.

            (1)                  parent map (child -> parent)
           /   \                 -------------------------
        (2)     (3)              2 -> 1     3 -> 1
        / \       \              4 -> 2     5 -> 2     6 -> 3
     (4)  (5)     (6)            7 -> 5     8 -> 5     9 -> 8
         /  \                    10 -> 6
      (7)   (8)
            /
         (9)

struct TreeNode {
    int val;
    TreeNode *left, *right;
    TreeNode(int v) : val(v), left(nullptr), right(nullptr) {}
};

// Map each node to its parent via BFS. root maps to nullptr.
unordered_map<TreeNode*, TreeNode*> buildParents(TreeNode* root) {
    unordered_map<TreeNode*, TreeNode*> parent;
    queue<TreeNode*> q;
    parent[root] = nullptr;
    q.push(root);
    while (!q.empty()) {
        TreeNode* node = q.front(); q.pop();
        if (node->left)  { parent[node->left]  = node; q.push(node->left);  }
        if (node->right) { parent[node->right] = node; q.push(node->right); }
    }
    return parent;  // now every node can reach left, right, AND parent
}

class TreeNode {
    int val;
    TreeNode left, right;
    TreeNode(int v) { val = v; }
}

// Map each node to its parent via BFS. root maps to null.
Map<TreeNode, TreeNode> buildParents(TreeNode root) {
    Map<TreeNode, TreeNode> parent = new HashMap<>();
    Queue<TreeNode> q = new LinkedList<>();
    parent.put(root, null);
    q.add(root);
    while (!q.isEmpty()) {
        TreeNode node = q.poll();
        if (node.left != null)  { parent.put(node.left, node);  q.add(node.left);  }
        if (node.right != null) { parent.put(node.right, node); q.add(node.right); }
    }
    return parent;  // now every node can reach left, right, AND parent
}

from collections import deque

class TreeNode:
    def __init__(self, val):
        self.val = val
        self.left = None
        self.right = None

# Map each node to its parent via BFS. root maps to None.
def build_parents(root):
    parent = {root: None}
    q = deque([root])
    while q:
        node = q.popleft()
        for child in (node.left, node.right):
            if child:
                parent[child] = node
                q.append(child)
    return parent  # now every node can reach left, right, AND parent

Building the map is O(n) time and O(n) space. Every problem below assumes you have run this once.

Why this matters. A downward-only tree can answer "how far down is X?" but not "what is K steps away in any direction?". The parent map is the bridge that turns a rooted tree into a navigable undirected graph — the single trick behind problems 2 and 3.

2. All Nodes at Distance K from a Target

Picture dropping a pebble into a pond at the target node. Ripples spread outward in rings: ring 0 is the target itself, ring 1 is its immediate neighbours, ring 2 is their neighbours, and so on. The nodes sitting on ring K are exactly the answer.

Because the tree is now undirected (thanks to the parent map), each node has up to three neighbours: left, right, and parent. We run a BFS from the target, tracking distance level by level, and collect everyone whose distance equals K. A visited set stops us from bouncing back the way we came.

// All node values exactly K edges from `target`. Uses buildParents() above.
vector<int> distanceK(TreeNode* root, TreeNode* target, int K) {
    auto parent = buildParents(root);
    unordered_set<TreeNode*> visited;
    queue<TreeNode*> q;
    q.push(target);
    visited.insert(target);
    int dist = 0;

    while (!q.empty()) {
        if (dist == K) {                 // this whole ring is distance K
            vector<int> res;
            while (!q.empty()) { res.push_back(q.front()->val); q.pop(); }
            return res;
        }
        int sz = q.size();
        for (int i = 0; i < sz; i++) {
            TreeNode* node = q.front(); q.pop();
            for (TreeNode* nb : {node->left, node->right, parent[node]}) {
                if (nb && !visited.count(nb)) { visited.insert(nb); q.push(nb); }
            }
        }
        dist++;
    }
    return {};  // K larger than tree radius
}

// All node values exactly K edges from `target`. Uses buildParents() above.
List<Integer> distanceK(TreeNode root, TreeNode target, int K) {
    Map<TreeNode, TreeNode> parent = buildParents(root);
    Set<TreeNode> visited = new HashSet<>();
    Queue<TreeNode> q = new LinkedList<>();
    q.add(target); visited.add(target);
    int dist = 0;

    while (!q.isEmpty()) {
        if (dist == K) {                 // this whole ring is distance K
            List<Integer> res = new ArrayList<>();
            for (TreeNode n : q) res.add(n.val);
            return res;
        }
        int sz = q.size();
        for (int i = 0; i < sz; i++) {
            TreeNode node = q.poll();
            for (TreeNode nb : new TreeNode[]{node.left, node.right, parent.get(node)}) {
                if (nb != null && visited.add(nb)) q.add(nb);
            }
        }
        dist++;
    }
    return new ArrayList<>();  // K larger than tree radius
}

from collections import deque

# All node values exactly K edges from `target`. Uses build_parents() above.
def distance_k(root, target, K):
    parent = build_parents(root)
    visited = {target}
    q = deque([target])
    dist = 0

    while q:
        if dist == K:                    # this whole ring is distance K
            return [n.val for n in q]
        for _ in range(len(q)):
            node = q.popleft()
            for nb in (node.left, node.right, parent[node]):
                if nb and nb not in visited:
                    visited.add(nb)
                    q.append(nb)
        dist += 1
    return []  # K larger than tree radius

Time O(n) (each node enqueued once), extra space O(n) for the parent map, visited set, and queue.

3. Burning Tree / Minimum Time to Burn the Whole Tree

This is the same ripple as problem 2, just without stopping at ring K. Set the target node on fire at minute 0. Every minute, the fire jumps from each burning node to all of its unburnt neighbours — left, right, and parent. The whole tree is ash the moment the farthest node finally catches, so the answer is simply the maximum distance from the target to any node.

So we BFS outward exactly as before, but instead of returning ring K we keep counting rings until the queue empties. The number of full rings we spread is the burning time.

Fire starts at node 5:

  minute 0:  {5}                   <- ignition
  minute 1:  {2, 7, 8}            neighbours of 5 (parent 2, kids 7, 8)
  minute 2:  {1, 4, 9}           neighbours of the above, still unburnt
  minute 3:  {3}                 1's other child
  minute 4:  {6}                 3's child
  minute 5:  {10}                last to catch  ->  total time = 5

// Minutes to burn the whole tree starting from `start`.
int burnTime(TreeNode* root, TreeNode* start) {
    auto parent = buildParents(root);
    unordered_set<TreeNode*> visited{start};
    queue<TreeNode*> q;
    q.push(start);
    int time = -1;  // the last ring adds no new minute

    while (!q.empty()) {
        int sz = q.size();
        for (int i = 0; i < sz; i++) {
            TreeNode* node = q.front(); q.pop();
            for (TreeNode* nb : {node->left, node->right, parent[node]})
                if (nb && !visited.count(nb)) { visited.insert(nb); q.push(nb); }
        }
        time++;  // one more ring caught fire
    }
    return time;  // == max distance from start to any node
}

// Minutes to burn the whole tree starting from `start`.
int burnTime(TreeNode root, TreeNode start) {
    Map<TreeNode, TreeNode> parent = buildParents(root);
    Set<TreeNode> visited = new HashSet<>();
    visited.add(start);
    Queue<TreeNode> q = new LinkedList<>();
    q.add(start);
    int time = -1;  // the last ring adds no new minute

    while (!q.isEmpty()) {
        int sz = q.size();
        for (int i = 0; i < sz; i++) {
            TreeNode node = q.poll();
            for (TreeNode nb : new TreeNode[]{node.left, node.right, parent.get(node)})
                if (nb != null && visited.add(nb)) q.add(nb);
        }
        time++;  // one more ring caught fire
    }
    return time;  // == max distance from start to any node
}

from collections import deque

# Minutes to burn the whole tree starting from `start`.
def burn_time(root, start):
    parent = build_parents(root)
    visited = {start}
    q = deque([start])
    time = -1  # the last ring adds no new minute

    while q:
        for _ in range(len(q)):
            node = q.popleft()
            for nb in (node.left, node.right, parent[node]):
                if nb and nb not in visited:
                    visited.add(nb)
                    q.append(nb)
        time += 1  # one more ring caught fire
    return time  # == max distance from start to any node

Time O(n), extra space O(n). Notice this is problem 2's BFS with the "stop at K" check removed and a ring counter added — one algorithm, two questions.

The unifying idea. "Nodes at distance K" and "time to burn the tree" are the same multi-source-free BFS from one node. Distance-K returns one specific ring; burning returns the index of the last ring. Master the ring-by-ring BFS once and you have both.

4. Cousins of a Node

In a family tree, two people are cousins when they are in the same generation but have different parents. Same idea here: nodes x and y are cousins when they sit at the same depth (same level) but do not share a parent. Siblings (same parent) are explicitly not cousins.

No undirected graph needed here — a single BFS or DFS that records each node's depth and parent answers it. Find both nodes, then check: equal depth AND different parent.

// True if values x and y are cousins: same depth, different parent.
bool isCousins(TreeNode* root, int x, int y) {
    queue<pair<TreeNode*, TreeNode*>> q;  // (node, its parent)
    q.push({root, nullptr});
    while (!q.empty()) {
        int sz = q.size();
        TreeNode *px = nullptr, *py = nullptr;
        for (int i = 0; i < sz; i++) {     // one full level
            auto [node, par] = q.front(); q.pop();
            if (node->val == x) px = par;
            if (node->val == y) py = par;
            if (node->left)  q.push({node->left, node});
            if (node->right) q.push({node->right, node});
        }
        if (px || py)  // at least one found on this level
            return px && py && px != py;  // both here, different parents
    }
    return false;
}

// True if values x and y are cousins: same depth, different parent.
boolean isCousins(TreeNode root, int x, int y) {
    Queue<TreeNode[]> q = new LinkedList<>();  // {node, parent}
    q.add(new TreeNode[]{root, null});
    while (!q.isEmpty()) {
        int sz = q.size();
        TreeNode px = null, py = null;
        for (int i = 0; i < sz; i++) {      // one full level
            TreeNode[] cur = q.poll();
            TreeNode node = cur[0], par = cur[1];
            if (node.val == x) px = par;
            if (node.val == y) py = par;
            if (node.left != null)  q.add(new TreeNode[]{node.left, node});
            if (node.right != null) q.add(new TreeNode[]{node.right, node});
        }
        if (px != null || py != null)
            return px != null && py != null && px != py;
    }
    return false;
}

from collections import deque

# True if values x and y are cousins: same depth, different parent.
def is_cousins(root, x, y):
    q = deque([(root, None)])  # (node, parent)
    while q:
        px = py = None
        for _ in range(len(q)):      # one full level
            node, par = q.popleft()
            if node.val == x: px = par
            if node.val == y: py = par
            if node.left:  q.append((node.left, node))
            if node.right: q.append((node.right, node))
        if px or py:                  # at least one found this level
            return px is not None and py is not None and px is not py
    return False

Time O(n), extra space O(w) where w is the maximum width of the tree (the largest level the queue holds).

Watch the edge cases. Two values that turn out to be siblings (same parent) are not cousins — the px != py check is doing the real work. If we find one value on a level but not the other, they are at different depths, so the answer is immediately false.

5. Distance Between Two Nodes

The shortest path between two nodes in a tree always bends at exactly one point: their Lowest Common Ancestor (LCA). Go up from a to the LCA, then down to b — there is no shortcut, because a tree has no cycles. So the distance is the two leg lengths added together:

dist(a, b) = depth(a) + depth(b) - 2 * depth(LCA(a, b))

         (depth measured from the root, root = depth 0)

  why "- 2 * depth(LCA)"?  the path from the root down to the LCA
  is counted once inside depth(a) and once inside depth(b), so we
  subtract that shared prefix twice to leave just the two legs.

So the recipe is: find the LCA (see the Lowest Common Ancestor page for the full set of techniques), then combine three depths. We compute depth as the number of edges from the root down to a node.

// Standard LCA for a general binary tree (see the LCA page).
TreeNode* lca(TreeNode* root, TreeNode* a, TreeNode* b) {
    if (!root || root == a || root == b) return root;
    TreeNode* L = lca(root->left,  a, b);
    TreeNode* R = lca(root->right, a, b);
    if (L && R) return root;
    return L ? L : R;
}

// Edges from root down to `target` (-1 if absent).
int depthOf(TreeNode* root, TreeNode* target, int d = 0) {
    if (!root) return -1;
    if (root == target) return d;
    int left = depthOf(root->left, target, d + 1);
    return left != -1 ? left : depthOf(root->right, target, d + 1);
}

int distance(TreeNode* root, TreeNode* a, TreeNode* b) {
    TreeNode* anc = lca(root, a, b);
    return depthOf(root, a) + depthOf(root, b) - 2 * depthOf(root, anc);
}

// Standard LCA for a general binary tree (see the LCA page).
TreeNode lca(TreeNode root, TreeNode a, TreeNode b) {
    if (root == null || root == a || root == b) return root;
    TreeNode L = lca(root.left,  a, b);
    TreeNode R = lca(root.right, a, b);
    if (L != null && R != null) return root;
    return L != null ? L : R;
}

// Edges from root down to `target` (-1 if absent).
int depthOf(TreeNode root, TreeNode target, int d) {
    if (root == null) return -1;
    if (root == target) return d;
    int left = depthOf(root.left, target, d + 1);
    return left != -1 ? left : depthOf(root.right, target, d + 1);
}

int distance(TreeNode root, TreeNode a, TreeNode b) {
    TreeNode anc = lca(root, a, b);
    return depthOf(root, a, 0) + depthOf(root, b, 0) - 2 * depthOf(root, anc, 0);
}

# Standard LCA for a general binary tree (see the LCA page).
def lca(root, a, b):
    if root is None or root is a or root is b:
        return root
    L = lca(root.left,  a, b)
    R = lca(root.right, a, b)
    if L and R:
        return root
    return L if L else R

# Edges from root down to `target` (-1 if absent).
def depth_of(root, target, d=0):
    if root is None:
        return -1
    if root is target:
        return d
    left = depth_of(root.left, target, d + 1)
    return left if left != -1 else depth_of(root.right, target, d + 1)

def distance(root, a, b):
    anc = lca(root, a, b)
    return depth_of(root, a) + depth_of(root, b) - 2 * depth_of(root, anc)

Each LCA and depth call is O(n), so this is O(n) time and O(h) recursion space. For many repeated queries, precompute depths and use binary-lifting LCA (covered on the LCA page) to drop each query to O(log n).

Alternative: you can also get the distance directly from the parent-map BFS of problem 2 — run BFS from a and read off the distance recorded when you reach b. The LCA formula is preferred when you already have depths or need many queries.

The takeaway. Four of these five problems collapse into the same move: build the parent map so the tree becomes undirected, then BFS outward in rings. Distance-K stops at a ring, burning counts all rings, and node-to-node distance can use the same BFS or the LCA depth formula. Learn the ripple, and the rest are variations.
