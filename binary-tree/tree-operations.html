Structural Operations on Binary Trees

This guide covers the classic structural operations on binary trees: inverting, comparing, flattening, merging, and counting. Nearly all of them share one shape — a recursion that visits each node once, does O(1) work, and combines the answers from its children. Every operation here runs in O(n) time.

All examples assume a standard node definition:

struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

class TreeNode {
    int val;
    TreeNode left, right;
    TreeNode(int x) { val = x; }
}

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

1. Invert / Mirror a Tree

Inverting a binary tree means swapping the left and right child of every node. The recursion is trivial: swap the children of the current node, then recurse into both subtrees.

        BEFORE                         AFTER (mirrored)

           4                              4
         /   \                          /   \
        2     7                        7     2
       / \   / \                      / \   / \
      1   3 6   9                    9   6 3   1

Every left/right pair is swapped, top to bottom.

// Swap children at every node, return new root
TreeNode* invertTree(TreeNode* root) {
    if (!root) return nullptr;
    TreeNode* tmp = root->left;
    root->left  = invertTree(root->right);
    root->right = invertTree(tmp);
    return root;
}

// Swap children at every node, return new root
TreeNode invertTree(TreeNode root) {
    if (root == null) return null;
    TreeNode tmp = root.left;
    root.left  = invertTree(root.right);
    root.right = invertTree(tmp);
    return root;
}

# Swap children at every node, return new root
def invertTree(root):
    if not root:
        return None
    root.left, root.right = invertTree(root.right), invertTree(root.left)
    return root

2. Check if Two Trees Are Identical

Two trees are identical when they have the same structure and the same node values. Walk both trees in lockstep: at each step, both nodes must be null together, or both non-null with equal values and identical subtrees.

bool isSameTree(TreeNode* p, TreeNode* q) {
    if (!p && !q) return true;        // both empty
    if (!p || !q) return false;       // only one empty
    return p->val == q->val
        && isSameTree(p->left,  q->left)
        && isSameTree(p->right, q->right);
}

boolean isSameTree(TreeNode p, TreeNode q) {
    if (p == null && q == null) return true;   // both empty
    if (p == null || q == null) return false;  // only one empty
    return p.val == q.val
        && isSameTree(p.left,  q.left)
        && isSameTree(p.right, q.right);
}

def isSameTree(p, q):
    if not p and not q:
        return True           # both empty
    if not p or not q:
        return False          # only one empty
    return (p.val == q.val
            and isSameTree(p.left,  q.left)
            and isSameTree(p.right, q.right))

3. Check if a Tree Is Symmetric

A tree is symmetric if its left subtree is a mirror image of its right subtree. This is a paired recursion with two pointers walking outward in opposite directions: compare left.left against right.right (the outer pair) and left.right against right.left (the inner pair).

    SYMMETRIC                     NOT SYMMETRIC

          1                            1
        /   \                        /   \
       2     2                      2     2
      / \   / \                      \     \
     3   4 4   3                       3     3

  left.left (3) == right.right (3)   values mirror, but
  left.right(4) == right.left (4)    structure does NOT:
  --> symmetric                      left has no left child
                                     while right does --> false

bool mirror(TreeNode* a, TreeNode* b) {
    if (!a && !b) return true;
    if (!a || !b) return false;
    return a->val == b->val
        && mirror(a->left,  b->right)   // outer pair
        && mirror(a->right, b->left);   // inner pair
}

bool isSymmetric(TreeNode* root) {
    return !root || mirror(root->left, root->right);
}

boolean mirror(TreeNode a, TreeNode b) {
    if (a == null && b == null) return true;
    if (a == null || b == null) return false;
    return a.val == b.val
        && mirror(a.left,  b.right)   // outer pair
        && mirror(a.right, b.left);   // inner pair
}

boolean isSymmetric(TreeNode root) {
    return root == null || mirror(root.left, root.right);
}

def isSymmetric(root):
    def mirror(a, b):
        if not a and not b:
            return True
        if not a or not b:
            return False
        return (a.val == b.val
                and mirror(a.left,  b.right)   # outer pair
                and mirror(a.right, b.left))   # inner pair
    return not root or mirror(root.left, root.right)

4. Subtree Check

Given trees s and t, decide whether t is a subtree of s — i.e. some node in s roots a tree identical to t. The straightforward approach reuses isSameTree: at every node of s, test whether the subtree rooted there equals t. This costs O(n × m) in the worst case (n, m = sizes of s, t).

bool isSameTree(TreeNode* p, TreeNode* q) {
    if (!p && !q) return true;
    if (!p || !q) return false;
    return p->val == q->val
        && isSameTree(p->left, q->left)
        && isSameTree(p->right, q->right);
}

bool isSubtree(TreeNode* s, TreeNode* t) {
    if (!s) return !t;                 // empty s matches only empty t
    if (isSameTree(s, t)) return true;
    return isSubtree(s->left, t) || isSubtree(s->right, t);
}

boolean isSameTree(TreeNode p, TreeNode q) {
    if (p == null && q == null) return true;
    if (p == null || q == null) return false;
    return p.val == q.val
        && isSameTree(p.left, q.left)
        && isSameTree(p.right, q.right);
}

boolean isSubtree(TreeNode s, TreeNode t) {
    if (s == null) return t == null;
    if (isSameTree(s, t)) return true;
    return isSubtree(s.left, t) || isSubtree(s.right, t);
}

def isSameTree(p, q):
    if not p and not q:
        return True
    if not p or not q:
        return False
    return (p.val == q.val
            and isSameTree(p.left, q.left)
            and isSameTree(p.right, q.right))

def isSubtree(s, t):
    if not s:
        return not t
    if isSameTree(s, t):
        return True
    return isSubtree(s.left, t) or isSubtree(s.right, t)
The O(n + m) serialization trick:
  • Serialize each tree into a string using a preorder traversal that includes explicit null markers (e.g. #) and a leading delimiter before each value (e.g. ^) so values like 1 and 12 cannot run together.
  • t is a subtree of s iff serialize(t) is a substring of serialize(s).
  • With KMP or Z-algorithm for the substring search, the whole check runs in O(n + m) instead of O(n × m). The delimiters are what make substring matching equivalent to subtree matching.

5. Flatten Binary Tree to a Linked List

Flatten the tree in place so that it becomes a right-skewed "linked list" following preorder order: each node's left is set to null and its right points to the next node in preorder.

        BEFORE                    AFTER (preorder via right pointers)

           1                       1
          / \                       \
         2   5                       2
        / \   \                       \
       3   4   6                       3
                                        \
   preorder: 1 2 3 4 5 6                 4
                                          \
                                           5
                                            \
                                             6

Approach A — Reverse-preorder recursion, O(n) time / O(h) stack

Process nodes in reverse preorder (right, then left, then current) while keeping a prev pointer to the last node already flattened. For each node, point its right at prev, clear its left, then set prev to itself. Because we visit in reverse preorder, prev is always exactly the node that should follow the current one.

TreeNode* prev = nullptr;

void flatten(TreeNode* root) {
    if (!root) return;
    flatten(root->right);      // reverse preorder:
    flatten(root->left);       // right, left, node
    root->right = prev;
    root->left  = nullptr;
    prev = root;
}

TreeNode prev = null;

void flatten(TreeNode root) {
    if (root == null) return;
    flatten(root.right);       // reverse preorder:
    flatten(root.left);        // right, left, node
    root.right = prev;
    root.left  = null;
    prev = root;
}

def flatten(root):
    prev = [None]
    def go(node):
        if not node:
            return
        go(node.right)          # reverse preorder:
        go(node.left)           # right, left, node
        node.right = prev[0]
        node.left  = None
        prev[0] = node
    go(root)

Approach B — Morris-like threading, O(1) extra space

This iterative method uses no recursion stack at all. Walk the chain via the right pointer. At each curr that has a left child, find the rightmost node of that left subtree (its preorder predecessor for the right subtree), splice the existing right subtree onto that node's right, then move the whole left subtree over to curr.right and null out curr.left. Advance to curr.right and repeat. The threading is what lets us reattach the dangling right subtree without storing anything.

curr = 1, left subtree = (2,3,4), right subtree = (5,6)

  rightmost of left subtree is node 4
  -> attach right subtree (5,6) after node 4
  -> move left subtree to curr.right, curr.left = null
  -> advance curr to its new right child

  Repeat until curr becomes null. No extra memory used.

void flatten(TreeNode* root) {
    TreeNode* curr = root;
    while (curr) {
        if (curr->left) {
            TreeNode* pred = curr->left;
            while (pred->right) pred = pred->right;  // rightmost
            pred->right = curr->right;               // splice right subtree
            curr->right = curr->left;                // move left over
            curr->left  = nullptr;
        }
        curr = curr->right;
    }
}

void flatten(TreeNode root) {
    TreeNode curr = root;
    while (curr != null) {
        if (curr.left != null) {
            TreeNode pred = curr.left;
            while (pred.right != null) pred = pred.right;  // rightmost
            pred.right = curr.right;                       // splice right subtree
            curr.right = curr.left;                        // move left over
            curr.left  = null;
        }
        curr = curr.right;
    }
}

def flatten(root):
    curr = root
    while curr:
        if curr.left:
            pred = curr.left
            while pred.right:
                pred = pred.right          # rightmost node
            pred.right = curr.right         # splice right subtree
            curr.right = curr.left          # move left over
            curr.left  = None
        curr = curr.right

6. Merge Two Binary Trees

Overlay two trees onto each other. Where both trees have a node, sum the values; where only one has a node, keep that node as-is. A clean recursion: if either node is null return the other, otherwise add the values and recurse on matching children.

   Tree 1          Tree 2          Merged

      1               2              3
     / \             / \           /   \
    3   2           1   3         4     5
   /                 \   \       / \     \
  5                   4   7     5   4     7

TreeNode* mergeTrees(TreeNode* a, TreeNode* b) {
    if (!a) return b;     // nothing in a -> take b
    if (!b) return a;     // nothing in b -> take a
    a->val += b->val;     // overlap -> sum
    a->left  = mergeTrees(a->left,  b->left);
    a->right = mergeTrees(a->right, b->right);
    return a;
}

TreeNode mergeTrees(TreeNode a, TreeNode b) {
    if (a == null) return b;   // nothing in a -> take b
    if (b == null) return a;   // nothing in b -> take a
    a.val += b.val;              // overlap -> sum
    a.left  = mergeTrees(a.left,  b.left);
    a.right = mergeTrees(a.right, b.right);
    return a;
}

def mergeTrees(a, b):
    if not a:
        return b           # nothing in a -> take b
    if not b:
        return a           # nothing in b -> take a
    a.val += b.val         # overlap -> sum
    a.left  = mergeTrees(a.left,  b.left)
    a.right = mergeTrees(a.right, b.right)
    return a

7. Count Good Nodes

A node is good if, on the path from the root down to it, no node has a value greater than it — in other words, it is at least as large as the maximum seen so far. This is another structural recursion, but instead of returning a rebuilt subtree it threads a running maximum down the tree and counts on the way.

          3   (good: max so far = 3)
         / \
        1   4   (4 good: 4 >= 3)
       /   / \
      3   1   5  (5 good: 5 >= 4)
   (3 good:        (1 not good: 1 < 4)
    3 >= 3)

  Good nodes: 3, 4, 5, 3  ->  count = 4

int goodNodes(TreeNode root) {
    return dfs(root, Integer.MIN_VALUE);
}

// maxSoFar = largest value on the path from root to here
int dfs(TreeNode node, int maxSoFar) {
    if (node == null) return 0;
    int count = (node.val >= maxSoFar) ? 1 : 0;
    int newMax = Math.max(maxSoFar, node.val);
    count += dfs(node.left,  newMax);
    count += dfs(node.right, newMax);
    return count;
}

The same pattern counts univalue subtrees (subtrees where every node shares one value): a postorder recursion returns whether the current subtree is univalue, and increments a counter each time both children report univalue and match the current node's value.

Complexity Summary

OperationTimeSpace (aux)
Invert / MirrorO(n)O(h) stack
Identical treesO(n)O(h) stack
Symmetric checkO(n)O(h) stack
Subtree (naive)O(n × m)O(h) stack
Subtree (serialization)O(n + m)O(n + m)
Flatten (recursive)O(n)O(h) stack
Flatten (Morris-like)O(n)O(1)
Merge two treesO(n)O(h) stack
Count good nodesO(n)O(h) stack

Here n (and m) is the number of nodes and h is the tree height. The recursion stack cost is O(h), which is O(log n) for a balanced tree and O(n) in the worst (skewed) case.

The unifying pattern:
  • Most structural operations are a single recursion that returns the rebuilt or checked subtree — invert, merge, and flatten all rebuild; identical and subtree all check.
  • Symmetric and identical use a paired recursion: two pointers advance together and you compare them step by step (symmetric crosses them, identical keeps them aligned).
  • Counting problems (good nodes, univalue subtrees) thread an accumulator down or bubble a boolean up, but the skeleton is the same node-once recursion.
  • When the naive approach is O(n × m), look for a serialization-to-string reformulation to drop to linear time.
