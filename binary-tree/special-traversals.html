Special Traversals

Once you know the classic ways to walk a tree — preorder, inorder, postorder, and level-order — a whole family of geometry-flavoured traversals opens up. Instead of asking "deep or wide?", these ask "where does each node sit on the page?" They group nodes by an invisible grid drawn over the tree.

This page covers the three you are most likely to be asked about that are not already on the other pages:

  • Vertical order — read the tree in columns, like reading newspaper columns top-to-bottom, left-to-right.
  • Diagonal — read the tree in slanted lines, like rain streaking down a window.
  • Zigzag / spiral — level-order, but snaking left and right (a quick recap; the full version lives on the Tree Traversals page).
Already covered elsewhere: preorder / inorder / postorder / level-order / Morris live on the Tree Traversals page, and top / bottom / left / right views plus boundary traversal live on the Tree Views & Properties page. This page is the rest of the family.

The Running Example

Throughout this page we use one fixed tree. It is not a BST — these traversals care about shape and position, not key order.

                1
              /   \
             2     3
            / \   / \
           4   5 6   7
                \   \
                 8   9

The big idea: lay a coordinate grid over this drawing. Give the root column 0. Every time you step left you move one column left (col - 1); every step right moves one column right (col + 1). The row is just the depth (root = row 0). Different traversals slice that grid differently.

1. Vertical Order Traversal

Analogy: imagine the tree drawn on a window and a low sun behind it casting vertical shadows straight down. Every node falls into a column. Vertical order reads those columns from the leftmost to the rightmost, and within a column, top to bottom.

Concretely, assign each node a column:

  • Root gets column 0.
  • Going left → col - 1.
  • Going right → col + 1.

For our tree the columns work out to:

   col:   -2    -1     0     1     2     3
          ----------------------------------
   row 0               1
   row 1         2           3
   row 2    4          5,6         7
   row 3                     8           9

   Vertical order (col -2 .. +3):
       [4]  [2]  [1, 5, 6]  [3, 8]  [7]  [9]
   Flattened:  4 2 1 5 6 3 8 7 9
The tie-break rule (the tricky part). Two nodes can land in the same column at the same row — here 5 and 6 both sit at column 0, row 2. The LeetCode "Vertical Order Traversal" rule says: sort all nodes by the triple (col, row, val). So within a column we go top row first, and when row also ties, the smaller value comes first. That is why the order is 5 then 6, not 6 then 5.
Watch out — two different problems share a name. The older "Vertical Order" problem (GeeksforGeeks / LeetCode 314) breaks same-column ties by insertion order from a BFS (left-to-right), not by value. The newer LeetCode 987 "Vertical Order Traversal" breaks ties by value via the (col, row, val) sort. Read the problem statement carefully — for a same row+column pair they can disagree. The code below implements the (col, row, val) rule.

The clean implementation: do any traversal (DFS or BFS), recording (row, col, val) for every node into a list, then sort that list by the triple and group by column.

vector<vector<int>> verticalTraversal(TreeNode* root) {
    // collect tuples of (col, row, val)
    vector<array<int, 3>> nodes;
    function<void(TreeNode*, int, int)> dfs =
        [&](TreeNode* n, int row, int col) {
            if (!n) return;
            nodes.push_back({col, row, n->val});
            dfs(n->left,  row + 1, col - 1);
            dfs(n->right, row + 1, col + 1);
        };
    dfs(root, 0, 0);
    // sort by (col, row, val)
    sort(nodes.begin(), nodes.end());
    vector<vector<int>> out;
    int prevCol = INT_MIN;
    for (auto& t : nodes) {
        if (t[0] != prevCol) { out.push_back({}); prevCol = t[0]; }
        out.back().push_back(t[2]);
    }
    return out;
}

List<List<Integer>> verticalTraversal(TreeNode root) {
    // each entry: {col, row, val}
    List<int[]> nodes = new ArrayList<>();
    dfs(root, 0, 0, nodes);
    // sort by col, then row, then val
    nodes.sort((a, b) ->
        a[0] != b[0] ? a[0] - b[0] :
        a[1] != b[1] ? a[1] - b[1] :
        a[2] - b[2]);
    List<List<Integer>> out = new ArrayList<>();
    Integer prevCol = null;
    for (int[] t : nodes) {
        if (prevCol == null || t[0] != prevCol) {
            out.add(new ArrayList<>());
            prevCol = t[0];
        }
        out.get(out.size() - 1).add(t[2]);
    }
    return out;
}

void dfs(TreeNode n, int row, int col, List<int[]> nodes) {
    if (n == null) return;
    nodes.add(new int[]{col, row, n.val});
    dfs(n.left,  row + 1, col - 1, nodes);
    dfs(n.right, row + 1, col + 1, nodes);
}

def vertical_traversal(root):
    nodes = []                       # list of (col, row, val)

    def dfs(n, row, col):
        if not n:
            return
        nodes.append((col, row, n.val))
        dfs(n.left,  row + 1, col - 1)
        dfs(n.right, row + 1, col + 1)

    dfs(root, 0, 0)
    nodes.sort()                     # sorts by col, then row, then val

    out, prev_col = [], None
    for col, row, val in nodes:
        if col != prev_col:
            out.append([])
            prev_col = col
        out[-1].append(val)
    return out

Complexity. We touch each of the n nodes once to collect, then sort the list of triples: O(n log n) time. Space is O(n) for the triple list and the output. (If a problem uses the BFS insertion-order tie-break instead of value, you can skip the value sort and use a column-keyed map plus a BFS, giving O(n) time after sorting only the column keys.)

2. Diagonal Traversal

Analogy: tilt your head and picture rain streaking down the tree at a 45° slant from upper-left to lower-right. Each raindrop trail is a diagonal. The rule: walking right keeps you on the same diagonal, but walking left drops you onto the next diagonal down.

So we give each node a diagonal index d:

  • Root: d = 0.
  • Right child: same diagonal → d.
  • Left child: next diagonal → d + 1.

Then read diagonal 0, then diagonal 1, and so on. Within a diagonal, nodes come out in the natural traversal order (top-to-bottom along the slant).

   Diagonals (right = same d, left = d + 1):

   d = 0 :  1  3  7  9      (root, keep going right)
   d = 1 :  2  5  8  6      (one left-step off d=0)
   d = 2 :  4               (two left-steps)

   Diagonal traversal:  1 3 7 9 2 5 8 6 4

The cleanest code keeps a queue per diagonal, or a single map from diagonal index to its list. Here is the map approach via DFS — group by d, then emit diagonals in increasing order.

vector<int> diagonalTraversal(TreeNode* root) {
    // map keeps diagonals sorted by key (0, 1, 2, ...)
    map<int, vector<int>> diag;
    function<void(TreeNode*, int)> dfs =
        [&](TreeNode* n, int d) {
            if (!n) return;
            diag[d].push_back(n->val);
            dfs(n->left,  d + 1);   // left -> next diagonal
            dfs(n->right, d);         // right -> same diagonal
        };
    dfs(root, 0);
    vector<int> out;
    for (auto& [d, vals] : diag)
        for (int v : vals) out.push_back(v);
    return out;
}

List<Integer> diagonalTraversal(TreeNode root) {
    // TreeMap keeps diagonals sorted by key
    TreeMap<Integer, List<Integer>> diag = new TreeMap<>();
    dfs(root, 0, diag);
    List<Integer> out = new ArrayList<>();
    for (List<Integer> vals : diag.values())
        out.addAll(vals);
    return out;
}

void dfs(TreeNode n, int d, TreeMap<Integer, List<Integer>> diag) {
    if (n == null) return;
    diag.computeIfAbsent(d, k -> new ArrayList<>()).add(n.val);
    dfs(n.left,  d + 1, diag);   // left -> next diagonal
    dfs(n.right, d, diag);         // right -> same diagonal
}

from collections import defaultdict

def diagonal_traversal(root):
    diag = defaultdict(list)

    def dfs(n, d):
        if not n:
            return
        diag[d].append(n.val)
        dfs(n.left,  d + 1)   # left -> next diagonal
        dfs(n.right, d)         # right -> same diagonal

    dfs(root, 0)
    out = []
    for d in sorted(diag):
        out.extend(diag[d])
    return out
Order within a diagonal. No extra sort is needed — the recursion order does the work. A node is appended to its diagonal before its own children, and a node always reaches its same-diagonal right-chain in order, so values come out in the natural traversal order along the slant (e.g. diagonal 1 above emits 2, 5, 8, 6: node 2 first, then everything reachable from its left/right before backtracking).

Complexity. Every node is visited once and appended to one diagonal list: O(n) work. With a hash map plus a final key sort it is O(n + k log k) where k is the number of diagonals (k ≤ n); with an ordered map (above) the keys stay sorted for free. Space is O(n).

3. Zigzag / Spiral Order (Recap)

Analogy: read the tree level by level like normal BFS, but make your eyes snake — left-to-right on one level, right-to-left on the next, like the way an old printer head sweeps back and forth.

For our tree the levels are [1], [2, 3], [4, 5, 6, 7], [8, 9]. Reversing every other level gives:

   level 0 (L→R):  1
   level 1 (R→L):  3 2
   level 2 (L→R):  4 5 6 7
   level 3 (R→L):  9 8

   Zigzag:  [[1], [3, 2], [4, 5, 6, 7], [9, 8]]

It is ordinary level-order with one extra boolean: track whether the current level runs left-to-right, build the level list, reverse it on alternate levels, and flip the flag.

vector<vector<int>> zigzag(TreeNode* root) {
    vector<vector<int>> out;
    if (!root) return out;
    queue<TreeNode*> q;
    q.push(root);
    bool leftToRight = true;
    while (!q.empty()) {
        int sz = q.size();
        vector<int> level(sz);
        for (int i = 0; i < sz; i++) {
            TreeNode* node = q.front(); q.pop();
            int idx = leftToRight ? i : sz - 1 - i;
            level[idx] = node->val;
            if (node->left)  q.push(node->left);
            if (node->right) q.push(node->right);
        }
        out.push_back(level);
        leftToRight = !leftToRight;
    }
    return out;
}

List<List<Integer>> zigzag(TreeNode root) {
    List<List<Integer>> out = new ArrayList<>();
    if (root == null) return out;
    Queue<TreeNode> q = new LinkedList<>();
    q.offer(root);
    boolean leftToRight = true;
    while (!q.isEmpty()) {
        int sz = q.size();
        Integer[] level = new Integer[sz];
        for (int i = 0; i < sz; i++) {
            TreeNode node = q.poll();
            int idx = leftToRight ? i : sz - 1 - i;
            level[idx] = node.val;
            if (node.left != null)  q.offer(node.left);
            if (node.right != null) q.offer(node.right);
        }
        out.add(Arrays.asList(level));
        leftToRight = !leftToRight;
    }
    return out;
}

from collections import deque

def zigzag(root):
    out = []
    if not root:
        return out
    q = deque([root])
    left_to_right = True
    while q:
        level = []
        for _ in range(len(q)):
            node = q.popleft()
            level.append(node.val)
            if node.left:
                q.append(node.left)
            if node.right:
                q.append(node.right)
        out.append(level if left_to_right else level[::-1])
        left_to_right = not left_to_right
    return out
For the full BFS walk-through (queue mechanics, level-by-level grouping, and the boolean-flip detail), see the Zigzag / Spiral Order section on the Tree Traversals page. Complexity is the same as plain level-order: O(n) time, O(w) space where w is the maximum width.

Complexity Summary

Here n = number of nodes, k = number of distinct columns/diagonals (k ≤ n), w = maximum tree width.

TraversalTimeSpaceGrouping keyTie-break within group
Vertical order (LC 987)O(n log n)O(n)column (left −1, right +1)(row, then value)
Vertical order (LC 314)O(n + k log k)O(n)columnBFS insertion order

How to spot which one a problem wants:
  • Vertical order — the prompt mentions "columns", "vertical lines", or assigns left/right a horizontal distance. Watch the tie-break wording closely.
  • Diagonal — the prompt talks about "diagonals", "slanting lines", or "45-degree" groupings; remember right keeps the diagonal, left starts a new one.
  • Zigzag / spiral — "level by level but alternate direction", "snake", or "boustrophedon". It is just BFS with a flip flag.
