Fenwick Tree (Binary Indexed Tree)

A Fenwick Tree (BIT) is an elegant data structure that supports point updates and prefix sum queries in O(log n) time with O(n) space. It's simpler and faster than a segment tree for problems that only need prefix queries.

Key Insight: Lowbit (Least Significant Bit)

The entire structure relies on the binary representation of indices. The lowbit of a number is its least significant set bit: lowbit(x) = x & (-x)

Index (decimal) -> Binary  -> lowbit -> Responsible for range
    1           -> 0001    -> 1      -> [1, 1]     (1 element)
    2           -> 0010    -> 2      -> [1, 2]     (2 elements)
    3           -> 0011    -> 1      -> [3, 3]     (1 element)
    4           -> 0100    -> 4      -> [1, 4]     (4 elements)
    5           -> 0101    -> 1      -> [5, 5]     (1 element)
    6           -> 0110    -> 2      -> [5, 6]     (2 elements)
    7           -> 0111    -> 1      -> [7, 7]     (1 element)
    8           -> 1000    -> 8      -> [1, 8]     (8 elements)

Each BIT[i] stores sum of elements in range [i - lowbit(i) + 1, i]

Operations

Prefix Query: sum(1..r)

Start at index r, add BIT[r], then strip the lowest set bit: r -= lowbit(r). Repeat until r = 0.

Query: prefix_sum(7) = sum of a[1..7]

Step 1: r = 7 (0111), add BIT[7] = 6,  r -= lowbit(7) = 7 - 1 = 6
Step 2: r = 6 (0110), add BIT[6] = 11, r -= lowbit(6) = 6 - 2 = 4
Step 3: r = 4 (0100), add BIT[4] = 11, r -= lowbit(4) = 4 - 4 = 0
Step 4: r = 0, stop.

Answer = 6 + 11 + 11 = 28 = 1+3+2+5+4+7+6 ✓

At most O(log n) steps (number of set bits in r)

Point Update: add(i, delta)

Start at index i, add delta to BIT[i], then move to parent: i += lowbit(i). Repeat until i > n.

Update: add(3, +5)  (add 5 to a[3])

Step 1: i = 3 (0011), BIT[3] += 5,  i += lowbit(3) = 3 + 1 = 4
Step 2: i = 4 (0100), BIT[4] += 5,  i += lowbit(4) = 4 + 4 = 8
Step 3: i = 8 (1000), BIT[8] += 5,  i += lowbit(8) = 8 + 8 = 16
Step 4: i = 16 > n, stop.

Updates all ancestors that include index 3 in their range

Range Query: sum(l..r)

Simply compute prefix_sum(r) - prefix_sum(l-1).

Building the BIT in O(n)

Instead of n individual updates (O(n log n)), we can build in O(n) by propagating each element to its immediate parent only:

O(n) Build Algorithm:
1. Copy array into BIT: BIT[i] = a[i]
2. For each i from 1 to n:
     parent = i + lowbit(i)
     if parent <= n: BIT[parent] += BIT[i]

This works because each node's parent is at i + lowbit(i),
and we process bottom-up, so when we reach a node,
all its children have already propagated their sums up.

Code Implementation

#include <bits/stdc++.h>
using namespace std;

struct FenwickTree {
    vector<int> bit;
    int n;

    FenwickTree(int n) : n(n), bit(n + 1, 0) {}

    // Build from array in O(n)
    FenwickTree(vector<int>& a) : n(a.size()), bit(a.size() + 1, 0) {
        for (int i = 1; i <= n; i++) bit[i] = a[i - 1];
        for (int i = 1; i <= n; i++) {
            int parent = i + (i & -i);
            if (parent <= n) bit[parent] += bit[i];
        }
    }

    // Point update: add delta to index i (1-indexed)
    void update(int i, int delta) {
        for (; i <= n; i += i & -i)
            bit[i] += delta;
    }

    // Prefix sum query: sum of [1..i]
    int query(int i) {
        int sum = 0;
        for (; i > 0; i -= i & -i)
            sum += bit[i];
        return sum;
    }

    // Range sum query: sum of [l..r]
    int query(int l, int r) {
        return query(r) - query(l - 1);
    }
};

int main() {
    vector<int> a = {1, 3, 2, 5, 4, 7, 6, 8};
    FenwickTree ft(a);

    cout << ft.query(1, 5) << endl;  // 15 = 1+3+2+5+4
    ft.update(3, 5);                   // a[3] += 5 -> a[3] = 7
    cout << ft.query(1, 5) << endl;  // 20 = 1+3+7+5+4
}

class FenwickTree {
    int[] bit;
    int n;

    FenwickTree(int n) {
        this.n = n;
        bit = new int[n + 1];
    }

    // Build from array in O(n)
    FenwickTree(int[] a) {
        this.n = a.length;
        bit = new int[n + 1];
        for (int i = 1; i <= n; i++) bit[i] = a[i - 1];
        for (int i = 1; i <= n; i++) {
            int parent = i + (i & -i);
            if (parent <= n) bit[parent] += bit[i];
        }
    }

    void update(int i, int delta) {
        for (; i <= n; i += i & -i)
            bit[i] += delta;
    }

    int query(int i) {
        int sum = 0;
        for (; i > 0; i -= i & -i)
            sum += bit[i];
        return sum;
    }

    int query(int l, int r) {
        return query(r) - query(l - 1);
    }
}

class FenwickTree:
    def __init__(self, n_or_arr):
        if isinstance(n_or_arr, int):
            self.n = n_or_arr
            self.bit = [0] * (self.n + 1)
        else:
            self.n = len(n_or_arr)
            self.bit = [0] + list(n_or_arr)  # 1-indexed
            for i in range(1, self.n + 1):
                parent = i + (i & -i)
                if parent <= self.n:
                    self.bit[parent] += self.bit[i]

    def update(self, i, delta):
        while i <= self.n:
            self.bit[i] += delta
            i += i & -i

    def query(self, i):
        s = 0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

    def range_query(self, l, r):
        return self.query(r) - self.query(l - 1)

# Usage
a = [1, 3, 2, 5, 4, 7, 6, 8]
ft = FenwickTree(a)
print(ft.range_query(1, 5))  # 15
ft.update(3, 5)
print(ft.range_query(1, 5))  # 20

BIT vs Segment Tree

Fenwick Tree

  • Simpler code (5-10 lines)
  • Faster constant factor
  • Less memory (n+1 array)
  • Only works with invertible operations (sum, XOR)
  • Cannot do arbitrary range queries directly (only prefix)

Segment Tree

  • More complex (~30-50 lines)
  • Slower constant factor
  • More memory (4n array)
  • Works with any associative operation (min, max, GCD)
  • Supports arbitrary range queries directly
When to use Fenwick Tree:
  • Point update + prefix/range sum queries
  • Inversion counting
  • Coordinate compression + frequency queries
  • When you need fast, minimal code in contests

Common Patterns

1. Find kth smallest (Order Statistics)

Use BIT as a frequency array. Binary search on prefix sums to find the kth element in O(log^2 n) or O(log n) with bit-descent.

2. Counting Inversions

Process elements right to left. For each element, query prefix sum up to its value minus 1 to count elements smaller that appear after it.

3. Range Frequency

With coordinate compression, maintain frequency counts and query ranges to count occurrences.
