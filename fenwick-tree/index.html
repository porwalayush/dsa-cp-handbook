When to Use Fenwick Tree

Quick decision guide

1

Point Update + Prefix Sum?

Classic BIT use case. Simpler and faster than segment tree. Only needs invertible operations (sum, XOR).

2

Counting Inversions?

Process array left to right. Use BIT as frequency array. Query prefix to count smaller elements seen so far.

3

Dynamic Order Statistics?

BIT on frequencies. Bit descent for O(log n) kth element. Add/remove elements dynamically.

4

Range Add + Range Sum?

Use two BITs trick. Maintains difference array decomposition for both range updates and range queries.

5

2D Grid with Updates?

2D BIT for point updates + rectangle sum queries. Nest the BIT loops for each dimension.

6

Need Min/Max/GCD?

DON'T use BIT. These aren't invertible. Use Segment Tree or Sparse Table instead.
