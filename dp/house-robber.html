House Robber

Find the maximum sum of non-adjacent elements. You cannot rob two adjacent houses because they have connected security systems.

Problem Formulation

   Houses: [2, 7, 9, 3, 1]

   Can't rob adjacent houses:
   Rob [2, 9, 1] = 12
   Rob [7, 3]    = 10
   Rob [2, 3]    = 5
   Rob [7, 1]    = 8
   Rob [9, 1]    = 10
   Rob [2, 9]    = 11

   Best: Rob houses 0, 2, 4 → 2 + 9 + 1 = 12 ✓

State & Transition

State: dp[i] = max sum considering the first i houses

Transition:
dp[i] = max(dp[i-1], dp[i-2] + nums[i])
      (skip house i) vs (rob house i + best from i-2)

Base: dp[0] = nums[0], dp[1] = max(nums[0], nums[1])

Intuition: At each house, we make a binary choice: skip it (take previous best) or rob it (add its value to the best sum ending 2+ houses back).

Interactive: Include/Exclude Decisions

Code: House Robber (Basic)

// O(n) time, O(1) space
int rob(vector<int>& nums) {
    int n = nums.size();
    if (n == 0) return 0;
    if (n == 1) return nums[0];

    int prev2 = nums[0];                // dp[i-2]
    int prev1 = max(nums[0], nums[1]); // dp[i-1]

    for (int i = 2; i < n; i++) {
        int curr = max(prev1, prev2 + nums[i]);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

// O(n) time, O(1) space
static int rob(int[] nums) {
    int n = nums.length;
    if (n == 0) return 0;
    if (n == 1) return nums[0];

    int prev2 = nums[0];
    int prev1 = Math.max(nums[0], nums[1]);

    for (int i = 2; i < n; i++) {
        int curr = Math.max(prev1, prev2 + nums[i]);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

# O(n) time, O(1) space
def rob(nums):
    if not nums:
        return 0
    if len(nums) == 1:
        return nums[0]

    prev2 = nums[0]
    prev1 = max(nums[0], nums[1])

    for i in range(2, len(nums)):
        curr = max(prev1, prev2 + nums[i])
        prev2 = prev1
        prev1 = curr

    return prev1

Variant: House Robber II (Circular)

Problem: Houses are arranged in a circle. House 0 and house n-1 are adjacent.

Key Insight: We can't rob both the first and last house. So solve two subproblems:
1. Rob from house 0 to n-2 (exclude last)
2. Rob from house 1 to n-1 (exclude first)
Answer = max of these two. 
// House Robber II: circular arrangement
int robRange(vector<int>& nums, int lo, int hi) {
    int prev2 = 0, prev1 = 0;
    for (int i = lo; i <= hi; i++) {
        int curr = max(prev1, prev2 + nums[i]);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

int robCircular(vector<int>& nums) {
    int n = nums.size();
    if (n == 1) return nums[0];
    return max(
        robRange(nums, 0, n - 2),  // exclude last
        robRange(nums, 1, n - 1)   // exclude first
    );
}

static int robCircular(int[] nums) {
    int n = nums.length;
    if (n == 1) return nums[0];
    return Math.max(
        robRange(nums, 0, n - 2),
        robRange(nums, 1, n - 1)

# House Robber II: circular arrangement
def rob_range(nums, lo, hi):
    prev2 = prev1 = 0
    for i in range(lo, hi + 1):
        curr = max(prev1, prev2 + nums[i])
        prev2 = prev1
        prev1 = curr
    return prev1

def rob_circular(nums):
    if len(nums) == 1:
        return nums[0]
    return max(
        rob_range(nums, 0, len(nums) - 2),  # exclude last
        rob_range(nums, 1, len(nums) - 1)   # exclude first
    )

Space Optimization Insight

   dp[i] only depends on dp[i-1] and dp[i-2]
   → We only need TWO variables!

   Array version:        Optimized:
   dp[0] = 2            prev2 = 2
   dp[1] = 7            prev1 = 7
   dp[2] = max(7, 2+9) = 11    curr = max(7, 2+9) = 11
   dp[3] = max(11, 7+3) = 11   prev2=7, prev1=11, curr=max(11,7+3)=11
   dp[4] = max(11, 11+1) = 12  prev2=11, prev1=11, curr=max(11,11+1)=12

   O(n) space → O(1) space!

Complexity

  • Time: O(n) - single pass through the array
  • Space: O(1) - only two variables needed
  • Circular variant: O(n) time, O(1) space (two passes, constant extra work)
