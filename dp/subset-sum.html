Subset Sum & Partition Problems

A family of problems built on the 0/1 Knapsack framework: determine if a subset can achieve a target sum, split an array into equal halves, or count subsets with a given sum.

1. Subset Sum Problem

Given an array of non-negative integers and a target sum, determine if there exists a subset whose elements sum to exactly the target.

   arr = [3, 34, 4, 12, 5, 2], target = 9

   Subset {4, 5} sums to 9 → TRUE
   Subset {3, 4, 2} sums to 9 → TRUE

   This is 0/1 Knapsack where:
     - weight[i] = arr[i]
     - value[i]  = arr[i]  (we only care about reaching target)
     - Answer is boolean: can we fill capacity = target exactly?

State & Transition

State: dp[i][s] = can we form sum s using first i elements?

Transition:
dp[i][s] = dp[i-1][s] (skip element i)
           OR dp[i-1][s - arr[i-1]] (include element i, if s >= arr[i-1])

Base Cases:
dp[0][0] = true (empty subset sums to 0)
dp[0][s] = false for s > 0 (no elements, can't form positive sum)

Visual Walkthrough

   arr = [2, 3, 7], target = 5

   dp[i][s]: T = true, . = false

         s:  0  1  2  3  4  5  6  7
   i=0 (--): T  .  .  .  .  .  .  .
   i=1 ( 2): T  .  T  .  .  .  .  .    ← can form {0, 2}
   i=2 ( 3): T  .  T  T  .  T  .  .    ← can form {0, 2, 3, 5}
   i=3 ( 7): T  .  T  T  .  T  .  T    ← can form {0, 2, 3, 5, 7}

   dp[3][5] = T → Answer: YES (subset {2,3})

2. Equal Partition Problem

Can we partition an array into two subsets with equal sum? This reduces to: find a subset with sum = totalSum / 2.

   arr = [1, 5, 11, 5], totalSum = 22

   target = 22 / 2 = 11
   Subset {1, 5, 5} = 11  →  remaining {11} = 11
   Both halves equal → TRUE

   Key insight: If totalSum is ODD, answer is always FALSE
Reduction: Equal Partition = Subset Sum with target = totalSum/2

Early exit: If totalSum % 2 != 0, return false immediately

3. Count Subsets with Given Sum

Instead of boolean (can/can't), count how many subsets sum to the target.

   arr = [1, 2, 3, 3], target = 6

   Subsets that sum to 6:
     {1, 2, 3} (using first 3)  = 6
     {1, 2, 3} (using second 3) = 6
     {3, 3}                     = 6

   Answer: 3

   Change: dp[i][s] = number of subsets (not boolean)
   Transition: dp[i][s] = dp[i-1][s] + dp[i-1][s - arr[i-1]]
                           (skip)        (include)

Code: Subset Sum (Boolean)

// Space optimized: O(target) space
bool subsetSum(vector<int>& arr, int target) {
    vector<bool> dp(target + 1, false);
    dp[0] = true;

    for (int num : arr) {
        for (int s = target; s >= num; s--) { // RIGHT to LEFT (0/1)
            dp[s] = dp[s] || dp[s - num];
        }
    }
    return dp[target];
}

// Space optimized: O(target) space
static boolean subsetSum(int[] arr, int target) {
    boolean[] dp = new boolean[target + 1];
    dp[0] = true;

    for (int num : arr) {
        for (int s = target; s >= num; s--) { // RIGHT to LEFT
            dp[s] = dp[s] || dp[s - num];
        }
    }
    return dp[target];
}

# Space optimized: O(target) space
def subset_sum(arr, target):
    dp = [False] * (target + 1)
    dp[0] = True

    for num in arr:
        for s in range(target, num - 1, -1):  # RIGHT to LEFT
            dp[s] = dp[s] or dp[s - num]
    return dp[target]

Code: Equal Partition

bool canPartition(vector<int>& nums) {
    int total = 0;
    for (int x : nums) total += x;
    if (total % 2 != 0) return false; // odd sum can't split

    int target = total / 2;
    vector<bool> dp(target + 1, false);
    dp[0] = true;

    for (int num : nums) {
        for (int s = target; s >= num; s--) {
            dp[s] = dp[s] || dp[s - num];
        }
    }
    return dp[target];
}

static boolean canPartition(int[] nums) {

}

def can_partition(nums):
    total = sum(nums)
    if total % 2 != 0:
        return False  # odd sum can't split

    target = total // 2
    dp = [False] * (target + 1)
    dp[0] = True

    for num in nums:
        for s in range(target, num - 1, -1):
            dp[s] = dp[s] or dp[s - num]
    return dp[target]

Code: Count Subsets with Given Sum

// Count subsets summing to target
int countSubsets(vector<int>& arr, int target) {
    vector<int> dp(target + 1, 0);
    dp[0] = 1; // empty subset

    for (int num : arr) {
        for (int s = target; s >= num; s--) {
            dp[s] += dp[s - num]; // add ways to include this num
        }
    }
    return dp[target];
}

// Count subsets summing to target
static int countSubsets(int[] arr, int target) {
    int[] dp = new int[target + 1];
    dp[0] = 1; // empty subset

    for (int num : arr) {
        for (int s = target; s >= num; s--) {
            dp[s] += dp[s - num];
        }
    }
    return dp[target];
}

# Count subsets summing to target
def count_subsets(arr, target):
    dp = [0] * (target + 1)
    dp[0] = 1  # empty subset

    for num in arr:
        for s in range(target, num - 1, -1):
            dp[s] += dp[s - num]
    return dp[target]

Key Observations

  • Subset Sum is a special case of 0/1 Knapsack where value = weight
  • Equal Partition reduces to Subset Sum with target = totalSum/2
  • Count of Subsets changes OR to + (count ways instead of check existence)
  • All use right-to-left iteration in 1D (each element used at most once)
  • Handle edge case: if arr contains 0s, each 0 doubles the count (include or exclude)

Complexity

ProblemTimeSpace (optimized)
Subset SumO(n * target)O(target)
Equal PartitionO(n * sum/2)O(sum/2)

  • All are pseudo-polynomial — polynomial in the numeric value of target, not in the input size
  • 2D version uses O(n * target) space; 1D optimization reduces to O(target)
  • For large targets but few elements, consider meet-in-the-middle O(2^(n/2)) approach
