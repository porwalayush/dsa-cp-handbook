Tree DP Problems Collection

Advanced tree DP problems featuring multi-state DFS, greedy-on-tree, and re-rooting techniques. Each problem uses O(n) time via a single or double DFS pass.

Problem 1: House Robber III

A binary tree where each node has a value. You cannot rob two directly connected nodes (parent-child). Find the maximum amount you can rob.

        3
       / \
      4   5
     / \   \
    1   3   1

Option A: Rob 4 + 5 = 9
Option B: Rob 3 + 1 + 3 + 1 = 8
Option C: Rob 4 + 1 = 5

Best: Rob nodes 4 and 5 -> 9

State per node: (not_rob, rob)
  not_rob[v]: max value if we do NOT rob node v
  rob[v]:     max value if we DO rob node v

Transition:
  rob[v] = val[v] + not_rob[left] + not_rob[right]
  not_rob[v] = max(rob[left], not_rob[left])
             + max(rob[right], not_rob[right])

State Definition: House Robber III

State: For each node v, compute a pair (not_rob, rob)

Transition:
rob[v] = val[v] + not_rob[left_child] + not_rob[right_child]
not_rob[v] = max(rob[left], not_rob[left]) + max(rob[right], not_rob[right])

Base Case: null node returns (0, 0)

Answer: max(rob[root], not_rob[root])

Step-by-Step: House Robber III

Step 1: Leaf nodes 1, 3, 1: rob = val, not_rob = 0.
Node 1 (left-left): (0, 1). Node 3 (left-right): (0, 3). Node 1 (right-right): (0, 1).
Step 2: Node 4: rob = 4 + 0 + 0 = 4, not_rob = max(0,1) + max(0,3) = 1+3 = 4.
Node 5: rob = 5 + 0 = 5, not_rob = max(0,1) = 1.
Step 3: Root 3: rob = 3 + 4 + 1 = 8, not_rob = max(4,4) + max(5,1) = 4+5 = 9.
Answer: max(8, 9) = 9.

Code: House Robber III

// House Robber III - Binary Tree
// Time: O(n), Space: O(h) recursion stack
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
};

// Returns {not_rob, rob}
pair<int,int> dfs(TreeNode* node) {
    if (!node) return {0, 0};

    auto [leftNot, leftRob] = dfs(node->left);
    auto [rightNot, rightRob] = dfs(node->right);

    // Rob this node: children must NOT be robbed
    int rob = node->val + leftNot + rightNot;

    // Don't rob this node: children are free to be robbed or not
    int notRob = max(leftNot, leftRob) + max(rightNot, rightRob);

    return {notRob, rob};
}

int rob(TreeNode* root) {
    auto [notRob, robRoot] = dfs(root);
    return max(notRob, robRoot);
}

// House Robber III - Binary Tree
// Time: O(n), Space: O(h) recursion stack
public int rob(TreeNode root) {
    int[] res = dfs(root);
    return Math.max(res[0], res[1]);
}

// Returns [not_rob, rob]
private int[] dfs(TreeNode node) {
    if (node == null) return new int[]{0, 0};

    int[] left = dfs(node.left);
    int[] right = dfs(node.right);

    // Rob this node: children must NOT be robbed
    int rob = node.val + left[0] + right[0];

    // Don't rob: children free choice
    int notRob = Math.max(left[0], left[1])
              + Math.max(right[0], right[1]);

    return new int[]{notRob, rob};
}

# House Robber III - Binary Tree
# Time: O(n), Space: O(h) recursion stack
def rob(root):
    def dfs(node):
        if not node:
            return (0, 0)  # (not_rob, rob)

        left_not, left_rob = dfs(node.left)
        right_not, right_rob = dfs(node.right)

        # Rob this node: children must NOT be robbed
        rob_val = node.val + left_not + right_not

        # Don't rob: children free choice
        not_rob_val = max(left_not, left_rob) + max(right_not, right_rob)

        return (not_rob_val, rob_val)

    not_rob, rob_val = dfs(root)
    return max(not_rob, rob_val)

Problem 2: Maximum Independent Set on Tree

Find the largest set of nodes in a tree such that no two selected nodes are adjacent (connected by an edge). This generalizes House Robber III to unweighted trees.

         1
        /|\
       2  3  4
      / \    |
     5   6   7
         |
         8

Maximum Independent Set: {1, 5, 6, 7} or {5, 8, 3, 4} etc.
We want the LARGEST such set.

State: dp[v][0] = size of MIS in subtree(v) if v is NOT selected
       dp[v][1] = size of MIS in subtree(v) if v IS selected

Transition (same as House Robber with val=1):
  dp[v][1] = 1 + sum(dp[child][0]) for all children
  dp[v][0] = sum(max(dp[child][0], dp[child][1])) for all children

This is exactly House Robber III with every node having value 1.
Key Insight: Maximum Independent Set on a tree is equivalent to House Robber III where all node values are 1. The same DP structure applies: include/exclude each node, with the constraint that adjacent nodes cannot both be included.

Problem 3: Binary Tree Cameras

Place the minimum number of cameras on tree nodes such that every node is monitored. A camera monitors itself, its parent, and its direct children.

Example:
      0              Place camera at node 1:
     /               monitors nodes 0, 1, 2, 3
    1
   / \
  2   3

Only 1 camera needed!

Three states per node:
  State 0: COVERED   - node is monitored (by child or parent)
  State 1: HAS_CAMERA - node has a camera installed
  State 2: NOT_COVERED - node is NOT yet monitored

Transition:
  If any child is NOT_COVERED -> this node MUST have a camera (HAS_CAMERA)
  If any child HAS_CAMERA -> this node is COVERED (no camera needed)
  If all children are COVERED -> this node is NOT_COVERED (needs parent)

Post-order DFS: process children first, then decide for parent.

State Definition: Binary Tree Cameras

States per node (greedy approach):
0 = COVERED (monitored by a child's camera)
1 = HAS_CAMERA (camera placed here)
2 = NOT_COVERED (needs monitoring from parent)

Greedy rules (bottom-up):
- If any child is NOT_COVERED (state 2): place camera here (return state 1)
- If any child HAS_CAMERA (state 1): this node is covered (return state 0)
- If all children are COVERED (state 0): this node is NOT covered (return state 2)

Special case: After DFS, if root is NOT_COVERED, add one more camera.

Why greedy works: By placing cameras as close to leaves' parents as possible, each camera covers maximum nodes.

Step-by-Step: Binary Tree Cameras

         0
        / \
       1   2
      / \
     3   4

DFS post-order: 3, 4, 1, 2, 0

Node 3 (leaf): no children uncovered, no children with camera
  -> all children "covered" (vacuously) -> NOT_COVERED (state 2)

Node 4 (leaf): same -> NOT_COVERED (state 2)

Node 1: children 3 and 4 are NOT_COVERED
  -> MUST place camera here -> HAS_CAMERA (state 1), cameras = 1

Node 2 (leaf): no children -> NOT_COVERED (state 2)

Node 0: child 1 HAS_CAMERA, child 2 is NOT_COVERED
  -> child NOT_COVERED -> MUST place camera -> HAS_CAMERA (state 1), cameras = 2

Answer: 2 cameras (at nodes 1 and 0)

Code: Binary Tree Cameras

// Binary Tree Cameras (Greedy Tree DP)
// Time: O(n), Space: O(h)
// States: 0=COVERED, 1=HAS_CAMERA, 2=NOT_COVERED
int cameras = 0;

int dfs(TreeNode* node) {
    if (!node) return 0; // null is considered COVERED

    int left = dfs(node->left);
    int right = dfs(node->right);

    // If any child is NOT_COVERED, must place camera here
    if (left == 2 || right == 2) {
        cameras++;
        return 1; // HAS_CAMERA
    }

    // If any child HAS_CAMERA, this node is covered
    if (left == 1 || right == 1) {
        return 0; // COVERED
    }

    // All children are covered but none has camera -> NOT_COVERED
    return 2; // NOT_COVERED (need parent to cover)
}

int minCameraCover(TreeNode* root) {
    cameras = 0;
    if (dfs(root) == 2)
        cameras++; // root not covered, add camera
    return cameras;
}

// Binary Tree Cameras (Greedy Tree DP)
// Time: O(n), Space: O(h)
int cameras = 0;

public int minCameraCover(TreeNode root) {
    cameras = 0;
    if (dfs(root) == 2)
        cameras++; // root not covered
    return cameras;
}

// 0=COVERED, 1=HAS_CAMERA, 2=NOT_COVERED
private int dfs(TreeNode node) {
    if (node == null) return 0; // null is covered

    int left = dfs(node.left);
    int right = dfs(node.right);

    if (left == 2 || right == 2) {
        cameras++;
        return 1; // must place camera
    }

    if (left == 1 || right == 1) {
        return 0; // covered by child
    }

    return 2; // not covered
}

# Binary Tree Cameras (Greedy Tree DP)
# Time: O(n), Space: O(h)
# States: 0=COVERED, 1=HAS_CAMERA, 2=NOT_COVERED
def minCameraCover(root):
    cameras = [0]

    def dfs(node):
        if not node:
            return 0  # null is covered

        left = dfs(node.left)
        right = dfs(node.right)

        # If any child is NOT_COVERED, must place camera here
        if left == 2 or right == 2:
            cameras[0] += 1
            return 1  # HAS_CAMERA

        # If any child HAS_CAMERA, this node is covered
        if left == 1 or right == 1:
            return 0  # COVERED

        # All children covered, none has camera -> NOT_COVERED
        return 2

    if dfs(root) == 2:
        cameras[0] += 1  # root not covered

    return cameras[0]

Problem 4: All Longest Paths (Re-rooting)

For every node in a tree, find the length of the longest path starting from that node. This requires the re-rooting technique to achieve O(n) time.

         1
        / \
       2   3
      / \   \
     4   5   6

Longest path from each node:
  Node 1: longest = 2 (to 4, 5, or 6)
  Node 2: longest = 3 (2 -> 1 -> 3 -> 6)
  Node 3: longest = 3 (3 -> 1 -> 2 -> 4 or 5)
  Node 4: longest = 4 (4 -> 2 -> 1 -> 3 -> 6)
  Node 5: longest = 4 (5 -> 2 -> 1 -> 3 -> 6)
  Node 6: longest = 4 (6 -> 3 -> 1 -> 2 -> 4 or 5)

Naive: DFS from each node = O(n^2)
Re-rooting: Two DFS passes = O(n)

Pass 1: Compute down[v] = longest path going DOWN in subtree(v)
Pass 2: Compute up[v] = longest path going UP through parent
Answer for v: max(down[v], up[v])

Re-rooting: Two-Pass Technique

Pass 1 (bottom-up): For each node v, compute:
- down1[v] = longest downward path (through best child)
- down2[v] = second longest downward path (through second-best child)
- best_child[v] = which child gives down1

Pass 2 (top-down): For each node v with parent p:
- If v is best_child[p]: up[v] = 1 + max(up[p], down2[p])
- Otherwise: up[v] = 1 + max(up[p], down1[p])

Why two downward values? When computing up[v], we go through parent p. The parent's longest path might go THROUGH v (down1), which we cannot reuse. So we take down2 instead.

Answer: For each node v, max(down1[v], up[v]) 
Pass 1 results (rooted at 1):
  Node 4: down1=0
  Node 5: down1=0
  Node 2: down1=1 (via 4 or 5), down2=1, best=4
  Node 6: down1=0
  Node 3: down1=1 (via 6), down2=0
  Node 1: down1=2 (via 2), down2=2 (via 3), best=2

Pass 2 results:
  Node 1: up=0 (root)
  Node 2: v IS best_child[1] -> up = 1 + max(up[1]=0, down2[1]=2) = 3
  Node 3: v is NOT best_child[1] -> up = 1 + max(up[1]=0, down1[1]=2) = 3
  Node 4: v IS best_child[2] -> up = 1 + max(up[2]=3, down2[2]=1) = 4
  Node 5: v not best_child[2] -> up = 1 + max(up[2]=3, down1[2]=1) = 4
  Node 6: v IS best_child[3] -> up = 1 + max(up[3]=3, down2[3]=0) = 4

Final answers: max(down1[v], up[v])
  1: max(2, 0) = 2
  2: max(1, 3) = 3
  3: max(1, 3) = 3
  4: max(0, 4) = 4
  5: max(0, 4) = 4
  6: max(0, 4) = 4

Code: All Longest Paths (Re-rooting)

// All Longest Paths via Re-rooting
// Time: O(n), Space: O(n)
vector<vector<int>> adj;
vector<int> down1, down2, bestChild, up, answer;

void dfs1(int v, int parent) {
    down1[v] = down2[v] = 0;
    bestChild[v] = -1;

    for (int u : adj[v]) {
        if (u == parent) continue;
        dfs1(u, v);
        int d = down1[u] + 1;
        if (d >= down1[v]) {
            down2[v] = down1[v];
            down1[v] = d;
            bestChild[v] = u;
        } else if (d > down2[v]) {
            down2[v] = d;
        }
    }
}

void dfs2(int v, int parent) {
    for (int u : adj[v]) {
        if (u == parent) continue;
        // If u is the best child, use second-best path from v
        if (u == bestChild[v])
            up[u] = 1 + max(up[v], down2[v]);
        else
            up[u] = 1 + max(up[v], down1[v]);
        dfs2(u, v);
    }
}

vector<int> allLongestPaths(int n) {
    down1.resize(n); down2.resize(n);
    bestChild.resize(n); up.resize(n, 0);
    answer.resize(n);

    dfs1(0, -1);
    dfs2(0, -1);

    for (int i = 0; i < n; i++)
        answer[i] = max(down1[i], up[i]);

    return answer;
}

# Time: O(n), Space: O(n)
import sys
sys.setrecursionlimit(200000)

def all_longest_paths(adj, n):
    down1 = [0] * n
    down2 = [0] * n
    best_child = [-1] * n
    up = [0] * n

    def dfs1(v, parent):
        for u in adj[v]:
            if u == parent:
                continue
            dfs1(u, v)
            d = down1[u] + 1
            if d >= down1[v]:
                down2[v] = down1[v]
                down1[v] = d
                best_child[v] = u
            elif d > down2[v]:
                down2[v] = d

    def dfs2(v, parent):
        for u in adj[v]:
            if u == parent:
                continue
            if u == best_child[v]:
                up[u] = 1 + max(up[v], down2[v])
            else:
                up[u] = 1 + max(up[v], down1[v])
            dfs2(u, v)

    dfs1(0, -1)
    dfs2(0, -1)

    return [max(down1[i], up[i]) for i in range(n)]

Complexity Summary

ProblemTimeSpaceStates per Node
House Robber IIIO(n)O(h)2 (rob/not rob)
Max Independent SetO(n)O(h)2 (include/exclude)
Binary Tree CamerasO(n)O(h)3 (covered/camera/not covered)

  • h = tree height: O(log n) for balanced, O(n) worst case for path/skewed tree
  • All problems are O(n): Each node is visited a constant number of times in DFS
  • Re-rooting pattern: Use whenever you need the answer for EVERY node as if it were the root. Avoids O(n^2) brute force.
  • Greedy on tree (cameras): Works because trees have no cycles, so local decisions propagate correctly bottom-up

Key Insights

Include/Exclude pattern: When you cannot select adjacent nodes, use 2 states: dp[v][0] (exclude v) and dp[v][1] (include v). This is the most common tree DP pattern.

Multi-state greedy: Binary Tree Cameras uses 3 states instead of 2. The greedy choice (place camera only when a child is uncovered) is provably optimal on trees.

Re-rooting: The key trick is storing the top-2 downward paths per node. When a child asks "what is the best path NOT going through me?", the answer is either the second-best downward path or the upward path through parent.
