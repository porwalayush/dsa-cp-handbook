Space Optimization Techniques

Many DP problems use O(n*m) space with a 2D table, but careful analysis of dependencies often allows reduction to O(m), O(2*m), or even O(1). Here are the three main techniques with before/after examples.

When to Apply Each Technique

   Dependency Analysis: What does dp[i][j] depend on?

   ┌───────────────────────────────────────────────────────────────┐
   │ Depends on             │ Technique            │ Space         │
   ├───────────────────────────────────────────────────────────────┤
   │ dp[i-1][...]           │ Rolling array (2 rows)│ O(2*m)       │
   │ dp[i-1][j], dp[i][j-1]│ Single row            │ O(m)         │
   │ dp[i-1] and dp[i-2]   │ Two variables          │ O(1)         │
   │   (scalar only)        │                       │              │
   └───────────────────────────────────────────────────────────────┘

   Key question: Does row i depend ONLY on row i-1?
   If yes → you only need 2 rows (or sometimes just 1)!

Technique 1: Rolling Array (2 Rows)

When the current row depends on the entire previous row, keep exactly 2 rows and alternate between them.

When it applies

Pattern: dp[i][j] depends on dp[i-1][j], dp[i-1][j-1], dp[i-1][j+1], etc.

Examples: LCS, Edit Distance, Grid paths (if looking at row above and diagonal)

Reduction: O(n*m) → O(2*m) = O(m)

Before: LCS with full 2D table

   dp[i][j] = LCS of first i chars of s1, first j chars of s2

   Dependency: dp[i][j] uses dp[i-1][j-1], dp[i-1][j], dp[i][j-1]
                              ↑ diag above   ↑ above     ↑ left

   Full table (m+1 x n+1):
   Row 0: [0, 0, 0, 0, 0, 0]
   Row 1: [0, 1, 1, 1, 1, 1]  ← depends on row 0
   Row 2: [0, 1, 1, 2, 2, 2]  ← depends on row 1 (NOT row 0!)
   Row 3: [0, 1, 1, 2, 2, 3]  ← depends on row 2

   Observation: Row i only needs row i-1. Rows 0..i-2 are waste!

After: Rolling array

// LCS: O(m*n) time, O(min(m,n)) space with rolling array
int lcs(string& s1, string& s2) {
    int m = s1.size(), n = s2.size();
    // Use shorter string for columns (further optimization)
    if (m < n) return lcs(s2, s1);

    vector<int> prev(n + 1, 0), curr(n + 1, 0);

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (s1[i-1] == s2[j-1]) {
                curr[j] = prev[j-1] + 1;
            } else {
                curr[j] = max(prev[j], curr[j-1]);
            }
        }
        swap(prev, curr); // rotate rows
        fill(curr.begin(), curr.end(), 0);
    }
    return prev[n];
}

// LCS: O(m*n) time, O(min(m,n)) space with rolling array
static int lcs(String s1, String s2) {
    int m = s1.length(), n = s2.length();
    if (m < n) return lcs(s2, s1);

    int[] prev = new int[n + 1];
    int[] curr = new int[n + 1];

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (s1.charAt(i-1) == s2.charAt(j-1)) {
                curr[j] = prev[j-1] + 1;
            } else {
                curr[j] = Math.max(prev[j], curr[j-1]);
            }
        }
        int[] temp = prev; prev = curr; curr = temp;
        Arrays.fill(curr, 0);
    }
    return prev[n];
}

# LCS: O(m*n) time, O(min(m,n)) space with rolling array
def lcs(s1, s2):
    if len(s1) < len(s2):
        s1, s2 = s2, s1
    m, n = len(s1), len(s2)

    prev = [0] * (n + 1)
    curr = [0] * (n + 1)

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                curr[j] = prev[j-1] + 1
            else:
                curr[j] = max(prev[j], curr[j-1])
        prev, curr = curr, [0] * (n + 1)
    return prev[n]

Technique 2: Single Row Optimization

When the current cell depends on the cell directly above AND the cell to its left in the same row, a single row suffices with careful update order.

When it applies

Pattern: dp[i][j] depends on dp[i-1][j] and dp[i-1][j-1] only (no dp[i][j-1])
OR: dp[i][j] depends on dp[i-1][j] and dp[i][j-1] (but NOT dp[i-1][j-1])

Examples: 0/1 Knapsack, Subset Sum, Coin Change

Key insight: Iterate right-to-left to preserve "previous row" values that haven't been overwritten yet

Before: 0/1 Knapsack 2D

   dp[i][w] = max(dp[i-1][w], dp[i-1][w-wt[i]] + val[i])
                  ↑ above       ↑ above-left

   When filling left-to-right in a single row:
   dp[w] = max(dp[w], dp[w-wt[i]] + val[i])
                ↑ still "old" value (acts as dp[i-1][w])

   Problem: dp[w-wt[i]] might already be updated!
   Solution: iterate RIGHT to LEFT

   w=7: dp[7] uses dp[7-wt[i]] which hasn't been touched yet ✓
   w=6: dp[6] uses dp[6-wt[i]] which hasn't been touched yet ✓
   ...continues safely

After: Single row, right-to-left

// 0/1 Knapsack: O(n*W) time, O(W) space
int knapsack(vector<int>& wt, vector<int>& val, int W) {
    vector<int> dp(W + 1, 0);

    for (int i = 0; i < wt.size(); i++) {
        for (int w = W; w >= wt[i]; w--) { // RIGHT to LEFT
            dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);
        }
    }
    return dp[W];
}

// Unbounded Knapsack: same structure, LEFT to RIGHT
int unbounded(vector<int>& wt, vector<int>& val, int W) {
    vector<int> dp(W + 1, 0);

    for (int i = 0; i < wt.size(); i++) {
        for (int w = wt[i]; w <= W; w++) { // LEFT to RIGHT
            dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);
        }
    }
    return dp[W];
}

// 0/1 Knapsack: O(n*W) time, O(W) space

        for (int w = W; w >= wt[i]; w--) { // RIGHT to LEFT
            dp[w] = Math.max(dp[w], dp[w - wt[i]] + val[i]);
        }
    }
    return dp[W];
}

# 0/1 Knapsack: O(n*W) time, O(W) space
def knapsack(wt, val, W):
    dp = [0] * (W + 1)

    for i in range(len(wt)):
        for w in range(W, wt[i] - 1, -1):  # RIGHT to LEFT
            dp[w] = max(dp[w], dp[w - wt[i]] + val[i])
    return dp[W]

# Unbounded: LEFT to RIGHT

    dp = [0] * (W + 1)

    for i in range(len(wt)):
        for w in range(wt[i], W + 1):  # LEFT to RIGHT
            dp[w] = max(dp[w], dp[w - wt[i]] + val[i])
    return dp[W]

Technique 3: Two Variables (Fibonacci-Style)

When dp[i] depends only on dp[i-1] and dp[i-2] (scalars, not arrays), just keep two variables.

When it applies

Pattern: dp[i] = f(dp[i-1], dp[i-2]) where each state is a single number

Examples: Fibonacci, Climbing Stairs, House Robber, Tiling problems

Reduction: O(n) → O(1)

Before: Fibonacci with array

   dp[0] = 0, dp[1] = 1
   dp[i] = dp[i-1] + dp[i-2]

   Array: [0, 1, 1, 2, 3, 5, 8, 13, 21, ...]
                           only need these two ──┘  └── to compute this

   We store n+1 values but only ever READ the last 2!

After: Two variables

// Fibonacci: O(n) time, O(1) space
int fib(int n) {
    if (n <= 1) return n;
    int prev2 = 0, prev1 = 1;

    for (int i = 2; i <= n; i++) {
        int curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

// House Robber: O(n) time, O(1) space
int rob(vector<int>& nums) {
    int prev2 = 0, prev1 = 0;

    for (int num : nums) {
        int curr = max(prev1, prev2 + num);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

# Fibonacci: O(n) time, O(1) space
def fib(n):
    if n <= 1:
        return n
    prev2, prev1 = 0, 1

    for _ in range(2, n + 1):
        prev2, prev1 = prev1, prev1 + prev2
    return prev1

# House Robber: O(n) time, O(1) space
def rob(nums):
    prev2, prev1 = 0, 0

    for num in nums:
        prev2, prev1 = prev1, max(prev1, prev2 + num)
    return prev1

Special Case: Single Row with Temp Variable

For LCS-style problems where you need dp[i-1][j-1] (diagonal), a single row works if you save the diagonal value before overwriting.

// LCS with truly single row + temp variable
int lcs(string& s1, string& s2) {
    int m = s1.size(), n = s2.size();
    vector<int> dp(n + 1, 0);

    for (int i = 1; i <= m; i++) {
        int prev = 0; // saves dp[i-1][j-1] (the diagonal)
        for (int j = 1; j <= n; j++) {
            int temp = dp[j]; // save before overwrite
            if (s1[i-1] == s2[j-1]) {
                dp[j] = prev + 1; // diagonal + 1
            } else {
                dp[j] = max(dp[j], dp[j-1]); // above vs left
            }
            prev = temp; // prev becomes diagonal for next j
        }
    }
    return dp[n];
}

// LCS with single row + temp variable
static int lcs(String s1, String s2) {
    int m = s1.length(), n = s2.length();
    int[] dp = new int[n + 1];

    for (int i = 1; i <= m; i++) {
        int prev = 0;
        for (int j = 1; j <= n; j++) {
            int temp = dp[j];
            if (s1.charAt(i-1) == s2.charAt(j-1)) {
                dp[j] = prev + 1;
            } else {
                dp[j] = Math.max(dp[j], dp[j-1]);
            }
            prev = temp;
        }
    }
    return dp[n];
}

# LCS with single row + temp variable
def lcs(s1, s2):
    m, n = len(s1), len(s2)
    dp = [0] * (n + 1)

    return dp[n]

Decision Flowchart

   Is your DP 1D (dp[i] depends on dp[i-1], dp[i-2])?
   └── YES → Two variables (O(1))

   Is your DP 2D where row i depends ONLY on row i-1?
   └── YES → Does it need the diagonal (dp[i-1][j-1])?
       ├── NO  → Single row, appropriate direction
       └── YES → Rolling array (2 rows) OR single row + temp var

   Does the inner loop direction matter?
   └── 0/1 (each item once): RIGHT to LEFT
       Unbounded (reuse OK): LEFT to RIGHT

   Is there a 3D DP (e.g., dp[i][j][k])?
   └── Often reducible to 2D by similar analysis

Common Pitfalls

  • Wrong direction: Right-to-left for 0/1 knapsack, left-to-right for unbounded. Mixing them up gives wrong answers silently.
  • Forgetting to reset: When using rolling arrays, remember to clear the "new" row before filling it.
  • Lost information: Space optimization sacrifices the ability to reconstruct the solution path. If you need the actual items/choices, keep the full table.
  • Diagonal dependency: For LCS/Edit Distance, save the diagonal value in a temp variable before overwriting.
  • Off-by-one with swap: After the loop, the answer is in prev (not curr) because we just swapped.
