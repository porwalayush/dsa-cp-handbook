State Machine DP

Model problems as transitions between states. The classic application is stock buy/sell problems where you track whether you're holding stock, not holding, or in cooldown.

The State Machine Idea

   Instead of one dp value per day, maintain MULTIPLE states:

   ┌─────────────┐     buy      ┌─────────────┐
   │  NOT HOLDING │ ──────────→ │   HOLDING    │
   │  (can buy)   │ ←────────── │  (can sell)  │
   └─────────────┘     sell      └─────────────┘
        ↓  ↑                          ↓  ↑
        rest                          rest

   Each day: choose an ACTION based on current STATE
   Track max profit in each state at each time step

1. Buy/Sell with Cooldown

After selling, you must wait one day before buying again (cooldown).

   prices = [1, 2, 3, 0, 2]

   States: HOLD, SOLD (just sold today), REST (not holding, can buy)

        ┌──────┐  sell   ┌──────┐  rest   ┌──────┐
        │ HOLD │ ──────→ │ SOLD │ ──────→ │ REST │
        └──────┘         └──────┘         └──────┘
           ↑                                   │
           └───────────── buy ─────────────────┘
           ↑               rest                 │
           └────────────────────────────────────┘
                         (HOLD keeps holding)

State & Transition

States on day i:
hold[i] = max profit while holding stock
sold[i] = max profit having just sold today
rest[i] = max profit not holding (free to buy)

Transitions:
hold[i] = max(hold[i-1], rest[i-1] - price[i])   # keep or buy
sold[i] = hold[i-1] + price[i]   # sell today
rest[i] = max(rest[i-1], sold[i-1])   # rest or was sold yesterday (cooldown done)

Base: hold[0] = -price[0], sold[0] = 0, rest[0] = 0
Answer: max(sold[n-1], rest[n-1])   # can't end holding

Visual Walkthrough

   prices = [1, 2, 3, 0, 2]

   Day 0 (price=1): hold=-1, sold=0, rest=0
   Day 1 (price=2): hold=max(-1, 0-2)=-1, sold=-1+2=1, rest=max(0,0)=0
   Day 2 (price=3): hold=max(-1, 0-3)=-1, sold=-1+3=2, rest=max(0,1)=1
   Day 3 (price=0): hold=max(-1, 1-0)=1,  sold=-1+0=-1, rest=max(1,2)=2
   Day 4 (price=2): hold=max(1, 2-2)=1,   sold=1+2=3,  rest=max(2,-1)=2

   Answer: max(sold[4], rest[4]) = max(3, 2) = 3
   Strategy: buy@1, sell@3 (profit 2), cooldown, buy@0, sell@2 (profit 2) = 3?
   Actually: buy@1, sell@2, cooldown, rest, ... OR buy@1, sell@3, cd, buy@0, sell@2
   Profit = (3-1) + (2-0) = 2 + 2 = ... wait sold[4]=hold[3]+price[4]=1+2=3
   So: buy@0 (price 0 on day 3), sell@2 (price 2 on day 4) from state where rest=2
   Total: 3

2. Buy/Sell with k Transactions

You may complete at most k buy-sell transactions.

State: dp[t][0] = max profit after t transactions, not holding
dp[t][1] = max profit after t transactions, holding

Transition (for each day's price):
dp[t][0] = max(dp[t][0], dp[t][1] + price)   # rest or sell
dp[t][1] = max(dp[t][1], dp[t-1][0] - price)   # rest or buy (uses t-1 for new txn)

Base: dp[t][0] = 0, dp[t][1] = -infinity for all t
Answer: dp[k][0] 
   prices = [3, 2, 6, 5, 0, 3], k = 2

   Transaction 1: buy@2, sell@6 → profit 4
   Transaction 2: buy@0, sell@3 → profit 3
   Total: 7

   State table evolution:
   t=0:  dp[0][0]=0                  (no transactions, no stock)
   t=1:  dp[1][0]=max profit with 1 sell done
         dp[1][1]=max profit holding with 1 buy done
   t=2:  dp[2][0]=max profit with 2 sells done

   Special case: if k >= n/2, unlimited transactions (greedy)

Code: Buy/Sell with Cooldown

// Stock with Cooldown: O(n) time, O(1) space
int maxProfit(vector<int>& prices) {
    int n = prices.size();
    if (n <= 1) return 0;

    int hold = -prices[0];  // holding stock
    int sold = 0;           // just sold
    int rest = 0;           // not holding, free to buy

    for (int i = 1; i < n; i++) {
        int prevHold = hold;
        hold = max(hold, rest - prices[i]);    // keep or buy
        rest = max(rest, sold);                 // rest or cooldown done
        sold = prevHold + prices[i];            // sell today
    }
    return max(sold, rest);
}

// Stock with Cooldown: O(n) time, O(1) space
static int maxProfit(int[] prices) {
    int n = prices.length;
    if (n <= 1) return 0;

    int hold = -prices[0];
    int sold = 0;
    int rest = 0;

    for (int i = 1; i < n; i++) {
        int prevHold = hold;
        hold = Math.max(hold, rest - prices[i]);
        rest = Math.max(rest, sold);
        sold = prevHold + prices[i];
    }
    return Math.max(sold, rest);
}

# Stock with Cooldown: O(n) time, O(1) space
def max_profit_cooldown(prices):
    if len(prices) <= 1:
        return 0

    hold = -prices[0]  # holding stock
    sold = 0            # just sold today
    rest = 0            # not holding, free to buy

    for price in prices[1:]:
        prev_hold = hold
        hold = max(hold, rest - price)   # keep or buy
        rest = max(rest, sold)            # rest or cooldown done
        sold = prev_hold + price          # sell today

    return max(sold, rest)

Code: Buy/Sell with k Transactions

// Stock with at most k transactions: O(n*k) time, O(k) space
int maxProfit(int k, vector<int>& prices) {
    int n = prices.size();
    if (n == 0) return 0;

    // If k >= n/2, unlimited transactions (greedy)
    if (k >= n / 2) {
        int profit = 0;
        for (int i = 1; i < n; i++)
            profit += max(0, prices[i] - prices[i-1]);
        return profit;
    }

    // dp[t][0] = max profit, t txns done, not holding
    // dp[t][1] = max profit, t txns done, holding
    vector<vector<int>> dp(k + 1, vector<int>(2, 0));
    for (int t = 0; t <= k; t++) dp[t][1] = INT_MIN;

    for (int price : prices) {
        for (int t = 1; t <= k; t++) {
            // sell: complete transaction t
            dp[t][0] = max(dp[t][0], dp[t][1] + price);
            // buy: start transaction t (requires t-1 done)
            dp[t][1] = max(dp[t][1], dp[t-1][0] - price);
        }
    }
    return dp[k][0];
}

// Stock with at most k transactions: O(n*k) time, O(k) space
static int maxProfit(int k, int[] prices) {
    int n = prices.length;
    if (n == 0) return 0;

    // If k >= n/2, unlimited transactions
    if (k >= n / 2) {
        int profit = 0;
        for (int i = 1; i < n; i++)
            profit += Math.max(0, prices[i] - prices[i-1]);
        return profit;
    }

    int[][] dp = new int[k + 1][2];
    for (int t = 0; t <= k; t++) dp[t][1] = Integer.MIN_VALUE;

    for (int price : prices) {
        for (int t = 1; t <= k; t++) {
            dp[t][0] = Math.max(dp[t][0], dp[t][1] + price);
            dp[t][1] = Math.max(dp[t][1], dp[t-1][0] - price);
        }
    }
    return dp[k][0];
}

# Stock with at most k transactions: O(n*k) time, O(k) space
def max_profit_k(k, prices):
    n = len(prices)
    if n == 0:
        return 0

    # If k >= n/2, unlimited transactions
    if k >= n // 2:
        return sum(max(0, prices[i] - prices[i-1])
                   for i in range(1, n))

    # dp[t][0] = not holding after t txns
    # dp[t][1] = holding during t-th txn
    dp = [[0, float('-inf')] for _ in range(k + 1)]

    for price in prices:
        for t in range(1, k + 1):
            dp[t][0] = max(dp[t][0], dp[t][1] + price)   # sell
            dp[t][1] = max(dp[t][1], dp[t-1][0] - price) # buy
    return dp[k][0]

Code: With Transaction Fee (Bonus)

// Unlimited transactions with fee: O(n) time, O(1) space
int maxProfit(vector<int>& prices, int fee) {
    int cash = 0;          // not holding
    int hold = -prices[0]; // holding

    for (int i = 1; i < prices.size(); i++) {
        cash = max(cash, hold + prices[i] - fee); // sell (pay fee)
        hold = max(hold, cash - prices[i]);        // buy
    }
    return cash;
}

// Unlimited transactions with fee: O(n) time, O(1) space
static int maxProfit(int[] prices, int fee) {
    int cash = 0;
    int hold = -prices[0];

    for (int i = 1; i < prices.length; i++) {
        cash = Math.max(cash, hold + prices[i] - fee);
        hold = Math.max(hold, cash - prices[i]);
    }
    return cash;
}

# Unlimited transactions with fee: O(n) time, O(1) space
def max_profit_fee(prices, fee):
    cash = 0             # not holding
    hold = -prices[0]   # holding

    for price in prices[1:]:
        cash = max(cash, hold + price - fee)  # sell
        hold = max(hold, cash - price)         # buy
    return cash

General State Machine Template

  • Identify states: What are the possible "modes" at each step? (holding, not-holding, cooldown, etc.)
  • Draw transitions: What actions move between states? (buy, sell, rest)
  • Write recurrences: Each state depends on previous states via transitions
  • Optimize space: Usually only need previous day's values (O(1) or O(k))
  • Watch for order: When updating multiple states, use temp variables to avoid clobbering

Complexity Summary

  • With Cooldown: O(n) time, O(1) space (3 variables)
  • With k Transactions: O(n*k) time, O(k) space
  • With Fee: O(n) time, O(1) space (2 variables)
  • Key optimization: when k >= n/2, reduce to unlimited case (greedy O(n))
