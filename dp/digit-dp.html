Digit DP

Count numbers in a range [L, R] satisfying some digit-based property. Build numbers digit by digit, tracking whether we are still constrained by the upper bound ("tight" constraint).

Core Concept

Problem: Count numbers from 1 to N with digit sum = S

Key Idea: Build the number from most significant digit to least.
At each position, decide which digit (0-9) to place.

The "tight" flag:
  tight = true  → previous digits exactly match N's prefix
                  → current digit can be at most N's digit at this position
  tight = false → previous digits are already less than N's prefix
                  → current digit can be anything 0-9

Example: N = 345, counting numbers ≤ 345
  Position 0: if we pick 3 (same as N[0]), tight stays true
              if we pick 0,1,2 → tight becomes false (number < N guaranteed)
              we CANNOT pick 4-9 (would exceed N)

  Position 1 (tight=true): limit is 4 (N[1])
  Position 1 (tight=false): limit is 9 (any digit OK)

State Definition

State: dp[pos][tight][state]

  • pos: current digit position (0 to len-1, left to right)
  • tight: boolean - are we still bounded by N?
  • state: problem-specific (e.g., digit sum so far, last digit, etc.)

Transition: For each valid digit d at position pos:
dp[pos][tight][state] += dp[pos+1][new_tight][new_state]
where new_tight = tight AND (d == N[pos])

Base Case: dp[len][*][target_state] = 1 (valid number found), else 0

Range [L, R]: solve(R) - solve(L-1)

Example: Count Numbers with Digit Sum = S

Problem: Count integers in [1, N] whose digit sum equals S.

State: dp[pos][tight][sum_so_far]
Transition: For digit d from 0 to limit:
  limit = N[pos] if tight, else 9
  dp[pos][tight][sum] += dp[pos+1][tight && d==limit][sum+d]
Base: If pos == len, return 1 if sum_so_far == S, else 0

Code: Digit Sum Count

// Count numbers in [1, N] with digit sum = S
// Time: O(digits * 10 * S), Space: O(digits * 2 * S)
string num;
int memo[20][2][200]; // pos, tight, sum

int solve(int pos, bool tight, int sum, int S) {
    if (sum > S) return 0; // pruning
    if (pos == num.size()) return sum == S ? 1 : 0;

    if (memo[pos][tight][sum] != -1)
        return memo[pos][tight][sum];

    int limit = tight ? (num[pos] - '0') : 9;
    int result = 0;

    for (int d = 0; d <= limit; d++) {
        result += solve(pos + 1,
                       tight && (d == limit),
                       sum + d, S);
    }

    return memo[pos][tight][sum] = result;
}

int countUpTo(long long N, int S) {
    num = to_string(N);
    memset(memo, -1, sizeof(memo));
    return solve(0, true, 0, S);
}

// For range [L, R]: countUpTo(R, S) - countUpTo(L-1, S)

// Count numbers in [1, N] with digit sum = S
// Time: O(digits * 10 * S), Space: O(digits * 2 * S)
static String num;
static int[][][] memo;

static int solve(int pos, int tight, int sum, int S) {
    if (sum > S) return 0;
    if (pos == num.length()) return sum == S ? 1 : 0;

    if (memo[pos][tight][sum] != -1)
        return memo[pos][tight][sum];

    int limit = (tight == 1) ? (num.charAt(pos) - '0') : 9;
    int result = 0;

    for (int d = 0; d <= limit; d++) {
        result += solve(pos + 1,
                       (tight == 1 && d == limit) ? 1 : 0,
                       sum + d, S);
    }

    return memo[pos][tight][sum] = result;
}

static int countUpTo(long N, int S) {
    num = Long.toString(N);
    memo = new int[20][2][200];
    for (int[][] a : memo)
        for (int[] b : a) Arrays.fill(b, -1);
    return solve(0, 1, 0, S);
}

# Count numbers in [1, N] with digit sum = S
# Time: O(digits * 10 * S), Space: O(digits * 2 * S)
from functools import lru_cache

def count_up_to(N, S):
    digits = str(N)
    n = len(digits)

    @lru_cache(maxsize=None)
    def solve(pos, tight, curr_sum):
        if curr_sum > S:
            return 0
        if pos == n:
            return 1 if curr_sum == S else 0

        limit = int(digits[pos]) if tight else 9
        result = 0

        for d in range(limit + 1):
            result += solve(pos + 1,
                           tight and (d == limit),
                           curr_sum + d)

        return result

    return solve(0, True, 0)

# For range [L, R]: count_up_to(R, S) - count_up_to(L-1, S)

Step-by-Step: N=345, S=12

Step 1: Convert N to string: "345". We process positions 0, 1, 2.
Step 2: At pos=0, tight=true, limit=3.
Try d=0: solve(1, false, 0) - free to place any digits later
Try d=1: solve(1, false, 1)
Try d=2: solve(1, false, 2)
Try d=3: solve(1, true, 3) - still tight!
Step 3: For the branch d=3, tight=true at pos=1, limit=4.
Try d=0 to d=4. For d=4, tight stays true.
For d=0..3, tight becomes false, giving full freedom at pos=2.
Step 4: At pos=2 (base case): return 1 if accumulated sum = 12, else 0.
Example valid number: 345 has sum 3+4+5=12. It gets counted!

Handling Range [L, R]

count(L, R) = count(1, R) - count(1, L-1)

Why this works:
  count(1, R) = numbers from 1 to R satisfying property
  count(1, L-1) = numbers from 1 to L-1 satisfying property
  Difference = numbers from L to R satisfying property

Edge case: if L = 0, handle separately (check if 0 satisfies property)

Complexity

  • Time: O(D * 10 * |states|) where D = number of digits in N
  • Space: O(D * 2 * |states|) for memoization table
  • D is small: for N up to 10^18, D ≤ 19
  • Key insight: Despite N being huge, the number of distinct states is small

// General Digit DP Template
string num;
int memo[20][2][MAX_STATE];

int solve(int pos, bool tight, int state) {
    // Base case: all digits placed
    if (pos == num.size())
        return isValid(state) ? 1 : 0;

    if (memo[pos][tight][state] != -1)
        return memo[pos][tight][state];

    int limit = tight ? (num[pos] - '0') : 9;
    int result = 0;

    for (int d = 0; d <= limit; d++) {
        int newState = transition(state, d);
        bool newTight = tight && (d == limit);
        result += solve(pos + 1, newTight, newState);
    }

    return memo[pos][tight][state] = result;
}

int countInRange(long long L, long long R) {
    num = to_string(R);
    memset(memo, -1, sizeof(memo));
    int high = solve(0, true, initialState);

    num = to_string(L - 1);
    memset(memo, -1, sizeof(memo));
    int low = solve(0, true, initialState);

    return high - low;
}

from functools import lru_cache

def count_in_range(L, R, target):
    def count_up_to(N):
        digits = str(N)
        n = len(digits)

        @lru_cache(maxsize=None)
        def solve(pos, tight, state):
            if pos == n:
                return 1 if is_valid(state) else 0

            limit = int(digits[pos]) if tight else 9
            result = 0

            for d in range(limit + 1):
                new_state = transition(state, d)
                new_tight = tight and (d == limit)
                result += solve(pos + 1, new_tight, new_state)

            return result

        return solve(0, True, initial_state)

    return count_up_to(R) - count_up_to(L - 1)

Key Insights

  • Leading zeros: Sometimes need a started flag to avoid counting leading zeros as actual zeros
  • Tight constraint: Once tight becomes false, it stays false for all subsequent positions
  • Memoization works: Because once tight=false, the subproblem is the same regardless of how we got there
  • Large N: Works for N up to 10^18 since we only track digit count (max 19 digits)
  • Multiple constraints: Add more state dimensions (e.g., tight_lower for lower bound directly)
