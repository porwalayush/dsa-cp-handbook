Unbounded Knapsack

A variant of the knapsack problem where each item can be selected unlimited times. The classic application is the Rod Cutting problem.

Problem Statement

   Given: n items, each with weight w[i] and value v[i]
   Knapsack capacity: W
   Each item can be used UNLIMITED times

   Example:
   items:    weight = [2, 3, 4, 5]
             value  = [3, 4, 5, 8]
   Capacity W = 8

   Take item 4 once (w=5,v=8) + item 1 once (w=2,v=3) = w:7, v:11
   Take item 1 four times (w=2*4=8, v=3*4=12) → Better!

   Answer: 12 (four copies of item 1)

Key Difference from 0/1 Knapsack

   0/1 Knapsack:     for w = W down to wt[i]   (RIGHT to LEFT)
   Unbounded:        for w = wt[i] up to W      (LEFT to RIGHT)

   Why LEFT to RIGHT works for unbounded:
   ┌─────────────────────────────────────────────────┐
   │ When computing dp[w], dp[w - wt[i]] may already │
   │ include item i (updated earlier in this pass).   │
   │ That's FINE — we WANT to reuse items!          │
   └─────────────────────────────────────────────────┘

   0/1: dp[w] depends on dp[w-wt[i]] from PREVIOUS row → go right-to-left
   Unbounded: dp[w] depends on dp[w-wt[i]] from SAME row → go left-to-right

State & Transition

State: dp[w] = maximum value achievable with capacity w

Transition:
For each item i: dp[w] = max(dp[w], dp[w - wt[i]] + val[i])
Process w from left to right to allow reuse

Base Case: dp[0] = 0 (zero capacity = zero value)

Visual Walkthrough

   wt = [2, 3, 5], val = [3, 4, 8], W = 7

   Process item 0 (w=2, v=3):  LEFT to RIGHT
   dp: [0, 0, 3, 3, 6, 6, 9, 9]
       dp[2]=3, dp[4]=dp[2]+3=6, dp[6]=dp[4]+3=9 (reuse!)

   Process item 1 (w=3, v=4):
   dp: [0, 0, 3, 4, 6, 7, 9, 10]
       dp[3]=max(3,4)=4, dp[5]=max(6, dp[2]+4)=7
       dp[7]=max(9, dp[4]+4)=10

   Process item 2 (w=5, v=8):
   dp: [0, 0, 3, 4, 6, 8, 9, 11]
       dp[5]=max(7,8)=8, dp[7]=max(10, dp[2]+8)=11

   Answer: dp[7] = 11 (item2 + item0 = w:5+2=7, v:8+3=11)

Rod Cutting: Classic Example

Given a rod of length n and prices for each length, find the maximum revenue from cutting and selling pieces.

   Rod length: 8
   Length:  1  2  3  4  5  6  7  8
   Price:   1  5  8  9 10 17 17 20

   This IS unbounded knapsack:
   - "items" = possible piece lengths (1 to n)
   - weight[i] = length i
   - value[i]  = price[i]
   - capacity  = rod length n

   Each piece length can be used unlimited times!

   Optimal for n=8: cut into 2+6 → price 5+17 = 22
   (better than selling whole rod for 20)

Code: Unbounded Knapsack

// Unbounded Knapsack: O(n*W) time, O(W) space
int unboundedKnapsack(vector<int>& wt, vector<int>& val, int W) {
    vector<int> dp(W + 1, 0);

    for (int i = 0; i < wt.size(); i++) {
        for (int w = wt[i]; w <= W; w++) { // LEFT to RIGHT!
            dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);
        }
    }
    return dp[W];
}

// Unbounded Knapsack: O(n*W) time, O(W) space
static int unboundedKnapsack(int[] wt, int[] val, int W) {
    int[] dp = new int[W + 1];

    for (int i = 0; i < wt.length; i++) {
        for (int w = wt[i]; w <= W; w++) { // LEFT to RIGHT!
            dp[w] = Math.max(dp[w], dp[w - wt[i]] + val[i]);
        }
    }
    return dp[W];
}

# Unbounded Knapsack: O(n*W) time, O(W) space
def unbounded_knapsack(wt, val, W):
    dp = [0] * (W + 1)

    for i in range(len(wt)):
        for w in range(wt[i], W + 1):  # LEFT to RIGHT!
            dp[w] = max(dp[w], dp[w - wt[i]] + val[i])
    return dp[W]

Code: Rod Cutting

// Rod Cutting: O(n^2) time, O(n) space
int rodCutting(vector<int>& price, int n) {
    // price[i] = value of rod of length (i+1)
    vector<int> dp(n + 1, 0);

    for (int len = 1; len <= n; len++) { // each piece length
        for (int w = len; w <= n; w++) { // LEFT to RIGHT
            dp[w] = max(dp[w], dp[w - len] + price[len - 1]);
        }
    }
    return dp[n];
}

// Rod Cutting: O(n^2) time, O(n) space
static int rodCutting(int[] price, int n) {
    int[] dp = new int[n + 1];

    for (int len = 1; len <= n; len++) {
        for (int w = len; w <= n; w++) { // LEFT to RIGHT
            dp[w] = Math.max(dp[w], dp[w - len] + price[len - 1]);
        }
    }
    return dp[n];
}

# Rod Cutting: O(n^2) time, O(n) space
def rod_cutting(price, n):
    dp = [0] * (n + 1)

    for length in range(1, n + 1):  # each piece length
        for w in range(length, n + 1):  # LEFT to RIGHT
            dp[w] = max(dp[w], dp[w - length] + price[length - 1])
    return dp[n]

Other Unbounded Knapsack Variants

Coin Change (Min coins): dp[w] = min coins to make amount w
dp[w] = min(dp[w], dp[w - coin] + 1) — left to right

Coin Change (Count ways): dp[w] = number of ways to make amount w
dp[w] += dp[w - coin] — left to right

Integer Break: maximize product of parts summing to n
dp[w] = max(dp[w], dp[w - i] * i) for i from 1 to w

0/1 vs Unbounded: Side by Side

Aspect0/1 KnapsackUnbounded Knapsack
Item usageAt most onceUnlimited times
1D loop directionw: W → wt[i] (right to left)w: wt[i] → W (left to right)
Why it worksUses previous row valuesUses current row values (reuse OK)
ExamplesSubset sum, partitionRod cutting, coin change
TimeO(n * W)O(n * W)
SpaceO(W)O(W)

Complexity

  • Time: O(n * W) where n = number of item types, W = capacity
  • Space: O(W) — only a single 1D array needed
  • Rod Cutting: O(n^2) time since items are lengths 1..n and capacity is n
  • Like 0/1 knapsack, this is pseudo-polynomial (depends on value of W)
