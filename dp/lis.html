Longest Increasing Subsequence

Find the length of the longest subsequence where elements are in strictly increasing order. A classic 1D DP problem with an elegant O(n log n) optimization.

Problem Statement

   arr = [10, 9, 2, 5, 3, 7, 101, 18]

   Some increasing subsequences:
   [2, 5, 7, 101]  length = 4 ✓ (longest)
   [2, 5, 7, 18]   length = 4 ✓ (also longest)
   [2, 3, 7, 18]   length = 4 ✓
   [10, 101]       length = 2
   [9, 101]        length = 2

   Answer: 4

Approach 1: O(n^2) DP

State: dp[i] = length of LIS ending at index i

Transition: dp[i] = max(dp[j] + 1) for all j < i where arr[j] < arr[i]

Base: dp[i] = 1 (each element is an LIS of length 1)

Interactive: Watch DP Fill

Step-by-Step Walkthrough

arr = [10, 9, 2, 5, 3, 7, 101, 18]

i=0: dp[0]=1 (just 10)
i=1: no j where arr[j]<9, dp[1]=1
i=2: no j where arr[j]<2, dp[2]=1
i=3: arr[2]=2<5, dp[3]=dp[2]+1=2 → [2,5]
i=4: arr[2]=2<3, dp[4]=dp[2]+1=2 → [2,3]
i=5: arr[2]<7,arr[3]<7,arr[4]<7, dp[5]=max(dp[2],dp[3],dp[4])+1=3 → [2,5,7] or [2,3,7]
i=6: many j work, dp[6]=dp[5]+1=4 → [2,5,7,101]
i=7: arr[5]=7<18, dp[7]=dp[5]+1=4 → [2,5,7,18]

Answer: max(dp) = 4

Code: O(n^2) DP Solution

// O(n^2) time, O(n) space
int lengthOfLIS(vector<int>& nums) {
    int n = nums.size();
    vector<int> dp(n, 1); // each element is LIS of length 1

    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (nums[j] < nums[i]) {
                dp[i] = max(dp[i], dp[j] + 1);
            }
        }
    }

    return *max_element(dp.begin(), dp.end());
}

// O(n^2) time, O(n) space
public int lengthOfLIS(int[] nums) {
    int n = nums.length;
    int[] dp = new int[n];
    Arrays.fill(dp, 1);

    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (nums[j] < nums[i]) {
                dp[i] = Math.max(dp[i], dp[j] + 1);
            }
        }
    }

    int ans = 0;
    for (int x : dp) ans = Math.max(ans, x);
    return ans;
}

# O(n^2) time, O(n) space
def length_of_lis(nums):
    n = len(nums)
    dp = [1] * n  # each element is LIS of length 1

    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)

    return max(dp)

Approach 2: O(n log n) with Binary Search

Maintain a tails array where tails[i] = smallest tail element for increasing subsequence of length i+1.

   arr = [10, 9, 2, 5, 3, 7, 101, 18]

   Process each element:
   10  → tails = [10]
   9   → 9 < 10, replace: tails = [9]
   2   → 2 < 9, replace: tails = [2]
   5   → 5 > 2, append: tails = [2, 5]
   3   → 3 < 5, replace: tails = [2, 3]
   7   → 7 > 3, append: tails = [2, 3, 7]
   101 → 101 > 7, append: tails = [2, 3, 7, 101]
   18  → 18 < 101, replace: tails = [2, 3, 7, 18]

   Answer: len(tails) = 4

   Key insight: tails is always sorted → use binary search!

Code: O(n log n) Solution

// O(n log n) time, O(n) space
int lengthOfLIS(vector<int>& nums) {
    vector<int> tails;

    for (int num : nums) {
        // Find first element in tails >= num
        auto it = lower_bound(tails.begin(), tails.end(), num);

        if (it == tails.end()) {
            tails.push_back(num); // extend longest subsequence
        } else {
            *it = num; // replace to keep smallest tail
        }
    }

    return tails.size();
}

// O(n log n) time, O(n) space
public int lengthOfLIS(int[] nums) {
    List<Integer> tails = new ArrayList<>();

    for (int num : nums) {
        int pos = Collections.binarySearch(tails, num);
        if (pos < 0) pos = -(pos + 1);

        if (pos == tails.size()) {
            tails.add(num);
        } else {
            tails.set(pos, num);
        }
    }

    return tails.size();
}

from bisect import bisect_left

# O(n log n) time, O(n) space
def length_of_lis(nums):
    tails = []

    for num in nums:
        pos = bisect_left(tails, num)

        if pos == len(tails):
            tails.append(num)  # extend
        else:
            tails[pos] = num   # replace

    return len(tails)

Variants

  • Longest Non-Decreasing: Use upper_bound instead of lower_bound
  • Longest Decreasing: Reverse the array, then find LIS
  • Print the LIS: Store parent pointers during DP fill
  • Count of LIS: Maintain count[] alongside dp[]
  • 2D LIS (Envelope): Sort by one dimension, LIS on other
