Probability & Expected Value DP

Compute expected values (average cost, time, steps) to reach a target state using DP. The key insight is that expected value satisfies linear equations that can be solved via DP or systems of equations when cycles exist.

Core Framework

State: dp[state] = expected number of steps (or cost) to reach the target from state

Transition (no cycles):
dp[state] = 1 + sum over next_states: P(next) * dp[next]

Base Case: dp[target] = 0 (already at goal, no more steps needed)

Direction: Compute backwards from the target state, or use memoization

Example 1: Expected Coin Flips to Get HH

Flip a fair coin repeatedly. What is the expected number of flips to see two consecutive Heads?

States based on suffix of flips:
  S0: no useful suffix (start, or last flip was T)
  S1: last flip was H
  S2: last two flips were HH (GOAL)

Transitions (each flip is H or T with probability 1/2):
  From S0: flip H → S1,  flip T → S0
  From S1: flip H → S2,  flip T → S0
  S2: absorbing state (done)

Expected value equations:
  E[S0] = 1 + (1/2)*E[S1] + (1/2)*E[S0]
  E[S1] = 1 + (1/2)*E[S2] + (1/2)*E[S0]
  E[S2] = 0

Solving:
  From E[S0]: E[S0] - (1/2)*E[S0] = 1 + (1/2)*E[S1]
              (1/2)*E[S0] = 1 + (1/2)*E[S1]
              E[S0] = 2 + E[S1]

  From E[S1]: E[S1] = 1 + 0 + (1/2)*E[S0]
              E[S1] = 1 + (1/2)*(2 + E[S1])
              E[S1] = 1 + 1 + (1/2)*E[S1]
              (1/2)*E[S1] = 2
              E[S1] = 4

  E[S0] = 2 + 4 = 6

Answer: Expected 6 flips to see HH

Handling Cycles

Problem: When the state graph has cycles, a state's expected value depends on itself. This creates a system of linear equations.

Approaches:
1. Algebraic: Write equations and solve symbolically (small state spaces)
2. Gaussian elimination: Solve the linear system numerically O(n3)
3. Reverse order: If a DAG ordering exists, compute in that order
4. Memoization with rearrangement: Move self-referencing terms to the left side

Example 2: Random Walk Expected Steps

Start at position 0 on a number line. Each step, move +1 with probability p or -1 with probability (1-p). Expected steps to reach position n?

States: positions 0, 1, 2, ..., n
  dp[i] = expected steps to reach n from position i

Equations:
  dp[n] = 0  (already there)
  dp[i] = 1 + p * dp[i+1] + (1-p) * dp[i-1]   for 0 < i < n

Boundary: dp[0] = 1 + dp[1]  (can only go right from 0, or reflect)

For p = 1/2 (unbiased walk with reflecting boundary at 0):
  dp[i] = 1 + (1/2)*dp[i+1] + (1/2)*dp[i-1]

  Solution: dp[i] = n^2 - i^2  (for reflecting boundary at 0)

For biased walk (p != 1/2):
  Solve the linear recurrence or use Gaussian elimination.

Example: n=3, p=1/2, reflecting at 0
  dp[0] = 9, dp[1] = 8, dp[2] = 5, dp[3] = 0

Step-by-Step: Dice Problem

Problem: Roll a fair 6-sided die repeatedly. What is the expected number of rolls to reach a sum ≥ n?
State: dp[s] = expected rolls to reach sum ≥ n from current sum s
Base Case: dp[s] = 0 for s ≥ n
Transition: dp[s] = 1 + (1/6) * sum(dp[s+1] + dp[s+2] + ... + dp[s+6])
Step 1: dp[n] = dp[n+1] = ... = dp[n+5] = 0 (all sums ≥ n are done)
Step 2: Compute backwards: dp[n-1] = 1 + (1/6)*(dp[n]+...+dp[n+4]+dp[n-1+6])
Since all terms on right are 0 for the last few: dp[n-1] = 1 + 0 = 1... but careful, dp[n-1+6] might not be 0 if n is small.
Step 3: For n=10: dp[10..15]=0, dp[9]=1, dp[8]=1+(1/6)*1=7/6, dp[7]=1+(1/6)*(1+7/6)=... Continue backwards to dp[0].

Code: Expected Coin Flips for Pattern

// Expected flips to see pattern (e.g., "HH")
// Using KMP failure function to define state transitions
// Time: O(m^2) where m = pattern length, Space: O(m)
double expectedFlips(string pattern) {
    int m = pattern.size();
    // Build KMP failure function
    vector<int> fail(m, 0);
    for (int i = 1; i < m; i++) {
        int j = fail[i - 1];
        while (j > 0 && pattern[i] != pattern[j]) j = fail[j - 1];
        if (pattern[i] == pattern[j]) j++;
        fail[i] = j;
    }

    // dp[i] = expected flips to reach state m from state i
    // state = length of longest prefix of pattern matched so far
    vector<double> dp(m + 1, 0);
    // dp[m] = 0 (pattern fully matched)

    // Solve from state m-1 down to 0
    // dp[i] = 1 + 0.5*dp[next(i,'H')] + 0.5*dp[next(i,'T')]
    // This may have forward references, so use Gaussian elimination
    // For simple patterns, can solve algebraically

    // General approach: system of m equations
    // For "HH": dp[0] = 1 + 0.5*dp[1] + 0.5*dp[0]
    //            dp[1] = 1 + 0.5*dp[2] + 0.5*dp[0]
    // Solving: dp[1] = 4, dp[0] = 6

    // For general pattern, use Gaussian elimination on m variables
    int n = m; // number of unknowns (states 0..m-1)
    vector<vector<double>> A(n, vector<double>(n + 1, 0));

    for (int i = 0; i < m; i++) {
        A[i][i] = 1.0; // dp[i] on left side
        for (char c : {'H', 'T'}) {
            // Compute next state from state i on character c
            int j = i;
            while (j > 0 && pattern[j] != c) j = fail[j - 1];
            if (pattern[j] == c) j++;
            if (j < m) A[i][j] -= 0.5; // subtract from left side
            // if j == m, it contributes 0.5 * 0 = 0
        }
        A[i][n] = 1.0; // right side: the +1 step cost
    }

    // Gaussian elimination
    for (int col = 0; col < n; col++) {
        int pivot = col;
        for (int row = col + 1; row < n; row++)
            if (abs(A[row][col]) > abs(A[pivot][col])) pivot = row;
        swap(A[col], A[pivot]);
        for (int row = 0; row < n; row++) {
            if (row == col) continue;
            double factor = A[row][col] / A[col][col];
            for (int j = col; j <= n; j++)
                A[row][j] -= factor * A[col][j];
        }
    }
    for (int i = 0; i < n; i++)
        dp[i] = A[i][n] / A[i][i];

    return dp[0];
}

// Expected flips to see pattern (e.g., "HH")
// Time: O(m^2 + m^3) for KMP + Gaussian elimination
static double expectedFlips(String pattern) {
    int m = pattern.length();
    int[] fail = new int[m];
    for (int i = 1; i < m; i++) {
        int j = fail[i - 1];
        while (j > 0 && pattern.charAt(i) != pattern.charAt(j))
            j = fail[j - 1];
        if (pattern.charAt(i) == pattern.charAt(j)) j++;
        fail[i] = j;
    }

    int n = m;
    double[][] A = new double[n][n + 1];

    for (int i = 0; i < m; i++) {
        A[i][i] = 1.0;
        for (char c : new char[]{'H', 'T'}) {
            int j = i;
            while (j > 0 && pattern.charAt(j) != c)
                j = fail[j - 1];
            if (pattern.charAt(j) == c) j++;
            if (j < m) A[i][j] -= 0.5;
        }
        A[i][n] = 1.0;
    }

    // Gaussian elimination
    for (int col = 0; col < n; col++) {
        int pivot = col;
        for (int row = col + 1; row < n; row++)
            if (Math.abs(A[row][col]) > Math.abs(A[pivot][col]))
                pivot = row;
        double[] tmp = A[col]; A[col] = A[pivot]; A[pivot] = tmp;
        for (int row = 0; row < n; row++) {
            if (row == col) continue;
            double factor = A[row][col] / A[col][col];
            for (int j = col; j <= n; j++)
                A[row][j] -= factor * A[col][j];
        }
    }

    return A[0][n] / A[0][0];
}

# Expected flips to see pattern (e.g., "HH")
# Time: O(m^2 + m^3) for KMP + Gaussian elimination
def expected_flips(pattern):
    m = len(pattern)
    # Build KMP failure function
    fail = [0] * m
    for i in range(1, m):
        j = fail[i - 1]
        while j > 0 and pattern[i] != pattern[j]:
            j = fail[j - 1]
        if pattern[i] == pattern[j]:
            j += 1
        fail[i] = j

    # Build system of equations
    n = m
    A = [[0.0] * (n + 1) for _ in range(n)]

    for i in range(m):
        A[i][i] = 1.0
        for c in ['H', 'T']:
            j = i
            while j > 0 and pattern[j] != c:
                j = fail[j - 1]
            if pattern[j] == c:
                j += 1
            if j < m:
                A[i][j] -= 0.5
        A[i][n] = 1.0  # +1 step cost

    # Gaussian elimination
    for col in range(n):
        pivot = max(range(col, n), key=lambda r: abs(A[r][col]))
        A[col], A[pivot] = A[pivot], A[col]
        for row in range(n):
            if row == col:
                continue
            factor = A[row][col] / A[col][col]
            for j in range(col, n + 1):
                A[row][j] -= factor * A[col][j]

    return A[0][n] / A[0][0]

# Example
print(expected_flips("HH"))   # Output: 6.0
print(expected_flips("HT"))   # Output: 4.0
print(expected_flips("HHH"))  # Output: 14.0

Code: Dice Sum Expected Rolls (No Cycles)

// Expected rolls of a fair die to reach sum ≥ n
// Time: O(n), Space: O(n)
double expectedRolls(int n) {
    // dp[s] = expected rolls to reach sum ≥ n from current sum s
    vector<double> dp(n + 6, 0.0);
    // dp[s] = 0 for s ≥ n

    for (int s = n - 1; s >= 0; s--) {
        dp[s] = 1.0; // one roll
        for (int die = 1; die <= 6; die++) {
            dp[s] += dp[s + die] / 6.0;
        }
    }
    return dp[0];
}

# Expected rolls of a fair die to reach sum ≥ n
# Time: O(n), Space: O(n)
def expected_rolls(n):
    dp = [0.0] * (n + 6)
    # dp[s] = 0 for s ≥ n

    for s in range(n - 1, -1, -1):
        dp[s] = 1.0  # one roll
        for die in range(1, 7):
            dp[s] += dp[s + die] / 6.0

    return dp[0]

# Example: expected rolls to reach sum ≥ 10
print(expected_rolls(10))  # ~2.85

  dp[0] ──(1-6)──→ dp[1..6]
  dp[1] ──(1-6)──→ dp[2..7]     (dp[4+] = 0)
  dp[2] ──(1-6)──→ dp[3..8]
  dp[3] ──(1-6)──→ dp[4..9]     (all 0)

  dp[3] = 1 + 0 = 1
  dp[2] = 1 + (1/6)*dp[3] = 1 + 1/6 = 7/6
  dp[1] = 1 + (1/6)*(dp[2]+dp[3]) = 1 + (1/6)*(7/6+1) = 49/36
  dp[0] = 1 + (1/6)*(dp[1]+dp[2]+dp[3]) = ...

Complexity

  • Without cycles: O(states · transitions) — standard DP backward computation
  • With cycles (Gaussian elimination): O(states3) to solve the linear system
  • Space: O(states) for DP array, O(states2) for Gaussian elimination
  • Alternative for cycles: Value iteration (converges geometrically) or matrix inversion

Key Insights

  • Linearity of expectation: E[X+Y] = E[X] + E[Y] regardless of dependence; often simplifies complex problems
  • Absorbing states: Target states have E=0; computation flows backward from them
  • Self-loops: If dp[s] = 1 + p*dp[s] + ..., rearrange: dp[s]*(1-p) = 1 + ..., so dp[s] = (1+...)/(1-p)
  • Probability DP vs Expected Value DP: Probability DP computes forward (P[reaching state]); Expected value DP computes backward (E[steps from state])
