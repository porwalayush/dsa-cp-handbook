BST Range Operations

Picture a BST as a sorted shelf of numbers that happens to be folded into a tree. A range like [low, high] is just a window onto that shelf — you only care about the values that fall between the two bookends. The magic is that you almost never have to look at the whole shelf: because every node already knows that everything to its left is smaller and everything to its right is larger, you can take one glance at a node and confidently throw away an entire branch that can't possibly hold anything in your window.

That one move — pruning branches that fall outside [low, high] — powers every operation on this page: trimming the tree to a range, summing the values inside it, deleting everything outside it, and counting what's left. They are all the same idea wearing different hats.

The pruning rule in one breath: at a node with value v —
  • if v < low, the node and its entire left subtree are below the window — only the right side can still matter.
  • if v > high, the node and its entire right subtree are above the window — only the left side can still matter.
  • if low ≤ v ≤ high, the node is a keeper, and both sides may still hold in-range values.

1. Trim a BST to a Range [low, high]

Think of trimming like pruning a real hedge: you want to keep only the part of the tree whose values land inside [low, high], and snip off everything else — but the result must still be a valid BST. The clever part is that you never delete one node at a time. When a node is too small, you don't just drop the node; you drop the node and its whole left subtree in one stroke, because in a BST everything left of a too-small node is even smaller.

The recursion almost writes itself once you see that:

  • If node.val < low: this node is out, and so is everything to its left. The only survivors live in the right subtree, so return trim(node.right) — it replaces the current node entirely.
  • If node.val > high: symmetric — everything to the right is gone, so return trim(node.left).
  • Otherwise the node stays; recursively trim both children and reattach the trimmed subtrees.
Trim to [5, 13]:

            8
           / \
          3   12
         / \    \
        1   6    15
             \
              7

node 8: 5 ≤ 8 ≤ 13  -> keep, trim both sides
  node 3: 3 < 5  -> drop 3 AND its left (node 1), return trim(right=6)
    node 6: in range -> keep, trim both sides
      node 7: in range -> keep
  node 12: in range -> keep, trim both sides
    node 15: 15 > 13 -> drop 15 AND its right, return trim(left=null) = null

Result:
            8
           / \
          6   12
           \
            7
Why returning the child works: trimming is "destructive rewiring." Each call returns the new root of the trimmed subtree, and the parent simply reattaches whatever comes back. When you return trim(node.right) for a too-small node, that returned subtree is spliced into the parent in place of the dropped node — which is exactly how the lone right child gets promoted up the tree. 
struct Node { int val; Node *left, *right; };

// Returns the new root of the subtree trimmed to [low, high].
Node* trimBST(Node* node, int low, int high) {
    if (!node) return nullptr;
    if (node->val < low)                 // node + entire LEFT subtree are too small
        return trimBST(node->right, low, high);
    if (node->val > high)                // node + entire RIGHT subtree are too large
        return trimBST(node->left, low, high);
    // in range: keep this node, trim and reattach both children
    node->left  = trimBST(node->left,  low, high);
    node->right = trimBST(node->right, low, high);
    return node;
}

class Node { int val; Node left, right; }

// Returns the new root of the subtree trimmed to [low, high].
Node trimBST(Node node, int low, int high) {
    if (node == null) return null;
    if (node.val < low)                 // node + entire LEFT subtree are too small
        return trimBST(node.right, low, high);
    if (node.val > high)                // node + entire RIGHT subtree are too large
        return trimBST(node.left, low, high);
    node.left  = trimBST(node.left,  low, high);
    node.right = trimBST(node.right, low, high);
    return node;
}

class Node:
    def __init__(self, val):
        self.val = val
        self.left = self.right = None

# Returns the new root of the subtree trimmed to [low, high].
def trim_bst(node, low, high):
    if node is None:
        return None
    if node.val < low:          # node + entire LEFT subtree are too small
        return trim_bst(node.right, low, high)
    if node.val > high:         # node + entire RIGHT subtree are too large
        return trim_bst(node.left, low, high)
    node.left  = trim_bst(node.left,  low, high)
    node.right = trim_bst(node.right, low, high)
    return node

2. Range Sum of a BST

Now suppose you don't want to reshape the tree — you just want to add up every value inside [low, high]. A naive solution visits all n nodes. But the BST ordering lets you be lazy in the best way: before stepping into a subtree, ask "could anything in there possibly be in my window?" If the answer is no, skip it entirely.

Concretely, at a node with value v: only go left if v > low (otherwise the left subtree is all ≤ v ≤ low, too small), and only go right if v < high (otherwise the right subtree is all too large). Add v to the running total only when it actually lands in [low, high].

rangeSum(low=5, high=13):

            8
           / \
          3   12
         / \    \
        1   6    15
             \
              7

node 8: in [5,13] -> add 8. 8>5 so explore left; 8<13 so explore right
  node 3: 3 < 5  -> do NOT add. 3 not > 5, so PRUNE left (node 1 skipped).
                    3 < 13, explore right.
    node 6: in range -> add 6, explore both
      node 7: in range -> add 7
  node 12: in range -> add 12. 12>5 explore left (none); 12<13 explore right
    node 15: 15 > 13 -> do NOT add, PRUNE right; 15 not < 13 either.
Sum = 8 + 6 + 7 + 12 = 33   (nodes 1 and 15's subtree pruned)

// Sum of all values in [low, high], pruning subtrees that can't help.
long long rangeSumBST(Node* node, int low, int high) {
    if (!node) return 0;
    long long total = 0;
    if (node->val >= low && node->val <= high)
        total += node->val;                 // node is inside the window
    if (node->val > low)                       // left may still hold in-range values
        total += rangeSumBST(node->left, low, high);
    if (node->val < high)                      // right may still hold in-range values
        total += rangeSumBST(node->right, low, high);
    return total;
}

// Sum of all values in [low, high], pruning subtrees that can't help.

    if (node.val < high)                       // right may still hold in-range values
        total += rangeSumBST(node.right, low, high);
    return total;
}

# Sum of all values in [low, high], pruning subtrees that can't help.
def range_sum_bst(node, low, high):
    if node is None:
        return 0
    total = 0
    if low <= node.val <= high:
        total += node.val               # node is inside the window
    if node.val > low:                   # left may still hold in-range values
        total += range_sum_bst(node.left, low, high)
    if node.val < high:                  # right may still hold in-range values
        total += range_sum_bst(node.right, low, high)
    return total

3. Delete Nodes Inside (or Outside) a Range

"Delete everything outside a range" is a riddle you already solved: that is precisely trim from Section 1 — keeping the inside is the same as deleting the outside. The more interesting cousin is delete everything inside a range (low, high), which is what the classic "delete nodes and return forest" problem is really asking when the deleted set happens to be a contiguous interval.

The key realization: when you delete a node, its left and right subtrees may become new standalone trees (a forest). A subtree becomes a root of its own only if its parent got deleted. So we carry a flag — "is my parent still around?" — and whenever a surviving node has a deleted parent, it joins the forest of roots.

Delete all values in (low=5, high=13)  (i.e. delete 6..12):

            8                         deleted: 6, 7, 8, 12
           / \                        survivors form a forest
          3   12
         / \    \          Result forest (roots):   3            15
        1   6    15                                 / \
             \                                      1  (6 gone)
              7

- node 8 deleted: its kids 3 and 12 may become roots
- node 3 survives, parent(8) deleted -> 3 is a forest root; trim its kids
    node 6 deleted (in range); node 1 survives, stays under 3
- node 12 deleted; node 15 survives, parent deleted -> 15 is a forest root
Relating it back to trim: deleting an open interval (low, high) keeps values ≤ low and ≥ high. Trim keeps a closed interval and returns one tree; delete-inside keeps the two outer pieces and returns several. Same pruning logic, different bookkeeping: trim re-splices into one root, delete collects a list of disconnected roots. 
// Delete every value strictly inside (low, high). Surviving subtrees
// whose parent was deleted become new roots collected in `forest`.
Node* deleteInRange(Node* node, int low, int high,
                    bool parentAlive, std::vector<Node*>& forest) {
    if (!node) return nullptr;
    bool deleted = node->val > low && node->val < high;
    // A surviving node with no living parent starts a new tree.
    if (!deleted && !parentAlive) forest.push_back(node);
    node->left  = deleteInRange(node->left,  low, high, !deleted, forest);
    node->right = deleteInRange(node->right, low, high, !deleted, forest);
    return deleted ? nullptr : node;   // detach deleted nodes from parent
}

// Delete every value strictly inside (low, high). Surviving subtrees
// whose parent was deleted become new roots collected in `forest`.
Node deleteInRange(Node node, int low, int high,
                   boolean parentAlive, List<Node> forest) {
    if (node == null) return null;
    boolean deleted = node.val > low && node.val < high;
    if (!deleted && !parentAlive) forest.add(node);
    node.left  = deleteInRange(node.left,  low, high, !deleted, forest);
    node.right = deleteInRange(node.right, low, high, !deleted, forest);
    return deleted ? null : node;   // detach deleted nodes from parent
}

# Delete every value strictly inside (low, high). Surviving subtrees
# whose parent was deleted become new roots collected in `forest`.
def delete_in_range(node, low, high, parent_alive, forest):
    if node is None:
        return None
    deleted = low < node.val < high
    if not deleted and not parent_alive:
        forest.append(node)          # survivor with a dead parent = new root
    node.left  = delete_in_range(node.left,  low, high, not deleted, forest)
    node.right = delete_in_range(node.right, low, high, not deleted, forest)
    return None if deleted else node   # detach deleted nodes from parent

4. Count Nodes in Range [low, high]

Counting is range-sum's twin — instead of adding values, you add 1 per in-range node, and you prune exactly the same way. This pruned traversal runs in O(h + k) time, where k is how many nodes are actually in the range.

Faster counting with order statistics: if your nodes are augmented with subtree sizes, you can count a range in O(h) flat — no dependence on k — using countLess(high+1) - countLess(low), where countLess uses the stored subtree sizes. See the order-statistics page (bst-order-statistics.html) for the rank / size-augmented version. The pruned traversal below needs no extra fields and is the right tool when your nodes aren't augmented. 
// Count of values in [low, high] via pruned traversal (no augmentation).
int rangeCount(Node* node, int low, int high) {
    if (!node) return 0;
    int cnt = 0;
    if (node->val >= low && node->val <= high) cnt = 1;
    if (node->val > low)                       // left can still contain matches
        cnt += rangeCount(node->left, low, high);
    if (node->val < high)                      // right can still contain matches
        cnt += rangeCount(node->right, low, high);
    return cnt;
}

// Count of values in [low, high] via pruned traversal (no augmentation).

    if (node.val < high)                       // right can still contain matches

}

# Count of values in [low, high] via pruned traversal (no augmentation).
def range_count(node, low, high):
    if node is None:
        return 0
    cnt = 1 if low <= node.val <= high else 0
    if node.val > low:                   # left can still contain matches
        cnt += range_count(node.left, low, high)
    if node.val < high:                  # right can still contain matches
        cnt += range_count(node.right, low, high)
    return cnt

6. Complexity Analysis

Here h is the tree height — O(log n) for a balanced BST, O(n) for a skewed one — and k is the number of nodes that actually fall inside the range.

OperationTimeSpace
Trim to rangeO(n) worst case, O(h + kept) typicalO(h) recursion stack
Range sum (with pruning)O(h + k)O(h)
Delete inside range (forest)O(n)O(h)
Count in range (pruned traversal)O(h + k)O(h)
Count in range (size-augmented)O(h)O(1)

The pruning is what turns a would-be O(n) scan into O(h + k): the algorithm only walks the boundary paths to low and high plus the in-range nodes between them, never touching the fully-out-of-range subtrees.

The unifying idea — prune what can't matter:
  • Trim drops a too-small node together with its entire left subtree (and the mirror for too-large), splicing the surviving child upward.
  • Range sum and count visit only the subtrees that could overlap [low, high], skipping the rest outright.
  • Delete-inside-range is trim turned inside out: keep the outer pieces, collect them as a forest.
Every operation here is the same observation — in a BST, one comparison at a node tells you a whole branch is irrelevant — applied with slightly different bookkeeping.
