BST Validation & Ordered Traversal

A Binary Search Tree obeys a single global rule: for every node, all keys in its left subtree are smaller and all keys in its right subtree are larger. This page covers how to correctly validate that property, find inorder successors and predecessors, locate floor and ceil, and answer range queries — all of which flow from one idea: an inorder traversal of a BST is sorted.

1. Validating a BST: the WRONG Approach

A tempting but incorrect idea is to only check each node against its two immediate children: left.val < node.val < right.val. This is a local check and it misses violations that occur deeper in the tree.

        5
       / \
      3   7
     / \
    2   6      <-- 6 is in the LEFT subtree of 5!

Local check at every node:
  node 5: 3 < 5 < 7   OK
  node 3: 2 < 3 < 6   OK   (6 > 3, looks fine locally)
  node 7: leaf        OK
  node 2: leaf        OK
  node 6: leaf        OK

=> Local check says "valid", but the tree is NOT a BST:
   6 sits in the left subtree of 5, yet 6 > 5.
   Inorder traversal: 2, 3, 6, 5, 7  -- NOT sorted (6 before 5).
Why the local check fails:
  • The BST property is global, not local: a node must respect the bounds set by every ancestor, not just its parent.
  • Checking only direct children verifies parent-child order but ignores the constraint that the entire left subtree must be below the node and the entire right subtree above it.
  • Correct approaches carry an allowed range down the tree (Section 2) or compare against the previous inorder value (Section 3).

2. Validation via Min/Max Bounds

Recurse from the root with an open interval (low, high). Each node's key must satisfy low < key < high (strict — duplicates are not allowed in a standard BST). When you descend left, the node becomes the new upper bound; when you descend right, it becomes the new lower bound.

Bounds propagate down the tree as (low, high):

                (-inf, +inf)
                     5
                    / \
        (-inf, 5)  /   \  (5, +inf)
                  3       8
                 / \     / \
      (-inf,3)  /   \   /   \  (8, +inf)
               2    4  6      9
                  (3,5) (5,8)

  Check node 2: -inf < 2 < 3   OK
  Check node 4:    3 < 4 < 5   OK
  Check node 6:    5 < 6 < 8   OK
  Check node 9:    8 < 9 < +inf OK   => valid BST

Going LEFT  from node v: high becomes v.key (low unchanged)
Going RIGHT from node v: low  becomes v.key (high unchanged)
Integer overflow caution: if node keys can be the full int range (e.g. INT_MIN / INT_MAX), passing the sentinel bounds as int breaks the strict comparison. Use a wider type such as long / long long for the bounds, or use nullable bounds (a null meaning "no constraint on this side") and skip the comparison when the bound is absent. 
struct Node { int key; Node *left, *right; };

// Use long long bounds so INT_MIN / INT_MAX keys never overflow.
bool validate(Node* node, long long low, long long high) {
    if (!node) return true;                 // empty subtree is a valid BST
    if (node->key <= low || node->key >= high)
        return false;                          // must lie strictly inside (low, high)
    return validate(node->left, low, node->key) &&
           validate(node->right, node->key, high);
}

bool isValidBST(Node* root) {
    // LLONG_MIN / LLONG_MAX are safely outside any int key range.
    return validate(root, LLONG_MIN, LLONG_MAX);
}

class Node { int key; Node left, right; }

// Nullable Long bounds: null means "no constraint on this side".
boolean validate(Node node, Long low, Long high) {
    if (node == null) return true;
    if (low != null && node.key <= low) return false;
    if (high != null && node.key >= high) return false;
    return validate(node.left, low, (long) node.key) &&
           validate(node.right, (long) node.key, high);
}

boolean isValidBST(Node root) {
    return validate(root, null, null);
}

class Node:
    def __init__(self, key):
        self.key = key
        self.left = self.right = None

# Python ints are unbounded, so float('-inf')/float('inf') are safe sentinels.
def validate(node, low, high):
    if node is None:
        return True
    if not (low < node.key < high):   # strict on both sides
        return False
    return (validate(node.left, low, node.key) and
            validate(node.right, node.key, high))

def is_valid_bst(root):
    return validate(root, float('-inf'), float('inf'))

3. Validation via Inorder Traversal

Since an inorder traversal (left, node, right) of a valid BST yields keys in strictly increasing order, we can validate by walking inorder and checking that each key is greater than the prev key visited. No bounds needed — just track one previous value.

Inorder visit order on a valid BST:

         5
        / \
       3   8
      / \   \
     2   4   9

Visit:  2, 3, 4, 5, 8, 9   (each > the one before)

Keep `prev` = last key seen (start at -inf / null).
At each node in inorder:
   if key <= prev  ->  NOT a BST
   else prev = key

// prev holds the last key visited inorder; passed by reference.
bool inorderCheck(Node* node, Node*& prev) {
    if (!node) return true;
    if (!inorderCheck(node->left, prev)) return false;
    if (prev && prev->key >= node->key) return false;  // must strictly increase
    prev = node;
    return inorderCheck(node->right, prev);
}

bool isValidBST(Node* root) {
    Node* prev = nullptr;
    return inorderCheck(root, prev);
}

private Node prev;  // last node visited inorder (instance field)

boolean isValidBST(Node root) {
    prev = null;
    return inorderCheck(root);
}

boolean inorderCheck(Node node) {
    if (node == null) return true;
    if (!inorderCheck(node.left)) return false;
    if (prev != null && prev.key >= node.key) return false;
    prev = node;
    return inorderCheck(node.right);
}

def is_valid_bst(root):
    prev = None      # last key seen inorder; use a list cell for closure
    state = [prev]

    def inorder(node):
        if node is None:
            return True
        if not inorder(node.left):
            return False
        if state[0] is not None and state[0] >= node.key:
            return False
        state[0] = node.key
        return inorder(node.right)

    return inorder(root)

4. Inorder Successor & Predecessor

The inorder successor of a node is the next-larger key (the node that comes right after it in sorted order); the predecessor is the next-smaller key.

(a) With parent pointers

If each node stores a pointer to its parent, you don't even need the root:

  • Successor: if the node has a right child, the answer is the leftmost node of the right subtree. Otherwise climb up parents until you move up from a left child — that parent is the successor (or null if you reach the top).
  • Predecessor: the mirror image — if there is a left child, take the rightmost node of the left subtree; otherwise climb until you move up from a right child.

(b) Without parent pointers (using BST structure)

Search down from the root. For the successor: whenever you go left (target is smaller), the current node is a candidate successor; record it and continue. The lowest recorded candidate is the answer. Equivalently:

  • If the node has a right child, the successor is the minimum of the right subtree.
  • Otherwise it is the lowest ancestor for which the node lies in that ancestor's left subtree.
                20
               /  \
             10    30
            /  \   /  \
           5   15 25   35

Successor of 15:  no right child of 15.
  Walk from root: 20 > 15 -> candidate=20, go left
                  10 < 15 -> go right (no candidate)
                  15 == target, stop.
  Lowest ancestor that has 15 in its LEFT subtree = 20.  Successor = 20.

Successor of 20:  has a right child (30).
  Successor = min of right subtree = leftmost of 30 = 25.

Predecessor of 20:  has a left child (10).
  Predecessor = max of left subtree = rightmost of 10 = 15.

// No parent pointers. Returns successor node, or nullptr if none.
Node* inorderSuccessor(Node* root, int key) {
    Node* succ = nullptr;
    Node* cur = root;
    while (cur) {
        if (key < cur->key) {        // cur could be the successor
            succ = cur;
            cur = cur->left;
        } else {                     // key >= cur->key, successor is to the right
            cur = cur->right;
        }
    }
    return succ;
}

// Symmetric: largest key strictly smaller than `key`.
Node* inorderPredecessor(Node* root, int key) {
    Node* pred = nullptr;
    Node* cur = root;
    while (cur) {
        if (key > cur->key) {        // cur could be the predecessor
            pred = cur;
            cur = cur->right;
        } else {
            cur = cur->left;
        }
    }
    return pred;
}

// No parent pointers. Returns successor node, or null if none.
Node inorderSuccessor(Node root, int key) {
    Node succ = null, cur = root;
    while (cur != null) {
        if (key < cur.key) {        // cur could be the successor
            succ = cur;
            cur = cur.left;
        } else {
            cur = cur.right;
        }
    }
    return succ;
}

Node inorderPredecessor(Node root, int key) {
    Node pred = null, cur = root;
    while (cur != null) {
        if (key > cur.key) {        // cur could be the predecessor
            pred = cur;
            cur = cur.right;
        } else {
            cur = cur.left;
        }
    }
    return pred;
}

# No parent pointers. Returns successor node, or None if none.
def inorder_successor(root, key):
    succ, cur = None, root
    while cur:
        if key < cur.key:          # cur could be the successor
            succ = cur
            cur = cur.left
        else:
            cur = cur.right
    return succ

def inorder_predecessor(root, key):
    pred, cur = None, root
    while cur:
        if key > cur.key:          # cur could be the predecessor
            pred = cur
            cur = cur.right
        else:
            cur = cur.left
    return pred

5. Floor & Ceil of a Key

The floor of a key x is the largest key in the tree that is <= x; the ceil is the smallest key >= x. Walk down from the root tracking the best candidate found so far — O(h) time, no extra space.

                20
               /  \
             10    30
            /  \   /  \
           5   15 25   35

floor(22):  walk 20 (<=22, best=20, go right)
                 30 (>22, go left)
                 25 (>22, go left -> null)  => floor = 20

ceil(22):   walk 20 (<22, go right)
                 30 (>=22, best=30, go left)
                 25 (>=22, best=25, go left -> null)  => ceil = 25

If x is present, floor(x) = ceil(x) = x.

// Largest key <= x. Returns LLONG_MIN sentinel if none exists.
long long floorKey(Node* root, int x) {
    long long best = LLONG_MIN;
    Node* cur = root;
    while (cur) {
        if (cur->key == x) return x;     // exact match is the floor
        if (cur->key < x) {              // candidate; look for something larger
            best = cur->key;
            cur = cur->right;
        } else {
            cur = cur->left;            // too big; go smaller
        }
    }
    return best;
}

// Smallest key >= x. Returns LLONG_MAX sentinel if none exists.
long long ceilKey(Node* root, int x) {
    long long best = LLONG_MAX;
    Node* cur = root;
    while (cur) {
        if (cur->key == x) return x;
        if (cur->key > x) {
            best = cur->key;
            cur = cur->left;
        } else {
            cur = cur->right;
        }
    }
    return best;
}

// Largest key <= x. Returns Long.MIN_VALUE if none exists.
long floorKey(Node root, int x) {
    long best = Long.MIN_VALUE;
    Node cur = root;
    while (cur != null) {
        if (cur.key == x) return x;
        if (cur.key < x) { best = cur.key; cur = cur.right; }
        else cur = cur.left;
    }
    return best;
}

// Smallest key >= x. Returns Long.MAX_VALUE if none exists.
long ceilKey(Node root, int x) {
    long best = Long.MAX_VALUE;
    Node cur = root;
    while (cur != null) {
        if (cur.key == x) return x;
        if (cur.key > x) { best = cur.key; cur = cur.left; }
        else cur = cur.right;
    }
    return best;
}

# Largest key <= x, or None if none exists.
def floor_key(root, x):
    best, cur = None, root
    while cur:
        if cur.key == x:
            return x
        if cur.key < x:        # candidate; try to find a larger one
            best = cur.key
            cur = cur.right
        else:
            cur = cur.left
    return best

# Smallest key >= x, or None if none exists.
def ceil_key(root, x):
    best, cur = None, root
    while cur:
        if cur.key == x:
            return x
        if cur.key > x:
            best = cur.key
            cur = cur.left
        else:
            cur = cur.right
    return best

6. Range Queries

To sum (or count) all keys in [lo, hi], exploit the BST ordering to prune entire subtrees: if the current key is <= lo there is nothing useful in its left subtree, and if it is >= hi there is nothing useful in its right subtree. Only descend where the range can still contain matches.

                20
               /  \
             10    30
            /  \   /  \
           5   15 25   35

rangeSum(lo=12, hi=27):
  node 20: in [12,27] -> add 20; 20>12 so explore left; 20<27 so explore right
    node 10: 10 < 12  -> PRUNE left subtree (5 too), only explore right
      node 15: in range -> add 15
    node 30: 30 > 27  -> PRUNE right subtree (35 too), only explore left
      node 25: in range -> add 25
  Sum = 20 + 15 + 25 = 60   (nodes 5 and 35 never visited)

// Sum of all keys in [lo, hi], pruning out-of-range subtrees.
long long rangeSumBST(Node* node, int lo, int hi) {
    if (!node) return 0;
    if (node->key < lo)              // whole left subtree is too small
        return rangeSumBST(node->right, lo, hi);
    if (node->key > hi)              // whole right subtree is too large
        return rangeSumBST(node->left, lo, hi);
    // lo <= key <= hi: count this node and recurse both sides
    return node->key
         + rangeSumBST(node->left, lo, hi)
         + rangeSumBST(node->right, lo, hi);
}

// Count of keys in [lo, hi].
int rangeCount(Node* node, int lo, int hi) {
    if (!node) return 0;
    if (node->key < lo) return rangeCount(node->right, lo, hi);
    if (node->key > hi) return rangeCount(node->left, lo, hi);
    return 1 + rangeCount(node->left, lo, hi)
             + rangeCount(node->right, lo, hi);
}

// Sum of all keys in [lo, hi], pruning out-of-range subtrees.
long rangeSumBST(Node node, int lo, int hi) {
    if (node == null) return 0;
    if (node.key < lo)              // whole left subtree is too small
        return rangeSumBST(node.right, lo, hi);
    if (node.key > hi)              // whole right subtree is too large
        return rangeSumBST(node.left, lo, hi);
    return node.key
         + rangeSumBST(node.left, lo, hi)
         + rangeSumBST(node.right, lo, hi);
}

// Count of keys in [lo, hi].
int rangeCount(Node node, int lo, int hi) {
    if (node == null) return 0;
    if (node.key < lo) return rangeCount(node.right, lo, hi);
    if (node.key > hi) return rangeCount(node.left, lo, hi);
    return 1 + rangeCount(node.left, lo, hi)
             + rangeCount(node.right, lo, hi);
}

# Sum of all keys in [lo, hi], pruning out-of-range subtrees.
def range_sum_bst(node, lo, hi):
    if node is None:
        return 0
    if node.key < lo:          # whole left subtree is too small
        return range_sum_bst(node.right, lo, hi)
    if node.key > hi:          # whole right subtree is too large
        return range_sum_bst(node.left, lo, hi)
    return (node.key
            + range_sum_bst(node.left, lo, hi)
            + range_sum_bst(node.right, lo, hi))

# Count of keys in [lo, hi].
def range_count(node, lo, hi):
    if node is None:
        return 0
    if node.key < lo:
        return range_count(node.right, lo, hi)
    if node.key > hi:
        return range_count(node.left, lo, hi)
    return 1 + range_count(node.left, lo, hi) \
             + range_count(node.right, lo, hi)

7. Complexity Analysis

Here h is the tree height — O(log n) for a balanced BST, O(n) in the worst case (a degenerate / skewed tree).

OperationTimeSpace
Validate (bounds, recursive)O(n)O(h) recursion stack
Validate (inorder)O(n)O(h) recursion stack
Successor / Predecessor (no parent)O(h)O(1)
Successor / Predecessor (with parent)O(h)O(1)
Floor / CeilO(h)O(1)
Range sum / count (with pruning)O(h + k)*O(h)

*k = number of nodes actually inside the range; pruning skips entire out-of-range subtrees so only relevant paths are walked.

The unifying idea — inorder = sorted:
  • Validation works because a valid BST's inorder sequence is strictly increasing; the bounds approach is just the same invariant expressed as per-node ranges.
  • Successor / predecessor are simply the next / previous elements in that sorted inorder order.
  • Floor / ceil are the neighbors of x in the sorted order (largest ≤ x, smallest ≥ x).
  • Range queries select a contiguous slice of the sorted order, which is exactly why we can prune everything outside [lo, hi].
Internalize "inorder of a BST is sorted" and every operation on this page becomes a variation on a single theme.
