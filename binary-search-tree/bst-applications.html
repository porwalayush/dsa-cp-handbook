BST Applications & Library Structures

A balanced Binary Search Tree keeps elements in sorted order while supporting O(log n) insert, delete, and search. This page covers classic BST interview problems and the real-world ordered structures (std::set, TreeMap, SortedList) that are built on top of balanced BSTs.

1. Two Sum in a BST

Given a BST and a target k, decide whether two distinct nodes sum to k. There are two natural approaches that both exploit the sorted nature of the tree.

Approach A: Inorder → sorted array + two pointers

An inorder traversal of a BST yields values in ascending sorted order. Flatten the tree into an array, then sweep two pointers from both ends: if the pair sum is too small advance the left pointer, if too large retreat the right pointer. This is O(n) time and O(n) space.

BST:            5
              /   \
             3     7
            / \   / \
           2   4 6   8

Inorder array:  [2, 3, 4, 5, 6, 7, 8]   target k = 11
                 L                 R
  2 + 8 = 10  < 11  -> L++
  3 + 8 = 11  == 11 -> FOUND (3, 8) ✓

struct Node { int val; Node *left, *right; };

void inorder(Node* root, vector<int>& out) {
    if (!root) return;
    inorder(root->left, out);
    out.push_back(root->val);
    inorder(root->right, out);
}

bool findTarget(Node* root, int k) {
    vector<int> a;
    inorder(root, a);                // sorted ascending
    int lo = 0, hi = (int)a.size() - 1;
    while (lo < hi) {
        int s = a[lo] + a[hi];
        if (s == k) return true;
        if (s < k) lo++;
        else hi--;
    }
    return false;
}

void inorder(TreeNode root, List<Integer> out) {
    if (root == null) return;
    inorder(root.left, out);
    out.add(root.val);
    inorder(root.right, out);
}

boolean findTarget(TreeNode root, int k) {
    List<Integer> a = new ArrayList<>();
    inorder(root, a);                // sorted ascending
    int lo = 0, hi = a.size() - 1;
    while (lo < hi) {
        int s = a.get(lo) + a.get(hi);
        if (s == k) return true;
        if (s < k) lo++;
        else hi--;
    }
    return false;
}

def find_target(root, k):
    a = []
    def inorder(node):
        if not node: return
        inorder(node.left)
        a.append(node.val)
        inorder(node.right)
    inorder(root)                    # sorted ascending
    lo, hi = 0, len(a) - 1
    while lo < hi:
        s = a[lo] + a[hi]
        if s == k: return True
        if s < k: lo += 1
        else: hi -= 1
    return False

Approach B: Two BST iterators (forward + backward)

To avoid the O(n) array we can run two stack-based iterators over the same tree. A forward iterator yields the next-smallest value (a normal inorder traversal); a backward iterator yields the next-largest value (a reverse-inorder traversal that descends right children first). We advance them like two pointers: if the current sum is too small, pull the next value from the forward iterator; if too large, pull from the backward iterator; stop when they meet. This keeps space at O(h) where h is the tree height, instead of O(n). It is the streaming equivalent of the two-pointer sweep, reading the smallest and largest remaining values on demand.

2. Closest Value to a Target

Given a BST and a (possibly non-integer) target, find the stored value closest to it. Because the tree is ordered we never need to look at both subtrees: at each node we move toward the target, recording the best candidate as we descend. This is O(h) time and O(1) space.

BST:            8
              /   \
             4     12
            / \   /  \
           2   6 10   14

target = 11.4    (walk down, track closest)

  node 8   -> |8-11.4|=3.4   best=8    11.4>8  -> go right
  node 12  -> |12-11.4|=0.6  best=12   11.4<12 -> go left
  node 10  -> |10-11.4|=1.4  (worse)   11.4>10 -> go right
  node = null -> stop

Closest value = 12 ✓

int closestValue(Node* root, double target) {
    int best = root->val;
    while (root) {
        if (abs(root->val - target) < abs(best - target))
            best = root->val;
        if (target < root->val) root = root->left;
        else if (target > root->val) root = root->right;
        else break;             // exact match
    }
    return best;
}

int closestValue(TreeNode root, double target) {
    int best = root.val;
    while (root != null) {
        if (Math.abs(root.val - target) < Math.abs(best - target))
            best = root.val;
        if (target < root.val) root = root.left;
        else if (target > root.val) root = root.right;
        else break;             // exact match
    }
    return best;
}

def closest_value(root, target):
    best = root.val
    node = root
    while node:
        if abs(node.val - target) < abs(best - target):
            best = node.val
        if target < node.val:
            node = node.left
        elif target > node.val:
            node = node.right
        else:
            break             # exact match
    return best

3. Merge Two BSTs

To merge two BSTs into one balanced BST containing all elements:

  1. Flatten each tree with an inorder traversal → two sorted lists (O(m) and O(n)).
  2. Merge the two sorted lists into one sorted list, exactly like the merge step of merge sort (O(m+n)).
  3. Build a height-balanced BST from the merged sorted list by recursively picking the middle element as the root (O(m+n)).

Total time is O(m + n) and space is O(m + n) for the lists plus O(m + n) for the resulting tree. This beats inserting every node of one tree into the other, which could degrade to O(n·h).

Tree A:   2          Tree B:   1
           \                  / \
            4                0   3

inorder(A) = [2, 4]      inorder(B) = [0, 1, 3]

merge   -> [0, 1, 2, 3, 4]

build balanced BST (mid = root):

            2
          /   \
         0     3
          \     \
           1     4

void inorder(Node* root, vector<int>& out) {
    if (!root) return;
    inorder(root->left, out);
    out.push_back(root->val);
    inorder(root->right, out);
}

vector<int> mergeSorted(vector<int>& a, vector<int>& b) {
    vector<int> res;
    size_t i = 0, j = 0;
    while (i < a.size() && j < b.size())
        res.push_back(a[i] <= b[j] ? a[i++] : b[j++]);
    while (i < a.size()) res.push_back(a[i++]);
    while (j < b.size()) res.push_back(b[j++]);
    return res;
}

// Build a height-balanced BST from sorted [lo, hi]
Node* build(vector<int>& v, int lo, int hi) {
    if (lo > hi) return nullptr;
    int mid = lo + (hi - lo) / 2;
    Node* root = new Node{v[mid], nullptr, nullptr};
    root->left  = build(v, lo, mid - 1);
    root->right = build(v, mid + 1, hi);
    return root;
}

Node* mergeBSTs(Node* r1, Node* r2) {
    vector<int> a, b;
    inorder(r1, a);
    inorder(r2, b);
    vector<int> merged = mergeSorted(a, b);   // O(m + n)
    return build(merged, 0, (int)merged.size() - 1);
}

4. Standard-Library Ordered Structures

You rarely hand-roll a balanced BST in production. Every major language ships a self-balancing tree:

  • C++: std::set / std::map (and the multiset/multimap variants) are typically Red-Black trees, giving sorted iteration and O(log n) operations.
  • Java: TreeSet / TreeMap are backed by a Red-Black tree with a rich navigable API.
  • Python: has no built-in ordered tree. Use the third-party sortedcontainers.SortedList / SortedDict (which use sorted blocks of arrays, not a BST, but expose the same ordered API), or use the bisect module on a plain sorted list.

Demo: ceiling / floor / range query

#include <set>
#include <iostream>
using namespace std;

int main() {
    set<int> s = {10, 20, 30, 40, 50};

    // ceiling(25): first element >= 25
    auto it = s.lower_bound(25);
    if (it != s.end()) cout << "ceiling(25)=" << *it << "\n";  // 30

    // floor(25): last element <= 25
    it = s.upper_bound(25);            // first > 25
    if (it != s.begin()) cout << "floor(25)=" << *prev(it) << "\n";  // 20

    // range query [20, 40]: iterate lower_bound(20) .. upper_bound(40)
    cout << "range[20,40]:";
    for (auto i = s.lower_bound(20); i != s.upper_bound(40); ++i)
        cout << " " << *i;     // 20 30 40
    cout << "\n";
}

import java.util.*;

class Demo {
    public static void main(String[] args) {
        TreeMap<Integer, String> m = new TreeMap<>();
        m.put(10, "a"); m.put(20, "b");
        m.put(30, "c"); m.put(40, "d"); m.put(50, "e");

        // ceilingKey(25): smallest key >= 25
        System.out.println("ceiling(25)=" + m.ceilingKey(25));  // 30

        // floorKey(25): largest key <= 25
        System.out.println("floor(25)=" + m.floorKey(25));      // 20

        // range query [20, 40] inclusive via subMap
        SortedMap<Integer, String> sub = m.subMap(20, true, 40, true);
        System.out.println("range[20,40]=" + sub.keySet());   // [20, 30, 40]
    }
}

from sortedcontainers import SortedList
import bisect

sl = SortedList([10, 20, 30, 40, 50])

# ceiling(25): first element >= 25
i = sl.bisect_left(25)
if i < len(sl):
    print("ceiling(25)=", sl[i])      # 30

# floor(25): last element <= 25
i = sl.bisect_right(25) - 1
if i >= 0:
    print("floor(25)=", sl[i])        # 20

# range query [20, 40] inclusive
lo = sl.bisect_left(20)
hi = sl.bisect_right(40)
print("range[20,40]=", list(sl[lo:hi]))  # [20, 30, 40]

# Pure-stdlib alternative on a plain sorted list:
arr = [10, 20, 30, 40, 50]
print(arr[bisect.bisect_left(arr, 25)])    # ceiling = 30

5. When to Reach for a Balanced BST

A balanced BST is not always the right tool. Choose based on which operations dominate:

StructurePoint ops (find/insert/delete)Ordered queriesBest for
Balanced BSTO(log n)predecessor, successor, floor, ceiling, range — all O(log n)Dynamic data needing both updates and order
Hash tableO(1) averageNone — no ordering, no range, no neighborsPure point lookup, no order needed
Sorted arrayfind O(log n), insert/delete O(n) (shift)O(log n) search, O(1) min/maxStatic or rarely-mutated ordered data
Heapfind-min/max O(1); arbitrary find/delete O(n)Only the min or max, never arbitrary orderPriority queue, repeated extract-min/max

BST vs Hash Table

  • Hash gives O(1) point access but no order.
  • BST gives sorted iteration, predecessor/successor, and range queries.
  • Pick BST whenever "what's the next bigger key?" matters.

BST vs Sorted Array

  • Array search is O(log n), but insert/delete shift elements in O(n).
  • BST does dynamic insert/delete in O(log n).
  • Pick BST when the set changes frequently.

BST vs Heap

  • A heap only exposes the min (or max) efficiently.
  • It cannot search for an arbitrary key in O(log n).
  • Pick a BST when you need ordered access to any element, not just the extreme.

Rule of Thumb

  • Need O(1) lookup only → hash table.
  • Need only min/max repeatedly → heap.
  • Static ordered data → sorted array.
  • Dynamic + ordered → balanced BST.

6. Real-World Uses

  • Ordered symbol tables: maps/sets that must iterate in key order (std::map, TreeMap).
  • Range queries: "all keys in [lo, hi]" via subMap / iterating between bounds.
  • Interval scheduling: keep active intervals ordered by endpoint; find overlaps or the next free slot with floor/ceiling.
  • k-th order statistics: an augmented BST that stores subtree sizes answers "k-th smallest" and "rank of x" in O(log n).
  • Databases & indexes: B-trees and B+ trees generalize the balanced BST to high fan-out, disk-friendly nodes — they are the backbone of relational-database indexes and filesystems, trading binary branching for hundreds of children per node to minimize disk reads.
Key takeaway:
  • Reach for a balanced BST (or its library form: std::set, TreeMap, SortedList) when you need BOTH dynamic updates AND ordered queries (predecessor, successor, floor, ceiling, range).
  • If you only need one of those, a hash table, heap, or sorted array will usually be simpler and faster.
