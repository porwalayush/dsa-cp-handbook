BST Construction & Conversion

Think of this page as a toolbox of closely related tricks, all built on one simple fact: if you walk a BST in "inorder" (left, then node, then right), the keys come out perfectly sorted. That single idea lets you turn sorted data into a balanced tree, flatten a tree into a sorted list and back, rebuild a tree from one traversal, and even repair a tree where two values got swapped.

1. Sorted Array → Height-Balanced BST

Here's the intuition first. Binary search on a sorted array always looks at the middle element to cut the search space in half. Building a balanced tree is the same move, frozen in place: pick the middle element as the root, then the left half becomes the left subtree and the right half becomes the right subtree — each built the exact same way. The tree you get is the binary search, drawn out as a structure.

Why bother? Because if instead you insert the sorted values one at a time (1, then 2, then 3...), every new value is bigger than the last and gets glued onto the right — you end up with a skewed tree that's really just a linked list, and searches go back to O(n). Always-pick-the-middle keeps both halves the same size, so the tree stays short: depth O(log n).

struct Node {
    int val;
    Node *left, *right;
    Node(int v) : val(v), left(nullptr), right(nullptr) {}
};

// Build a balanced BST from a[lo..hi] (inclusive)
Node* build(vector<int>& a, int lo, int hi) {
    if (lo > hi) return nullptr;
    int mid = lo + (hi - lo) / 2;
    Node* root = new Node(a[mid]);
    root->left  = build(a, lo, mid - 1);
    root->right = build(a, mid + 1, hi);
    return root;
}

Node* sortedArrayToBST(vector<int>& a) {
    return build(a, 0, (int)a.size() - 1);
}

class Node {
    int val;
    Node left, right;
    Node(int v) { val = v; }
}

}

class Node:
    def __init__(self, v):
        self.val = v
        self.left = self.right = None

# Build a balanced BST from a[lo..hi] (inclusive)
def build(a, lo, hi):
    if lo > hi:
        return None
    mid = (lo + hi) // 2
    root = Node(a[mid])
    root.left  = build(a, lo, mid - 1)
    root.right = build(a, mid + 1, hi)
    return root

def sorted_array_to_bst(a):
    return build(a, 0, len(a) - 1)
Why the middle gives minimum height:
  • Choosing the middle splits the remaining keys into two parts whose sizes differ by at most one. The height satisfies H(n) = 1 + H(n/2), which resolves to O(log n).
  • Any choice off-center makes one subtree larger, which can only increase the maximum depth, never decrease it. The middle is therefore the height-minimizing pivot at every level.

2. Sorted Linked List → Balanced BST

The same divide-and-conquer works on a sorted linked list, but a list has no O(1) random access, so locating the middle each time costs O(n) and the whole thing degrades to O(n log n). The trick is to build the tree in inorder order while walking the list once.

Since an inorder traversal visits nodes in sorted order, and the list is already sorted, we can build the left subtree first, then consume the current list head as the root (advancing a shared pointer), then build the right subtree. Counting the number of nodes up front tells us how to split; the list pointer is consumed exactly in sorted order, giving O(n) total.

List:  1 -> 2 -> 3 -> 4 -> 5 -> 6 -> 7   (n = 7)

convert(7):
   left  = convert(3)        builds subtree over first 3 nodes
   root  = take list head    (the 4th node, value 4)
   right = convert(3)        builds subtree over last 3 nodes

The moving pointer is consumed left-to-right: 1,2,3,4,5,6,7
so each node is "placed" exactly when inorder reaches it.

                  4
                /   \
               2     6
              / \   / \
             1   3 5   7

struct ListNode { int val; ListNode* next; };

ListNode* head;  // shared moving pointer

int length(ListNode* h) {
    int n = 0;
    for (; h; h = h->next) n++;
    return n;
}

// Build a balanced BST over the next `n` list nodes, inorder.
Node* convert(int n) {
    if (n <= 0) return nullptr;
    Node* left = convert(n / 2);          // left half first
    Node* root = new Node(head->val);     // consume head as root
    head = head->next;                       // advance pointer
    root->left  = left;
    root->right = convert(n - n / 2 - 1);  // remaining = right half
    return root;
}

Node* sortedListToBST(ListNode* h) {
    head = h;
    return convert(length(h));
}

class ListNode:
    def __init__(self, v, nxt=None):
        self.val, self.next = v, nxt

def sorted_list_to_bst(head):
    def length(h):
        n = 0
        while h:
            n += 1; h = h.next
        return n

    cur = [head]  # box so inner fn can mutate the pointer

    def convert(n):
        if n <= 0:
            return None
        left = convert(n // 2)            # left half first
        root = Node(cur[0].val)           # consume head as root
        cur[0] = cur[0].next            # advance pointer
        root.left  = left
        root.right = convert(n - n // 2 - 1)
        return root

    return convert(length(head))
Why this beats the repeated-middle approach: Finding the middle with slow/fast pointers and recursing costs T(n) = 2T(n/2) + O(n) = O(n log n). The inorder-simulation visits each list node once and does O(1) work per node, so it is O(n) time with O(log n) recursion stack. A doubly linked list works identically; the prev pointers are simply ignored during the build.

3. BST → Sorted Doubly Linked List (in-place)

The reverse direction: flatten a BST into a sorted doubly linked list reusing the existing nodes — left becomes prev and right becomes next. An inorder traversal visits keys in sorted order, so we keep a running prev pointer and thread each visited node onto the tail. In the circular variant we finally join head and tail.

BST:            4
              /   \
             2     6
            / \   / \
           1   3 5   7

Inorder threading sets prev/next as nodes are visited:

   head                                            tail
    |                                                |
    1 <-> 2 <-> 3 <-> 4 <-> 5 <-> 6 <-> 7

Circular variant: also link  tail.next = head  and  head.prev = tail
    +-----------------------------------------------+
    |                                               |
    v                                               |
    1 <-> 2 <-> 3 <-> 4 <-> 5 <-> 6 <-> 7 ----------+

// Uses left = prev, right = next. Returns the list head.
Node *head = nullptr, *prev = nullptr;

void inorder(Node* node) {
    if (!node) return;
    inorder(node->left);
    if (!prev) head = node;          // leftmost node = head
    else { prev->right = node; node->left = prev; }
    prev = node;                     // node becomes new tail
    inorder(node->right);
}

Node* treeToDoublyList(Node* root, bool circular) {
    head = prev = nullptr;
    inorder(root);
    if (circular && head) {           // prev is now the tail
        prev->right = head;
        head->left  = prev;
    }
    return head;
}

// Uses left = prev, right = next. Returns the list head.
Node head = null, prev = null;

void inorder(Node node) {
    if (node == null) return;
    inorder(node.left);
    if (prev == null) head = node;   // leftmost node = head
    else { prev.right = node; node.left = prev; }
    prev = node;                       // node becomes new tail
    inorder(node.right);
}

Node treeToDoublyList(Node root, boolean circular) {
    head = prev = null;
    inorder(root);
    if (circular && head != null) {  // prev is now the tail
        prev.right = head;
        head.left  = prev;
    }
    return head;
}

# Uses left = prev, right = next. Returns the list head.
def tree_to_doubly_list(root, circular=False):
    state = {"head": None, "prev": None}

    def inorder(node):
        if not node:
            return
        inorder(node.left)
        if state["prev"] is None:
            state["head"] = node        # leftmost node = head
        else:
            state["prev"].right = node
            node.left = state["prev"]
        state["prev"] = node              # node becomes new tail
        inorder(node.right)

    inorder(root)
    head, tail = state["head"], state["prev"]
    if circular and head:
        tail.right = head
        head.left  = tail
    return head

4. Construct BST from Preorder Traversal

A preorder array [root, ...left subtree..., ...right subtree...] uniquely determines a BST (the BST property fills in the ambiguity). The naive split-by-search is O(n^2); the O(n) approach passes an upper bound down the recursion: a node belongs to the current subtree only if its value is below the bound.

Preorder: [8, 5, 1, 7, 10, 12]

build(bound = +inf):
  take 8 -> root.        recurse left with bound = 8
    take 5 -> 8.left.    recurse left with bound = 5
      take 1 -> 5.left.  next is 7, 7 > 5 -> stop left
      next is 7, 7 < 8 -> recurse right with bound = 8
      take 7 -> 5.right. next is 10, 10 > 8 -> stop
    next is 10, 10 < inf -> recurse right with bound = +inf
    take 10 -> 8.right.  recurse right, take 12 -> 10.right

                  8
                /   \
               5     10
              / \      \
             1   7      12

int idx;
vector<int> pre;

Node* build(int bound) {
    if (idx == (int)pre.size() || pre[idx] > bound)
        return nullptr;
    Node* root = new Node(pre[idx++]);
    root->left  = build(root->val);   // left keys < root
    root->right = build(bound);       // right keys < inherited bound
    return root;
}

Node* bstFromPreorder(vector<int>& preorder) {
    pre = preorder; idx = 0;
    return build(INT_MAX);
}

int idx;
int[] pre;

Node build(int bound) {
    if (idx == pre.length || pre[idx] > bound)
        return null;
    Node root = new Node(pre[idx++]);
    root.left  = build(root.val);    // left keys < root
    root.right = build(bound);       // right keys < inherited bound
    return root;
}

Node bstFromPreorder(int[] preorder) {
    pre = preorder; idx = 0;
    return build(Integer.MAX_VALUE);
}

def bst_from_preorder(preorder):
    idx = [0]

    def build(bound):
        if idx[0] == len(preorder) or preorder[idx[0]] > bound:
            return None
        val = preorder[idx[0]]
        idx[0] += 1
        root = Node(val)
        root.left  = build(val)        # left keys < root
        root.right = build(bound)      # right keys < inherited bound
        return root

    return build(float("inf"))
Monotonic-stack alternative: Maintain a stack of ancestors with decreasing values. For each value, while it exceeds the stack top, pop — the last popped node is its parent and the new node is that parent's right child; otherwise it is the left child of the current top. Each node is pushed and popped once, so this is also O(n).

5. Recover a BST (Two Nodes Swapped)

Exactly two nodes of a valid BST were swapped by mistake. An inorder traversal of a correct BST is strictly increasing, so the swap creates one or two descending adjacent pairs. Identify the offenders, then swap their values back (the structure is untouched).

  • first = the larger element of the first descending pair.
  • second = the smaller element of the last descending pair.
  • If the two swapped nodes are adjacent in the inorder order there is only one descending pair, so second is set from that same pair.
Correct inorder:  1  2  3  4  5  6  7
Swap 2 and 6:     1  6  3  4  5  2  7
                     ^--first         (6 > 3: first descending pair)
                                ^--second (5 > 2: last descending pair)

first  = 6  (larger of first dip),  second = 2 (smaller of last dip)
Swap their values back -> 1 2 3 4 5 6 7   restored.

Adjacent case, swap 3 and 4:  1 2 4 3 5 6 7
Only one dip (4 > 3): first = 4, second = 3.

Node *first = nullptr, *second = nullptr, *prev = nullptr;

void inorder(Node* node) {
    if (!node) return;
    inorder(node->left);
    if (prev && prev->val > node->val) {  // a descending pair
        if (!first) first = prev;        // larger of first dip
        second = node;                   // smaller of last dip
    }
    prev = node;
    inorder(node->right);
}

void recoverTree(Node* root) {
    first = second = prev = nullptr;
    inorder(root);
    if (first && second) swap(first->val, second->val);
}

Node first = null, second = null, prev = null;

void inorder(Node node) {
    if (node == null) return;
    inorder(node.left);
    if (prev != null && prev.val > node.val) {  // descending pair
        if (first == null) first = prev;       // larger of first dip
        second = node;                          // smaller of last dip
    }
    prev = node;
    inorder(node.right);
}

void recoverTree(Node root) {
    first = second = prev = null;
    inorder(root);
    if (first != null && second != null) {
        int t = first.val; first.val = second.val; second.val = t;
    }
}

def recover_tree(root):
    state = {"first": None, "second": None, "prev": None}

    def inorder(node):
        if not node:
            return
        inorder(node.left)
        prev = state["prev"]
        if prev and prev.val > node.val:    # descending pair
            if state["first"] is None:
                state["first"] = prev        # larger of first dip
            state["second"] = node           # smaller of last dip
        state["prev"] = node
        inorder(node.right)

    inorder(root)
    f, s = state["first"], state["second"]
    if f and s:
        f.val, s.val = s.val, f.val
O(1) space — Morris traversal: The recursive/stack version uses O(h) space. A Morris inorder traversal temporarily threads each node's predecessor right pointer to itself, achieving constant extra space while applying the exact same first/second/prev logic. It is the canonical follow-up for this problem.

6. Balance an Unbalanced BST

To rebalance any BST, combine the inorder traversal with section 1: walk the tree inorder to extract a sorted array, then rebuild a height-balanced BST from that array. Two linear passes give O(n) time and O(n) space.

void inorder(Node* n, vector<int>& out) {
    if (!n) return;
    inorder(n->left, out);
    out.push_back(n->val);
    inorder(n->right, out);
}

Node* balanceBST(Node* root) {
    vector<int> sorted;
    inorder(root, sorted);             // sorted keys
    return build(sorted, 0, (int)sorted.size() - 1);  // reuse section 1
}

def balance_bst(root):
    keys = []
    def inorder(n):
        if not n:
            return
        inorder(n.left)
        keys.append(n.val)
        inorder(n.right)
    inorder(root)                      # sorted keys
    return build(keys, 0, len(keys) - 1)  # reuse section 1

The unifying principle — inorder ↔ sorted bijection:
  • For any BST, the inorder traversal yields its keys in strictly sorted order, and conversely any sorted sequence laid out by inorder reconstructs a BST.
  • This bijection is why building from a sorted array, flattening to a sorted list, rebalancing, and detecting a corrupted node are all variations of "walk in inorder and either read or write the sorted stream."
  • Preorder reconstruction is the one outlier: it relies on the BST ordering invariant (via the upper bound) rather than the inorder stream, because preorder alone fixes the root-first shape.
