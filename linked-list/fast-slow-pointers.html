Fast & Slow Pointers

Picture a tortoise and a hare on a track. The hare runs twice as fast. On a straight track, when the hare reaches the end the tortoise is exactly halfway. On a circular track, the hare eventually laps the tortoise and they meet. Those two facts solve a whole family of linked list problems in a single pass with O(1) memory.

Use case 1 — Find the middle

Move slow by 1 and fast by 2. When fast falls off the end, slow sits on the middle node. One pass, no length counting needed.

Node* middle(Node* head) {
    Node* slow = head;
    Node* fast = head;
    while (fast && fast->next) {
        slow = slow->next;        // +1
        fast = fast->next->next;   // +2
    }
    return slow;                  // middle (2nd of two for even n)
}

Node middle(Node head) {
    Node slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;         // +1
        fast = fast.next.next;    // +2
    }
    return slow;
}

def middle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next          # +1
        fast = fast.next.next     # +2
    return slow

Use case 2 — Detect a cycle (Floyd's algorithm)

If the list loops back on itself, fast will eventually lap slow and they'll land on the same node. If fast reaches null, there's no cycle. This is Floyd's cycle detection.

bool hasCycle(Node* head) {
    Node* slow = head;
    Node* fast = head;
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) return true;  // they met
    }
    return false;                 // fast hit null
}

boolean hasCycle(Node head) {
    Node slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true;
    }
    return false;
}

def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:
            return True
    return False
Finding where the cycle starts: after slow and fast meet, reset one pointer to the head and move both one step at a time. They meet again exactly at the cycle's entry node. (It's a neat number-theory result — the distance from head to the loop start equals the distance from the meeting point to the loop start.)

Use case 3 — Nth node from the end

A close cousin: advance a lead pointer N steps first, then move both together. When the lead hits null, the trailing pointer is on the Nth-from-last node — one pass, no length count.

Node* nthFromEnd(Node* head, int n) {
    Node* lead = head;
    for (int i = 0; i < n; i++) lead = lead->next;  // gap of n
    Node* trail = head;
    while (lead) { lead = lead->next; trail = trail->next; }
    return trail;
}

Node nthFromEnd(Node head, int n) {
    Node lead = head;
    for (int i = 0; i < n; i++) lead = lead.next;  // gap of n
    Node trail = head;
    while (lead != null) { lead = lead.next; trail = trail.next; }
    return trail;
}

def nth_from_end(head, n):
    lead = head
    for _ in range(n):
        lead = lead.next          # gap of n
    trail = head
    while lead:
        lead, trail = lead.next, trail.next
    return trail
Always guard fast.next before fast.next.next. The loop condition fast != null && fast.next != null is what stops you from dereferencing null at the end of an even-length list.
