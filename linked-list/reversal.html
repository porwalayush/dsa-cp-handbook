Reversing a Linked List

This is the rite-of-passage linked list problem. The goal: turn 1 → 2 → 3 → null into 3 → 2 → 1 → null. The trick is to flip each arrow to point backward, one at a time — but you must save the next node before you flip, or you'll lose the rest of the chain.

The three-pointer dance

We carry three pointers as we walk:

  • prev — the part already reversed (starts at null).
  • curr — the node we're flipping right now (starts at head).
  • next — a temporary handle on the rest, so we don't lose it when we flip.

Iterative reversal

Four lines inside the loop, in this exact order. The mnemonic: save → flip → slide → slide.

Node* reverse(Node* head) {
    Node* prev = nullptr;
    Node* curr = head;
    while (curr != nullptr) {
        Node* next = curr->next;  // 1. save the rest
        curr->next = prev;        // 2. flip the arrow
        prev = curr;              // 3. slide prev forward
        curr = next;              // 4. slide curr forward
    }
    return prev;                  // prev is the new head
}

Node reverse(Node head) {
    Node prev = null;
    Node curr = head;
    while (curr != null) {
        Node next = curr.next;    // 1. save the rest
        curr.next = prev;         // 2. flip the arrow
        prev = curr;              // 3. slide prev forward
        curr = next;              // 4. slide curr forward
    }
    return prev;                  // prev is the new head
}

def reverse(head):
    prev, curr = None, head
    while curr:
        nxt = curr.next           # 1. save the rest
        curr.next = prev          # 2. flip the arrow
        prev = curr               # 3. slide prev forward
        curr = nxt                # 4. slide curr forward
    return prev                   # prev is the new head
If you forget step 1 (next = curr.next) and flip first, you overwrite curr.next and lose the entire unreversed tail. Always save before you flip.

Recursive reversal

The recursive version reverses the rest of the list first, then fixes up the link between the current node and its old successor. Elegant, but uses O(n) stack space.

Node* reverse(Node* head) {
    if (!head || !head->next) return head;  // base case
    Node* newHead = reverse(head->next);
    head->next->next = head;   // make successor point back
    head->next = nullptr;       // old head becomes tail
    return newHead;
}

Node reverse(Node head) {
    if (head == null || head.next == null) return head;
    Node newHead = reverse(head.next);
    head.next.next = head;    // successor points back
    head.next = null;          // old head becomes tail
    return newHead;
}

def reverse(head):
    if not head or not head.next:
        return head
    new_head = reverse(head.next)
    head.next.next = head         # successor points back
    head.next = None             # old head becomes tail
    return new_head
Both run in O(n) time. Iterative uses O(1) extra space; recursive uses O(n) on the call stack. In interviews, prefer iterative unless asked otherwise — and know that "reverse part of the list" (between positions m and n) is just this technique applied to a sublist after a dummy node.
