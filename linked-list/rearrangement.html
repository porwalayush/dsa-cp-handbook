Rearranging a Linked List

Reversal flips a whole chain. But a huge family of interview problems asks you to rearrange a list into a new shape — swap neighbors, rotate the tail to the front, or interleave the front and back. The good news: every one of these is just careful pointer rewiring. Save what you'll need, splice, move on. This page walks through five classics, each with a plain-language picture first.

One mental model for all of them: a dummy node in front of the head gives you a stable handle so you never special-case the first element. Many of these solutions start with dummy.next = head and return dummy.next at the end.

1. Reverse Nodes in k-Groups

The picture: imagine a deck of cards laid in a row. You scoop up the first k cards, flip that little pile over, set it back down, then move to the next k. If the last pile has fewer than k cards, you leave it untouched.

So 1→2→3→4→5 with k=2 becomes 2→1→4→3→5 — pairs reversed, the lonely 5 stays put.

The trick is two-part: (a) before reversing a group, peek ahead k nodes to confirm a full group exists; (b) reverse exactly k nodes, then stitch the reversed group's tail to the head of whatever the next group reverses into.

Node* reverseKGroup(Node* head, int k) {
    Node* node = head;
    for (int i = 0; i < k; i++) {       // 1. is there a full group?
        if (!node) return head;          // leftover tail < k: leave as-is
        node = node->next;
    }
    Node* prev = reverseKGroup(node, k);   // 2. reverse the rest first
    Node* curr = head;
    for (int i = 0; i < k; i++) {       // 3. reverse this group of k
        Node* nxt = curr->next;
        curr->next = prev;
        prev = curr;
        curr = nxt;
    }
    return prev;                          // new head of this group
}

Node reverseKGroup(Node head, int k) {
    Node node = head;
    for (int i = 0; i < k; i++) {       // 1. is there a full group?
        if (node == null) return head;   // leftover tail < k: leave as-is
        node = node.next;
    }
    Node prev = reverseKGroup(node, k);   // 2. reverse the rest first
    Node curr = head;
    for (int i = 0; i < k; i++) {       // 3. reverse this group of k
        Node nxt = curr.next;
        curr.next = prev;
        prev = curr;

}

def reverseKGroup(head, k):
    node = head
    for _ in range(k):               # 1. is there a full group?
        if not node:
            return head                # leftover tail < k: leave as-is
        node = node.next
    prev = reverseKGroup(node, k)     # 2. reverse the rest first
    curr = head
    for _ in range(k):               # 3. reverse this group of k
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt
    return prev                       # new head of this group

Complexity: O(n) time — each node is visited a constant number of times. O(n/k) extra stack space for the recursion (or O(1) with an iterative version).

2. Swap Nodes in Pairs

The picture: this is just "reverse in k-groups" with k = 2 — partners on a dance floor switching places. 1→2→3→4 becomes 2→1→4→3. Because the group size is fixed at two, you can write it cleanly without the group-counting machinery.

For each pair we have three actors: prev (the node before the pair, starting at a dummy), and the two partners a and b. We make prev → b → a, then re-anchor prev at a for the next pair.

Node* swapPairs(Node* head) {
    Node dummy(0); dummy.next = head;
    Node* prev = &dummy;
    while (prev->next && prev->next->next) {
        Node* a = prev->next;
        Node* b = a->next;
        a->next = b->next;     // a points past b
        b->next = a;           // b points back to a
        prev->next = b;        // prev points to new front b
        prev = a;              // a is now before the next pair
    }
    return dummy.next;
}

Node swapPairs(Node head) {
    Node dummy = new Node(0); dummy.next = head;
    Node prev = dummy;
    while (prev.next != null && prev.next.next != null) {
        Node a = prev.next;
        Node b = a.next;
        a.next = b.next;       // a points past b
        b.next = a;            // b points back to a
        prev.next = b;         // prev points to new front b
        prev = a;              // a is now before the next pair
    }
    return dummy.next;
}

def swapPairs(head):
    dummy = Node(0); dummy.next = head
    prev = dummy
    while prev.next and prev.next.next:
        a = prev.next
        b = a.next
        a.next = b.next        # a points past b
        b.next = a             # b points back to a
        prev.next = b          # prev points to new front b
        prev = a               # a is now before the next pair
    return dummy.next

Complexity: O(n) time, O(1) space — one pass, constant pointers.

3. Rotate List to the Right by k

The picture: picture the list as a loop of people holding hands in a circle. "Rotate right by k" means: walk the join point backward by k hands and cut the circle there. The chunk that was at the back swings around to the front.

1→2→3→4→5 rotated right by 2 gives 4→5→1→2→3 — the last two slide to the front.

Three steps: (1) measure the length n and connect the tail to the head to form a ring; (2) reduce k mod n (rotating by the full length changes nothing); (3) the new tail is at index n - k - 1 from the old head — walk there, set the new head to its next, and break the ring.

Node* rotateRight(Node* head, int k) {
    if (!head || !head->next || k == 0) return head;
    int n = 1;
    Node* tail = head;
    while (tail->next) { tail = tail->next; n++; }  // find length + tail
    tail->next = head;                            // close the ring
    k %= n;
    int stepsToNewTail = n - k;
    Node* newTail = head;
    for (int i = 1; i < stepsToNewTail; i++) newTail = newTail->next;
    Node* newHead = newTail->next;
    newTail->next = nullptr;                      // break the ring
    return newHead;
}

Node rotateRight(Node head, int k) {
    if (head == null || head.next == null || k == 0) return head;
    int n = 1;
    Node tail = head;
    while (tail.next != null) { tail = tail.next; n++; }  // length + tail
    tail.next = head;                              // close the ring
    k %= n;
    int stepsToNewTail = n - k;
    Node newTail = head;
    for (int i = 1; i < stepsToNewTail; i++) newTail = newTail.next;
    Node newHead = newTail.next;
    newTail.next = null;                          // break the ring
    return newHead;
}

def rotateRight(head, k):
    if not head or not head.next or k == 0:
        return head
    n, tail = 1, head
    while tail.next:                  # find length + tail
        tail = tail.next; n += 1
    tail.next = head                  # close the ring
    k %= n
    steps_to_new_tail = n - k
    new_tail = head
    for _ in range(steps_to_new_tail - 1):
        new_tail = new_tail.next
    new_head = new_tail.next
    new_tail.next = None             # break the ring
    return new_head
Always take k % n first. Test cases love to pass k far larger than the list length. Without the modulo you'd walk off the end (or loop forever on the ring).

Complexity: O(n) time, O(1) space.

4. Odd-Even Linked List

The picture: children line up and you ask everyone in an odd spot (1st, 3rd, 5th…) to step to the left, and everyone in an even spot to the right. Order within each group is preserved; then you glue the even line onto the end of the odd line.

Note this is about position, not value: 1→2→3→4→5 becomes 1→3→5→2→4 (odd-indexed nodes first, then even-indexed).

Weave two chains at once: an odd pointer and an even pointer that each hop forward by two. Remember the head of the even chain (evenHead) so you can attach it after the odd chain finishes.

Node* oddEvenList(Node* head) {
    if (!head || !head->next) return head;
    Node* odd = head;
    Node* even = head->next;
    Node* evenHead = even;            // remember where evens start
    while (even && even->next) {
        odd->next = even->next;        // skip the even node
        odd = odd->next;
        even->next = odd->next;        // skip the odd node
        even = even->next;
    }
    odd->next = evenHead;             // stitch evens after odds
    return head;
}

Node oddEvenList(Node head) {
    if (head == null || head.next == null) return head;
    Node odd = head;
    Node even = head.next;
    Node evenHead = even;             // remember where evens start
    while (even != null && even.next != null) {
        odd.next = even.next;         // skip the even node
        odd = odd.next;
        even.next = odd.next;         // skip the odd node
        even = even.next;
    }
    odd.next = evenHead;              // stitch evens after odds
    return head;
}

def oddEvenList(head):
    if not head or not head.next:
        return head
    odd = head
    even = head.next
    even_head = even                  # remember where evens start
    while even and even.next:
        odd.next = even.next          # skip the even node
        odd = odd.next
        even.next = odd.next          # skip the odd node
        even = even.next
    odd.next = even_head              # stitch evens after odds
    return head

Complexity: O(n) time, O(1) space — single pass, no new nodes.

5. Reorder List

The picture: fold the list in half like a strip of paper. The front and the back zip together: L0 → Ln → L1 → Ln-1 → L2 → …. So 1→2→3→4→5 becomes 1→5→2→4→3.

This is the grand finale because it composes three earlier techniques:

  1. Find the middle with slow/fast pointers (tortoise and hare).
  2. Reverse the second half (the reversal pattern from the previous page).
  3. Merge the two halves by alternating one node from each.
void reorderList(Node* head) {
    if (!head || !head->next) return;
    // 1. find middle (slow ends at mid)
    Node *slow = head, *fast = head;
    while (fast->next && fast->next->next) {
        slow = slow->next; fast = fast->next->next;
    }
    // 2. reverse the second half
    Node* second = slow->next;
    slow->next = nullptr;
    Node* prev = nullptr;
    while (second) {
        Node* nxt = second->next;
        second->next = prev; prev = second; second = nxt;
    }
    second = prev;            // head of reversed half
    // 3. merge two halves alternately
    Node* first = head;
    while (second) {
        Node* f = first->next, *s = second->next;
        first->next = second; second->next = f;
        first = f; second = s;
    }
}

void reorderList(Node head) {
    if (head == null || head.next == null) return;
    // 1. find middle (slow ends at mid)
    Node slow = head, fast = head;
    while (fast.next != null && fast.next.next != null) {
        slow = slow.next; fast = fast.next.next;
    }
    // 2. reverse the second half
    Node second = slow.next;
    slow.next = null;
    Node prev = null;
    while (second != null) {
        Node nxt = second.next;
        second.next = prev; prev = second; second = nxt;
    }
    second = prev;            // head of reversed half
    // 3. merge two halves alternately
    Node first = head;
    while (second != null) {
        Node f = first.next, s = second.next;
        first.next = second; second.next = f;
        first = f; second = s;
    }
}

def reorderList(head):
    if not head or not head.next:
        return
    # 1. find middle (slow ends at mid)
    slow = fast = head
    while fast.next and fast.next.next:
        slow = slow.next; fast = fast.next.next
    # 2. reverse the second half
    second = slow.next
    slow.next = None
    prev = None
    while second:
        nxt = second.next
        second.next = prev; prev = second; second = nxt
    second = prev             # head of reversed half
    # 3. merge two halves alternately
    first = head
    while second:
        f, s = first.next, second.next
        first.next = second; second.next = f
        first, second = f, s

Complexity: O(n) time (three linear passes), O(1) space.

The universal bug: losing a pointer mid-rewire. Before you overwrite any .next, ask "do I still have a handle on the node it used to point to?" If not, save it in a temp variable first. This single discipline solves every problem on this page.
