Advanced Patterns

These are the problems that show up in real interviews and real systems. Each one is just the basics — traverse, splice, dummy nodes, fast/slow — combined in a clever way.

LRU Cache

An LRU (Least Recently Used) cache keeps the k most recently used items and evicts the oldest when it's full. The classic design pairs a doubly linked list (ordering by recency) with a hash map (O(1) lookup):

  • Hash map: key → node, so get is O(1).
  • Doubly linked list: most-recently-used at the front, least-recently-used at the back. On any access, splice the node to the front. When full, drop the back node.
🗄️ LRU Cache (capacity 3)
"Use" a key to move it to the front. When a 4th distinct key arrives, the least-recently-used one (at the back) is evicted.
← most recent (front)least recent (back) →
Empty cache. Use a key to add it.
# Python's OrderedDict makes this almost free,
# but here's the idea with the move-to-end trick.
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity):
        self.cap = capacity
        self.cache = OrderedDict()

    def get(self, key):
        if key not in self.cache:
            return -1
        self.cache.move_to_end(key)      # mark as recent
        return self.cache[key]

    def put(self, key, value):
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.cap:
            self.cache.popitem(last=False)  # evict oldest

// LinkedHashMap gives access-order for free.
class LRUCache extends LinkedHashMap<Integer, Integer> {
    private final int cap;
    LRUCache(int capacity) {
        super(capacity, 0.75f, true);  // access order
        cap = capacity;
    }
    public int get(int key) {
        return getOrDefault(key, -1);
    }
    public void put(int key, int value) { super.put(key, value); }
    protected boolean removeEldestEntry(Map.Entry e) {
        return size() > cap;          // evict oldest
    }
}

// list<> is a doubly linked list; map key -> iterator.
class LRUCache {
    int cap;
    list<pair<int,int>> dll;                // front = most recent
    unordered_map<int, list<pair<int,int>>::iterator> pos;
public:
    LRUCache(int capacity) : cap(capacity) {}
    int get(int key) {
        if (!pos.count(key)) return -1;
        dll.splice(dll.begin(), dll, pos[key]);  // move to front
        return pos[key]->second;
    }
    void put(int key, int value) {
        if (pos.count(key)) dll.erase(pos[key]);
        dll.push_front({key, value});
        pos[key] = dll.begin();
        if (dll.size() > cap) {        // evict back
            pos.erase(dll.back().first);
            dll.pop_back();
        }
    }
};

Copy a list with random pointers

Each node has a next and a random pointer to any node. To deep-copy it, the elegant O(1)-space trick is to weave clones in: put each clone right after its original, wire up the randoms using that adjacency, then unweave.

def copyRandomList(head):
    if not head: return None
    # 1. weave: A -> A' -> B -> B' -> ...
    cur = head
    while cur:
        clone = Node(cur.val)
        clone.next, cur.next = cur.next, clone
        cur = clone.next
    # 2. assign randoms via adjacency
    cur = head
    while cur:
        if cur.random:
            cur.next.random = cur.random.next
        cur = cur.next.next
    # 3. unweave the two lists
    cur, copy_head = head, head.next
    while cur:
        clone = cur.next
        cur.next = clone.next
        clone.next = clone.next.next if clone.next else None
        cur = cur.next
    return copy_head

Node copyRandomList(Node head) {
    if (head == null) return null;
    // 1. weave clones in
    for (Node c = head; c != null; c = c.next.next) {
        Node clone = new Node(c.val);
        clone.next = c.next; c.next = clone;
    }
    // 2. wire randoms
    for (Node c = head; c != null; c = c.next.next)
        if (c.random != null) c.next.random = c.random.next;
    // 3. unweave
    Node copy = head.next;
    for (Node c = head; c != null; c = c.next) {
        Node clone = c.next;
        c.next = clone.next;
        if (clone.next != null) clone.next = clone.next.next;
    }
    return copy;
}

Pattern radar: almost every linked list problem is one of — (a) rewire pointers carefully, (b) use a dummy node to dodge the head edge case, (c) use two pointers at different speeds or with a fixed gap, or (d) pair the list with a hash map for O(1) lookup. Recognize which one and you're most of the way there.
