Doubly & Circular Linked Lists

A singly linked list is a one-way street: every node knows the next house, but none of them remember the one behind. That's fine until you're standing at some node and need to back up — or splice it out — and realize you have no way to reach its predecessor. The two variations on this page fix that, each in its own way: doubly linked lists add a backward arrow, and circular lists join the ends so the chain loops forever.

Doubly Linked Lists

Think of a doubly linked list like a train: each car is coupled to the one in front and the one behind. From any car you can walk toward the engine or toward the caboose. In list terms, every node carries two pointers instead of one — a next and a prev.

// A node that remembers both neighbors
struct Node {
    int data;
    Node* prev;

    Node(int d) : data(d), prev(nullptr), next(nullptr) {}

// A node that remembers both neighbors
class Node {
    int data;
    Node prev, next;
    Node(int d) { data = d; }

# A node that remembers both neighbors
class Node:
    def __init__(self, data):
        self.data = data
        self.prev = None
        self.next = None

Inserting at the head or tail

Inserting still means "rewire pointers in the right order" — there are just twice as many to keep straight. The trick: connect the newcomer's prev and next first, then fix up the neighbors that should now point back at it. With a tail pointer kept alongside head, both ends are O(1).

// Push a new value onto the front. Returns new head.
Node* insertHead(Node* head, int value) {
    Node* node = new Node(value);
    node->next = head;            // newcomer points forward
    if (head) head->prev = node;   // old head looks back
    return node;                   // node is the new head
}

// Append to the back. tail is updated by reference.
void insertTail(Node*& head, Node*& tail, int value) {
    Node* node = new Node(value);
    node->prev = tail;            // newcomer looks back at old tail
    if (tail) tail->next = node;   // old tail points forward
    else head = node;             // list was empty
    tail = node;                  // node is the new tail
}

// Insert value right after an existing node p.
void insertAfter(Node* p, int value) {
    Node* node = new Node(value);
    node->prev = p;
    node->next = p->next;
    if (p->next) p->next->prev = node;
    p->next = node;
}

// Push a new value onto the front. Returns new head.

The killer feature: O(1) deletion of a known node

This is why doubly linked lists exist. In a singly linked list, to delete a node you must first find its predecessor by walking from the head — that's O(n). In a doubly linked list, the node already holds a pointer to its predecessor (prev), so you can unhook it in constant time without scanning anything. You hand the function the node itself and it splices its two neighbors together.

// Remove node x in O(1). Updates head/tail if needed.
void deleteNode(Node*& head, Node*& tail, Node* x) {
    if (x->prev) x->prev->next = x->next;  // left neighbor skips x
    else        head = x->next;            // x was the head
    if (x->next) x->next->prev = x->prev;  // right neighbor skips x
    else        tail = x->prev;            // x was the tail
    delete x;
}

// Remove node x in O(1). (head/tail kept in a wrapper class)
void deleteNode(Node x) {
    if (x.prev != null) x.prev.next = x.next;  // left skips x
    else                  head = x.next;        // x was head
    if (x.next != null) x.next.prev = x.prev;  // right skips x
    else                  tail = x.prev;        // x was tail
}

# Remove node x in O(1).
def delete_node(self, x):
    if x.prev: x.prev.next = x.next   # left neighbor skips x
    else:      self.head = x.next     # x was the head
    if x.next: x.next.prev = x.prev   # right neighbor skips x
    else:      self.tail = x.prev     # x was the tail
Why O(1) matters: data structures like an LRU cache rely on yanking an arbitrary node out of the middle the instant it's used and moving it to the front. With a singly linked list that's O(n) every time; with a doubly linked list it's O(1) — which is exactly why LinkedHashMap and most LRU implementations use one.

Circular Linked Lists

Picture a group of friends sitting around a campfire. Go around handing out marshmallows and after the last person you're right back at the first — there's no "end," the seating just loops. A circular linked list is the same idea: the last node's next points back to the head instead of to null.

The catch: there is no null to tell you you're done. To know when you've gone all the way around, you compare back to the head — keep walking curr = curr.next until curr == head again. (Use a do-while-style loop so you visit the head once before the check fires.)

// Traverse exactly once around the ring.
void traverse(Node* head) {
    if (!head) return;
    Node* curr = head;
    do {
        cout << curr->data << " ";
        curr = curr->next;
    } while (curr != head);   // stop when we loop back
}

// Insert at the front of a circular list (tail->next == head).
Node* insertFront(Node* head, Node* tail, int value) {
    Node* node = new Node(value);
    node->next = head ? head : node;  // single node points to itself
    if (tail) tail->next = node;       // last node closes the loop
    return node;                       // new head
}

// Traverse exactly once around the ring.
void traverse(Node head) {
    if (head == null) return;
    Node curr = head;
    do {
        System.out.print(curr.data + " ");
        curr = curr.next;
    } while (curr != head);   // stop when we loop back
}

// Insert at the front (tail.next == head).
Node insertFront(Node head, Node tail, int value) {
    Node node = new Node(value);
    node.next = (head != null) ? head : node;
    if (tail != null) tail.next = node;
    return node;
}

# Traverse exactly once around the ring.
def traverse(head):
    if not head: return
    curr = head
    while True:
        print(curr.data, end=" ")
        curr = curr.next
        if curr is head:   # looped all the way around
            break

# Insert at the front (tail.next is head).
def insert_front(head, tail, value):
    node = Node(value)
    node.next = head if head else node  # lone node points to itself
    if tail: tail.next = node          # close the loop
    return node
The infinite-loop trap: any code written for a null-terminated list (like while (curr != nullptr)) will spin forever on a circular list. Always anchor your loop to head and remember to update the tail's next on every insert/delete so the loop stays closed.

Circular Doubly Linked Lists

Combine both ideas and you get the most flexible shape of all: every node has prev and next, and the ends wrap around — head.prev is the tail, and tail.next is the head. From any node you can step in either direction and never hit a dead end. This is the backbone of real LRU caches: a hash map gives O(1) lookup of a node, and the circular doubly linked list gives O(1) "move this node to the front" and "evict the back." See the advanced techniques page for how that cache is wired up end to end.

A neat simplification: circular doubly linked lists are often built with a single sentinel (dummy) node whose next and prev stand in for the head and tail. Because the ring is never truly empty, you delete the "head is null" and "tail is null" special cases entirely — every insert and delete is the same handful of pointer assignments.

The Josephus Problem

Here's the classic story: n people stand in a circle. Starting from a fixed point you count off, and every k-th person is eliminated and leaves the circle. Counting then resumes from the next person. People keep getting removed until only one survivor remains. The question: where should you stand to be that survivor? A circle of people who keep getting removed is a circular linked list almost by definition — so it's a perfect fit.

Approach 1 — Simulate it with a circular linked list

The most literal solution: build a circular list of n people, then repeatedly walk k-1 steps and unlink the next node, until one node points to itself. This mirrors exactly what the simulator above does. It's O(n·k) time — fine for small inputs and great for understanding.

// Returns the 0-indexed survivor by simulating the circle.
int josephusSim(int n, int k) {
    // Build circular list 0,1,...,n-1
    Node* head = new Node(0);
    Node* tail = head;
    for (int i = 1; i < n; i++) {
        tail->next = new Node(i);
        tail = tail->next;
    }
    tail->next = head;            // close the ring

    Node* prev = tail;            // node before the counter
    while (prev->next != prev) {    // more than one left
        for (int step = 1; step < k; step++)
            prev = prev->next;    // count k-1 hops
        Node* victim = prev->next;
        prev->next = victim->next; // unlink the k-th person
        delete victim;
    }
    return prev->data;
}

// Returns the 0-indexed survivor by simulating the circle.
int josephusSim(int n, int k) {
    Node head = new Node(0), tail = head;
    for (int i = 1; i < n; i++) {
        tail.next = new Node(i);
        tail = tail.next;
    }
    tail.next = head;            // close the ring

    Node prev = tail;
    while (prev.next != prev) {   // more than one left
        for (int step = 1; step < k; step++)
            prev = prev.next;    // count k-1 hops
        prev.next = prev.next.next;  // unlink the k-th person
    }
    return prev.data;
}

# Returns the 0-indexed survivor by simulating the circle.
def josephus_sim(n, k):
    head = Node(0); tail = head
    for i in range(1, n):
        tail.next = Node(i)
        tail = tail.next
    tail.next = head             # close the ring

    prev = tail
    while prev.next is not prev:    # more than one left
        for _ in range(k - 1):
            prev = prev.next     # count k-1 hops
        prev.next = prev.next.next  # unlink the k-th person
    return prev.data

Approach 2 — The O(n) recurrence

There's a beautiful shortcut. Let J(n, k) be the survivor's 0-indexed position. After the first person (at index k-1) is eliminated, you're left with the same problem on n-1 people, but the "start" has shifted forward by k seats. That gives the recurrence:

  • J(1, k) = 0 — one person, they survive at index 0.
  • J(n, k) = (J(n-1, k) + k) % n — shift the smaller answer forward and wrap.

Unrolled into a loop, that's O(n) time and O(1) space — no list to build, no k-counting. (Add 1 at the end if your problem uses 1-indexed seats.)

// O(n) time, O(1) space. 0-indexed survivor.
int josephus(int n, int k) {
    int result = 0;              // J(1, k) = 0
    for (int m = 2; m <= n; m++)
        result = (result + k) % m;  // J(m, k)
    return result;             // add 1 for 1-indexed seats
}

// O(n) time, O(1) space. 0-indexed survivor.
int josephus(int n, int k) {

    for (int m = 2; m <= n; m++)
        result = (result + k) % m;  // J(m, k)

# O(n) time, O(1) space. 0-indexed survivor.
def josephus(n, k):
    result = 0                  # J(1, k) = 0
    for m in range(2, n + 1):
        result = (result + k) % m   # J(m, k)
    return result             # add 1 for 1-indexed seats
Sanity check it against the simulator: with n = 7, k = 3 the recurrence returns survivor index 3 (0-indexed). Run the ring above to the end and the green survivor will be the person labeled 3 — the two approaches agree.
