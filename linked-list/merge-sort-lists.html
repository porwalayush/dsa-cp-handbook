Merging & Sorting Lists

Merging two already-sorted lists is like a zipper: you keep two fingers, one on each list's smallest remaining item, and always pull the smaller one into the result. Master this and merge sort on a linked list falls out almost for free.

The dummy-node trick

Building a list from scratch is annoying because the first node is a special case. The fix: start with a throwaway dummy node and a tail pointer. Attach everything after dummy, and return dummy.next at the end. No special case for the first element.

Merge two sorted lists

Node* mergeTwo(Node* a, Node* b) {
    Node dummy(0);            // throwaway head
    Node* tail = &dummy;
    while (a && b) {
        if (a->data <= b->data) { tail->next = a; a = a->next; }
        else                   { tail->next = b; b = b->next; }
        tail = tail->next;
    }
    tail->next = a ? a : b;     // attach the leftover tail
    return dummy.next;
}

Node mergeTwo(Node a, Node b) {
    Node dummy = new Node(0);
    Node tail = dummy;
    while (a != null && b != null) {
        if (a.data <= b.data) { tail.next = a; a = a.next; }
        else                 { tail.next = b; b = b.next; }
        tail = tail.next;
    }
    tail.next = (a != null) ? a : b;   // leftover tail
    return dummy.next;
}

def merge_two(a, b):
    dummy = Node(0)
    tail = dummy
    while a and b:
        if a.data <= b.data:
            tail.next, a = a, a.next
        else:
            tail.next, b = b, b.next
        tail = tail.next
    tail.next = a if a else b    # leftover tail
    return dummy.next
Why <= and not <? Using <= keeps equal elements in their original order — that makes the merge stable, which matters when you build merge sort on top of it.

Merge sort on a linked list

Merge sort is the best general sort for linked lists: O(n log n) time and only O(log n) stack space — no random access needed, unlike quicksort. Three steps:

  1. Split the list into two halves using the slow/fast pointer trick.
  2. Recursively sort each half.
  3. Merge the two sorted halves with the zipper above.
Node* sortList(Node* head) {
    if (!head || !head->next) return head;   // 0 or 1 node

    // 1. split: find middle with slow/fast
    Node* slow = head; Node* fast = head->next;
    while (fast && fast->next) { slow = slow->next; fast = fast->next->next; }
    Node* mid = slow->next;
    slow->next = nullptr;            // cut into two halves

    // 2. sort each half, 3. merge
    return mergeTwo(sortList(head), sortList(mid));
}

Node sortList(Node head) {
    if (head == null || head.next == null) return head;

    Node slow = head, fast = head.next;
    while (fast != null && fast.next != null) {
        slow = slow.next; fast = fast.next.next;
    }
    Node mid = slow.next;
    slow.next = null;                 // cut

    return mergeTwo(sortList(head), sortList(mid));
}

def sort_list(head):
    if not head or not head.next:
        return head

    slow, fast = head, head.next
    while fast and fast.next:
        slow, fast = slow.next, fast.next.next
    mid = slow.next
    slow.next = None                  # cut

    return merge_two(sort_list(head), sort_list(mid))
Don't forget to cut the list (slow.next = null) before recursing. If you skip it, the "first half" still runs all the way to the end and you recurse forever.
