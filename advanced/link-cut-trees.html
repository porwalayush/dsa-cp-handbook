Link-Cut Trees

A structure for a forest of trees that changes over time — you can link two trees, cut an edge, and query the path between any two nodes, all in O(log n) amortized. Built from splay trees over "preferred paths".

The Operations

   link(u, v)    — attach tree of u under v (must be in different trees)
   cut(u, v)     — remove the edge u–v, splitting one tree into two
   connected(u,v)— are they in the same tree?
   pathQuery(u,v)— aggregate (sum/max) on the path u..v
   makeRoot(u)   — re-root the tree at u

   All in O(log n) amortized via splay trees on "preferred paths".

Code: Link-Cut Tree (splay-based)

struct LCT {
    struct Node { int ch[2] = {0,0}, par = 0; bool rev = false; };
    vector<Node> t;
    LCT(int n) : t(n + 1) {}

    bool isRoot(int x) {
        return t[t[x].par].ch[0] != x && t[t[x].par].ch[1] != x;
    }
    void push(int x) {
        if (t[x].rev) {
            swap(t[x].ch[0], t[x].ch[1]);
            for (int c : t[x].ch) if (c) t[c].rev ^= 1;
            t[x].rev = false;
        }
    }
    void rotate(int x) {
        int p = t[x].par, g = t[p].par;
        int d = t[p].ch[1] == x;
        if (!isRoot(p)) t[g].ch[t[g].ch[1]==p] = x;
        t[x].par = g;
        t[p].ch[d] = t[x].ch[d^1];
        if (t[x].ch[d^1]) t[t[x].ch[d^1]].par = p;
        t[x].ch[d^1] = p; t[p].par = x;
    }
    void splay(int x) {
        while (!isRoot(x)) {
            int p = t[x].par, g = t[p].par;
            if (!isRoot(p)) rotate((t[g].ch[1]==p) == (t[p].ch[1]==x) ? p : x);
            rotate(x);
        }
    }
    int access(int x) {
        int last = 0;
        for (int y = x; y; y = t[y].par) { splay(y); t[y].ch[1] = last; last = y; }
        splay(x); return last;
    }
    void makeRoot(int x) { access(x); t[x].rev ^= 1; }
    void link(int u, int v) { makeRoot(u); t[u].par = v; }
    void cut(int u, int v) {
        makeRoot(u); access(v);
        t[v].ch[0] = t[u].par = 0;
    }
};
# Link-Cut Tree (splay-based), index 0 = null sentinel
class LCT:
    def __init__(self, n):
        self.ch = [[0, 0] for _ in range(n + 1)]
        self.par = [0] * (n + 1)
        self.rev = [False] * (n + 1)
    def is_root(self, x):
        p = self.par[x]
        return self.ch[p][0] != x and self.ch[p][1] != x
    def push(self, x):
        if self.rev[x]:
            self.ch[x][0], self.ch[x][1] = self.ch[x][1], self.ch[x][0]
            for c in self.ch[x]:
                if c: self.rev[c] = not self.rev[c]
            self.rev[x] = False
    def access(self, x):
        last = 0; y = x
        while y:
            self.splay(y); self.ch[y][1] = last; last = y; y = self.par[y]
        self.splay(x); return last
    def make_root(self, x):
        self.access(x); self.rev[x] = not self.rev[x]
    def link(self, u, v):
        self.make_root(u); self.par[u] = v
    def connected(self, u, v):
        self.make_root(u); self.access(v)
        return u == v or self.par[u] != 0

The Core Concept

  • The tree is split into preferred paths; each path is a splay tree keyed by depth.
  • access(x) makes the root→x path "preferred" — the key primitive everything builds on.
  • Amortized O(log n) per operation, proven via the same potential argument as splay trees.
  • Unlike HLD, the tree structure itself can change (link/cut) between queries.

When You Need LCT

Dynamic connectivity on a forest (edges added/removed).

Online MST / link-cut problems where the tree changes.

Path aggregates with structural updates — HLD can't re-root or cut.

Dynamic LCA, subtree-to-path transformations.

Common Pitfalls

1. The lazy rev flag: makeRoot reverses a path — push it down before touching children.

2. isRoot check: a splay-tree root (path root) is detected by neither-child-of-parent — distinct from the represented tree's root.

3. link/cut preconditions: link only different trees; cut only an existing edge — validate first.

4. It's hard: use HLD or Euler-tour trees if the tree is static or you only need connectivity — LCT is overkill unless edges truly change.
