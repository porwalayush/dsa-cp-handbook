Treap & Balanced BST

A treap is a BST by key and a heap by a random priority. The random priorities keep it balanced (expected height O(log n)) without complex rotations — and the split/merge pair makes it astonishingly versatile.

BST + Heap = Treap

   Each node has:  key (BST order)  +  priority (heap order, random)

   • In-order by KEY      → sorted sequence (it's a BST).
   • Parent priority ≥ child priority → it's a max-heap on priority.

   Because priorities are random, the tree shape is random →
   expected height O(log n). No balance bookkeeping needed!

Interactive: Split & Merge

Everything reduces to two operations. Split(k) cuts the treap into "keys ≤ k" (blue) and "keys > k" (purple). Merge glues two treaps back (left keys all < right keys).

Code: Treap with Split/Merge

struct Node {
    int key, pri, sz;
    Node *l = 0, *r = 0;
    Node(int k) : key(k), pri(rand()), sz(1) {}
};
int getSz(Node* t) { return t ? t->sz : 0; }
void upd(Node* t) { if (t) t->sz = 1 + getSz(t->l) + getSz(t->r); }

// split by KEY: left has keys <= k, right has keys > k
void split(Node* t, int k, Node*& l, Node*& r) {
    if (!t) { l = r = 0; return; }
    if (t->key <= k) { split(t->r, k, t->r, r); l = t; }
    else            { split(t->l, k, l, t->l); r = t; }
    upd(t);
}
// merge: ALL keys in a < ALL keys in b
Node* merge(Node* a, Node* b) {
    if (!a || !b) return a ? a : b;
    if (a->pri > b->pri) { a->r = merge(a->r, b); upd(a); return a; }
    else               { b->l = merge(a, b->l); upd(b); return b; }
}
Node* insert(Node* t, int key) {
    Node *l, *r; split(t, key, l, r);
    return merge(merge(l, new Node(key)), r);
}
class Node {
    int key, pri, sz;
    Node l, r;
    Node(int k) { key = k; pri = rng.nextInt(); sz = 1; }
}
int getSz(Node t) { return t == null ? 0 : t.sz; }
void upd(Node t) { if (t != null) t.sz = 1 + getSz(t.l) + getSz(t.r); }

Node[] split(Node t, int k) {       // returns {left, right}
    if (t == null) return new Node[]{null, null};
    if (t.key <= k) {
        Node[] s = split(t.r, k); t.r = s[0]; upd(t);
        return new Node[]{t, s[1]};
    } else {
        Node[] s = split(t.l, k); t.l = s[1]; upd(t);
        return new Node[]{s[0], t};
    }
}
Node merge(Node a, Node b) {
    if (a == null || b == null) return a == null ? b : a;
    if (a.pri > b.pri) { a.r = merge(a.r, b); upd(a); return a; }
    else { b.l = merge(a, b.l); upd(b); return b; }
}
import random

class Node:
    __slots__ = ('key', 'pri', 'sz', 'l', 'r')
    def __init__(self, key):
        self.key = key; self.pri = random.random()
        self.sz = 1; self.l = self.r = None

def sz(t): return t.sz if t else 0
def upd(t):
    if t: t.sz = 1 + sz(t.l) + sz(t.r)

def split(t, k):                  # left: keys <= k
    if not t: return (None, None)
    if t.key <= k:
        l, r = split(t.r, k); t.r = l; upd(t); return (t, r)
    else:
        l, r = split(t.l, k); t.l = r; upd(t); return (l, t)

def merge(a, b):                  # all keys a < all keys b
    if not a or not b: return a or b
    if a.pri > b.pri:
        a.r = merge(a.r, b); upd(a); return a
    else:
        b.l = merge(a, b.l); upd(b); return b

Implicit Treap (indexed by position)

  • Use subtree size as an implicit key → the treap behaves like an array you can split/merge at any index.
  • Enables O(log n): insert/erase at a position, reverse a subarray (lazy flag), move a block, range queries — things arrays and balanced BSTs struggle with.
  • This is the "rope" data structure for huge editable text.

What Treaps Give You

Ordered set/map with insert/erase/rank/k-th in O(log n).

Order statistics — find the k-th smallest, count less-than.

Implicit treap — array with insert/delete/reverse/range-ops.

Persistent treap — versioned sequences via path copying.

Common Pitfalls

1. Forgetting upd(): sizes/aggregates must be recomputed after every split and merge.

2. Split semantics: be precise whether "≤ k" or "< k" goes left — duplicates depend on it.

3. Weak randomness: low-entropy priorities can unbalance the tree — use a good RNG.

4. Lazy propagation: for implicit treaps with reverse/add, push down lazy tags before recursing.
