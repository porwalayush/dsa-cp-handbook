Persistent Data Structures

A persistent structure keeps every past version alive after each update — you can query any historical version. The trick: an update copies only the path it touches (O(log n) new nodes) and shares everything else.

Path Copying

   Updating one leaf of a segment tree changes only the
   O(log n) nodes on the root→leaf path.

   So: CREATE NEW copies of just those nodes, and POINT them
   at the OLD (unchanged) children. Old root still works →
   old version intact. New root → new version.

   Each update: O(log n) time AND O(log n) extra memory.

Interactive: Persistent Array (versioned)

Each update creates a new root sharing untouched subtrees with the previous version. Set an index to a value → a new version appears; only the path to that leaf is freshly colored, the rest is shared.

Code: Persistent Segment Tree

struct Node { int sum, l, r; };
vector<Node> t = {{0,0,0}}; // node 0 = null
vector<int> roots;                // roots[v] = version v

int build(int lo, int hi, vector<int>& a) {
    int id = t.size(); t.push_back({0,0,0});
    if (lo == hi) { t[id].sum = a[lo]; return id; }
    int mid = (lo+hi)/2;
    t[id].l = build(lo, mid, a);
    t[id].r = build(mid+1, hi, a);
    t[id].sum = t[t[id].l].sum + t[t[id].r].sum;
    return id;
}
// returns a NEW node id for the updated version
int update(int prev, int lo, int hi, int pos, int val) {
    int id = t.size(); t.push_back(t[prev]); // COPY this node
    if (lo == hi) { t[id].sum = val; return id; }
    int mid = (lo+hi)/2;
    if (pos <= mid) t[id].l = update(t[prev].l, lo, mid, pos, val);
    else           t[id].r = update(t[prev].r, mid+1, hi, pos, val);
    t[id].sum = t[t[id].l].sum + t[t[id].r].sum;
    return id; // only O(log n) nodes copied; rest shared
}
int[] sum, lc, rc; int cnt = 1;  // node 0 = null
int[] roots;

int build(int lo, int hi, int[] a) {
    int id = cnt++;
    if (lo == hi) { sum[id] = a[lo]; return id; }
    int mid = (lo+hi)/2;
    lc[id] = build(lo, mid, a);
    rc[id] = build(mid+1, hi, a);
    sum[id] = sum[lc[id]] + sum[rc[id]];
    return id;
}
int update(int prev, int lo, int hi, int pos, int val) {
    int id = cnt++;
    lc[id] = lc[prev]; rc[id] = rc[prev]; sum[id] = sum[prev]; // copy
    if (lo == hi) { sum[id] = val; return id; }
    int mid = (lo+hi)/2;
    if (pos <= mid) lc[id] = update(lc[prev], lo, mid, pos, val);
    else           rc[id] = update(rc[prev], mid+1, hi, pos, val);
    sum[id] = sum[lc[id]] + sum[rc[id]];
    return id;
}
# Persistent segment tree via tuples (immutable nodes)
# node = (sum, left, right)   leaves: (value, None, None)

def build(lo, hi, a):
    if lo == hi:
        return (a[lo], None, None)
    mid = (lo + hi) // 2
    L = build(lo, mid, a)
    R = build(mid + 1, hi, a)
    return (L[0] + R[0], L, R)

def update(node, lo, hi, pos, val):
    if lo == hi:
        return (val, None, None)        # new leaf
    mid = (lo + hi) // 2
    if pos <= mid:
        L = update(node[1], lo, mid, pos, val)
        R = node[2]                       # SHARE right subtree
    else:
        L = node[1]                       # SHARE left subtree
        R = update(node[2], mid + 1, hi, pos, val)
    return (L[0] + R[0], L, R)

# keep a list of roots; roots[v] is version v

Famous Application: K-th Smallest in a Range

  • Build a persistent segment tree over prefixes: version i = counts of values in a[0..i].
  • The structure for range [l, r] = version r "minus" version l−1 (walk both roots together).
  • Walk down by counts to find the k-th smallest in [l, r] in O(log n). A staple of competitive programming.

What Becomes Persistent

Persistent segment tree — range k-th, range distinct counts, offline queries.

Persistent array / trie — versioned state, functional programming.

Persistent DSU — union-find with rollback / time travel.

Persistent treap — versioned sequences with split/merge.

Common Pitfalls

1. Mutating shared nodes: never modify an old node in place — always copy, or you corrupt past versions.

2. Memory: n updates × O(log n) nodes — preallocate generously (≈ n log n nodes).

3. Storing roots: keep an array of version roots; the root id is the version handle.

4. Recursion to leaves: only one root→leaf path is new per update; the sibling pointers must reuse old ids.
