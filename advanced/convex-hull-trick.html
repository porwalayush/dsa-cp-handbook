Convex Hull Trick

A DP optimization for transitions of the form dp[i] = min(m·x + b) over a set of lines. Instead of checking every line, keep only the ones forming the lower envelope — querying the minimum at any x becomes O(log n) (or O(1) amortized).

The Setup

   Each candidate transition is a LINE:  y = m·x + b
   A query asks: "at x = q, which line gives the minimum y?"

   Only lines on the LOWER ENVELOPE can ever be the minimum.
   Lines fully above others are useless and get discarded.

   Add lines (sorted by slope) → maintain the hull with a stack.
   Query → binary search (or a moving pointer if x is monotonic).

Code: Convex Hull Trick (min, monotonic)

// Minimize. Add lines with DECREASING slope; query increasing x.
struct CHT {
    vector<long long> m, b;   // line i: y = m[i]*x + b[i]
    int ptr = 0;

    bool bad(int l1, int l2, int l3) {
        // l2 is useless if its intersection with l1 is past l3's
        return (__int128)(b[l3]-b[l1])*(m[l1]-m[l2])
            <= (__int128)(b[l2]-b[l1])*(m[l1]-m[l3]);
    }
    void add(long long slope, long long intercept) {
        m.push_back(slope); b.push_back(intercept);
        while (m.size() >= 3 && bad(m.size()-3, m.size()-2, m.size()-1)) {
            m.erase(m.end()-2); b.erase(b.end()-2);
        }
    }
    long long query(long long x) {   // x non-decreasing
        if (ptr >= (int)m.size()) ptr = m.size()-1;
        while (ptr + 1 < (int)m.size() &&
               m[ptr+1]*x + b[ptr+1] <= m[ptr]*x + b[ptr]) ptr++;
        return m[ptr]*x + b[ptr];
    }
};
class CHT {
    long[] m = new long[N]; long[] b = new long[N];
    int size = 0, ptr = 0;

    boolean bad(int l1, int l2, int l3) {
        return (double)(b[l3]-b[l1])*(m[l1]-m[l2])
            <= (double)(b[l2]-b[l1])*(m[l1]-m[l3]);
    }
    void add(long slope, long intercept) {
        m[size] = slope; b[size] = intercept; size++;
        while (size >= 3 && bad(size-3, size-2, size-1)) {
            m[size-2] = m[size-1]; b[size-2] = b[size-1]; size--;
        }
    }
    long query(long x) {
        if (ptr >= size) ptr = size - 1;
        while (ptr + 1 < size && m[ptr+1]*x + b[ptr+1] <= m[ptr]*x + b[ptr]) ptr++;
        return m[ptr]*x + b[ptr];
    }
}
class CHT:
    def __init__(self):
        self.m = []; self.b = []; self.ptr = 0

    def _bad(self, l1, l2, l3):
        return ((self.b[l3]-self.b[l1])*(self.m[l1]-self.m[l2])
                <= (self.b[l2]-self.b[l1])*(self.m[l1]-self.m[l3]))

    def add(self, slope, intercept):    # slopes decreasing
        self.m.append(slope); self.b.append(intercept)
        while len(self.m) >= 3 and self._bad(-3, -2, -1):
            self.m.pop(-2); self.b.pop(-2)

    def query(self, x):                  # x non-decreasing
        if self.ptr >= len(self.m): self.ptr = len(self.m) - 1
        while (self.ptr + 1 < len(self.m) and
               self.m[self.ptr+1]*x + self.b[self.ptr+1]
               <= self.m[self.ptr]*x + self.b[self.ptr]):
            self.ptr += 1
        return self.m[self.ptr]*x + self.b[self.ptr]

Li Chao Tree (the flexible cousin)

  • A segment tree over the x-range where each node stores the line that's best at its midpoint.
  • Handles lines added in any order and arbitrary query points — O(log range) each.
  • Easier to reason about than the stack CHT when slopes aren't monotonic.

When DP Becomes CHT

   Look for transitions like:
       dp[i] = min over j  ( dp[j] + cost(j, i) )
   where cost factors into  (something_j) · (something_i) + ...

   Rewrite as a line:  y = m·x + b   with
       m = function of j,  b = dp[j] + const,  x = function of i.
   Then "min over j" = "min over lines at x" → CHT.
   Drops O(n²) DP to O(n log n) or O(n).

Common Pitfalls

1. Min vs max: for maximum, flip the comparisons (upper envelope) or negate slopes/intercepts.

2. Monotonicity assumptions: the pointer-based query needs sorted slopes AND non-decreasing x — otherwise use binary search or Li Chao.

3. Overflow in the "bad" test: the cross-multiplication can overflow — use __int128 or doubles carefully.

4. Duplicate slopes: keep only the lower intercept among equal slopes.
