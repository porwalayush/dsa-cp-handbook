Centroid Decomposition

Recursively split a tree at its centroid — the node whose removal leaves all pieces of size ≤ n/2. The resulting "centroid tree" has depth O(log n), so path/distance queries that pass through centroids run in O(n log n) overall.

What's a Centroid?

   The centroid of a tree is a node such that, when removed,
   every remaining component has ≤ n/2 nodes.

   • Every tree has one (sometimes two) centroids.
   • Find it: compute subtree sizes, then walk toward the
     heaviest child while that child's subtree > n/2.

   Remove the centroid → recurse on each piece → its centroid
   becomes a CHILD in the centroid tree. Depth = O(log n).

def calc_size(u, p):
    sz[u] = 1
    for v in adj[u]:
        if v != p and not removed[v]:
            sz[u] += calc_size(v, u)
    return sz[u]

def find_centroid(u, p, n):
    for v in adj[u]:
        if v != p and not removed[v] and sz[v] > n // 2:
            return find_centroid(v, u, n)
    return u

    c = find_centroid(entry, -1, n)

The Core Insight

  • Any path in the tree passes through the centroid of some level — and there are only O(log n) levels.
  • So you process all paths "through centroid c" while c is removed, then recurse — each node appears in O(log n) levels.
  • This gives O(n log n) (or O(n log²n)) algorithms for distance-based tree problems.

Where It's Used

Count paths of length k (or weight ≤ k) in a tree.

Closest marked node with updates — store distances to each centroid ancestor.

Number of pairs at distance d — classic centroid problem.

Tree + distance queries generally — the go-to "divide & conquer on trees".

Common Pitfalls

1. Recompute sizes each level: after removing a centroid, subtree sizes change — recalc within the current component.

2. The removed[] flag: never descend into already-removed centroids.

3. Double-counting paths: use inclusion-exclusion (subtract paths staying within a single child subtree).

4. Recursion depth: the centroid tree is shallow (log n), but the size DFS can be deep — guard the stack.
