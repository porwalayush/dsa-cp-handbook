Square Root Decomposition

Split an array into blocks of size ≈ √n. Precompute an answer per block, and any range query touches at most two partial blocks + a few full blocks — giving O(√n) per query, a sweet spot between brute force and segment trees.

The Block Idea

   n = 16, block size = 4

   [ 3 1 4 1 | 5 9 2 6 | 5 3 5 8 | 9 7 9 3 ]
     block 0    block 1   block 2   block 3
     sum=9      sum=22    sum=21    sum=28

   Query sum[2..13]:
     • partial: indices 2,3      (walk individually)
     • full:    block 1, block 2 (use precomputed sums)
     • partial: indices 12,13    (walk individually)
   → O(√n) work instead of O(n).

Code: Block Range-Sum + Point Update

int n, B;                 // B = block size ~ sqrt(n)
vector<long long> a, blockSum;

void build(vector<long long>& arr) {
    a = arr; n = a.size(); B = sqrt(n) + 1;
    blockSum.assign((n + B - 1) / B, 0);
    for (int i = 0; i < n; i++) blockSum[i / B] += a[i];
}
long long query(int l, int r) {   // inclusive
    long long s = 0;
    while (l <= r && l % B != 0) s += a[l++];   // left partial
    while (l + B - 1 <= r) { s += blockSum[l / B]; l += B; } // full blocks
    while (l <= r) s += a[l++];                  // right partial
    return s;
}
void update(int i, long long val) {
    blockSum[i / B] += val - a[i];
    a[i] = val;
}

import math

class SqrtDecomp:
    def __init__(self, arr):
        self.a = arr[:]
        self.n = len(arr)
        self.B = int(math.isqrt(self.n)) + 1
        self.block = [0] * ((self.n + self.B - 1) // self.B)
        for i, v in enumerate(arr):
            self.block[i // self.B] += v
    def query(self, l, r):           # inclusive
        s = 0
        while l <= r and l % self.B != 0:
            s += self.a[l]; l += 1
        while l + self.B - 1 <= r:
            s += self.block[l // self.B]; l += self.B
        while l <= r:
            s += self.a[l]; l += 1
        return s
    def update(self, i, val):
        self.block[i // self.B] += val - self.a[i]
        self.a[i] = val

Mo's Algorithm (offline queries)

  • Sort queries by (block of left endpoint, right endpoint), then move two pointers to answer each.
  • Total pointer movement is O((n + q)√n) — magical for "count distinct in range", frequency, etc.
  • Requires offline queries (all known in advance) and an answer you can update by adding/removing one element.

When to Use Sqrt Decomposition

Simpler than a segment tree and often fast enough (n ≤ 10⁵–10⁶).

Weird operations a segment tree can't easily do → blocks are flexible.

Mo's algorithm for offline range queries with add/remove answers.

Sqrt on sequences/trees — block decomposition generalizes widely.

Common Pitfalls

1. Block size: √n balances the two costs; tune it (sometimes n/√q) for the query count.

2. Boundary blocks: the partial-block loops must stop exactly at r — off-by-one is easy here.

3. Mo's ordering: sort by block, then by right (alternate direction per block to shave constant).

4. Updates in Mo's: plain Mo's is query-only; "Mo's with updates" adds a time dimension (O(n^(5/3))).
