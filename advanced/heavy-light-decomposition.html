Heavy-Light Decomposition

Turns path queries on a tree into a handful of range queries on an array. Break the tree into "heavy" chains; any root-to-node path crosses at most O(log n) chains, so a segment tree over the chains answers path queries in O(log²n).

Heavy vs Light Edges

   For each node, its HEAVY child = the child with the
   largest subtree. That edge is "heavy"; all others "light".

   Key fact: any path from the root DOWN to a leaf crosses
   at most O(log n) LIGHT edges — because each light edge
   at least HALVES the subtree size as you go down.

   Heavy edges form chains → lay each chain contiguously in
   an array → segment tree handles each chain segment.

Code: HLD Skeleton

int par[N], dep[N], sz[N], heavy[N], head[N], pos[N], cur = 0;
vector<int> adj[N];

int dfsSize(int u, int p) {
    sz[u] = 1; heavy[u] = -1;
    int mx = 0;
    for (int v : adj[u]) if (v != p) {
        par[v] = u; dep[v] = dep[u] + 1;
        int s = dfsSize(v, u); sz[u] += s;
        if (s > mx) { mx = s; heavy[u] = v; } // biggest child
    }
    return sz[u];
}
void decompose(int u, int h) {
    head[u] = h; pos[u] = cur++;          // position in base array
    if (heavy[u] != -1) decompose(heavy[u], h);  // extend chain
    for (int v : adj[u]) if (v != par[u] && v != heavy[u])
        decompose(v, v);                  // new chain
}
// path query u..v: jump chains by head, query segtree on [pos..]
long long queryPath(int u, int v) {
    long long res = 0;
    while (head[u] != head[v]) {
        if (dep[head[u]] < dep[head[v]]) swap(u, v);
        res += segQuery(pos[head[u]], pos[u]);
        u = par[head[u]];
    }
    if (dep[u] > dep[v]) swap(u, v);
    res += segQuery(pos[u], pos[v]);       // same chain
    return res;
}

import sys
sys.setrecursionlimit(300000)

def dfs_size(u, p):
    sz[u] = 1; heavy[u] = -1; mx = 0
    for v in adj[u]:
        if v != p:
            par[v] = u; dep[v] = dep[u] + 1
            s = dfs_size(v, u); sz[u] += s
            if s > mx: mx = s; heavy[u] = v
    return sz[u]

def decompose(u, h):
    global cur
    head[u] = h; pos[u] = cur; cur += 1
    if heavy[u] != -1: decompose(heavy[u], h)
    for v in adj[u]:
        if v != par[u] and v != heavy[u]:
            decompose(v, v)

def query_path(u, v):
    res = 0
    while head[u] != head[v]:
        if dep[head[u]] < dep[head[v]]: u, v = v, u
        res += seg_query(pos[head[u]], pos[u])
        u = par[head[u]]
    if dep[u] > dep[v]: u, v = v, u
    res += seg_query(pos[u], pos[v])
    return res

What HLD Answers

  • Path sum / max / min between any two nodes, with point or range updates.
  • Subtree queries too (subtrees are contiguous in the same Euler order).
  • Combine with a segment tree (sum/max), or with lazy propagation for path updates.
  • Per query: O(log²n) (O(log n) chains × O(log n) segtree).

Common Pitfalls

1. Edge vs node values: for edge weights, store each edge at its child node and skip the LCA in the same-chain query.

2. Recursion depth: a chain (or a stick-shaped tree) can be deep — raise the recursion limit / use iterative DFS.

3. Heavy child choice: it's the child with the largest subtree size, not the largest weight.

4. Chain jump direction: always lift the node whose chain head is deeper first.
