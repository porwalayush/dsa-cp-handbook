FFT & NTT

The Fast Fourier Transform multiplies two polynomials (or big numbers) in O(n log n) instead of the naive O(n²). NTT is its integer cousin — same idea, done modulo a prime, so results stay exact.

The Big Trick: Evaluate → Multiply → Interpolate

   Multiplying two degree-n polynomials directly = O(n²).

   FFT route (O(n log n)):
     1. EVALUATE both polynomials at 2n special points
        (the complex "roots of unity") — fast via FFT.
     2. MULTIPLY the point-values pairwise — O(n).
     3. INTERPOLATE back to coefficients — inverse FFT.

   A polynomial is determined by its values at enough points,
   and roots of unity make evaluation a divide & conquer.

Interactive: Polynomial Multiplication (Convolution)

Multiplying polynomials = convolving their coefficient arrays. Each output coefficient is a sum of cross products. Type coefficients (space-separated) and step through — FFT computes exactly this, just in O(n log n).

Code: Iterative FFT & NTT

using cd = complex<double>;
const double PI = acos(-1);

void fft(vector<cd>& a, bool inv) {
    int n = a.size();
    for (int i = 1, j = 0; i < n; i++) {   // bit-reversal
        int bit = n >> 1;
        for (; j & bit; bit >>= 1) j ^= bit;
        j ^= bit;
        if (i < j) swap(a[i], a[j]);
    }
    for (int len = 2; len <= n; len <<= 1) {
        double ang = 2 * PI / len * (inv ? -1 : 1);
        cd wlen(cos(ang), sin(ang));
        for (int i = 0; i < n; i += len) {
            cd w(1);
            for (int k = 0; k < len / 2; k++) {
                cd u = a[i+k], v = a[i+k+len/2] * w;
                a[i+k] = u + v; a[i+k+len/2] = u - v;
                w *= wlen;
            }
        }
    }
    if (inv) for (cd& x : a) x /= n;
}
vector<long long> multiply(vector<int>& A, vector<int>& B) {
    vector<cd> fa(A.begin(), A.end()), fb(B.begin(), B.end());
    int n = 1; while (n < A.size() + B.size()) n <<= 1;
    fa.resize(n); fb.resize(n);
    fft(fa, false); fft(fb, false);
    for (int i = 0; i < n; i++) fa[i] *= fb[i]; // pointwise
    fft(fa, true);
    vector<long long> res(n);
    for (int i = 0; i < n; i++) res[i] = round(fa[i].real());
    return res;
}
// NTT: exact integer convolution mod a prime
const long long MOD = 998244353, G = 3; // primitive root

long long power(long long a, long long b) {
    long long r = 1; a %= MOD;
    while (b) { if (b & 1) r = r * a % MOD; a = a * a % MOD; b >>= 1; }
    return r;
}
void ntt(vector<long long>& a, bool inv) {
    int n = a.size();
    for (int i = 1, j = 0; i < n; i++) {
        int bit = n >> 1;
        for (; j & bit; bit >>= 1) j ^= bit;
        j ^= bit; if (i < j) swap(a[i], a[j]);
    }
    for (int len = 2; len <= n; len <<= 1) {
        long long wlen = power(G, (MOD - 1) / len);
        if (inv) wlen = power(wlen, MOD - 2);
        for (int i = 0; i < n; i += len) {
            long long w = 1;
            for (int k = 0; k < len / 2; k++) {
                long long u = a[i+k], v = a[i+k+len/2] * w % MOD;
                a[i+k] = (u + v) % MOD;
                a[i+k+len/2] = (u - v + MOD) % MOD;
                w = w * wlen % MOD;
            }
        }
    }
    if (inv) { long long ni = power(n, MOD - 2); for (auto& x : a) x = x * ni % MOD; }
}
# numpy makes FFT convolution a one-liner
import numpy as np

def multiply(A, B):
    n = 1
    while n < len(A) + len(B):
        n <<= 1
    fa = np.fft.fft(A, n)
    fb = np.fft.fft(B, n)
    res = np.fft.ifft(fa * fb).real   # pointwise then inverse
    return [round(x) for x in res[:len(A)+len(B)-1]]

# Exact integer convolution without floats: pure-Python NTT
MOD, G = 998244353, 3
def ntt(a, inv):
    n = len(a); j = 0
    for i in range(1, n):
        bit = n >> 1
        while j & bit: j ^= bit; bit >>= 1
        j ^= bit
        if i < j: a[i], a[j] = a[j], a[i]
    length = 2
    while length <= n:
        w = pow(G, (MOD - 1) // length, MOD)
        if inv: w = pow(w, MOD - 2, MOD)
        for i in range(0, n, length):
            wn = 1
            for k in range(length // 2):
                u = a[i+k]; v = a[i+k+length//2] * wn % MOD
                a[i+k] = (u + v) % MOD
                a[i+k+length//2] = (u - v) % MOD
                wn = wn * w % MOD
        length <<= 1

FFT vs NTT

  • FFT uses complex roots of unity → floating-point → tiny rounding error (round at the end). Works for any real coefficients.
  • NTT uses a primitive root mod a special prime (e.g. 998244353) → exact integers, no rounding. Needs answers under that modulus.
  • Big-number multiplication, string matching with wildcards, and many convolution sums all reduce to this.

Where It's Used

Polynomial / big-integer multiplication in O(n log n).

Counting: "how many pairs sum to k" via convolution of frequency arrays.

String matching with wildcards (FFT correlation).

Generating functions — combinatorial counting at scale.

Common Pitfalls

1. Size to a power of two ≥ lenA + lenB before transforming.

2. FFT rounding: large coefficients overflow double precision — use NTT or split into halves.

3. NTT modulus: must be of the form c·2^k + 1 with enough 2-power for your size.

4. Forgetting the 1/n in inverse: the inverse transform must divide by n.
