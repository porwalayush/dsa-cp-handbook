Segment Tree Beats

The Segment Tree Beats (also called the Ji Driver Segment Tree, named after Ji Chen who introduced it in 2016) is a powerful extension of the segment tree that efficiently supports range chmin/chmax (clamping) operations together with range sum queries. A naive lazy propagation approach fails here because the sum update depends on how many elements actually change, but Segment Tree Beats solves this elegantly.

The Key Insight: Break, Tag, and Beats

The core idea is to maintain extra information per node (the second maximum and count of maximum) so we can decide at each node whether to:

  • Break condition: Stop recursion early because no element in this range needs updating
  • Tag condition: Apply the update to the entire node because only the maximum elements are affected (and we know exactly how many there are)
  • Recurse deeper: When neither condition is met, we must split and recurse into children

This three-way decision at each node is what gives the technique its name -- we "beat" the complexity by avoiding unnecessary recursion in most cases.

Node Information

Each node in the segment tree stores four values for the range it covers:

FieldDescriptionPurpose
maxMaximum value in the rangeDetermines if chmin(x) affects this node
second_maxStrict second maximum (largest value < max)Determines whether to apply tag or recurse
count_maxNumber of elements equal to maxAllows O(1) sum adjustment when clamping
sumSum of all elements in the rangeAnswers range sum queries

The Three Cases for chmin(x)

When we apply chmin(l, r, x) (set every element a[i] = min(a[i], x) for i in [l, r]), at each node we check:

chmin(x) on node with range [l, r]:

Case (a): max <= x
  --> Break! No element exceeds x, nothing to do.
  --> This is the "break condition"

Case (b): second_max < x < max
  --> Tag! Only the max elements are reduced to x.
  --> sum decreases by (max - x) * count_max
  --> Update: max = x, sum -= (old_max - x) * count_max
  --> This is the "tag condition"

Case (c): x <= second_max
  --> Recurse into both children.
  --> We cannot determine the sum change without going deeper.
  --> After recursing, pull up: recalculate node from children.

Pushdown (Propagation)

When a node has been tagged with a chmin value, we must propagate it to children before recursing into them. The pushdown applies chmin(parent.max) to each child:

pushdown(node):
  for each child in {left, right}:
    if child.max > node.max:
      // The parent's max acts as the chmin tag
      child.sum -= (child.max - node.max) * child.count_max
      child.max = node.max
      // Note: child.second_max is unchanged (it's < node.max)

Code Implementation

#include <bits/stdc++.h>
using namespace std;

struct SegTreeBeats {
    struct Node {
        long long sum;
        int mx, second_mx, cnt_mx;
    };

    int n;
    vector<Node> tree;

    SegTreeBeats(int n) : n(n), tree(4 * n) {}

    void pull(int v) {
        int l = 2 * v, r = 2 * v + 1;
        tree[v].sum = tree[l].sum + tree[r].sum;
        if (tree[l].mx == tree[r].mx) {
            tree[v].mx = tree[l].mx;
            tree[v].cnt_mx = tree[l].cnt_mx + tree[r].cnt_mx;
            tree[v].second_mx = max(tree[l].second_mx, tree[r].second_mx);
        } else if (tree[l].mx > tree[r].mx) {
            tree[v].mx = tree[l].mx;
            tree[v].cnt_mx = tree[l].cnt_mx;
            tree[v].second_mx = max(tree[l].second_mx, tree[r].mx);
        } else {
            tree[v].mx = tree[r].mx;
            tree[v].cnt_mx = tree[r].cnt_mx;
            tree[v].second_mx = max(tree[l].mx, tree[r].second_mx);
        }
    }

    // Push the chmin tag down to children
    void push_down(int v) {
        for (int child : {2 * v, 2 * v + 1}) {
            if (tree[child].mx > tree[v].mx) {
                tree[child].sum -= (long long)(tree[child].mx - tree[v].mx) * tree[child].cnt_mx;
                tree[child].mx = tree[v].mx;
            }
        }
    }

    void build(vector<int>& arr, int v, int l, int r) {
        if (l == r) {
            tree[v].sum = tree[v].mx = arr[l];
            tree[v].second_mx = -1;  // no second max for a single element
            tree[v].cnt_mx = 1;
            return;
        }
        int mid = (l + r) / 2;
        build(arr, 2 * v, l, mid);
        build(arr, 2 * v + 1, mid + 1, r);
        pull(v);
    }

    void build(vector<int>& arr) { build(arr, 1, 0, n - 1); }

    // Range chmin: set a[i] = min(a[i], x) for i in [ql, qr]
    void update_chmin(int v, int l, int r, int ql, int qr, int x) {
        if (ql > r || qr < l || tree[v].mx <= x)
            return;  // Case (a): break condition
        if (ql <= l && r <= qr && tree[v].second_mx < x) {
            // Case (b): tag condition -- only max elements are affected
            tree[v].sum -= (long long)(tree[v].mx - x) * tree[v].cnt_mx;
            tree[v].mx = x;
            return;
        }
        // Case (c): recurse deeper
        push_down(v);
        int mid = (l + r) / 2;
        update_chmin(2 * v, l, mid, ql, qr, x);
        update_chmin(2 * v + 1, mid + 1, r, ql, qr, x);
        pull(v);
    }

    void update_chmin(int ql, int qr, int x) {
        update_chmin(1, 0, n - 1, ql, qr, x);
    }

    // Range sum query
    long long query_sum(int v, int l, int r, int ql, int qr) {
        if (ql > r || qr < l) return 0;
        if (ql <= l && r <= qr) return tree[v].sum;
        push_down(v);
        int mid = (l + r) / 2;
        return query_sum(2 * v, l, mid, ql, qr) +
               query_sum(2 * v + 1, mid + 1, r, ql, qr);
    }

    long long query_sum(int ql, int qr) {
        return query_sum(1, 0, n - 1, ql, qr);
    }

    // Range max query
    int query_max(int v, int l, int r, int ql, int qr) {
        if (ql > r || qr < l) return INT_MIN;
        if (ql <= l && r <= qr) return tree[v].mx;
        push_down(v);
        int mid = (l + r) / 2;
        return max(query_max(2 * v, l, mid, ql, qr),
                   query_max(2 * v + 1, mid + 1, r, ql, qr));
    }

    int query_max(int ql, int qr) {
        return query_max(1, 0, n - 1, ql, qr);
    }
};

int main() {
    vector<int> arr = {3, 7, 2, 8, 6, 4, 9, 1};
    SegTreeBeats st(arr.size());
    st.build(arr);

    cout << st.query_sum(0, 7) << "\n";  // 40
    st.update_chmin(0, 7, 5);             // chmin(5) on entire array
    cout << st.query_sum(0, 7) << "\n";  // 30
    cout << st.query_max(0, 7) << "\n";  // 5
}

class SegTreeBeats {
    long[] sum;
    int[] mx, secondMx, cntMx;
    int n;

    SegTreeBeats(int[] arr) {
        n = arr.length;
        sum = new long[4 * n];
        mx = new int[4 * n];
        secondMx = new int[4 * n];
        cntMx = new int[4 * n];
        build(arr, 1, 0, n - 1);
    }

    void pull(int v) {
        int l = 2 * v, r = 2 * v + 1;
        sum[v] = sum[l] + sum[r];
        if (mx[l] == mx[r]) {
            mx[v] = mx[l];
            cntMx[v] = cntMx[l] + cntMx[r];
            secondMx[v] = Math.max(secondMx[l], secondMx[r]);
        } else if (mx[l] > mx[r]) {
            mx[v] = mx[l]; cntMx[v] = cntMx[l];
            secondMx[v] = Math.max(secondMx[l], mx[r]);
        } else {
            mx[v] = mx[r]; cntMx[v] = cntMx[r];
            secondMx[v] = Math.max(mx[l], secondMx[r]);
        }
    }

    void pushDown(int v) {
        for (int child : new int[]{2 * v, 2 * v + 1}) {
            if (mx[child] > mx[v]) {
                sum[child] -= (long)(mx[child] - mx[v]) * cntMx[child];
                mx[child] = mx[v];
            }
        }
    }

    void build(int[] arr, int v, int l, int r) {
        if (l == r) {
            sum[v] = mx[v] = arr[l];
            secondMx[v] = -1;
            cntMx[v] = 1;
            return;
        }
        int mid = (l + r) / 2;
        build(arr, 2 * v, l, mid);
        build(arr, 2 * v + 1, mid + 1, r);
        pull(v);
    }

    void updateChmin(int v, int l, int r, int ql, int qr, int x) {
        if (ql > r || qr < l || mx[v] <= x) return;
        if (ql <= l && r <= qr && secondMx[v] < x) {
            sum[v] -= (long)(mx[v] - x) * cntMx[v];
            mx[v] = x;
            return;
        }
        pushDown(v);
        int mid = (l + r) / 2;
        updateChmin(2 * v, l, mid, ql, qr, x);
        updateChmin(2 * v + 1, mid + 1, r, ql, qr, x);
        pull(v);
    }

    void updateChmin(int ql, int qr, int x) {
        updateChmin(1, 0, n - 1, ql, qr, x);
    }

    long querySum(int v, int l, int r, int ql, int qr) {
        if (ql > r || qr < l) return 0;
        if (ql <= l && r <= qr) return sum[v];
        pushDown(v);
        int mid = (l + r) / 2;
        return querySum(2 * v, l, mid, ql, qr) +
               querySum(2 * v + 1, mid + 1, r, ql, qr);
    }

    long querySum(int ql, int qr) {
        return querySum(1, 0, n - 1, ql, qr);
    }
}

class SegTreeBeats:
    def __init__(self, arr):
        self.n = len(arr)
        self.sum = [0] * (4 * self.n)
        self.mx = [0] * (4 * self.n)
        self.second_mx = [0] * (4 * self.n)
        self.cnt_mx = [0] * (4 * self.n)
        self._build(arr, 1, 0, self.n - 1)

    def _pull(self, v):
        l, r = 2 * v, 2 * v + 1
        self.sum[v] = self.sum[l] + self.sum[r]
        if self.mx[l] == self.mx[r]:
            self.mx[v] = self.mx[l]
            self.cnt_mx[v] = self.cnt_mx[l] + self.cnt_mx[r]
            self.second_mx[v] = max(self.second_mx[l], self.second_mx[r])
        elif self.mx[l] > self.mx[r]:
            self.mx[v] = self.mx[l]
            self.cnt_mx[v] = self.cnt_mx[l]
            self.second_mx[v] = max(self.second_mx[l], self.mx[r])
        else:
            self.mx[v] = self.mx[r]
            self.cnt_mx[v] = self.cnt_mx[r]
            self.second_mx[v] = max(self.mx[l], self.second_mx[r])

    def _push_down(self, v):
        for child in (2 * v, 2 * v + 1):
            if self.mx[child] > self.mx[v]:
                self.sum[child] -= (self.mx[child] - self.mx[v]) * self.cnt_mx[child]
                self.mx[child] = self.mx[v]

    def _build(self, arr, v, l, r):
        if l == r:
            self.sum[v] = self.mx[v] = arr[l]
            self.second_mx[v] = -1
            self.cnt_mx[v] = 1
            return
        mid = (l + r) // 2
        self._build(arr, 2 * v, l, mid)
        self._build(arr, 2 * v + 1, mid + 1, r)
        self._pull(v)

    def update_chmin(self, ql, qr, x, v=1, l=0, r=None):
        if r is None: r = self.n - 1
        if ql > r or qr < l or self.mx[v] <= x:
            return  # break condition
        if ql <= l and r <= qr and self.second_mx[v] < x:
            # tag condition
            self.sum[v] -= (self.mx[v] - x) * self.cnt_mx[v]
            self.mx[v] = x
            return
        self._push_down(v)
        mid = (l + r) // 2
        self.update_chmin(ql, qr, x, 2 * v, l, mid)
        self.update_chmin(ql, qr, x, 2 * v + 1, mid + 1, r)
        self._pull(v)

    def query_sum(self, ql, qr, v=1, l=0, r=None):
        if r is None: r = self.n - 1
        if ql > r or qr < l: return 0
        if ql <= l and r <= qr: return self.sum[v]
        self._push_down(v)
        mid = (l + r) // 2
        return (self.query_sum(ql, qr, 2 * v, l, mid) +
                self.query_sum(ql, qr, 2 * v + 1, mid + 1, r))

# Usage
st = SegTreeBeats([3, 7, 2, 8, 6, 4, 9, 1])
print(st.query_sum(0, 7))   # 40
st.update_chmin(0, 7, 5)   # chmin(5) on entire array
print(st.query_sum(0, 7))   # 30

Amortized Complexity

The amortized time complexity per operation is O(log2 n). The proof relies on a potential function argument:

  • Define the potential as the total number of distinct values across all nodes in the tree
  • Each chmin operation that triggers Case (c) reduces the number of distinct maximums along the path
  • The total work done across all operations is bounded by the total decrease in potential plus O(q log n) for q operations
OperationAmortized TimeNote
chmin(l, r, x)O(log2 n)Range clamp minimum
chmax(l, r, x)O(log2 n)Symmetric to chmin
query_sum(l, r)O(log n)Standard range query
query_max(l, r)O(log n)Standard range query
BuildO(n)Standard build

Supporting chmax (Symmetric Case)

To support chmax(l, r, x) (set a[i] = max(a[i], x)), maintain the symmetric information:

  • Store min, second_min, count_min in addition to the max fields
  • The three cases become: (a) min >= x: break, (b) second_min > x >= min: tag, (c) x >= second_min: recurse
  • When both chmin and chmax are needed simultaneously, careful ordering during pushdown is required

Applications

Common Use Cases:
  • Range min/max assignment with sum queries: The classic application -- clamp values in a range and query their sum
  • Historical queries: Track the historical maximum/minimum of each element across all operations (requires additional fields per node)
  • Chtholly-style problems: When intervals are assigned values and you need aggregate statistics
  • Competitive programming: Problems like "Gorgeous Sequence" (HDU 5306), CF 1290E, and range operations that mix clamp + sum
Key Takeaways:
  • Segment Tree Beats works because the extra metadata (second_max, count_max) lets us avoid recursing in most cases
  • The technique generalizes: any operation where you can identify a "tag condition" using O(1) extra info per node can use this pattern
  • In practice, the constant factor is small and the structure performs well for n up to 106
  • Cannot be combined with arbitrary lazy propagation -- only works for specific operation types
