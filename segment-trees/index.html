Tree structure and indexing. Build from array. Range min/max/sum queries. Point updates.

Lazy Propagation

Range updates in O(log n). Pending updates propagation. Range set, range add, range XOR.

Hard O(log n)

Iterative Segment Tree

Bottom-up implementation. Faster constant factor. Memory efficient. Contest-friendly template.

Advanced Segment Trees

Persistent Segment Tree

Version history. Kth smallest in range. Copy-on-write nodes. O(log n) per version.

Hard O(log n) per op

Merge Sort Tree

Count elements in range less than k. Sorted arrays at each node. Fractional cascading.

Hard O(log^2 n)

Segment Tree Beats

Range chmin/chmax operations. Ji driver segment tree. Break & tag conditions.

Hard O(log^2 n) amortized

Dynamic / Sparse Segment Tree

Create nodes on demand. Large value ranges without coordinate compression. Pointer-based.

Hard O(log N)

Which Structure to Use?

Quick decision guide for range query problems

1

Point Update + Prefix Query?

Fenwick Tree (BIT). Simplest, fastest constant. Use when associative + has inverse (sum, XOR).

2

Range Update + Range Query?

Segment Tree + Lazy Propagation. Or two BITs for additive updates. Lazy for complex ops.

3

Static Array, Range Min/Max?

Sparse Table for O(1) query with O(n log n) build. No updates supported.

4

Kth Smallest in Range?

Persistent Segment Tree or Merge Sort Tree. PST for online, MST for offline.

5

2D Range Queries?

2D BIT for point update + rectangle sum. 2D segment tree for complex operations.

6

Huge Value Range?

Coordinate compression + BIT. Or dynamic/sparse segment tree. Or persistent ST.
