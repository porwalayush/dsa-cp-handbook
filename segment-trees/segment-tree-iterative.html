Iterative Segment Tree

The iterative segment tree is a bottom-up implementation that uses a flat array of size 2n. It eliminates recursion overhead, is cache-friendly, and produces compact code ideal for competitive programming.

Why Iterative?

  • Faster constant factor — no function call overhead or stack frame allocation from recursion
  • Cache-friendly — sequential memory access patterns; parent nodes are adjacent to children in the array
  • Simpler for contests — the entire implementation fits in ~20 lines; easy to type under time pressure
  • Exactly 2n space — no need for the 4n upper bound required by recursive versions

Bottom-Up Array Layout

The tree is stored in an array t[0..2n-1]. Leaves occupy positions n to 2n-1, and internal nodes occupy positions 1 to n-1. Position 0 is unused.

Array: [5, 8, 6, 3, 2, 7, 2, 6]    n = 8

Tree array t[] (sum segment tree):

Value: [39] [22] [17] [13]  [9]  [9]  [8]  [5] [8] [6] [3] [2] [7] [2] [6]

Parent-child relationships:
  parent(i) = i / 2     (integer division)
  left(i)   = 2 * i
  right(i)  = 2 * i + 1

  t[i] = t[2*i] + t[2*i+1]   (for internal nodes)

Build Operation

Copy array values into the leaf positions, then compute each internal node bottom-up from its two children.

Step 1: Place leaves at positions n..2n-1
  t[8]=5, t[9]=8, t[10]=6, t[11]=3, t[12]=2, t[13]=7, t[14]=2, t[15]=6

Step 2: Fill internal nodes from i = n-1 down to 1
  i=7: t[7] = t[14] + t[15] = 2 + 6 = 8
  i=6: t[6] = t[12] + t[13] = 2 + 7 = 9
  i=5: t[5] = t[10] + t[11] = 6 + 3 = 9
  i=4: t[4] = t[8]  + t[9]  = 5 + 8 = 13
  i=3: t[3] = t[6]  + t[7]  = 9 + 8 = 17
  i=2: t[2] = t[4]  + t[5]  = 13 + 9 = 22
  i=1: t[1] = t[2]  + t[3]  = 22 + 17 = 39

Time: O(n)  |  Space: O(2n)

Point Update

Update the leaf directly, then walk up to the root recalculating each ancestor.

Update arr[2] = 10  (was 6)
Leaf position: i = 2 + n = 2 + 8 = 10

  t[10] = 10                  (set new value)
  i = 10 / 2 = 5
  t[5] = t[10] + t[11] = 10 + 3 = 13

  i = 5 / 2 = 2
  t[2] = t[4] + t[5] = 13 + 13 = 26

  i = 2 / 2 = 1
  t[1] = t[2] + t[3] = 26 + 17 = 43

Done! Only O(log n) nodes updated.

Code pattern:
  i = pos + n
  t[i] = new_val
  while i > 1:
      i = i / 2
      t[i] = t[2*i] + t[2*i+1]

Range Query

Use two pointers l and r starting at the leaf level. At each level, if a pointer is a right child (odd index), include it and move inward. Then move both pointers to their parents.

Query: sum(2, 6)  -- sum of arr[2..6] (0-indexed)

Initialize: l = 2 + n = 10,  r = 6 + n = 14
             (point to leaves for arr[2] and arr[6])
Result = 0

Iteration 1: l=10, r=14
  l is even  -> skip (it's a left child, parent covers it)
  r is even  -> r is right child? 14 is even -> it's a left child
               Actually: odd = right child. 14 is even = left child.
               Wait -- let's use the standard approach:
               r is inclusive, so we use r+1 for the open interval.

--- Cleaner explanation using half-open [l, r) ---

Query: sum(2, 6) means arr[2]+arr[3]+arr[4]+arr[5]+arr[6]
Convert to tree indices: l = 2+n = 10,  r = 6+n+1 = 15

Iteration 1: l=10, r=15
  l=10 is even -> move up: l = 5
  r=15 is odd  -> r-=1 => r=14, res += t[14]=2, then r = 7
Result = 2

Iteration 2: l=5, r=7
  l=5 is odd  -> res += t[5]=9, l+=1 => l=6, then l = 3
  r=7 is odd  -> r-=1 => r=6, res += t[6]=9, then r = 3
Result = 2 + 9 + 9 = 20

Iteration 3: l=3, r=3
  l >= r -> STOP

Final answer = 20
Verify: arr[2]+arr[3]+arr[4]+arr[5]+arr[6] = 6+3+2+7+2 = 20  ✓

Query algorithm (half-open interval [l, r)):

  res = 0
  l += n        // move to leaf layer
  r += n + 1    // one past the right leaf

  while l < r:
      if l is odd:     // l is a right child
          res += t[l]  // include this node
          l += 1       // move to next subtree
      if r is odd:     // r-1 is a right child
          r -= 1       // include this node
          res += t[r]
      l /= 2           // move up one level
      r /= 2

Why it works:
  - If l is a right child, its parent covers elements outside
    our range (to the left), so we take l alone and skip its parent.
  - If r-1 is a right child, we include it before moving up.
  - Each level, the range shrinks until l >= r.

Full Implementation

#include <bits/stdc++.h>
using namespace std;

struct SegTree {
    int n;
    vector<int> t;

    SegTree(int n) : n(n), t(2 * n, 0) {}

    SegTree(vector<int>& arr) : n(arr.size()), t(2 * arr.size()) {
        // Place leaves at positions [n, 2n)
        for (int i = 0; i < n; i++)
            t[i + n] = arr[i];
        // Build internal nodes bottom-up
        for (int i = n - 1; i >= 1; i--)
            t[i] = t[2 * i] + t[2 * i + 1];
    }

    // Point update: set arr[pos] = val
    void update(int pos, int val) {
        pos += n;            // go to leaf
        t[pos] = val;        // update leaf
        for (pos /= 2; pos >= 1; pos /= 2)
            t[pos] = t[2 * pos] + t[2 * pos + 1];
    }

    // Range query: sum of [l, r] (inclusive)
    int query(int l, int r) {
        int res = 0;
        for (l += n, r += n + 1; l < r; l /= 2, r /= 2) {
            if (l & 1) res += t[l++];
            if (r & 1) res += t[--r];
        }
        return res;
    }
};

int main() {
    vector<int> arr = {5, 8, 6, 3, 2, 7, 2, 6};
    SegTree st(arr);

    cout << st.query(2, 6) << "\n";  // 20
    st.update(2, 10);                 // arr[2] = 10
    cout << st.query(2, 6) << "\n";  // 24
    cout << st.query(0, 7) << "\n";  // 43
}

class SegTree {
    int[] t;
    int n;

    SegTree(int[] arr) {
        n = arr.length;
        t = new int[2 * n];
        // Place leaves
        for (int i = 0; i < n; i++)
            t[i + n] = arr[i];
        // Build internal nodes
        for (int i = n - 1; i >= 1; i--)
            t[i] = t[2 * i] + t[2 * i + 1];
    }

    // Point update: set arr[pos] = val
    void update(int pos, int val) {
        pos += n;
        t[pos] = val;
        for (pos /= 2; pos >= 1; pos /= 2)
            t[pos] = t[2 * pos] + t[2 * pos + 1];
    }

    // Range query: sum of [l, r] (inclusive)
    int query(int l, int r) {
        int res = 0;
        for (l += n, r += n + 1; l < r; l /= 2, r /= 2) {
            if ((l & 1) == 1) res += t[l++];
            if ((r & 1) == 1) res += t[--r];
        }
        return res;
    }

    public static void main(String[] args) {
        int[] arr = {5, 8, 6, 3, 2, 7, 2, 6};
        SegTree st = new SegTree(arr);

        System.out.println(st.query(2, 6));  // 20
        st.update(2, 10);                      // arr[2] = 10
        System.out.println(st.query(2, 6));  // 24
        System.out.println(st.query(0, 7));  // 43
    }
}

class SegTree:
    def __init__(self, arr):
        self.n = len(arr)
        self.t = [0] * (2 * self.n)
        # Place leaves at positions [n, 2n)
        for i in range(self.n):
            self.t[i + self.n] = arr[i]
        # Build internal nodes bottom-up
        for i in range(self.n - 1, 0, -1):
            self.t[i] = self.t[2 * i] + self.t[2 * i + 1]

    def update(self, pos, val):
        """Set arr[pos] = val"""
        pos += self.n
        self.t[pos] = val
        pos //= 2
        while pos >= 1:
            self.t[pos] = self.t[2 * pos] + self.t[2 * pos + 1]
            pos //= 2

    def query(self, l, r):
        """Sum of arr[l..r] (inclusive)"""
        res = 0
        l += self.n
        r += self.n + 1
        while l < r:
            if l & 1:
                res += self.t[l]
                l += 1
            if r & 1:
                r -= 1
                res += self.t[r]
            l //= 2
            r //= 2
        return res

Limitations of Iterative Segment Trees:
  • Lazy propagation is harder — requires pushing updates top-down before queries, adding O(log n) overhead and complexity
  • Non-commutative operations — the query loop collects values from left and right separately; you must combine them in the correct order
  • Persistent trees — the iterative structure does not support path-copying, making persistence impractical
  • Power-of-two assumption — works cleanly when n is a power of 2; for arbitrary n, you can pad the array or handle edge cases carefully
