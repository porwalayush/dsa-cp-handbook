Segment Tree with Coordinate Compression

Coordinate compression is a technique that maps large, sparse values into a compact range of contiguous integers (ranks). This lets us use array-based structures like segment trees and BITs even when the value domain is huge (up to 109) but the number of distinct values is small (up to 105).

Why We Need Coordinate Compression

Consider counting inversions in an array where values can be up to 109. A BIT or segment tree indexed by value would need 109 cells — far too much memory. But if the array has only n elements, there are at most n distinct values. We can remap them to the range [0, n-1] and build our data structure over just n positions.

Key Insight:
  • Values can span up to 109, but we only have up to 105 distinct values
  • We only care about the relative order of values, not their absolute magnitudes
  • Mapping to ranks preserves all ordering relationships needed for comparisons

Visual: Mapping Large Values to Contiguous Ranks

Original array: [1000000, 3, 999999999, 50, 3, 1000000]

Step 1: Collect all values
  vals = [1000000, 3, 999999999, 50, 3, 1000000]

Step 2: Sort
  sorted = [3, 50, 1000000, 999999999]

Step 3: Deduplicate
  unique = [3, 50, 1000000, 999999999]

Step 4: Assign ranks (0-indexed via binary search)
  3           -> rank 0
  50          -> rank 1
  1000000     -> rank 2
  999999999   -> rank 3

Step 5: Replace original values with ranks
  Original:   [1000000, 3, 999999999, 50, 3, 1000000]
  Compressed: [   2,    0,     3,      1,  0,    2   ]

Now we only need a BIT/segment tree of size 4 instead of 10^9!

Step-by-Step Algorithm

  1. Collect all relevant values into an auxiliary array
  2. Sort the auxiliary array
  3. Deduplicate (remove consecutive duplicates)
  4. Binary search for each original value to get its rank (0-indexed position in the sorted unique array)
compress(arr):
  sorted_unique = sort(deduplicate(arr))
  for each element x in arr:
    rank[x] = lower_bound(sorted_unique, x)  // O(log n) per lookup
  return rank

Total time: O(n log n) for sort + O(n log n) for all lookups = O(n log n)
Space: O(n)

Classic Problem: Count Inversions with Large Values

An inversion is a pair (i, j) where i < j but arr[i] > arr[j]. When values are up to 109, we compress them first, then use a BIT to count how many previously seen elements have a larger rank than the current one.

Array: [5, 3, 8, 1, 4]   (after compression: [2, 1, 4, 0, 3])
                           ranks use 0-based indexing

Process left to right, maintaining BIT of "elements seen so far":

i=0: rank=2, elements seen with rank > 2: query(3,4) = 0
     Insert rank 2 into BIT.  inversions += 0

i=1: rank=1, elements seen with rank > 1: query(2,4) = 1  (rank 2 is present)
     Insert rank 1 into BIT.  inversions += 1

i=2: rank=4, elements seen with rank > 4: query(5,4) = 0
     Insert rank 4 into BIT.  inversions += 0

i=3: rank=0, elements seen with rank > 0: query(1,4) = 3  (ranks 1,2,4 present)
     Insert rank 0 into BIT.  inversions += 3

i=4: rank=3, elements seen with rank > 3: query(4,4) = 1  (rank 4 is present)
     Insert rank 3 into BIT.  inversions += 1

Total inversions = 0 + 1 + 0 + 3 + 1 = 5

Classic Problem: Count Elements Less Than k in a Range (Offline)

Given an array and queries of the form "how many elements in arr[l..r] are less than k?", we can solve this offline by sorting queries by k and sweeping through compressed values.

Array: [10, 30, 20, 50, 40]
Queries: (l=0, r=3, k=25), (l=1, r=4, k=45)

Step 1: Compress array values AND query thresholds together
  All values: [10, 20, 25, 30, 40, 45, 50]
  Ranks:        0   1   2   3   4   5   6

Step 2: Sort queries by k (ascending)
  Q1: (l=0, r=3, k=25, rank_k=2)
  Q2: (l=1, r=4, k=45, rank_k=5)

Step 3: Sort array elements by value: (10,idx=0), (20,idx=2), (30,idx=1), ...

Step 4: Sweep - for each query, insert all elements with value < k into BIT by index
  For Q1 (k=25): insert positions of values < 25 -> insert idx 0 (val 10), idx 2 (val 20)
    BIT query(0,3) = 2   (positions 0 and 2 are in [0,3])

  For Q2 (k=45): insert positions of values < 45 -> also insert idx 1 (val 30), idx 4 (val 40)
    BIT query(1,4) = 3   (positions 1, 2, 4 are in [1,4])

Answers: Q1 = 2, Q2 = 3

Full Code: Inversion Count with Coordinate Compression + BIT

#include <bits/stdc++.h>
using namespace std;

struct BIT {
    vector<int> tree;
    int n;

    BIT(int n) : n(n), tree(n + 1, 0) {}

    void update(int i, int delta) {
        for (++i; i <= n; i += i & (-i))
            tree[i] += delta;
    }

    int query(int i) {  // prefix sum [0, i]
        int s = 0;
        for (++i; i > 0; i -= i & (-i))
            s += tree[i];
        return s;
    }

    int query(int l, int r) {  // range sum [l, r]
        return query(r) - (l ? query(l - 1) : 0);
    }
};

// Coordinate compression: maps values to ranks [0, m-1]
vector<int> compress(vector<int>& arr) {
    vector<int> sorted_vals(arr);
    sort(sorted_vals.begin(), sorted_vals.end());
    sorted_vals.erase(unique(sorted_vals.begin(), sorted_vals.end()), sorted_vals.end());

    vector<int> ranks(arr.size());
    for (int i = 0; i < arr.size(); i++) {
        ranks[i] = lower_bound(sorted_vals.begin(), sorted_vals.end(), arr[i])
                   - sorted_vals.begin();
    }
    return ranks;
}

long long countInversions(vector<int>& arr) {
    int n = arr.size();
    vector<int> ranks = compress(arr);
    int m = *max_element(ranks.begin(), ranks.end()) + 1;

    BIT bit(m);
    long long inversions = 0;

    for (int i = 0; i < n; i++) {
        // Count elements already inserted with rank > ranks[i]
        inversions += bit.query(ranks[i] + 1, m - 1);
        // Insert current rank
        bit.update(ranks[i], 1);
    }
    return inversions;
}

int main() {
    vector<int> arr = {1000000000, 3, 999999999, 50, 7};
    cout << "Inversions: " << countInversions(arr) << endl;
    // Output: Inversions: 6
}

import java.util.*;

class BIT {
    int[] tree;
    int n;

    BIT(int n) {
        this.n = n;
        tree = new int[n + 1];
    }

    void update(int i, int delta) {
        for (++i; i <= n; i += i & (-i))
            tree[i] += delta;
    }

    int query(int i) { // prefix sum [0, i]
        int s = 0;
        for (++i; i > 0; i -= i & (-i))
            s += tree[i];
        return s;
    }

    int query(int l, int r) { // range sum [l, r]
        return query(r) - (l > 0 ? query(l - 1) : 0);
    }
}

public class InversionCount {
    static int[] compress(int[] arr) {
        int[] sorted = arr.clone();
        Arrays.sort(sorted);
        // Deduplicate
        int m = 0;
        for (int i = 0; i < sorted.length; i++)
            if (i == 0 || sorted[i] != sorted[i - 1])
                sorted[m++] = sorted[i];
        sorted = Arrays.copyOf(sorted, m);

        int[] ranks = new int[arr.length];
        for (int i = 0; i < arr.length; i++)
            ranks[i] = Arrays.binarySearch(sorted, arr[i]);
        return ranks;
    }

    static long countInversions(int[] arr) {
        int n = arr.length;
        int[] ranks = compress(arr);
        int m = 0;
        for (int r : ranks) m = Math.max(m, r);
        m++;

        BIT bit = new BIT(m);
        long inversions = 0;

        for (int i = 0; i < n; i++) {
            // Count elements already inserted with rank > ranks[i]
            if (ranks[i] + 1 <= m - 1)
                inversions += bit.query(ranks[i] + 1, m - 1);
            // Insert current rank
            bit.update(ranks[i], 1);
        }
        return inversions;
    }

    public static void main(String[] args) {
        int[] arr = {1000000000, 3, 999999999, 50, 7};
        System.out.println("Inversions: " + countInversions(arr));
        // Output: Inversions: 6
    }
}

from bisect import bisect_left

class BIT:
    def __init__(self, n):
        self.n = n
        self.tree = [0] * (n + 1)

    def update(self, i, delta=1):
        i += 1  # 1-indexed internally
        while i <= self.n:
            self.tree[i] += delta
            i += i & (-i)

    def query(self, i):
        """Prefix sum [0, i]"""
        s = 0
        i += 1
        while i > 0:
            s += self.tree[i]
            i -= i & (-i)
        return s

    def range_query(self, l, r):
        """Range sum [l, r]"""
        return self.query(r) - (self.query(l - 1) if l > 0 else 0)


def compress(arr):
    """Map values to contiguous ranks [0, m-1]"""
    sorted_unique = sorted(set(arr))
    return [bisect_left(sorted_unique, x) for x in arr], len(sorted_unique)


def count_inversions(arr):
    ranks, m = compress(arr)
    bit = BIT(m)
    inversions = 0

    for rank in ranks:
        # Count elements already inserted with rank > current rank
        if rank + 1 <= m - 1:
            inversions += bit.range_query(rank + 1, m - 1)
        # Insert current rank
        bit.update(rank)

    return inversions


# Example with large values
arr = [1000000000, 3, 999999999, 50, 7]
print(f"Inversions: {count_inversions(arr)}")
# Output: Inversions: 6

When to Use Coordinate Compression vs Dynamic Segment Tree

CriteriaCoordinate CompressionDynamic Segment Tree
All values known upfront?Yes (offline)Not required (online)
MemoryO(n) — compactO(n log V) — allocates nodes on demand
ImplementationSimple — sort + binary searchMore complex — pointer-based or implicit
Value rangeIrrelevant after compressionMust handle full range [0, V]
New values at runtime?Cannot handle — ranks are fixedCan insert new values dynamically
Constant factorLow — uses arraysHigher — pointer chasing, allocation
Best forOffline problems, competitive programmingOnline queries with unknown future values
Rule of Thumb:
  • If all queries and updates are known in advance, prefer coordinate compression for its simplicity and speed
  • If queries arrive online and may introduce new values never seen before, use a dynamic segment tree or a balanced BST (e.g., policy-based tree in C++)

Offline vs Online Trade-offs

Offline Approach (Coordinate Compression)

  • Requirement: All values (from the array AND queries) must be known before processing
  • Advantage: Minimal memory, fast constant factor, simple to implement
  • Technique: Collect all values, compress, then process queries using the compressed indices
  • Limitation: Cannot handle truly interactive problems where future values are unknown

Online Approach (Dynamic Segment Tree / Merge Sort Tree)

  • Requirement: None — handles arbitrary values at any time
  • Advantage: Fully general, works with streaming data
  • Technique: Allocate tree nodes only when accessed; the tree spans [0, 109] but only materializes O(n log V) nodes
  • Limitation: Higher memory overhead, slower due to allocation and pointer indirection
ScenarioRecommended Approach
Count inversions in a static arrayCoordinate compression + BIT
Range frequency queries (all queries given)Offline compression + persistent seg tree
Online insert + count less than kDynamic segment tree or balanced BST
2D queries with large coordinatesCompress both dimensions independently
Interactive judge with unknown valuesDynamic segment tree (no choice)
Competitive Programming Tip:
  • Most contest problems allow offline processing — always check if coordinate compression works before reaching for heavier structures
  • When both query thresholds and array values need compression, combine them into a single sorted unique array
  • Time complexity is the same (O(n log n)) for both approaches — the difference is in constant factor and memory
