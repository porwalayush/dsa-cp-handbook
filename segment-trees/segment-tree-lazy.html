Lazy Propagation in Segment Trees

Lazy Propagation is a technique that defers updates to segment tree nodes until they are actually needed. This enables range updates in O(log n) instead of O(n), making it essential for problems that combine range updates with range queries.

Why Lazy Propagation?

Consider the problem: "Add a value v to every element in range [l, r]." Without lazy propagation, you would need to update each of the r - l + 1 elements individually, each taking O(log n), giving O(n log n) per range update. With lazy propagation, this drops to O(log n).

Problem: Range Add + Range Sum Query

Without Lazy Propagation:
  range_add(l, r, val) = point_update each index in [l,r]
  Time per range update: O((r-l+1) * log n) = O(n log n) worst case

With Lazy Propagation:
  range_add(l, r, val) = mark nodes covering [l,r] as "pending"
  Time per range update: O(log n)

Example: 100,000 operations on array of size 100,000
  Without lazy: up to 10^10 operations (TLE)
  With lazy:    up to 10^5 * 17 = 1.7 * 10^6 operations (fast)

Core Idea: Deferred Updates

Instead of updating every leaf node in a range, we store a pending update (lazy value) at the highest possible nodes. The update is only pushed down to children when we need to access those children.

Operation: Add 3 to range [0, 3]
Array: [2, 1, 5, 3, 4, 7, 6, 8]

BEFORE: Standard segment tree (sum)
                     [36]                 node 1: [0,7]
                   /      \
              [11]           [25]         nodes 2,3
             /    \         /    \
          [3]     [8]    [11]    [14]    nodes 4-7
          / \     / \     / \     / \
        [2] [1] [5] [3] [4] [7] [6] [8] leaves

AFTER: With lazy propagation (add 3 to [0,3])
                     [36+12=48]           tree updated
                   /      \
              [11+12=23]    [25]          tree updated
             /    \         /    \
          [3]     [8]    [11]    [14]    NOT visited!
          / \     / \     / \     / \
        [2] [1] [5] [3] [4] [7] [6] [8] NOT visited!

Lazy array:
  lazy[2] = 3   (node 2 covers [0,3], pending +3 for children)
  All other lazy values = 0

Key insight: Node 2 covers [0,3] exactly, so we:
  1. Update tree[2] += 3 * 4 = 12  (val * count of elements)
  2. Store lazy[2] = 3  (pending for children)
  3. STOP - don't recurse deeper!

The Push-Down Operation

The push_down (or propagate) function transfers pending lazy values from a parent to its children. This is called before we recurse into children during updates or queries.

push_down(node, l, r):
  If lazy[node] != 0:
    mid = (l + r) / 2
    left_count  = mid - l + 1
    right_count = r - mid

    // Push to left child
    tree[2*node]     += lazy[node] * left_count
    lazy[2*node]     += lazy[node]

    // Push to right child
    tree[2*node + 1] += lazy[node] * right_count
    lazy[2*node + 1] += lazy[node]

    // Clear parent's lazy
    lazy[node] = 0

Visualization of Push-Down

State: lazy[2] = 3, node 2 covers [0,3]

Now we query sum([0,1]). Must recurse into children of node 2.
BEFORE push_down(node=2, l=0, r=3):

         Node 2
        sum=23, lazy=3
        /          \
    Node 4          Node 5
   sum=3, lazy=0   sum=8, lazy=0
   covers [0,1]    covers [2,3]

AFTER push_down(node=2, l=0, r=3):

         Node 2
        sum=23, lazy=0      <- lazy cleared
        /          \
    Node 4          Node 5
   sum=3+3*2=9     sum=8+3*2=14    <- tree updated
   lazy=0+3=3      lazy=0+3=3     <- lazy inherited
   covers [0,1]    covers [2,3]

The +3 is propagated ONE level down, not all the way to leaves.
If we later need node 4's children, we push_down again.

Range Add + Range Sum Query

This is the most common lazy propagation variant. We support two operations:

  • range_add(l, r, val): Add val to every element in [l, r]
  • range_sum(l, r): Query the sum of elements in [l, r]
Example walkthrough: Array = [1, 3, 5, 7, 9, 11]

Initial tree (sum):
                  [36]
                /      \
           [9]           [27]
          /   \         /    \
       [4]    [5]    [16]    [11]
       / \     |     /  \      |
     [1] [3] [5]  [7]  [9]  [11]

Step 1: range_add(1, 4, 2)  -- add 2 to indices 1..4

Visit node 1 [0,5]: partial overlap, push_down (lazy=0, nothing), recurse
  Visit node 2 [0,2]: partial overlap, push_down, recurse
    Visit node 4 [0,1]: partial overlap, push_down, recurse
      Visit node 8 [0,0]: outside [1,4], return
      Visit node 9 [1,1]: fully inside
        tree[9] += 2*1 = 2  -> tree[9] = 5
        lazy[9] += 2 (leaf, no children, but consistent)
    tree[4] = tree[8] + tree[9] = 1 + 5 = 6
    Visit node 5 [2,2]: fully inside [1,4]
      tree[5] += 2*1 = 2  -> tree[5] = 7
      lazy[5] += 2
  tree[2] = tree[4] + tree[5] = 6 + 7 = 13
  Visit node 3 [3,5]: partial overlap, push_down, recurse
    Visit node 6 [3,4]: fully inside [1,4]
      tree[6] += 2*2 = 4  -> tree[6] = 20
      lazy[6] += 2          <- STOP here, don't go deeper!
    Visit node 7 [5,5]: outside [1,4], return
  tree[3] = tree[6] + tree[7] = 20 + 11 = 31
tree[1] = tree[2] + tree[3] = 13 + 31 = 44

Result: tree[1] = 44 (was 36, added 2 to 4 elements = +8)

Step 2: range_sum(3, 5)  -- query sum of indices 3..5

Visit node 1 [0,5]: partial, recurse
  Visit node 2 [0,2]: outside [3,5], return 0
  Visit node 3 [3,5]: fully inside, return tree[3] = 31

Answer = 31
(Original: 7+9+11=27, after +2 to [1,4]: 9+11+11=31) Correct!

Note: We never needed to push_down into node 6's children!

Range Set + Range Query Variant

Another common variant where range updates set all elements to a value (assignment) rather than adding. The key difference is that new lazy values override (replace) old ones instead of accumulating.

Range Set: set all elements in [l,r] to val

Differences from Range Add:
  1. lazy stores the assigned value (use sentinel like -1 or INT_MIN for "no pending")
  2. push_down REPLACES children's values (not adds)
  3. New set OVERWRITES any existing lazy

push_down for Range Set:
  if lazy[node] != NO_PENDING:
    mid = (l + r) / 2
    tree[2*node]     = lazy[node] * (mid - l + 1)
    tree[2*node + 1] = lazy[node] * (r - mid)
    lazy[2*node]     = lazy[node]       // override child lazy
    lazy[2*node + 1] = lazy[node]       // override child lazy
    lazy[node] = NO_PENDING

Example: set [0,3] = 5, then set [2,5] = 2

After set [0,3] = 5:  Array conceptually: [5, 5, 5, 5, 9, 11]
After set [2,5] = 2:  Array conceptually: [5, 5, 2, 2, 2, 2]

The second set at index 2,3 overwrites the first set.
lazy is replaced, not accumulated.

Full Implementation: Range Add + Range Sum

#include <bits/stdc++.h>
using namespace std;

struct LazySegTree {
    vector<long long> tree, lazy;
    int n;

    LazySegTree(vector<int>& arr) {
        n = arr.size();
        tree.resize(4 * n, 0);
        lazy.resize(4 * n, 0);
        build(arr, 1, 0, n - 1);
    }

    void build(vector<int>& arr, int node, int l, int r) {
        if (l == r) {
            tree[node] = arr[l];
            return;
        }
        int mid = (l + r) / 2;
        build(arr, 2 * node, l, mid);
        build(arr, 2 * node + 1, mid + 1, r);
        tree[node] = tree[2 * node] + tree[2 * node + 1];
    }

    // Push pending lazy value down to children
    void push_down(int node, int l, int r) {
        if (lazy[node] != 0) {
            int mid = (l + r) / 2;
            // Update left child
            tree[2 * node] += lazy[node] * (mid - l + 1);
            lazy[2 * node] += lazy[node];
            // Update right child
            tree[2 * node + 1] += lazy[node] * (r - mid);
            lazy[2 * node + 1] += lazy[node];
            // Clear current node's lazy
            lazy[node] = 0;
        }
    }

    // Range update: add val to all elements in [ql, qr]
    void range_add(int node, int l, int r, int ql, int qr, long long val) {
        if (qr < l || r < ql) return;          // no overlap
        if (ql <= l && r <= qr) {              // full overlap
            tree[node] += val * (r - l + 1);
            lazy[node] += val;
            return;
        }
        push_down(node, l, r);                  // partial overlap
        int mid = (l + r) / 2;
        range_add(2 * node, l, mid, ql, qr, val);
        range_add(2 * node + 1, mid + 1, r, ql, qr, val);
        tree[node] = tree[2 * node] + tree[2 * node + 1];
    }

    void range_add(int ql, int qr, long long val) {
        range_add(1, 0, n - 1, ql, qr, val);
    }

    // Range query: sum of elements in [ql, qr]
    long long range_sum(int node, int l, int r, int ql, int qr) {
        if (qr < l || r < ql) return 0;        // no overlap
        if (ql <= l && r <= qr) return tree[node]; // full overlap
        push_down(node, l, r);                  // partial overlap
        int mid = (l + r) / 2;
        return range_sum(2 * node, l, mid, ql, qr) +
               range_sum(2 * node + 1, mid + 1, r, ql, qr);
    }

    long long range_sum(int ql, int qr) {
        return range_sum(1, 0, n - 1, ql, qr);
    }
};

int main() {
    vector<int> arr = {1, 3, 5, 7, 9, 11};
    LazySegTree st(arr);

    cout << st.range_sum(0, 5) << endl;   // 36
    st.range_add(1, 4, 2);                // add 2 to [1,4]
    cout << st.range_sum(1, 4) << endl;   // 5+7+9+11 + 4*2 = 32
    cout << st.range_sum(0, 5) << endl;   // 44
    st.range_add(0, 2, -1);               // subtract 1 from [0,2]
    cout << st.range_sum(0, 2) << endl;   // 0+4+6 = 10
}

    int n;

    LazySegTree(int[] arr) {

        tree = new long[4 * n];
        lazy = new long[4 * n];
        build(arr, 1, 0, n - 1);

        int mid = (l + r) / 2;
        build(arr, 2 * node, l, mid);
        build(arr, 2 * node + 1, mid + 1, r);
        tree[node] = tree[2 * node] + tree[2 * node + 1];
    }

    void pushDown(int node, int l, int r) {
        if (lazy[node] != 0) {
            int mid = (l + r) / 2;
            tree[2 * node] += lazy[node] * (mid - l + 1);
            lazy[2 * node] += lazy[node];
            tree[2 * node + 1] += lazy[node] * (r - mid);
            lazy[2 * node + 1] += lazy[node];
            lazy[node] = 0;
        }
    }

    void rangeAdd(int node, int l, int r, int ql, int qr, long val) {
        if (qr < l || r < ql) return;
        if (ql <= l && r <= qr) {
            tree[node] += val * (r - l + 1);
            lazy[node] += val;
            return;
        }
        pushDown(node, l, r);
        int mid = (l + r) / 2;
        rangeAdd(2 * node, l, mid, ql, qr, val);
        rangeAdd(2 * node + 1, mid + 1, r, ql, qr, val);
        tree[node] = tree[2 * node] + tree[2 * node + 1];
    }

    void rangeAdd(int ql, int qr, long val) {
        rangeAdd(1, 0, n - 1, ql, qr, val);
    }

    long rangeSum(int node, int l, int r, int ql, int qr) {
        if (qr < l || r < ql) return 0;
        if (ql <= l && r <= qr) return tree[node];
        pushDown(node, l, r);
        int mid = (l + r) / 2;
        return rangeSum(2 * node, l, mid, ql, qr) +
               rangeSum(2 * node + 1, mid + 1, r, ql, qr);
    }

    long rangeSum(int ql, int qr) {
        return rangeSum(1, 0, n - 1, ql, qr);
    }

    public static void main(String[] args) {
        int[] arr = {1, 3, 5, 7, 9, 11};
        LazySegTree st = new LazySegTree(arr);

        System.out.println(st.rangeSum(0, 5));  // 36
        st.rangeAdd(1, 4, 2);                     // add 2 to [1,4]
        System.out.println(st.rangeSum(1, 4));  // 32
        System.out.println(st.rangeSum(0, 5));  // 44
        st.rangeAdd(0, 2, -1);                    // subtract 1 from [0,2]
        System.out.println(st.rangeSum(0, 2));  // 10
    }
}

class LazySegTree:
    def __init__(self, arr):
        self.n = len(arr)
        self.tree = [0] * (4 * self.n)
        self.lazy = [0] * (4 * self.n)
        self._build(arr, 1, 0, self.n - 1)

    def _build(self, arr, node, l, r):
        if l == r:
            self.tree[node] = arr[l]
            return
        mid = (l + r) // 2
        self._build(arr, 2 * node, l, mid)
        self._build(arr, 2 * node + 1, mid + 1, r)
        self.tree[node] = self.tree[2 * node] + self.tree[2 * node + 1]

    def _push_down(self, node, l, r):
        if self.lazy[node] != 0:
            mid = (l + r) // 2
            # Push to left child
            self.tree[2 * node] += self.lazy[node] * (mid - l + 1)
            self.lazy[2 * node] += self.lazy[node]
            # Push to right child
            self.tree[2 * node + 1] += self.lazy[node] * (r - mid)
            self.lazy[2 * node + 1] += self.lazy[node]
            # Clear current lazy
            self.lazy[node] = 0

    def range_add(self, ql, qr, val, node=1, l=0, r=None):
        if r is None: r = self.n - 1
        if qr < l or r < ql:
            return                          # no overlap
        if ql <= l and r <= qr:
            self.tree[node] += val * (r - l + 1)
            self.lazy[node] += val
            return                          # full overlap
        self._push_down(node, l, r)         # partial overlap
        mid = (l + r) // 2
        self.range_add(ql, qr, val, 2 * node, l, mid)
        self.range_add(ql, qr, val, 2 * node + 1, mid + 1, r)
        self.tree[node] = self.tree[2 * node] + self.tree[2 * node + 1]

    def range_sum(self, ql, qr, node=1, l=0, r=None):
        if r is None: r = self.n - 1
        if qr < l or r < ql:
            return 0                       # no overlap
        if ql <= l and r <= qr:
            return self.tree[node]          # full overlap
        self._push_down(node, l, r)         # partial overlap
        mid = (l + r) // 2
        return (self.range_sum(ql, qr, 2 * node, l, mid) +
                self.range_sum(ql, qr, 2 * node + 1, mid + 1, r))

Range Set Implementation

#include <bits/stdc++.h>
using namespace std;

struct LazySegTreeSet {
    vector<long long> tree;
    vector<long long> lazy;
    int n;
    const long long NO_PENDING = LLONG_MIN;

    LazySegTreeSet(vector<int>& arr) {
        n = arr.size();
        tree.resize(4 * n, 0);
        lazy.assign(4 * n, NO_PENDING);
        build(arr, 1, 0, n - 1);
    }

    void build(vector<int>& arr, int node, int l, int r) {
        if (l == r) { tree[node] = arr[l]; return; }
        int mid = (l + r) / 2;
        build(arr, 2 * node, l, mid);
        build(arr, 2 * node + 1, mid + 1, r);
        tree[node] = tree[2 * node] + tree[2 * node + 1];
    }

    void push_down(int node, int l, int r) {
        if (lazy[node] != NO_PENDING) {
            int mid = (l + r) / 2;
            // Set (not add) children's values
            tree[2 * node] = lazy[node] * (mid - l + 1);
            lazy[2 * node] = lazy[node];
            tree[2 * node + 1] = lazy[node] * (r - mid);
            lazy[2 * node + 1] = lazy[node];
            lazy[node] = NO_PENDING;
        }
    }

    void range_set(int node, int l, int r, int ql, int qr, long long val) {
        if (qr < l || r < ql) return;
        if (ql <= l && r <= qr) {
            tree[node] = val * (r - l + 1);
            lazy[node] = val;
            return;
        }
        push_down(node, l, r);
        int mid = (l + r) / 2;
        range_set(2 * node, l, mid, ql, qr, val);
        range_set(2 * node + 1, mid + 1, r, ql, qr, val);
        tree[node] = tree[2 * node] + tree[2 * node + 1];
    }

    void range_set(int ql, int qr, long long val) {
        range_set(1, 0, n - 1, ql, qr, val);
    }

    long long range_sum(int node, int l, int r, int ql, int qr) {
        if (qr < l || r < ql) return 0;
        if (ql <= l && r <= qr) return tree[node];
        push_down(node, l, r);
        int mid = (l + r) / 2;
        return range_sum(2 * node, l, mid, ql, qr) +
               range_sum(2 * node + 1, mid + 1, r, ql, qr);
    }

    long long range_sum(int ql, int qr) {
        return range_sum(1, 0, n - 1, ql, qr);
    }
};

class LazySegTreeSet {
    long[] tree, lazy;
    boolean[] hasLazy;  // track if lazy is pending
    int n;

    LazySegTreeSet(int[] arr) {
        n = arr.length;
        tree = new long[4 * n];
        lazy = new long[4 * n];
        hasLazy = new boolean[4 * n];
        build(arr, 1, 0, n - 1);
    }

    void build(int[] arr, int node, int l, int r) {
        if (l == r) { tree[node] = arr[l]; return; }
        int mid = (l + r) / 2;
        build(arr, 2 * node, l, mid);
        build(arr, 2 * node + 1, mid + 1, r);
        tree[node] = tree[2 * node] + tree[2 * node + 1];
    }

    void pushDown(int node, int l, int r) {
        if (hasLazy[node]) {
            int mid = (l + r) / 2;
            tree[2 * node] = lazy[node] * (mid - l + 1);
            lazy[2 * node] = lazy[node];
            hasLazy[2 * node] = true;
            tree[2 * node + 1] = lazy[node] * (r - mid);
            lazy[2 * node + 1] = lazy[node];
            hasLazy[2 * node + 1] = true;
            hasLazy[node] = false;
        }
    }

    void rangeSet(int node, int l, int r, int ql, int qr, long val) {
        if (qr < l || r < ql) return;
        if (ql <= l && r <= qr) {
            tree[node] = val * (r - l + 1);
            lazy[node] = val;
            hasLazy[node] = true;
            return;
        }
        pushDown(node, l, r);
        int mid = (l + r) / 2;
        rangeSet(2 * node, l, mid, ql, qr, val);
        rangeSet(2 * node + 1, mid + 1, r, ql, qr, val);
        tree[node] = tree[2 * node] + tree[2 * node + 1];
    }

    }

        if (ql <= l && r <= qr) return tree[node];
        pushDown(node, l, r);

        return rangeSum(2 * node, l, mid, ql, qr) +
               rangeSum(2 * node + 1, mid + 1, r, ql, qr);
    }

    long rangeSum(int ql, int qr) {
        return rangeSum(1, 0, n - 1, ql, qr);
    }
}

class LazySegTreeSet:
    def __init__(self, arr):
        self.n = len(arr)
        self.tree = [0] * (4 * self.n)
        self.lazy = [None] * (4 * self.n)  # None = no pending
        self._build(arr, 1, 0, self.n - 1)

    def _build(self, arr, node, l, r):
        if l == r:
            self.tree[node] = arr[l]
            return
        mid = (l + r) // 2
        self._build(arr, 2 * node, l, mid)
        self._build(arr, 2 * node + 1, mid + 1, r)
        self.tree[node] = self.tree[2 * node] + self.tree[2 * node + 1]

    def _push_down(self, node, l, r):
        if self.lazy[node] is not None:
            mid = (l + r) // 2
            val = self.lazy[node]
            self.tree[2 * node] = val * (mid - l + 1)
            self.lazy[2 * node] = val
            self.tree[2 * node + 1] = val * (r - mid)
            self.lazy[2 * node + 1] = val
            self.lazy[node] = None

    def range_set(self, ql, qr, val, node=1, l=0, r=None):
        if r is None: r = self.n - 1
        if qr < l or r < ql: return
        if ql <= l and r <= qr:
            self.tree[node] = val * (r - l + 1)
            self.lazy[node] = val
            return
        self._push_down(node, l, r)
        mid = (l + r) // 2
        self.range_set(ql, qr, val, 2 * node, l, mid)
        self.range_set(ql, qr, val, 2 * node + 1, mid + 1, r)
        self.tree[node] = self.tree[2 * node] + self.tree[2 * node + 1]

    def range_sum(self, ql, qr, node=1, l=0, r=None):
        if r is None: r = self.n - 1
        if qr < l or r < ql: return 0
        if ql <= l and r <= qr: return self.tree[node]
        self._push_down(node, l, r)
        mid = (l + r) // 2
        return (self.range_sum(ql, qr, 2 * node, l, mid) +
                self.range_sum(ql, qr, 2 * node + 1, mid + 1, r))

Complexity

OperationWithout LazyWith LazySpace

Range UpdateO(n log n)O(log n)-

When do you need Lazy Propagation?
  • You NEED lazy when you have range updates + range queries together (e.g., add val to [l,r], then query sum of [l,r])
  • You DON'T need lazy for point updates + range queries (standard segment tree suffices)
  • You DON'T need lazy for range updates + point queries (can use difference arrays or BIT with range update)
  • Key rule: Lazy is required when the update operation is "compatible" with the query -- i.e., you can compute the new aggregate for a range without visiting every element (e.g., adding v to n elements increases sum by v*n)
  • Common lazy variants: Range Add + Sum, Range Set + Sum, Range Add + Min/Max, Range Set + Min/Max, Range XOR + Sum (for bits)
  • Cannot use simple lazy: Range Set + Count of Distinct Values (need more complex node info)

How Updates and Queries Interact

Execution flow of range_add and range_sum:

range_add(node, l, r, ql, qr, val):
  ┌─────────────────────────────────┐
  │ 1. No overlap? -> return        │
  │ 2. Full overlap?                │
  │    -> tree[node] += val*(r-l+1) │
  │    -> lazy[node] += val         │
  │    -> return (STOP here!)       │
  │ 3. Partial overlap?             │
  │    -> push_down(node)           │
  │    -> recurse left child        │
  │    -> recurse right child       │
  │    -> tree[node] = left + right │
  └─────────────────────────────────┘

range_sum(node, l, r, ql, qr):
  ┌─────────────────────────────────┐
  │ 1. No overlap? -> return 0      │
  │ 2. Full overlap? -> return      │
  │    tree[node]  (already correct │
  │    because update maintained it)│
  │ 3. Partial overlap?             │
  │    -> push_down(node)           │
  │    -> return left + right       │
  └─────────────────────────────────┘

Critical insight: push_down is called ONLY on partial overlap.
  - Full overlap: node's tree value is already correct
  - No overlap: we don't touch the node at all
  - Partial: we MUST push before accessing children

Multiple Pending Updates

What happens with multiple range_adds before any query?

Array: [1, 2, 3, 4]  (n=4)

Operation 1: range_add(0, 3, 5)   -- add 5 to all
Operation 2: range_add(0, 1, 3)   -- add 3 to first half

After Op 1:
         Node 1 [0,3]
        tree=10+5*4=30, lazy=5
        /              \
    Node 2 [0,1]      Node 3 [2,3]
   tree=3, lazy=0     tree=7, lazy=0

  Node 1 fully covers [0,3], so we just update node 1.
  Children are NOT touched.

After Op 2 (add 3 to [0,1]):
  Node 1 [0,3]: partial overlap with [0,1]
    -> push_down node 1!
       Node 2: tree=3+5*2=13, lazy=0+5=5
       Node 3: tree=7+5*2=17, lazy=0+5=5
       Node 1: lazy=0
    -> recurse left: Node 2 [0,1] fully inside [0,1]
       tree=13+3*2=19, lazy=5+3=8
    -> recurse right: Node 3 [2,3] outside [0,1]
       return
    -> tree[1] = tree[2] + tree[3] = 19 + 17 = 36

Final state:
         Node 1 [0,3]
        tree=36, lazy=0
        /              \
    Node 2 [0,1]      Node 3 [2,3]
   tree=19, lazy=8    tree=17, lazy=5
   (1+5+3)+(2+5+3)   (3+5)+(4+5)
     = 9 + 10 = 19     = 8 + 9 = 17

Lazy values accumulate: node 2 has lazy=8 (5 from op1 + 3 from op2)
