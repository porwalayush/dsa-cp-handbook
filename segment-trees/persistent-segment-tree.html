Persistent Segment Tree

A Persistent Segment Tree maintains the full version history of a segment tree across updates. Instead of modifying nodes in-place, each update creates a new version by copying only the nodes along the path from the root to the updated leaf — a technique called copy-on-write.

Concept: Copy-on-Write

In a standard segment tree, an update overwrites nodes in-place. In a persistent segment tree:

  • Each update creates O(log n) new nodes (the path from root to leaf)
  • Unchanged subtrees are shared between versions via pointers
  • Every previous root remains valid and accessible
  • We store an array of roots: roots[v] gives the root of version v

This gives us the ability to query any historical version in O(log n) without storing a full copy of the tree for each version.

Kth Smallest Element in Range [l, r]

The classic application of persistent segment trees. Given an array, answer queries: "What is the kth smallest element in the subarray arr[l..r]?"

Approach

  1. Coordinate compress the array values into range [0, m-1]
  2. Build the persistent segment tree on prefix frequency arrays:
    • Version 0: empty tree (all zeros)
    • Version i: insert arr[i] into Version i-1 (increment count at compressed index)
  3. To query kth smallest in [l, r]: subtract roots[l-1] from roots[r] to get the frequency tree for the subarray, then walk down to find kth element
Array: [5, 1, 3, 2, 4]
Sorted unique: [1, 2, 3, 4, 5] -> compressed indices 0..4

Build versions by inserting elements left to right:
  Version 0: []           -> all zeros
  Version 1: [5]         -> increment index 4
  Version 2: [5,1]       -> increment index 0
  Version 3: [5,1,3]     -> increment index 2
  Version 4: [5,1,3,2]   -> increment index 1
  Version 5: [5,1,3,2,4] -> increment index 3

Query: 2nd smallest in arr[2..4] = subarray [3, 2, 4]

  Use roots[4] - roots[1] to get frequency in range [2,4]:
  Freq tree for [3,2,4]: indices [0,1,2,3,4] -> [0,1,1,1,0]

  Walk down to find kth = 2:
    Root [3]: left_count = 1 (indices 0..2)
              Since k=2 > left_count=1? No, left has [0,1,1]=2
              k=2 <= left_count=2, go left
    Left [2]: left_count = 1 (indices 0..1)
              k=2 > left_count=1, go right, k = 2-1 = 1
    Leaf at index 2 -> sorted[2] = 3

  Answer: 3 (the 2nd smallest in [3,2,4] is 3)

Building from Sorted Values

The persistent segment tree for kth queries is built over the value domain (sorted unique values), not the index domain. Each version i represents the frequency histogram of the first i elements.

This is closely related to merge sort trees: a merge sort tree stores sorted sublists at each node, while the persistent segment tree stores prefix frequency counts over the value domain. Both solve the same problems, but the persistent approach uses O(n log n) space vs O(n log n) for merge sort tree, with better query time O(log n) vs O(log^2 n).

Code Implementation

#include <bits/stdc++.h>
using namespace std;

struct Node {
    int cnt;
    Node *left, *right;
    Node(int c, Node* l, Node* r) : cnt(c), left(l), right(r) {}
};

Node* NIL;

void init() {
    NIL = new Node(0, nullptr, nullptr);
    NIL->left = NIL->right = NIL;
}

// Build empty tree over value range [l, r]
Node* build(int l, int r) {
    if (l == r) return NIL;
    int mid = (l + r) / 2;
    return new Node(0, build(l, mid), build(mid + 1, r));
}

// Insert: increment count at position pos, return new root
Node* update(Node* prev, int l, int r, int pos) {
    if (l == r)
        return new Node(prev->cnt + 1, NIL, NIL);
    int mid = (l + r) / 2;
    if (pos <= mid)
        return new Node(prev->cnt + 1,
                       update(prev->left, l, mid, pos),
                       prev->right);
    else
        return new Node(prev->cnt + 1,
                       prev->left,
                       update(prev->right, mid + 1, r, pos));
}

// Query: kth smallest in range using roots[r] - roots[l-1]
int query(Node* lt, Node* rt, int l, int r, int k) {
    if (l == r) return l;
    int mid = (l + r) / 2;
    int left_count = rt->left->cnt - lt->left->cnt;
    if (k <= left_count)
        return query(lt->left, rt->left, l, mid, k);
    else
        return query(lt->right, rt->right, mid + 1, r, k - left_count);
}

int main() {
    init();
    int n, q;
    cin >> n >> q;
    vector<int> arr(n);
    for (int& x : arr) cin >> x;

    // Coordinate compression
    vector<int> sorted_vals(arr);
    sort(sorted_vals.begin(), sorted_vals.end());
    sorted_vals.erase(unique(sorted_vals.begin(), sorted_vals.end()), sorted_vals.end());
    int m = sorted_vals.size();

    auto compress = [&](int x) {
        return lower_bound(sorted_vals.begin(), sorted_vals.end(), x) - sorted_vals.begin();
    };

    // Build persistent segment tree
    vector<Node*> roots(n + 1);
    roots[0] = build(0, m - 1);
    for (int i = 0; i < n; i++)
        roots[i + 1] = update(roots[i], 0, m - 1, compress(arr[i]));

    // Answer queries: kth smallest in arr[l..r] (1-indexed)
    while (q--) {
        int l, r, k;
        cin >> l >> r >> k;
        int idx = query(roots[l - 1], roots[r], 0, m - 1, k);
        cout << sorted_vals[idx] << "\n";
    }
}

import java.util.*;

class PersistentSegTree {
    static int MAXNODES = 20_000_000;
    static int[] cnt = new int[MAXNODES];
    static int[] left = new int[MAXNODES];
    static int[] right = new int[MAXNODES];
    static int tot = 0;

    static int newNode(int c, int l, int r) {
        int id = ++tot;
        cnt[id] = c; left[id] = l; right[id] = r;
        return id;
    }

    // Build empty tree over [l, r]
    static int build(int l, int r) {
        if (l == r) return newNode(0, 0, 0);
        int mid = (l + r) / 2;
        return newNode(0, build(l, mid), build(mid + 1, r));
    }

    // Insert: increment at pos, return new root id
    static int update(int prev, int l, int r, int pos) {
        if (l == r)
            return newNode(cnt[prev] + 1, 0, 0);
        int mid = (l + r) / 2;
        if (pos <= mid)
            return newNode(cnt[prev] + 1,
                          update(left[prev], l, mid, pos),
                          right[prev]);
        else
            return newNode(cnt[prev] + 1,
                          left[prev],
                          update(right[prev], mid + 1, r, pos));
    }

    // Query kth smallest using roots[r] - roots[l-1]
    static int query(int lt, int rt, int l, int r, int k) {
        if (l == r) return l;
        int mid = (l + r) / 2;
        int leftCount = cnt[left[rt]] - cnt[left[lt]];
        if (k <= leftCount)
            return query(left[lt], left[rt], l, mid, k);
        else
            return query(right[lt], right[rt], mid + 1, r, k - leftCount);
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt(), q = sc.nextInt();
        int[] arr = new int[n];
        for (int i = 0; i < n; i++) arr[i] = sc.nextInt();

        // Coordinate compression
        int[] sorted = arr.clone();
        Arrays.sort(sorted);
        int m = 0;
        for (int i = 0; i < n; i++)
            if (i == 0 || sorted[i] != sorted[i - 1])
                sorted[m++] = sorted[i];

        // Build persistent segment tree
        int[] roots = new int[n + 1];
        roots[0] = build(0, m - 1);
        for (int i = 0; i < n; i++) {
            int pos = Arrays.binarySearch(sorted, 0, m, arr[i]);
            roots[i + 1] = update(roots[i], 0, m - 1, pos);
        }

        // Answer queries
        StringBuilder sb = new StringBuilder();
        while (q-- > 0) {
            int l = sc.nextInt(), r = sc.nextInt(), k = sc.nextInt();
            int idx = query(roots[l - 1], roots[r], 0, m - 1, k);
            sb.append(sorted[idx]).append("\n");
        }
        System.out.print(sb);
    }
}

import sys
from bisect import bisect_left
input = sys.stdin.readline

class Node:
    __slots__ = ['cnt', 'left', 'right']
    def __init__(self, cnt=0, left=None, right=None):
        self.cnt = cnt
        self.left = left
        self.right = right

def build(l, r):
    if l == r:
        return Node()
    mid = (l + r) // 2
    return Node(0, build(l, mid), build(mid + 1, r))

def update(prev, l, r, pos):
    if l == r:
        return Node(prev.cnt + 1)
    mid = (l + r) // 2
    if pos <= mid:
        return Node(prev.cnt + 1,
                   update(prev.left, l, mid, pos),
                   prev.right)
    else:
        return Node(prev.cnt + 1,
                   prev.left,
                   update(prev.right, mid + 1, r, pos))

def query(lt, rt, l, r, k):
    if l == r:
        return l
    mid = (l + r) // 2
    left_count = rt.left.cnt - lt.left.cnt
    if k <= left_count:
        return query(lt.left, rt.left, l, mid, k)
    else:
        return query(lt.right, rt.right, mid + 1, r, k - left_count)

def main():
    n, q = map(int, input().split())
    arr = list(map(int, input().split()))

    # Coordinate compression
    sorted_vals = sorted(set(arr))
    m = len(sorted_vals)
    compress = {v: i for i, v in enumerate(sorted_vals)}

    # Build persistent segment tree
    roots = [None] * (n + 1)
    roots[0] = build(0, m - 1)
    for i in range(n):
        roots[i + 1] = update(roots[i], 0, m - 1, compress[arr[i]])

    # Answer queries: kth smallest in arr[l..r] (1-indexed)
    out = []
    for _ in range(q):
        l, r, k = map(int, input().split())
        idx = query(roots[l - 1], roots[r], 0, m - 1, k)
        out.append(str(sorted_vals[idx]))
    print("\n".join(out))

Memory Analysis

Each update creates exactly O(log n) new nodes (one per level of the tree). For n insertions:

  • Initial tree: O(m) nodes where m is the value range (after compression, m ≤ n)
  • Each of n updates: O(log m) new nodes
  • Total nodes: O(n log n)

In practice, for n = 10^5 with values up to 10^9 (compressed to 10^5 unique values), total nodes are approximately n * log2(n) + n ≈ 10^5 * 17 + 10^5 ≈ 1.8 * 10^6 nodes. Each node stores a count and two pointers (or indices), so memory is manageable.

Implementation tip: For competitive programming, use a flat array with indices instead of pointers to avoid memory allocation overhead. Pre-allocate an array of size n * 20 (since log2(10^5) < 17, and adding some buffer). The Java implementation above demonstrates this approach.

Applications

Key applications of Persistent Segment Trees:
  • Version queries: Query or rollback to any previous state of the data structure in O(log n)
  • Kth smallest in range [l, r]: The classic application using prefix version subtraction (described above)
  • Count elements in range less than k: Walk the persistent tree to count frequencies in [0, k-1] for a given subarray
  • Online queries: Unlike merge sort tree approaches that need offline processing, persistent ST handles online queries naturally
  • Persistent data on trees: Combined with Euler tour or HLD for path/subtree version queries on trees
  • 2D problems: Replace one dimension with persistence (e.g., sweep line + persistent structure)
