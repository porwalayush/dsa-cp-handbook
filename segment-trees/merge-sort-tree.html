Merge Sort Tree

A Merge Sort Tree is a segment tree where each node stores the sorted array of all elements in its range. This allows answering order-statistic queries like "count of elements in [l, r] that are less than or equal to k" efficiently using binary search at each visited node.

What is a Merge Sort Tree?

In a standard segment tree, each node stores a single aggregate value (sum, min, max). In a Merge Sort Tree, each node stores the entire sorted subarray of elements in its range. The name comes from the fact that the build process mirrors merge sort: leaf nodes hold single elements, and internal nodes merge their children's sorted arrays.

  • Each leaf node stores a single-element array [arr[i]]
  • Each internal node stores the sorted merge of its two children's arrays
  • The root stores the fully sorted array (same as the output of merge sort)
  • The structure is static (no point updates supported efficiently)

Build Operation

Building a Merge Sort Tree follows the same recursive structure as a segment tree. At each internal node, we merge the sorted arrays from the left and right children (exactly like the merge step in merge sort).

build(node, l, r):
  if l == r:
    tree[node] = [arr[l]]        // leaf: single element
    return
  mid = (l + r) / 2
  build(2*node, l, mid)          // build left child
  build(2*node+1, mid+1, r)     // build right child
  tree[node] = merge(tree[2*node], tree[2*node+1])  // merge sorted arrays

Time:  O(n log n)  -- each element appears in O(log n) nodes
Space: O(n log n)  -- total storage across all levels
Space Analysis:
  • At each level of the tree, every element appears exactly once
  • There are O(log n) levels in the tree
  • Total storage: n elements per level x log n levels = O(n log n)

Query: Count Elements ≤ k in [l, r]

The primary query on a Merge Sort Tree is: given a range [l, r] and a value k, count how many elements in that range are less than or equal to k. Since each node stores a sorted array, we use upper_bound (binary search) at each visited node.

Query: count elements ≤ 4 in range [1, 6]
Array: [3, 1, 4, 1, 5, 9, 2, 6]

Traversal (same as standard segment tree query):
  Node 1 [0,7]: partial overlap with [1,6] -> recurse both
    Node 2 [0,3]: partial overlap -> recurse both
      Node 4 [0,1]: partial overlap -> recurse both
        Node 8 [0,0]: outside [1,6] -> return 0
        Node 9 [1,1]: fully inside -> upper_bound([1], 4) = 1
      Node 5 [2,3]: fully inside -> upper_bound([1,4], 4) = 2
    Node 3 [4,7]: partial overlap -> recurse both
      Node 6 [4,5]: fully inside -> upper_bound([5,9], 4) = 0
      Node 7 [6,7]: partial overlap -> recurse both
        Node 14 [6,6]: fully inside -> upper_bound([2], 4) = 1
        Node 15 [7,7]: outside [1,6] -> return 0

Answer = 0 + 1 + 2 + 0 + 1 + 0 = 4
Verify: elements in [1,6] are {1,4,1,5,9,2}, those ≤ 4: {1,4,1,2} = 4 items

Query Algorithm

query(node, l, r, ql, qr, k):
  if qr < l or r < ql:          // no overlap
    return 0
  if ql <= l and r <= qr:       // full overlap
    return upper_bound(tree[node], k)  // binary search
  mid = (l + r) / 2
  return query(2*node, l, mid, ql, qr, k) +
         query(2*node+1, mid+1, r, ql, qr, k)

Time per query: O(log^2 n)
  - O(log n) nodes visited
  - O(log n) binary search at each node

Complexity

OperationTimeSpace
BuildO(n log n)O(n log n)
Query (count ≤ k)O(log2 n)-
Query (kth smallest via binary search)O(log3 n)-

Full Implementation

#include <bits/stdc++.h>
using namespace std;

struct MergeSortTree {
    int n;
    vector<vector<int>> tree;

    MergeSortTree(vector<int>& arr) {
        n = arr.size();
        tree.resize(4 * n);
        build(arr, 1, 0, n - 1);
    }

    void build(vector<int>& arr, int node, int l, int r) {
        if (l == r) {
            tree[node] = {arr[l]};
            return;
        }
        int mid = (l + r) / 2;
        build(arr, 2 * node, l, mid);
        build(arr, 2 * node + 1, mid + 1, r);
        // Merge two sorted arrays
        merge(tree[2 * node].begin(), tree[2 * node].end(),
              tree[2 * node + 1].begin(), tree[2 * node + 1].end(),
              back_inserter(tree[node]));
    }

    // Count elements in [ql, qr] that are <= k
    int query(int node, int l, int r, int ql, int qr, int k) {
        if (qr < l || r < ql) return 0;
        if (ql <= l && r <= qr) {
            // upper_bound gives count of elements <= k
            return upper_bound(tree[node].begin(),
                               tree[node].end(), k) - tree[node].begin();
        }
        int mid = (l + r) / 2;
        return query(2 * node, l, mid, ql, qr, k) +
               query(2 * node + 1, mid + 1, r, ql, qr, k);
    }

    int countLessOrEqual(int ql, int qr, int k) {
        return query(1, 0, n - 1, ql, qr, k);
    }

    // Kth smallest in [ql, qr] using binary search on answer
    int kthSmallest(int ql, int qr, int k) {
        int lo = *min_element(tree[1].begin(), tree[1].end());
        int hi = *max_element(tree[1].begin(), tree[1].end());
        int ans = lo;
        while (lo <= hi) {
            int mid = lo + (hi - lo) / 2;
            if (countLessOrEqual(ql, qr, mid) >= k) {
                ans = mid;
                hi = mid - 1;
            } else {
                lo = mid + 1;
            }
        }
        return ans;
    }
};

int main() {
    vector<int> arr = {3, 1, 4, 1, 5, 9, 2, 6};
    MergeSortTree mst(arr);

    // Count elements <= 4 in range [1, 6]
    cout << mst.countLessOrEqual(1, 6, 4) << endl;  // 4

    // 3rd smallest in range [0, 7]
    cout << mst.kthSmallest(0, 7, 3) << endl;  // 2
}

import java.util.*;

class MergeSortTree {
    int n;
    int[][] tree;

    MergeSortTree(int[] arr) {
        n = arr.length;
        tree = new int[4 * n][];
        build(arr, 1, 0, n - 1);
    }

    void build(int[] arr, int node, int l, int r) {
        if (l == r) {
            tree[node] = new int[]{arr[l]};
            return;
        }
        int mid = (l + r) / 2;
        build(arr, 2 * node, l, mid);
        build(arr, 2 * node + 1, mid + 1, r);
        // Merge two sorted arrays
        tree[node] = mergeSorted(tree[2 * node], tree[2 * node + 1]);
    }

    int[] mergeSorted(int[] a, int[] b) {
        int[] res = new int[a.length + b.length];
        int i = 0, j = 0, k = 0;
        while (i < a.length && j < b.length)
            res[k++] = (a[i] <= b[j]) ? a[i++] : b[j++];
        while (i < a.length) res[k++] = a[i++];
        while (j < b.length) res[k++] = b[j++];
        return res;
    }

    // Count elements in [ql, qr] that are <= k
    int query(int node, int l, int r, int ql, int qr, int k) {
        if (qr < l || r < ql) return 0;
        if (ql <= l && r <= qr) {
            // Binary search: count of elements <= k
            return upperBound(tree[node], k);
        }
        int mid = (l + r) / 2;
        return query(2 * node, l, mid, ql, qr, k) +
               query(2 * node + 1, mid + 1, r, ql, qr, k);
    }

    // Returns index of first element > k (i.e., count of elements <= k)
    int upperBound(int[] arr, int k) {
        int lo = 0, hi = arr.length;
        while (lo < hi) {
            int mid = (lo + hi) / 2;
            if (arr[mid] <= k) lo = mid + 1;
            else hi = mid;
        }
        return lo;
    }

    int countLessOrEqual(int ql, int qr, int k) {
        return query(1, 0, n - 1, ql, qr, k);
    }

    // Kth smallest in [ql, qr] using binary search on answer
    int kthSmallest(int ql, int qr, int k) {
        int lo = tree[1][0];
        int hi = tree[1][tree[1].length - 1];
        int ans = lo;
        while (lo <= hi) {
            int mid = lo + (hi - lo) / 2;
            if (countLessOrEqual(ql, qr, mid) >= k) {
                ans = mid;
                hi = mid - 1;
            } else {
                lo = mid + 1;
            }
        }
        return ans;
    }

    public static void main(String[] args) {
        int[] arr = {3, 1, 4, 1, 5, 9, 2, 6};
        MergeSortTree mst = new MergeSortTree(arr);

        // Count elements <= 4 in range [1, 6]
        System.out.println(mst.countLessOrEqual(1, 6, 4));  // 4

        // 3rd smallest in range [0, 7]
        System.out.println(mst.kthSmallest(0, 7, 3));  // 2
    }
}

from bisect import bisect_right

class MergeSortTree:
    def __init__(self, arr):
        self.n = len(arr)
        self.tree = [[] for _ in range(4 * self.n)]
        self._build(arr, 1, 0, self.n - 1)

    def _build(self, arr, node, l, r):
        if l == r:
            self.tree[node] = [arr[l]]
            return
        mid = (l + r) // 2
        self._build(arr, 2 * node, l, mid)
        self._build(arr, 2 * node + 1, mid + 1, r)
        # Merge two sorted lists
        left = self.tree[2 * node]
        right = self.tree[2 * node + 1]
        merged = []
        i = j = 0
        while i < len(left) and j < len(right):
            if left[i] <= right[j]:
                merged.append(left[i]); i += 1
            else:
                merged.append(right[j]); j += 1
        merged.extend(left[i:])
        merged.extend(right[j:])
        self.tree[node] = merged

    def _query(self, node, l, r, ql, qr, k):
        if qr < l or r < ql:
            return 0
        if ql <= l and r <= qr:
            # bisect_right gives count of elements <= k
            return bisect_right(self.tree[node], k)
        mid = (l + r) // 2
        return (self._query(2 * node, l, mid, ql, qr, k) +
                self._query(2 * node + 1, mid + 1, r, ql, qr, k))

    def count_less_or_equal(self, ql, qr, k):
        return self._query(1, 0, self.n - 1, ql, qr, k)

    def kth_smallest(self, ql, qr, k):
        """Find kth smallest element in arr[ql..qr]"""
        lo = self.tree[1][0]
        hi = self.tree[1][-1]
        ans = lo
        while lo <= hi:
            mid = (lo + hi) // 2
            if self.count_less_or_equal(ql, qr, mid) >= k:
                ans = mid
                hi = mid - 1
            else:
                lo = mid + 1
        return ans

Comparison with Persistent Segment Tree

Both Merge Sort Trees and Persistent Segment Trees can answer order-statistic queries on ranges. Here is how they compare:

AspectMerge Sort TreePersistent Segment Tree
Build TimeO(n log n)O(n log n)
Build SpaceO(n log n)O(n log n)
Count ≤ k in [l,r]O(log2 n)O(log n)
Kth smallest in [l,r]O(log3 n)O(log n)
ImplementationSimple (arrays + binary search)Complex (pointer-based nodes)
Point UpdatesNot efficientO(log2 n) with persistence
Memory LocalityGood (contiguous arrays)Poor (scattered nodes)
Constant FactorSmallLarge (pointer overhead)
When to use which:
  • Merge Sort Tree: When implementation simplicity matters, or when the log2 n factor is acceptable for the constraints (n, q ≤ 105)
  • Persistent Segment Tree: When you need O(log n) per query, need updates, or constraints are tight (n, q ≤ 106)

Applications

1. Kth Smallest in a Range (Binary Search on Answer)

Binary search on the answer value. For a candidate value mid, count how many elements in [l, r] are ≤ mid. If that count is ≥ k, the answer is at most mid; otherwise it is larger.

kthSmallest(l, r, k):
  lo = min(arr), hi = max(arr)
  while lo <= hi:
    mid = (lo + hi) / 2
    if countLessOrEqual(l, r, mid) >= k:
      ans = mid
      hi = mid - 1          // try smaller
    else:
      lo = mid + 1          // need larger
  return ans

Time: O(log(max_val) * log^2(n)) = O(log^3 n) for values in [1, n]

2. Count Elements in Value Range [a, b] within Index Range [l, r]

Count elements with value in [a, b] within positions [l, r]:

countInRange(l, r, a, b):
  return countLessOrEqual(l, r, b) - countLessOrEqual(l, r, a - 1)

Example: count elements in [2, 5] within arr[1..6]
  Array: [3, 1, 4, 1, 5, 9, 2, 6]
  countLessOrEqual(1, 6, 5) = 5  (elements: 1,4,1,5,2)
  countLessOrEqual(1, 6, 1) = 2  (elements: 1,1)
  Answer = 5 - 2 = 3             (elements: 4,5,2)

3. Other Applications

  • Number of inversions in a range: for each element at position i in [l, r], count elements in [l, i-1] that are greater
  • Count of distinct elements ≤ k: combine with coordinate compression
  • Offline queries with sorting: process queries sorted by k to avoid repeated binary searches
Key Takeaways:
  • Merge Sort Tree trades query speed (log2 n vs log n) for implementation simplicity
  • Best suited for problems with moderate constraints where a Persistent Segment Tree would be overkill
  • The "binary search on answer + count ≤ k" pattern is extremely versatile
  • No efficient point updates -- use offline processing or a Persistent Segment Tree if updates are needed
