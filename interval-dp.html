Interval DP (Range DP)

Solve problems on contiguous intervals [i, j] by trying all possible split points k. Build solutions for larger intervals from smaller ones.

Core Concept

Interval DP pattern:
  dp[i][j] = optimal answer for the subproblem on interval [i, j]

For every split point k where i ≤ k < j:
  dp[i][j] = optimize over  dp[i][k] + dp[k+1][j] + merge_cost(i, k, j)

Fill order: by increasing interval LENGTH (len = j - i + 1)
  len=1: dp[i][i] = base case
  len=2: dp[i][i+1] = ...
  len=3: dp[i][i+2] = try k=i, k=i+1
  ...
  len=n: dp[0][n-1] = final answer

       [0 ─────────────── n-1]        ← final answer
      /          |            \
  [0──k]    [0──k]  [k+1──n-1]   ... try all k

Problem 1: Matrix Chain Multiplication

Given matrices A1, A2, ..., An with dimensions p[0]×p[1], p[1]×p[2], ..., p[n-1]×p[n], find the minimum number of scalar multiplications to compute the product A1*A2*...*An.

Example: dimensions p = [10, 30, 5, 60]
  A1: 10x30,  A2: 30x5,  A3: 5x60

  Option 1: (A1*A2)*A3
    A1*A2: 10*30*5 = 1500 ops → result is 10x5
    (A1*A2)*A3: 10*5*60 = 3000 ops
    Total: 4500

  Option 2: A1*(A2*A3)
    A2*A3: 30*5*60 = 9000 ops → result is 30x60
    A1*(A2*A3): 10*30*60 = 18000 ops
    Total: 27000

  Optimal: Option 1 with 4500 multiplications
  Parenthesization matters enormously!

State & Transition (MCM)

State: dp[i][j] = minimum cost to multiply matrices i through j

Transition: Split at k (multiply [i..k] and [k+1..j], then combine):
dp[i][j] = min over k in [i, j-1] of dp[i][k] + dp[k+1][j] + p[i-1]*p[k]*p[j]

Base Case: dp[i][i] = 0 (single matrix, no multiplication needed)

Answer: dp[1][n]

p = [10, 30, 5, 60],  Matrices: M1(10x30), M2(30x5), M3(5x60)

         j=1    j=2    j=3
  i=1  [  0   1500   4500 ]    ← dp[1][3] = answer
  i=2  [  -     0    9000 ]
  i=3  [  -     -      0  ]

Fill order:
  Length 1: dp[1][1]=0, dp[2][2]=0, dp[3][3]=0
  Length 2: dp[1][2] = p[0]*p[1]*p[2] = 10*30*5 = 1500
            dp[2][3] = p[1]*p[2]*p[3] = 30*5*60 = 9000
  Length 3: dp[1][3] = min(
              k=1: dp[1][1] + dp[2][3] + p[0]*p[1]*p[3] = 0+9000+18000 = 27000
              k=2: dp[1][2] + dp[3][3] + p[0]*p[2]*p[3] = 1500+0+3000 = 4500 ✓
            ) = 4500

Code: Matrix Chain Multiplication

// Matrix Chain Multiplication
// Time: O(n^3), Space: O(n^2)
int matrixChain(vector<int>& p) {
    int n = p.size() - 1; // number of matrices
    vector<vector<int>> dp(n + 1, vector<int>(n + 1, 0));

    // Fill by increasing interval length
    for (int len = 2; len <= n; len++) {
        for (int i = 1; i <= n - len + 1; i++) {
            int j = i + len - 1;
            dp[i][j] = INT_MAX;

            // Try all split points
            for (int k = i; k < j; k++) {
                int cost = dp[i][k] + dp[k+1][j]
                           + p[i-1] * p[k] * p[j];
                dp[i][j] = min(dp[i][j], cost);
            }
        }
    }
    return dp[1][n];
}

# Matrix Chain Multiplication
# Time: O(n^3), Space: O(n^2)
def matrix_chain(p):
    n = len(p) - 1  # number of matrices
    dp = [[0] * (n + 1) for _ in range(n + 1)]

    # Fill by increasing interval length
    for length in range(2, n + 1):
        for i in range(1, n - length + 2):
            j = i + length - 1
            dp[i][j] = float('inf')

            for k in range(i, j):
                cost = (dp[i][k] + dp[k+1][j]
                        + p[i-1] * p[k] * p[j])
                dp[i][j] = min(dp[i][j], cost)

    return dp[1][n]

Problem 2: Burst Balloons

Given n balloons with values nums[0..n-1]. Bursting balloon i gives nums[i-1] * nums[i] * nums[i+1] coins. Find the maximum coins you can collect by bursting all balloons. (Boundary values are 1.)

Example: nums = [3, 1, 5, 8]  (with boundary 1s: [1, 3, 1, 5, 8, 1])

Key Insight: Think of which balloon to burst LAST in interval [i,j].
If balloon k is burst last in [i,j]:
  - All balloons in [i,k-1] and [k+1,j] are already gone
  - When k is burst, its neighbors are the boundaries: nums[i-1] and nums[j+1]
  - Cost = nums[i-1] * nums[k] * nums[j+1]

dp[i][j] = max coins from bursting all balloons in (i, j) exclusive
         = max over k in (i,j) of:
           dp[i][k] + dp[k][j] + nums[i]*nums[k]*nums[j]

Answer: dp[0][n+1] where nums is padded with 1s

Code: Burst Balloons

// Burst Balloons - which balloon to burst LAST
// Time: O(n^3), Space: O(n^2)
int maxCoins(vector<int>& nums) {
    int n = nums.size();
    // Pad with boundary 1s
    vector<int> a(1, 1);
    a.insert(a.end(), nums.begin(), nums.end());
    a.push_back(1);  // a = [1, nums..., 1]

    int m = a.size(); // n + 2
    vector<vector<int>> dp(m, vector<int>(m, 0));

    // len is the gap between i and j (exclusive endpoints)
    for (int len = 2; len < m; len++) {
        for (int i = 0; i + len < m; i++) {
            int j = i + len;
            // k is the last balloon to burst in (i, j)
            for (int k = i + 1; k < j; k++) {
                dp[i][j] = max(dp[i][j],
                    dp[i][k] + dp[k][j] + a[i] * a[k] * a[j]);
            }
        }
    }
    return dp[0][m - 1];
}

// Burst Balloons - which balloon to burst LAST
// Time: O(n^3), Space: O(n^2)
static int maxCoins(int[] nums) {
    int n = nums.length;
    int[] a = new int[n + 2];
    a[0] = a[n + 1] = 1;
    for (int i = 0; i < n; i++) a[i + 1] = nums[i];

    int m = n + 2;
    int[][] dp = new int[m][m];

    for (int len = 2; len < m; len++) {
        for (int i = 0; i + len < m; i++) {
            int j = i + len;
            for (int k = i + 1; k < j; k++) {
                dp[i][j] = Math.max(dp[i][j],
                    dp[i][k] + dp[k][j] + a[i] * a[k] * a[j]);
            }
        }
    }
    return dp[0][m - 1];
}

# Burst Balloons - which balloon to burst LAST
# Time: O(n^3), Space: O(n^2)
def max_coins(nums):
    a = [1] + nums + [1]  # pad with boundary 1s
    m = len(a)
    dp = [[0] * m for _ in range(m)]

    for length in range(2, m):
        for i in range(m - length):
            j = i + length
            for k in range(i + 1, j):
                dp[i][j] = max(dp[i][j],
                    dp[i][k] + dp[k][j] + a[i] * a[k] * a[j])

    return dp[0][m - 1]

Walkthrough: Burst Balloons

Setup: nums = [3, 1, 5, 8], padded a = [1, 3, 1, 5, 8, 1]
dp[i][j] = max coins by bursting all balloons strictly between index i and j in array a.
Length 2 (gap of 2, one balloon between i and j):
dp[0][2]: burst k=1 (val 3): 1*3*1 = 3
dp[1][3]: burst k=2 (val 1): 3*1*5 = 15
dp[2][4]: burst k=3 (val 5): 1*5*8 = 40
dp[3][5]: burst k=4 (val 8): 5*8*1 = 40
Length 3 (two balloons between i and j):
dp[0][3]: max(k=1: dp[0][1]+dp[1][3]+1*3*5=0+15+15=30,
               k=2: dp[0][2]+dp[2][3]+1*1*5=3+0+5=8) = 30
dp[1][4]: max(k=2: 0+40+3*1*8=64, k=3: 15+0+3*5*8=135) = 135
dp[2][5]: max(k=3: 0+40+1*5*1=45, k=4: 40+0+1*8*1=48) = 48
Continue until dp[0][5] = 167 (the answer).

Each cell dp[i][j] depends on cells in the same row (left)
and same column (below) → already computed!

Complexity

  • Time: O(n^3) - three nested loops (length, start, split point)
  • Space: O(n^2) - the 2D dp table
  • Practical limit: n up to ~500 for O(n^3)
  • Key pattern: Always fill by increasing interval length to ensure subproblems are solved first

// General Interval DP Template
int n = ...;

// Base case: intervals of length 1
for (int i = 0; i < n; i++)
    dp[i][i] = base_value(i);

// Fill by increasing length
for (int len = 2; len <= n; len++) {
    for (int i = 0; i + len - 1 < n; i++) {
        int j = i + len - 1;
        dp[i][j] = WORST_VALUE; // INT_MAX or 0

        for (int k = i; k < j; k++) {
            dp[i][j] = optimize(dp[i][j],

        }
    }
}

Key Insights

  • Fill order is crucial: Must process smaller intervals before larger ones
  • "Last operation" trick: In Burst Balloons, think about which element is processed LAST (not first) in the interval
  • Inclusive vs exclusive: Some problems use open intervals (i,j) vs closed [i,j]. Be careful with indexing.
  • Prefix sums: If merge cost involves a range sum, precompute prefix sums for O(1) range queries
  • Knuth optimization: Some interval DPs can be sped up to O(n^2) if the optimal split point is monotone
