Wildcard & Regex Matching

Two pattern matching problems solved with 2D DP. Wildcard uses ? and *, while regex uses . and * with different semantics.

Problem 1: Wildcard Pattern Matching

Given a pattern p and a text s, implement wildcard matching where:

  • ? matches exactly one character
  • * matches any sequence of characters (including empty)
Examples:
  s = "adceb", p = "*a*b"   -> true  (* matches "", then "dce")
  s = "acdcb", p = "a*c?b"  -> false (c?b needs 3 chars, "dcb" -> c?b = c+d+b, no)
  s = "abcde", p = "a*e"    -> true  (* matches "bcd")
  s = "abc",   p = "a?c"    -> true  (? matches 'b')
  s = "",      p = "*"       -> true  (* matches empty)
  s = "",      p = "?"       -> false (? needs one char)

State Definition: Wildcard

State: dp[i][j] = whether pattern[0..i-1] matches text[0..j-1]

Transition:
If p[i-1] == '?' or p[i-1] == s[j-1]: dp[i][j] = dp[i-1][j-1]
If p[i-1] == '*': dp[i][j] = dp[i-1][j] || dp[i][j-1]
  dp[i-1][j]: * matches empty (consume * from pattern)
  dp[i][j-1]: * matches one more character (consume char from text)
Otherwise: dp[i][j] = false

Base Cases:
dp[0][0] = true (empty pattern matches empty text)
dp[i][0] = true only if pattern[0..i-1] is all '*'
dp[0][j] = false for j > 0 (empty pattern cannot match non-empty text)

Visual: How * Expands

Pattern: "a*d"   Text: "abcd"

dp table (rows = pattern, cols = text):

         ""    a    b    c    d
  ""   [ T    F    F    F    F ]
  a    [ F    T    F    F    F ]
  *    [ F    T    T    T    T ]   <- * can match "", "b", "bc", "bcd"
  d    [ F    F    F    F    T ]

For '*' row (i=2):
  dp[2][1]: dp[1][1]=T (match empty) OR dp[2][0]=F -> T
  dp[2][2]: dp[1][2]=F (match empty) OR dp[2][1]=T (extend) -> T
  dp[2][3]: dp[1][3]=F (match empty) OR dp[2][2]=T (extend) -> T
  dp[2][4]: dp[1][4]=F (match empty) OR dp[2][3]=T (extend) -> T

Key insight: dp[i][j-1] means "* already matched up to j-2,
so extend it to also match s[j-1]" - the * grows to the right!

Step-by-Step Walkthrough

Step 1: dp[0][0] = true. For the first column, dp[i][0] = true only while p[0..i-1] are all '*'. Once we hit a non-* character, the rest are false.
Step 2: For each cell (i,j), check what p[i-1] is:
- Regular char or '?': needs exact match or any-one-char match with s[j-1]
- '*': try matching empty (look up) or extending match (look left)
Step 3: The '*' propagation works left-to-right: once dp[i][j-1] is true, dp[i][j] becomes true too (the * "consumes" another character).
Step 4: Answer is dp[m][n] where m = len(pattern), n = len(text).

Code: Wildcard Matching

// Wildcard Pattern Matching
// Time: O(m*n), Space: O(m*n) or O(n) optimized
bool isMatch(string s, string p) {
    int m = p.size(), n = s.size();
    vector<vector<bool>> dp(m + 1, vector<bool>(n + 1, false));

    dp[0][0] = true;

    // First column: pattern of all '*' matches empty string
    for (int i = 1; i <= m; i++) {
        if (p[i - 1] == '*')
            dp[i][0] = dp[i - 1][0];
    }

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (p[i - 1] == '*') {
                // * matches empty (dp[i-1][j]) or one more char (dp[i][j-1])
                dp[i][j] = dp[i - 1][j] || dp[i][j - 1];
            } else if (p[i - 1] == '?' || p[i - 1] == s[j - 1]) {
                // ? matches any single char, or exact match
                dp[i][j] = dp[i - 1][j - 1];
            }
            // else: dp[i][j] stays false
        }
    }

    return dp[m][n];
}

// Space-optimized O(n) version
bool isMatchOpt(string s, string p) {
    int m = p.size(), n = s.size();
    vector<bool> dp(n + 1, false);
    dp[0] = true;

    for (int i = 1; i <= m; i++) {
        vector<bool> ndp(n + 1, false);
        ndp[0] = dp[0] && (p[i - 1] == '*');
        for (int j = 1; j <= n; j++) {
            if (p[i - 1] == '*')
                ndp[j] = dp[j] || ndp[j - 1];
            else if (p[i - 1] == '?' || p[i - 1] == s[j - 1])
                ndp[j] = dp[j - 1];
        }
        dp = ndp;
    }

    return dp[n];
}

# Wildcard Pattern Matching
# Time: O(m*n), Space: O(m*n) or O(n) optimized
def isMatch(s: str, p: str) -> bool:
    m, n = len(p), len(s)
    dp = [[False] * (n + 1) for _ in range(m + 1)]

    dp[0][0] = True

    # First column: pattern of all '*' matches empty string
    for i in range(1, m + 1):
        if p[i - 1] == '*':
            dp[i][0] = dp[i - 1][0]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if p[i - 1] == '*':
                # * matches empty OR one more char
                dp[i][j] = dp[i - 1][j] or dp[i][j - 1]
            elif p[i - 1] == '?' or p[i - 1] == s[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]

    return dp[m][n]

Problem 2: Regular Expression Matching

Given a pattern p and a text s, implement regex matching where:

  • . matches exactly one character (any character)
  • * matches zero or more of the preceding element
Key difference from Wildcard:
  Wildcard '*' = matches ANY sequence independently
  Regex 'x*'  = matches zero or more of character 'x'

Examples:
  s = "aa",  p = "a*"   -> true  (a* = zero or more 'a')
  s = "ab",  p = ".*"   -> true  (.* = zero or more of any char)
  s = "aab", p = "c*a*b" -> true (c*=empty, a*="aa", b="b")
  s = "ab",  p = "a*b"  -> true  (a*="a", b="b")
  s = "abc", p = "a.c"  -> true  (.='b')
  s = "abc", p = "a*c"  -> false (a*="a" or "aa"..., no way to match 'b')

State Definition: Regex

State: dp[i][j] = whether pattern[0..i-1] matches text[0..j-1]

Transition:
If p[i-1] == '.' or p[i-1] == s[j-1]: dp[i][j] = dp[i-1][j-1]

If p[i-1] == '*': (applies to p[i-2] + '*' as a unit)
  Case 1: Zero occurrences of p[i-2]: dp[i][j] = dp[i-2][j]
  Case 2: One or more occurrences: dp[i][j] = dp[i][j-1] if (p[i-2]=='.' or p[i-2]==s[j-1])

Otherwise: dp[i][j] = false

Base Cases:
dp[0][0] = true
dp[i][0]: true if pattern[0..i-1] can match empty (e.g., "a*b*" matches empty)
dp[0][j] = false for j > 0

Visual: Regex * Expansion

Pattern: "a*b"   Text: "aab"

         ""    a    a    b
  ""   [ T    F    F    F ]
  a    [ F    .    .    . ]     (a alone, not reached directly)
  *    [ T    T    T    F ]     a* can match "", "a", "aa"
  b    [ F    F    F    T ]

How a* row works:
  dp[2][0] = dp[0][0] = T  (zero 'a's)
  dp[2][1]: dp[0][1]=F (zero a's) OR (p[0]='a'==s[0]='a' AND dp[2][0]=T) -> T
  dp[2][2]: dp[0][2]=F (zero a's) OR (p[0]='a'==s[1]='a' AND dp[2][1]=T) -> T
  dp[2][3]: dp[0][3]=F (zero a's) OR (p[0]='a'!= s[2]='b') -> F

Key: "a*" is treated as a UNIT (2 characters in pattern).
  dp[i-2][j]: skip the entire "x*" (zero occurrences)
  dp[i][j-1]: extend "x*" by one more (if x matches s[j-1])

---

Pattern: ".*"   Text: "abc"

         ""    a    b    c
  ""   [ T    F    F    F ]
  .    [ F    .    .    . ]
  *    [ T    T    T    T ]    .* matches everything!

dp[2][j]: dp[0][j]=F (zero) OR (p[0]='.' matches ANY, dp[2][j-1]) -> T, T, T

Code: Regular Expression Matching

// Regular Expression Matching
// Time: O(m*n), Space: O(m*n)
bool isMatch(string s, string p) {
    int m = p.size(), n = s.size();
    vector<vector<bool>> dp(m + 1, vector<bool>(n + 1, false));

    dp[0][0] = true;

    // Base case: patterns like a*, a*b*, a*b*c* can match empty
    for (int i = 2; i <= m; i += 2) {
        if (p[i - 1] == '*')
            dp[i][0] = dp[i - 2][0];
    }

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (p[i - 1] == '*') {
                // Zero occurrences of preceding element
                dp[i][j] = dp[i - 2][j];
                // One or more: preceding char matches current text char
                if (p[i - 2] == '.' || p[i - 2] == s[j - 1])
                    dp[i][j] = dp[i][j] || dp[i][j - 1];
            } else if (p[i - 1] == '.' || p[i - 1] == s[j - 1]) {
                // Exact match or . wildcard
                dp[i][j] = dp[i - 1][j - 1];
            }
        }
    }

    return dp[m][n];
}

# Time: O(m*n), Space: O(m*n)
def isMatch(s: str, p: str) -> bool:
    m, n = len(p), len(s)
    dp = [[False] * (n + 1) for _ in range(m + 1)]

    dp[0][0] = True

    # Base case: patterns like a*, a*b* can match empty
    for i in range(2, m + 1, 2):
        if p[i - 1] == '*':
            dp[i][0] = dp[i - 2][0]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if p[i - 1] == '*':
                # Zero occurrences of preceding element
                dp[i][j] = dp[i - 2][j]
                # One or more occurrences
                if p[i - 2] == '.' or p[i - 2] == s[j - 1]:
                    dp[i][j] = dp[i][j] or dp[i][j - 1]
            elif p[i - 1] == '.' or p[i - 1] == s[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]

    return dp[m][n]

Complexity Analysis

  • Time: O(m * n) for both problems where m = pattern length, n = text length
  • Space: O(m * n) for the DP table, optimizable to O(n) using rolling array
  • Wildcard space optimization: Straightforward since each row only depends on previous row and current row (left neighbor)
  • Regex space optimization: Slightly trickier because * needs dp[i-2][j], so keep two previous rows

Key Insights

Wildcard *: Think of it as a "greedy consumer" that can eat any number of characters. The dp[i][j-1] transition means "I already started matching with *, let me eat one more character."

Regex *: Always think in pairs (x*). The transition dp[i-2][j] = "pretend x* doesn't exist" and dp[i][j-1] = "use x* to consume one more matching character."

Common mistake: In regex, '*' never appears at position 0 (it always follows another character). Make sure your loop handles this correctly.

Edge Cases

Wildcard:
  p = "*"   -> matches everything
  p = "?"   -> matches exactly one char
  p = "***" -> same as single "*" (can collapse consecutive *'s)
  p = ""    -> matches only empty string

Regex:
  p = ".*"  -> matches everything (. matches any, * repeats)
  p = "a*"  -> matches "", "a", "aa", "aaa", ...
  p = ".*.*" -> same as ".*" (redundant)
  p = "a*a" -> matches "a", "aa", "aaa", ... (a* + literal a)
