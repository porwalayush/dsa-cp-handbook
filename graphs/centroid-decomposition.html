Centroid Decomposition

Centroid Decomposition is a divide-and-conquer technique on trees. It recursively finds the centroid (a node whose removal splits the tree into subtrees each of size at most n/2), processes paths through it, then recurses on the remaining subtrees. The resulting "centroid tree" has height O(log n).

What is a Centroid?

A centroid of a tree is a node whose removal results in no remaining subtree having more than n/2 nodes. Every tree has at least one centroid (and at most two).

Visual Walkthrough

    Original Tree (7 nodes):

          1
         / \
        2   3
       / \   \
      4   5   6
      |
      7

    Finding centroid:
    Subtree sizes from node 1: [7, 4, 2, 1, 1, 1, 1]
    Node 1: max subtree after removal = max(4, 2) = 4 > 7/2=3.5
    Node 2: max subtree after removal = max(3, 2, 1) = 3 <= 3.5
    Centroid = 2!

    Decomposition Process:
    ─────────────────────
    Step 1: Centroid of full tree = 2
            Process all paths through 2
            Remove node 2, get subtrees: {4,7}, {5}, {1,3,6}

    Step 2: Centroid of {4,7} = 4
            Centroid of {5} = 5
            Centroid of {1,3,6} = 1 (or 3)

    Step 3: Continue recursively...

    Centroid Tree:
              2
           /  |  \
          4   5   1
          |       |
          7       3
                  |
                  6

    Height of centroid tree: O(log n)
    Every path in original tree passes through
    some ancestor in centroid tree!

Algorithm Steps

Step 1: Compute subtree sizes using DFS.
Step 2: Find the centroid: a node where max(subtree sizes after removal) is at most n/2.
Step 3: Process all paths that pass through the centroid (problem-specific computation).
Step 4: Mark centroid as removed. Recurse on each remaining subtree.

Why O(log n) Height?

Each time we pick a centroid, the remaining subtrees have at most n/2 nodes. So the depth of recursion is at most log2(n). This means every node appears in at most O(log n) levels of the decomposition, giving us O(n log n) total work.

// Example: Count paths of length exactly k
int ans = 0;
int k;
vector<int> paths; // distances from centroid

void getDists(int v, int p, int dist) {
    paths.push_back(dist);
    for (int u : adj[v]) {
        if (u != p && !removed[u])
            getDists(u, v, dist + 1);
    }
}

void solve(int v) {
    int sz = getSize(v, -1);
    int centroid = getCentroid(v, -1, sz);
    removed[centroid] = true;

    // Count paths through centroid of length k
    vector<int> all_dists;
    all_dists.push_back(0); // centroid itself

    for (int u : adj[centroid]) {
        if (removed[u]) continue;
        paths.clear();
        getDists(u, centroid, 1);

        // Check pairs between current subtree and previous
        for (int d : paths) {
            int need = k - d;
            if (need >= 0) {
                ans += count(all_dists.begin(), all_dists.end(), need);
            }
        }
        for (int d : paths)
            all_dists.push_back(d);
    }

    for (int u : adj[centroid]) {
        if (!removed[u])
            solve(u);
    }
}

class CentroidDecomposition:
    def __init__(self, n, adj):
        self.n = n
        self.adj = adj
        self.subtree_size = [0] * (n + 1)
        self.removed = [False] * (n + 1)
        self.centroid_parent = [-1] * (n + 1)

    def get_size(self, v, p):
        self.subtree_size[v] = 1
        for u in self.adj[v]:
            if u != p and not self.removed[u]:
                self.subtree_size[v] += self.get_size(u, v)
        return self.subtree_size[v]

    def get_centroid(self, v, p, tree_size):
        for u in self.adj[v]:
            if u != p and not self.removed[u]:
                if self.subtree_size[u] > tree_size // 2:
                    return self.get_centroid(u, v, tree_size)
        return v

    def decompose(self, v, parent=-1):
        sz = self.get_size(v, -1)
        centroid = self.get_centroid(v, -1, sz)

        self.removed[centroid] = True
        self.centroid_parent[centroid] = parent

        # Process paths through centroid (problem-specific)
        self.process(centroid)

        # Recurse on remaining subtrees
        for u in self.adj[centroid]:
            if not self.removed[u]:
                self.decompose(u, centroid)

        """Override for problem-specific logic"""
        pass

# Example: Count paths of length exactly k
class PathCounter(CentroidDecomposition):
    def __init__(self, n, adj, k):
        super().__init__(n, adj)
        self.k = k
        self.answer = 0

    def get_dists(self, v, p, dist, dists):
        dists.append(dist)
        for u in self.adj[v]:
            if u != p and not self.removed[u]:
                self.get_dists(u, v, dist + 1, dists)

    def process(self, centroid):
        all_dists = [0]  # centroid itself at distance 0
        for u in self.adj[centroid]:
            if self.removed[u]:
                continue
            dists = []
            self.get_dists(u, centroid, 1, dists)
            # Count pairs from this subtree + previous subtrees
            for d in dists:
                need = self.k - d
                if need >= 0:
                    self.answer += all_dists.count(need)
            all_dists.extend(dists)

# Example usage
n = 7
adj = [[] for _ in range(n + 1)]
edges = [(1,2),(1,3),(2,4),(2,5),(3,6),(4,7)]
for u, v in edges:
    adj[u].append(v)
    adj[v].append(u)

counter = PathCounter(n, adj, k=3)
counter.decompose(1)
print(f"Paths of length 3: {counter.answer}")

Applications

  • Count/find paths of specific length: For each centroid, compute distances to all nodes in its subtree and find pairs that sum to target.
  • Distance queries with updates: Maintain distances to centroid ancestors; update/query in O(log n).
  • Closest marked node: For each centroid ancestor, maintain minimum distance to a marked node.
  • Path with specific property: Paths passing through centroid can be composed from two half-paths.
  • xor/sum on paths: Decompose path queries through centroid hierarchy.

Complexity Analysis

  • Decomposition time: O(n log n). Each node participates in O(log n) levels.
  • Centroid tree height: O(log n). Guaranteed because each subtree has at most n/2 nodes.
  • Space: O(n) for the decomposition structure itself.
  • Per-query (distance queries): O(log n) by traversing centroid ancestors.
Key insight: Any path in the original tree passes through exactly one node in the centroid tree that is an ancestor of both endpoints. This means we can decompose any path problem into O(log n) subproblems, one per level of the centroid tree.

Key Takeaways

  1. Centroid decomposition is divide-and-conquer on trees.
  2. The centroid tree has O(log n) height, enabling efficient path queries.
  3. Every path passes through some centroid ancestor of its endpoints.
  4. Common pattern: for each centroid, aggregate information from different subtrees to count/compute paths through it.
  5. Works well when combined with other data structures (BIT, segment tree) for online queries.
