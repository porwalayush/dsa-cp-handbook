Advanced DP Optimizations

Techniques to speed up DP transitions from O(n^2) or O(n^3) to O(n log n) or O(n). These apply when the DP recurrence has special structure that can be exploited.

Overview of Techniques

TechniqueReduces FromReduces ToCondition
Divide & ConquerO(kn^2)O(kn log n)opt(i) ≤ opt(i+1)
Knuth's OptimizationO(n^3)O(n^2)Quadrangle inequality
Convex Hull TrickO(n^2)O(n) or O(n log n)Linear transition form
Monotone QueueO(n*W)O(n)Sliding window constraint

1. Divide and Conquer Optimization

Applies when computing dp[k][i] = min over j<i of (dp[k-1][j] + cost(j+1, i)) and the optimal split point opt(i) is monotone: opt(i) ≤ opt(i+1).

Standard DP: dp[k][i] = min { dp[k-1][j] + cost(j+1, i) } for j < i
Naive: O(kn^2) - for each of kn states, try all n splits

Monotonicity condition:
  Let opt(i) = the j that minimizes dp[k-1][j] + cost(j+1, i)
  If opt(i) <= opt(i+1) for all i, we can use D&C!

Divide & Conquer idea:
  To compute dp[k][lo..hi]:
    1. Find opt(mid) by trying all valid j values
    2. Recurse on dp[k][lo..mid-1] with j in [jlo..opt(mid)]
    3. Recurse on dp[k][mid+1..hi] with j in [opt(mid)..jhi]

  Each level of recursion does O(n) total work
  Depth = O(log n)
  Total: O(kn log n)

Example problem: Divide n items into k groups to minimize total cost.

When Does D&C Optimization Apply?

Condition: The cost function satisfies the "monotone minima" property. This holds when:
- cost(a, c) + cost(b, d) ≤ cost(a, d) + cost(b, c) for a ≤ b ≤ c ≤ d
- This is the "inverse quadrangle inequality" or "concave SMAWK" condition

Common examples:
- cost(i, j) = (prefix_sum[j] - prefix_sum[i])^2
- cost(i, j) = sum of squared deviations in range [i, j]

How to verify: If cost satisfies the quadrangle inequality, then opt(i) is monotone.

Code: Divide and Conquer Optimization

// Divide and Conquer DP Optimization
// dp[k][i] = min { dp[k-1][j] + cost(j+1, i) } for j < i
// Time: O(k * n * log n)

const int INF = 1e18;
vector<vector<long long>> dp;
// cost(l, r) = cost of putting elements l..r in one group

long long cost(int l, int r); // problem-specific

void solve(int k, int lo, int hi, int jlo, int jhi) {
    if (lo > hi) return;

    int mid = (lo + hi) / 2;
    int opt = jlo;
    dp[k][mid] = INF;

    // Find optimal split point for mid
    for (int j = jlo; j <= min(mid - 1, jhi); j++) {
        long long val = dp[k - 1][j] + cost(j + 1, mid);
        if (val < dp[k][mid]) {
            dp[k][mid] = val;
            opt = j;
        }
    }

    // Recurse: left half uses j in [jlo, opt]
    //          right half uses j in [opt, jhi]
    solve(k, lo, mid - 1, jlo, opt);
    solve(k, mid + 1, hi, opt, jhi);
}

// Usage:
void computeDP(int n, int K) {
    dp.assign(K + 1, vector<long long>(n + 1, INF));
    dp[0][0] = 0;

    for (int k = 1; k <= K; k++) {
        solve(k, 1, n, 0, n - 1);
    }
    // Answer: dp[K][n]
}

2. Knuth's Optimization

Applies to interval DP of the form dp[i][j] = min over i≤k<j of (dp[i][k] + dp[k+1][j] + cost(i,j)) when the cost satisfies the quadrangle inequality.

Standard interval DP:
  dp[i][j] = min { dp[i][k] + dp[k+1][j] + cost(i,j) } for i <= k < j
  Naive: O(n^3)

Knuth's condition (quadrangle inequality):
  cost(a,c) + cost(b,d) <= cost(a,d) + cost(b,c)  for a <= b <= c <= d
  AND cost is monotone: cost(a,d) >= cost(b,c) for a <= b <= c <= d

If these hold, then:
  opt[i][j-1] <= opt[i][j] <= opt[i+1][j]

This bounds the search range for k, reducing O(n^3) to O(n^2)!

Proof sketch: Over all (i,j), total work =
  sum of (opt[i+1][j] - opt[i][j-1] + 1) which telescopes to O(n^2)

Example problems:
  - Optimal Binary Search Tree
  - Minimum cost to merge stones
  - Breaking a stick optimally

Code: Knuth's Optimization

// Knuth's Optimization
// dp[i][j] = min { dp[i][k] + dp[k+1][j] + cost(i,j) }
// Time: O(n^2) instead of O(n^3)

const int MAXN = 5005;
long long dp[MAXN][MAXN];
int opt[MAXN][MAXN]; // optimal split point
long long prefix[MAXN]; // for computing cost(i,j)

// cost(i,j) = prefix[j] - prefix[i-1] (example: sum of range)
long long cost(int i, int j) {
    return prefix[j] - prefix[i - 1];
}

void knuthDP(int n) {
    // Base case: single elements
    for (int i = 1; i <= n; i++) {
        dp[i][i] = 0;
        opt[i][i] = i;
    }

    // Fill by increasing interval length
    for (int len = 2; len <= n; len++) {
        for (int i = 1; i + len - 1 <= n; i++) {
            int j = i + len - 1;
            dp[i][j] = 1e18;

            // Key: only search k in [opt[i][j-1], opt[i+1][j]]
            int lo = opt[i][j - 1];
            int hi = (i + 1 <= j) ? opt[i + 1][j] : j;

            for (int k = lo; k <= min(hi, j - 1); k++) {
                long long val = dp[i][k] + dp[k + 1][j] + cost(i, j);
                if (val < dp[i][j]) {
                    dp[i][j] = val;
                    opt[i][j] = k;
                }
            }
        }
    }
    // Answer: dp[1][n]
}

# Time: O(n^2) instead of O(n^3)
def knuth_dp(n, prefix):
    INF = float('inf')
    dp = [[0] * (n + 1) for _ in range(n + 1)]
    opt = [[0] * (n + 1) for _ in range(n + 1)]

    def cost(i, j):
        return prefix[j] - prefix[i - 1]

    # Base case
    for i in range(1, n + 1):
        dp[i][i] = 0
        opt[i][i] = i

    # Fill by increasing length
    for length in range(2, n + 1):
        for i in range(1, n - length + 2):
            j = i + length - 1
            dp[i][j] = INF

            lo = opt[i][j - 1]
            hi = opt[i + 1][j] if i + 1 <= j else j

            for k in range(lo, min(hi, j - 1) + 1):
                val = dp[i][k] + dp[k + 1][j] + cost(i, j)
                if val < dp[i][j]:
                    dp[i][j] = val
                    opt[i][j] = k

    return dp[1][n]

3. Convex Hull Trick (CHT)

Applies when the DP transition has the form dp[i] = min/max over j of (dp[j] + b[j] * a[i]) where the "lines" dp[j] + b[j]*x can be maintained on a convex hull.

Standard form:
  dp[i] = min { dp[j] + b[j] * a[i] + c[i] }  for j < i
         = min { (b[j] * a[i] + dp[j]) } + c[i]

Think of it as:  dp[i] = min over lines y = b[j]*x + dp[j]
                 evaluated at x = a[i], then add c[i]

Each j defines a LINE: slope = b[j], intercept = dp[j]
Query: find the line with minimum value at x = a[i]

Convex Hull Trick:
  Maintain lines on lower convex hull
  Query in O(1) if x values are monotone (pointer trick)
  Query in O(log n) otherwise (binary search)

Example: dp[i] = min { dp[j] + (prefix[i] - prefix[j])^2 }
  Expand: dp[j] + prefix[i]^2 - 2*prefix[i]*prefix[j] + prefix[j]^2
  = (dp[j] + prefix[j]^2) + (-2*prefix[j]) * prefix[i] + prefix[i]^2
  = intercept[j] + slope[j] * x_i + constant_i

  slope = -2*prefix[j], intercept = dp[j] + prefix[j]^2
  Query at x = prefix[i]

When Does CHT Apply?

Requirements:
1. Transition can be rewritten as: dp[i] = min/max of (m_j * x_i + b_j) + stuff independent of j
2. Where m_j (slopes) depends only on j, and x_i depends only on i

Variants:
- If slopes are monotone (sorted): Use deque, O(n) total (amortized O(1) per query)
- If query x values are monotone: Use pointer on hull, O(n) total
- General case: Use Li Chao tree or binary search on hull, O(n log n)

Common patterns:
- dp[i] = min(dp[j] + (a[i] - b[j])^2) -- expand the square
- dp[i] = min(dp[j] + cost[j] * weight[i]) -- already in linear form

Code: Convex Hull Trick

// Convex Hull Trick (minimum query, slopes decreasing)
// Time: O(n) amortized with monotone queries
struct Line {
    long long m, b; // y = m*x + b
    long long eval(long long x) { return m * x + b; }
};

struct ConvexHullTrick {
    deque<Line> hull;

    // Check if line b is unnecessary (dominated by a and c)
    bool bad(Line a, Line b, Line c) {
        // Intersection of a,c is to the left of intersection of a,b
        return (__int128)(c.b - a.b) * (a.m - b.m)
            <= (__int128)(b.b - a.b) * (a.m - c.m);
    }

    // Add line with slope m, intercept b
    // Requirement: slopes added in decreasing order for min query
    void addLine(long long m, long long b) {
        Line l = {m, b};
        while (hull.size() >= 2 && bad(hull[hull.size()-2], hull.back(), l))
            hull.pop_back();
        hull.push_back(l);
    }

    // Query minimum at x (x values must be increasing)
    long long query(long long x) {
        while (hull.size() >= 2 && hull[0].eval(x) >= hull[1].eval(x))
            hull.pop_front();
        return hull.front().eval(x);
    }
};

// Example: dp[i] = min { dp[j] + (prefix[i] - prefix[j])^2 }
void solve(vector<long long>& prefix, int n) {
    vector<long long> dp(n + 1);
    ConvexHullTrick cht;

    // Line for j: slope = -2*prefix[j], intercept = dp[j] + prefix[j]^2
    cht.addLine(-2 * prefix[0], dp[0] + prefix[0] * prefix[0]);

    for (int i = 1; i <= n; i++) {
        // Query at x = prefix[i]
        dp[i] = cht.query(prefix[i]) + prefix[i] * prefix[i];
        // Add line for i
        cht.addLine(-2 * prefix[i], dp[i] + prefix[i] * prefix[i]);
    }
}

// Convex Hull Trick (minimum query)
// Time: O(n) with monotone slopes and queries
static long[] hullM, hullB; // slopes and intercepts
static int head, tail;

static long eval(int idx, long x) {
    return hullM[idx] * x + hullB[idx];
}

static boolean bad(int a, int b, long m, long bNew) {
    // Check if line b is dominated by lines a and new
    return (double)(bNew - hullB[a]) * (hullM[a] - hullM[b])
        <= (double)(hullB[b] - hullB[a]) * (hullM[a] - m);
}

static void addLine(long m, long b) {
    while (tail - head >= 2 && bad(tail - 2, tail - 1, m, b))
        tail--;
    hullM[tail] = m;
    hullB[tail] = b;
    tail++;
}

static long query(long x) {
    while (tail - head >= 2 && eval(head, x) >= eval(head + 1, x))
        head++;
    return eval(head, x);
}

static long[] solve(long[] prefix, int n) {
    hullM = new long[n + 1];
    hullB = new long[n + 1];
    head = 0; tail = 0;
    long[] dp = new long[n + 1];

    addLine(-2 * prefix[0], dp[0] + prefix[0] * prefix[0]);

    for (int i = 1; i <= n; i++) {
        dp[i] = query(prefix[i]) + prefix[i] * prefix[i];
        addLine(-2 * prefix[i], dp[i] + prefix[i] * prefix[i]);
    }
    return dp;
}

# Convex Hull Trick (minimum query, monotone)
# Time: O(n) amortized
from collections import deque

class ConvexHullTrick:
    def __init__(self):
        self.hull = deque()  # list of (slope, intercept)

    def bad(self, a, b, c):
        # Is line b unnecessary given lines a and c?
        return (c[1] - a[1]) * (a[0] - b[0]) \
            <= (b[1] - a[1]) * (a[0] - c[0])

    def add_line(self, m, b):
        # Add line y = m*x + b (slopes must be decreasing for min)
        line = (m, b)
        while len(self.hull) >= 2 and self.bad(self.hull[-2], self.hull[-1], line):
            self.hull.pop()
        self.hull.append(line)

    def query(self, x):
        # Query minimum at x (x must be increasing)
        while len(self.hull) >= 2:
            m0, b0 = self.hull[0]
            m1, b1 = self.hull[1]
            if m0 * x + b0 >= m1 * x + b1:
                self.hull.popleft()
            else:
                break
        m, b = self.hull[0]
        return m * x + b

# Example: dp[i] = min { dp[j] + (prefix[i] - prefix[j])^2 }
def solve(prefix, n):
    dp = [0] * (n + 1)
    cht = ConvexHullTrick()
    cht.add_line(-2 * prefix[0], dp[0] + prefix[0] ** 2)

    for i in range(1, n + 1):
        dp[i] = cht.query(prefix[i]) + prefix[i] ** 2
        cht.add_line(-2 * prefix[i], dp[i] + prefix[i] ** 2)

    return dp[n]

4. Monotone Queue (Deque) Optimization

Applies when the DP transition looks at a sliding window: dp[i] = min/max over j in [i-W, i-1] of (dp[j] + something).

Standard form:
  dp[i] = min { dp[j] + cost(i) }  for  i-W <= j <= i-1

Without optimization: O(n*W) - check W candidates for each i
With monotone deque:  O(n)   - each element enters/leaves deque once

How it works:
  Maintain a deque of candidate j values
  Front of deque = current best j for the minimum

  For each i:
    1. Remove expired indices from front (j < i-W)
    2. Answer: dp[i] = dp[deque.front()] + cost(i)
    3. Remove from back while dp[j] >= dp[i] (they will never be optimal)
    4. Push i to back

Example: Maximum sum subarray of length at most W
  dp[i] = prefix[i] - min(prefix[j]) for j in [i-W, i-1]
  Use deque to maintain minimum of prefix sums in sliding window

Example: Jump Game with limited jump length
  dp[i] = min(dp[j]) + cost[i]  for  max(0, i-K) <= j < i

Code: Monotone Queue Optimization

// Monotone Queue DP Optimization
// dp[i] = min { dp[j] } + cost[i]  for j in [i-W, i-1]
// Time: O(n), Space: O(n)

vector<int> solve(vector<int>& cost, int n, int W) {
    vector<int> dp(n, 0);
    deque<int> dq; // stores indices, front = index of min dp value

    dp[0] = cost[0];
    dq.push_back(0);

    for (int i = 1; i < n; i++) {
        // Remove expired indices (outside window)
        while (!dq.empty() && dq.front() < i - W)
            dq.pop_front();

        // Compute dp[i] using front of deque (minimum in window)
        dp[i] = dp[dq.front()] + cost[i];

        // Maintain monotonicity: remove from back if dp[back] >= dp[i]
        while (!dq.empty() && dp[dq.back()] >= dp[i])
            dq.pop_back();

        dq.push_back(i);
    }

    return dp;
}

// Maximum subarray sum of length at most W
long long maxSubarrayW(vector<int>& arr, int W) {
    int n = arr.size();
    vector<long long> prefix(n + 1, 0);
    for (int i = 0; i < n; i++)
        prefix[i + 1] = prefix[i] + arr[i];

    // ans = max(prefix[i] - prefix[j]) for i-W <= j < i
    long long ans = LLONG_MIN;
    deque<int> dq; // min deque for prefix values
    dq.push_back(0);

    for (int i = 1; i <= n; i++) {
        // Remove expired
        while (!dq.empty() && dq.front() < i - W)
            dq.pop_front();

        // Best = prefix[i] - min prefix in window
        ans = max(ans, prefix[i] - prefix[dq.front()]);

        // Maintain min deque
        while (!dq.empty() && prefix[dq.back()] >= prefix[i])
            dq.pop_back();
        dq.push_back(i);
    }

    return ans;
}

// Monotone Queue DP Optimization
// Time: O(n), Space: O(n)
static int[] solve(int[] cost, int n, int W) {
    int[] dp = new int[n];
    Deque<Integer> dq = new ArrayDeque<>();

    dp[0] = cost[0];
    dq.addLast(0);

    for (int i = 1; i < n; i++) {
        // Remove expired indices
        while (!dq.isEmpty() && dq.peekFirst() < i - W)
            dq.pollFirst();

        // dp[i] = min in window + cost[i]
        dp[i] = dp[dq.peekFirst()] + cost[i];

        // Maintain monotonicity
        while (!dq.isEmpty() && dp[dq.peekLast()] >= dp[i])
            dq.pollLast();

        dq.addLast(i);
    }

    return dp;
}

# Time: O(n), Space: O(n)
from collections import deque

def solve(cost, n, W):
    # dp[i] = min(dp[j] for j in [i-W, i-1]) + cost[i]
    dp = [0] * n
    dq = deque()  # indices, front = min dp value in window

    dp[0] = cost[0]
    dq.append(0)

    for i in range(1, n):
        # Remove expired indices
        while dq and dq[0] < i - W:
            dq.popleft()

        # Compute dp[i]
        dp[i] = dp[dq[0]] + cost[i]

        # Maintain monotonicity: remove dominated elements from back
        while dq and dp[dq[-1]] >= dp[i]:
            dq.pop()

        dq.append(i)

    return dp

# Max subarray sum with length at most W
def max_subarray_w(arr, W):
    n = len(arr)
    prefix = [0] * (n + 1)
    for i in range(n):
        prefix[i + 1] = prefix[i] + arr[i]

    ans = float('-inf')
    dq = deque([0])  # min deque for prefix

    for i in range(1, n + 1):
        while dq and dq[0] < i - W:
            dq.popleft()

        ans = max(ans, prefix[i] - prefix[dq[0]])

        while dq and prefix[dq[-1]] >= prefix[i]:
            dq.pop()
        dq.append(i)

    return ans

Summary: When to Apply Each Technique

Decision flowchart:

1. Is the transition: dp[i] = min/max over j in [i-W, i-1]?
   -> Monotone Queue (deque), O(n)

2. Can the transition be written as dp[i] = min(slope_j * x_i + intercept_j)?
   -> Convex Hull Trick
   -> O(n) if slopes and queries are monotone
   -> O(n log n) otherwise (binary search or Li Chao tree)

3. Is it interval DP: dp[i][j] = min_k(dp[i][k] + dp[k+1][j] + cost(i,j))?
   AND cost satisfies quadrangle inequality?
   -> Knuth's Optimization, O(n^2)

4. Is it: dp[k][i] = min_j(dp[k-1][j] + cost(j+1,i))?
   AND opt(i) is monotone in i?
   -> Divide & Conquer Optimization, O(kn log n)

5. None of the above? -> Check for SMAWK, Aliens trick, or lambda optimization

  • Divide & Conquer: Most general. Works whenever optimal split point is monotone. Relatively easy to implement.
  • Knuth's: Specific to interval DP. Needs both quadrangle inequality and monotonicity of cost. Tricky to verify conditions.
  • Convex Hull Trick: Very powerful for 1D DP with linear transitions. The key step is algebraic manipulation to extract slopes and intercepts.
  • Monotone Queue: Simplest of the four. Applies whenever there is a fixed-size sliding window constraint on the DP transition.
  • Verification: When unsure if an optimization applies, compare results with the brute-force O(n^2) or O(n^3) solution on small inputs.

Key Insights

Algebraic manipulation is key: For CHT, you need to expand and rearrange the DP transition to isolate the linear form. Practice expanding squares and grouping terms by j vs i.

Monotonicity is the core insight: D&C optimization and Knuth's both exploit that the "best decision" moves rightward as we move to larger subproblems.

Amortization argument: Monotone queue and CHT are O(n) because each element enters and leaves the deque/hull at most once. The total work across all iterations is O(n), not O(n) per iteration.

Implementation tip: Use long long for all intermediate computations. These optimizations are typically applied to problems with large values where overflow is a real concern.
