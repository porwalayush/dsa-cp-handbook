Recursion & Memoization

The top-down approach to Dynamic Programming. Start with the big problem, recursively break it down, and cache results to avoid redundant computation.

The Core Idea

DP problems have two key properties:

  1. Overlapping Subproblems: Same subproblems are solved multiple times
  2. Optimal Substructure: Optimal solution is built from optimal solutions of subproblems

Recursion Tree: Why Naive Recursion is Slow

Consider computing fib(5) with naive recursion:

                         fib(5)
                       /        \
                  fib(4)          fib(3)
                /      \         /      \
           fib(3)    fib(2)   fib(2)   fib(1)
           /    \    /    \    /    \
      fib(2) fib(1) fib(1) fib(0) fib(1) fib(0)
      /    \
   fib(1) fib(0)

   Total calls: 15 (exponential!)
   fib(2) computed 3 times
   fib(3) computed 2 times
  • Without memo: O(2^n) time - exponential explosion!
  • With memo: O(n) time - each subproblem solved exactly once
  • Space: O(n) for memo table + O(n) for recursion stack

Memoization: Cache Results

Store the result of each subproblem. Before computing, check if already solved:

   Memoization Process for fib(5):

   Call fib(5) → not cached → compute
     Call fib(4) → not cached → compute
       Call fib(3) → not cached → compute
         Call fib(2) → not cached → compute
           Call fib(1) → return 1 (base)
           Call fib(0) → return 0 (base)
         memo[2] = 1 ✓
         Call fib(1) → return 1 (base)
       memo[3] = 2 ✓
       Call fib(2) → CACHED! return 1 ⚡
     memo[4] = 3 ✓
     Call fib(3) → CACHED! return 2 ⚡
   memo[5] = 5 ✓

   Only 9 calls instead of 15!
   With larger n, savings are dramatic.

Code: Fibonacci with Memoization

#include <bits/stdc++.h>
using namespace std;

// Naive recursion: O(2^n) time
int fibNaive(int n) {
    if (n <= 1) return n;
    return fibNaive(n - 1) + fibNaive(n - 2);
}

// Memoized recursion: O(n) time, O(n) space
int memo[1001];
int fibMemo(int n) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n]; // already computed!
    memo[n] = fibMemo(n - 1) + fibMemo(n - 2);
    return memo[n];
}

int main() {
    memset(memo, -1, sizeof(memo));
    cout << fibMemo(10) << endl; // 55
    return 0;
}

import java.util.Arrays;

public class FibMemo {
    static int[] memo = new int[1001];

    // Naive recursion: O(2^n) time
    static int fibNaive(int n) {
        if (n <= 1) return n;
        return fibNaive(n - 1) + fibNaive(n - 2);
    }

    // Memoized: O(n) time, O(n) space
    static int fibMemo(int n) {
        if (n <= 1) return n;
        if (memo[n] != -1) return memo[n]; // cached!
        memo[n] = fibMemo(n - 1) + fibMemo(n - 2);
        return memo[n];
    }

    public static void main(String[] args) {
        Arrays.fill(memo, -1);
        System.out.println(fibMemo(10)); // 55
    }
}

# Naive recursion: O(2^n) time
def fib_naive(n):
    if n <= 1:
        return n
    return fib_naive(n - 1) + fib_naive(n - 2)

# Memoized: O(n) time, O(n) space
def fib_memo(n, memo={}):
    if n <= 1:
        return n
    if n in memo:
        return memo[n]  # cached!
    memo[n] = fib_memo(n - 1) + fib_memo(n - 2)
    return memo[n]

# Using @lru_cache (Pythonic way)
from functools import lru_cache

@lru_cache(maxsize=None)
def fib(n):
    if n <= 1:
        return n
    return fib(n - 1) + fib(n - 2)

print(fib(10))  # 55

When to Use Memoization (Top-Down)

Advantages

  • Natural recursive thinking
  • Only computes needed states
  • Easy to implement from recurrence
  • Good for sparse state spaces

Disadvantages

  • Recursion stack overhead
  • Stack overflow for deep recursion
  • Slightly slower due to function calls
  • Harder to optimize space

Complexity Analysis

  • Time: O(number of unique states × work per state)
  • Space: O(number of unique states) for memo + O(max recursion depth) for stack
  • Example: Fibonacci → O(n) states × O(1) per state = O(n) time, O(n) space

Common Pitfalls

1. Forgetting Base Cases: Always handle the smallest subproblems first.

2. Wrong State Definition: Make sure the state captures ALL information needed.

3. Stack Overflow: For n > 10^4, consider iterative (tabulation) approach.

4. Memo Initialization: Use -1 or null to distinguish "not computed" from valid 0.
