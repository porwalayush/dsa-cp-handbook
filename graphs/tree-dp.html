Tree DP Fundamentals

Compute DP values on tree structures using DFS. Each node's answer is derived from its children's answers, computed bottom-up during post-order traversal.

Core Concept

Tree DP pattern:
  1. Root the tree (pick any node as root)
  2. DFS from root
  3. For each node v, compute dp[v] from dp[children of v]
  4. Process children BEFORE the parent (post-order)

         1 (root)
        / \
       2   3
      / \   \
     4   5   6

DFS order: 4 → 5 → 2 → 6 → 3 → 1
dp computed: dp[4], dp[5] → dp[2], dp[6] → dp[3] → dp[1]

Key: dp[v] = answer for the subtree rooted at v

Example 1: Tree Diameter

Find the longest path between any two nodes in an unweighted tree.

         1
        / \
       2   3
      / \   \
     4   5   6
    /
   7

Diameter = longest path = 7 → 4 → 2 → 1 → 3 → 6 (length 5)

Idea: For each node v, track the longest path going DOWN through v.
  depth[v] = longest downward path from v to any leaf in its subtree

The diameter THROUGH node v:
  = sum of the two longest downward paths among v's children
  = top two values of (depth[child] + 1)

Global answer = max over all nodes of diameter through that node

State & Transition (Diameter)

State: depth[v] = length of longest path from v going downward

Transition: depth[v] = max(depth[child] + 1) over all children

Base Case: depth[leaf] = 0

Answer: Maintain a global variable. For each node, the diameter through v = sum of the two largest (depth[child] + 1). Update global max.

Step-by-Step: Tree Diameter

Step 1: DFS reaches leaves: depth[7]=0, depth[5]=0, depth[6]=0
Step 2: Node 4: child is 7. depth[4] = depth[7]+1 = 1.
Diameter through 4: only one child, so path = 1.
Step 3: Node 2: children are 4 (depth 1) and 5 (depth 0).
depth[2] = max(1+1, 0+1) = 2.
Diameter through 2: (1+1) + (0+1) = 2+1 = 3. Global max = 3.
Step 4: Node 3: child is 6 (depth 0). depth[3] = 1.
Step 5: Node 1: children are 2 (depth 2) and 3 (depth 1).
depth[1] = max(2+1, 1+1) = 3.
Diameter through 1: (2+1) + (1+1) = 3+2 = 5. Global max = 5.

Code: Tree Diameter

// Tree Diameter using Tree DP
// Time: O(n), Space: O(n)
int diameter = 0;
vector<vector<int>> adj;

int dfs(int v, int parent) {
    int maxDepth1 = 0, maxDepth2 = 0; // top two depths

    for (int u : adj[v]) {
        if (u == parent) continue;
        int d = dfs(u, v) + 1;
        if (d >= maxDepth1) {
            maxDepth2 = maxDepth1;
            maxDepth1 = d;
        } else if (d > maxDepth2) {
            maxDepth2 = d;
        }
    }

    // Diameter through v = sum of two longest arms
    diameter = max(diameter, maxDepth1 + maxDepth2);

    return maxDepth1; // return longest downward path
}

// Call: dfs(0, -1); answer is in 'diameter'

Example 2: House Robber on Tree

Each node has a value. You cannot rob two adjacent nodes (connected by an edge). Find the maximum total value you can rob.

        3
       / \
      2   3
       \   \
        3   1

If we rob node 3 (root): cannot rob children 2 and 3
  Rob root(3) + grandchildren(3 + 1) = 7

If we skip root: rob children
  Rob child 2's subtree optimally + child 3's subtree optimally

State: dp[v][0] = max rob from subtree v, NOT robbing v
       dp[v][1] = max rob from subtree v, ROBBING v

Transition:
  dp[v][0] = sum of max(dp[child][0], dp[child][1]) for all children
  dp[v][1] = val[v] + sum of dp[child][0] for all children

Answer: max(dp[root][0], dp[root][1])

Code: House Robber on Tree

// House Robber III - Tree DP
// Time: O(n), Space: O(n)
vector<vector<int>> adj;
vector<int> val;

// Returns {not_rob, rob} for node v
pair<int,int> dfs(int v, int parent) {
    int notRob = 0, rob = val[v];

    for (int u : adj[v]) {
        if (u == parent) continue;
        auto [childNot, childRob] = dfs(u, v);
        notRob += max(childNot, childRob); // skip v: children free
        rob += childNot;                   // rob v: children skipped
    }

    return {notRob, rob};
}

// Call:
auto [notRob, rob] = dfs(0, -1);
int answer = max(notRob, rob);

// House Robber III - Tree DP
// Time: O(n), Space: O(n)
static List<List<Integer>> adj;
static int[] val;

// Returns int[]{not_rob, rob} for node v
static int[] dfs(int v, int parent) {
    int notRob = 0, rob = val[v];

    for (int u : adj.get(v)) {
        if (u == parent) continue;
        int[] child = dfs(u, v);
        notRob += Math.max(child[0], child[1]);
        rob += child[0];
    }

    return new int[]{notRob, rob};
}

// Call:
int[] res = dfs(0, -1);
int answer = Math.max(res[0], res[1]);

# House Robber III - Tree DP
# Time: O(n), Space: O(n)
def tree_rob(adj, val, root=0):
    def dfs(v, parent):
        not_rob = 0
        rob = val[v]

        for u in adj[v]:
            if u == parent:
                continue
            child_not, child_rob = dfs(u, v)
            not_rob += max(child_not, child_rob)
            rob += child_not

        return not_rob, rob

    not_rob, rob = dfs(root, -1)
    return max(not_rob, rob)

Example 3: Maximum Path Sum

Find the path in a weighted tree with maximum total node value (path can start and end at any nodes).

State: maxDown[v] = maximum sum path starting at v going downward

Transition: maxDown[v] = val[v] + max(0, max(maxDown[child]) for child in children)
(Take 0 if all children are negative — path can just be node v)

Answer update: For each node v, best path through v = val[v] + sum of top 2 positive maxDown[child] values. Update global max.

Base Case: maxDown[leaf] = val[leaf]

Code: Maximum Path Sum in Tree

// Maximum Path Sum in Tree
// Time: O(n), Space: O(n)
vector<vector<int>> adj;
vector<int> val;
int maxSum;

int dfs(int v, int parent) {
    int best = val[v]; // max path sum going down from v
    int top1 = 0, top2 = 0; // top two child contributions

    for (int u : adj[v]) {
        if (u == parent) continue;
        int d = dfs(u, v);
        if (d > 0) { // only include positive contributions
            if (d >= top1) { top2 = top1; top1 = d; }
            else if (d > top2) { top2 = d; }
        }
    }

    // Path through v using top two children
    maxSum = max(maxSum, val[v] + top1 + top2);

    return val[v] + top1; // return best single-direction path
}

Visual: DFS Order and Bottom-Up Computation

Tree with values:
        [5]
       /   \
     [3]   [4]
     / \     \
   [2] [1]   [6]

DFS traversal (post-order): 2, 1, 3, 6, 4, 5

Computation order:
  dp[2] = computed (leaf)          → depth=0
  dp[1] = computed (leaf)          → depth=0
  dp[3] = f(dp[2], dp[1])         → uses children
  dp[6] = computed (leaf)          → depth=0
  dp[4] = f(dp[6])                → uses child
  dp[5] = f(dp[3], dp[4])         → uses children (LAST)

Key property: When we compute dp[v], ALL descendants
of v have already been computed!

Complexity

  • Time: O(n) - each node visited exactly once during DFS
  • Space: O(n) - recursion stack + dp array
  • Why O(n): Tree has exactly n-1 edges, each traversed twice (down and up)
  • Stack depth: O(n) worst case (path graph), O(log n) for balanced tree

Re-rooting Technique (Brief)

Problem: Find dp[v] for EVERY node as root, not just one fixed root.

Naive: Run DFS from each node = O(n^2).

Re-rooting: O(n) total with two passes:
  1. First DFS: Compute dp[v] with node 0 as root (standard tree DP)
  2. Second DFS: For each edge (parent, child), "move" the root from parent to child. Update dp[child] using dp[parent] and the contribution from child's subtree.

Key idea: When re-rooting from parent to child, child gains the "rest of tree" contribution that was previously the parent's answer minus child's contribution. 
Re-rooting example (sum of distances to all nodes):

First pass (root at 1):      Second pass (re-root):
  dp[1] = 8                    dp[1] = 8 (base)
  dp[2] = computed as child    dp[2] = dp[1] - subtree_size[2]
  dp[3] = computed as child              + (n - subtree_size[2])
  ...                          ...

The "contribution flip": subtree of child becomes
the "rest of tree" from child's perspective.

Key Insights

  • Always use parent tracking: Pass parent parameter in DFS to avoid revisiting the parent as a "child"
  • Post-order = bottom-up: Children are computed before parent, which is what we need
  • Multiple states per node: Often need dp[v][0] and dp[v][1] (e.g., include/exclude node v)
  • Global answer vs return value: The answer might not be dp[root] but max over all nodes (like diameter)
  • Iterative DFS: For very deep trees, convert recursion to iterative with explicit stack to avoid stack overflow
