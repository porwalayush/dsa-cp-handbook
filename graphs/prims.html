Prim's Algorithm

Prim's algorithm finds the Minimum Spanning Tree (MST) of a weighted, connected, undirected graph. It grows the MST one vertex at a time, always picking the cheapest edge that connects a vertex already in the MST to one that is not.

How It Works (Intuition)

Think of it like building a road network for a group of cities. You start at one city and repeatedly build the cheapest road that connects a new city to your existing network.

  1. Start from any vertex. Mark it as part of the MST.
  2. Look at all edges going from MST vertices to non-MST vertices.
  3. Pick the edge with the minimum weight.
  4. Add that edge and the new vertex to the MST.
  5. Repeat until all vertices are in the MST.

Efficient Implementation (Priority Queue)

The naive approach checks all edges each time. Using a min-heap (priority queue), we efficiently get the minimum weight edge in O(log V) time.

Key Idea: Push all edges of a newly added vertex into the priority queue. Pop the minimum edge. If it connects to an unvisited vertex, add it to the MST. Otherwise, discard and pop again. 
import java.util.*;

class PrimsAlgorithm {
    // Edge: {weight, destinationVertex}
    static int primsMST(int V, List<List<int[]>> adj) {
        // Min-heap: {weight, vertex}
        PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[0] - b[0]);
        boolean[] inMST = new boolean[V];
        int totalWeight = 0;

        // Start from vertex 0 with weight 0
        pq.offer(new int[]{0, 0});

        while (!pq.isEmpty()) {
            int[] top = pq.poll();
            int weight = top[0], u = top[1];

            // Skip if already in MST
            if (inMST[u]) continue;

            // Add vertex to MST
            inMST[u] = true;
            totalWeight += weight;

            // Push all edges from u to non-MST vertices
            for (int[] edge : adj.get(u)) {
                int v = edge[0], w = edge[1];
                if (!inMST[v]) {
                    pq.offer(new int[]{w, v});
                }
            }
        }
        return totalWeight;
    }

    public static void main(String[] args) {
        int V = 5;
        List<List<int[]>> adj = new ArrayList<>();
        for (int i = 0; i < V; i++) adj.add(new ArrayList<>());

        // Add edges: {destination, weight}
        adj.get(0).add(new int[]{1, 2});
        adj.get(1).add(new int[]{0, 2});
        adj.get(0).add(new int[]{3, 8});
        adj.get(3).add(new int[]{0, 8});
        adj.get(1).add(new int[]{2, 6});
        adj.get(2).add(new int[]{1, 6});
        adj.get(1).add(new int[]{3, 3});
        adj.get(3).add(new int[]{1, 3});
        adj.get(2).add(new int[]{3, 4});
        adj.get(3).add(new int[]{2, 4});
        adj.get(3).add(new int[]{4, 7});
        adj.get(4).add(new int[]{3, 7});

        System.out.println("MST weight: " + primsMST(V, adj));
        // Output: MST weight: 16
    }
}
#include <bits/stdc++.h>
using namespace std;

int primsMST(int V, vector<vector<pair<int,int>>>& adj) {
    // Min-heap: {weight, vertex}
    priority_queue<pair<int,int>, vector<pair<int,int>>, greater<pair<int,int>>> pq;
    vector<bool> inMST(V, false);
    int totalWeight = 0;

    // Start from vertex 0
    pq.push({0, 0});

    while (!pq.empty()) {
        auto [weight, u] = pq.top();
        pq.pop();

        // Skip if already in MST
        if (inMST[u]) continue;

        // Add vertex to MST
        inMST[u] = true;
        totalWeight += weight;

        // Push all edges from u to non-MST vertices
        for (auto& [v, w] : adj[u]) {
            if (!inMST[v]) {
                pq.push({w, v});
            }
        }
    }
    return totalWeight;
}

import heapq

def prims(V, adj):
    # adj[u] = [(v, weight), ...]
    in_mst = [False] * V
    # Min-heap: (weight, node)
    pq = [(0, 0)]  # start from node 0 with cost 0
    mst_cost = 0

    while pq:
        w, u = heapq.heappop(pq)
        if in_mst[u]:
            continue
        in_mst[u] = True
        mst_cost += w

        for v, weight in adj[u]:
            if not in_mst[v]:
                heapq.heappush(pq, (weight, v))

    return mst_cost

# Example
V = 5
adj = [[] for _ in range(V)]
edges = [(0,1,2), (0,3,6), (1,2,3), (1,3,8), (1,4,5), (2,4,7), (3,4,9)]
for u, v, w in edges:
    adj[u].append((v, w))
    adj[v].append((u, w))

print("MST cost:", prims(V, adj))

Time and Space Complexity

  • Time: O((V + E) log V) with a binary heap. Each vertex is extracted once (V extractions), and each edge causes at most one insertion into the heap (E insertions). Each heap operation is O(log V).
  • Space: O(V + E) for the adjacency list and O(E) for the priority queue in the worst case.

Prim's vs Kruskal's

  • Prim's grows from a single source vertex -- better for dense graphs (many edges), especially with adjacency matrix + O(V^2) simple loop.
  • Kruskal's sorts all edges globally -- better for sparse graphs (few edges) since sorting E edges is cheap when E is small.
  • Both produce the same MST (if edge weights are unique). If weights have ties, they may produce different valid MSTs.
  • Kruskal's needs Union-Find (DSU); Prim's needs a priority queue.

When to Use Prim's

  • Dense graphs where E is close to V^2
  • When you already have an adjacency list/matrix representation
  • When you want to grow the MST incrementally from a starting point
  • Problems where you need to track which edges are in the MST as you build it
