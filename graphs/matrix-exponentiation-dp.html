Matrix Exponentiation DP

Accelerate linear recurrences from O(n) to O(k3 · log n) by expressing the recurrence as matrix multiplication and using fast exponentiation. Here k is the order of the recurrence and n is the target index.

Problem Statement

Given a linear recurrence of order k:

f(n) = a1*f(n-1) + a2*f(n-2) + ... + ak*f(n-k)

with base cases f(0), f(1), ..., f(k-1)

Compute f(n) for very large n (e.g., n = 10^18) modulo some prime.

Naive DP: O(n) time — too slow for n = 10^18
Matrix Exponentiation: O(k^3 * log n) time

Core Idea: Recurrence as Matrix Multiplication

Key Insight: Express the state vector [f(n), f(n-1), ..., f(n-k+1)] as a matrix times the previous state vector [f(n-1), f(n-2), ..., f(n-k)].

Transition Matrix M (k x k):
Row 0: [a1, a2, a3, ..., ak] (the recurrence coefficients)
Row 1: [1, 0, 0, ..., 0] (shift f(n-1) down)
Row 2: [0, 1, 0, ..., 0] (shift f(n-2) down)
...
Row k-1: [0, 0, ..., 1, 0]

Then: [f(n), f(n-1), ..., f(n-k+1)]T = Mn-k+1 · [f(k-1), f(k-2), ..., f(0)]T

Visual: Fibonacci Example

Fibonacci: f(n) = f(n-1) + f(n-2),  f(0)=0, f(1)=1

Transition matrix (k=2):
  ⌊ f(n)   ⌋   ⌊ 1  1 ⌋   ⌊ f(n-1) ⌋
  ⌊ f(n-1) ⌋ = ⌊ 1  0 ⌋ * ⌊ f(n-2) ⌋

Computing f(5):
  ⌊ f(5) ⌋   ⌊ 1  1 ⌋^4   ⌊ f(1) ⌋   ⌊ 1  1 ⌋^4   ⌊ 1 ⌋
  ⌊ f(4) ⌋ = ⌊ 1  0 ⌋   * ⌊ f(0) ⌋ = ⌊ 1  0 ⌋   * ⌊ 0 ⌋

Fast exponentiation of M^4:
  M^1 = ⌊ 1  1 ⌋    M^2 = ⌊ 2  1 ⌋    M^4 = ⌊ 5  3 ⌋
        ⌊ 1  0 ⌋          ⌊ 1  1 ⌋          ⌊ 3  2 ⌋

  Result: M^4 * [1, 0]^T = [5, 3]^T
  So f(5) = 5, f(4) = 3  ✓

Fast Matrix Exponentiation

Algorithm: Compute Mn using binary exponentiation (repeated squaring):

1. Start with result = Identity matrix
2. While n > 0:
    - If n is odd: result = result * M
    - M = M * M
    - n = n / 2

Matrix multiplication: O(k3) for k x k matrices
Number of multiplications: O(log n)
Total: O(k3 · log n)

Step-by-Step: General Recurrence

Example: f(n) = 2*f(n-1) + 3*f(n-2) + f(n-3), with f(0)=1, f(1)=1, f(2)=4
Step 1: Build transition matrix (k=3):
M = [[2, 3, 1], [1, 0, 0], [0, 1, 0]]
Step 2: Initial state vector: v = [f(2), f(1), f(0)]T = [4, 1, 1]T
Step 3: To find f(n): compute Mn-2 · v, read top element
Verify f(3): M · [4, 1, 1]T = [2*4+3*1+1*1, 4, 1]T = [12, 4, 1]T
Check: f(3) = 2*4 + 3*1 + 1*1 = 12 ✓

// Matrix Exponentiation for Linear Recurrences
// Time: O(k^3 * log n), Space: O(k^2)
const int MOD = 1e9 + 7;
typedef vector<vector<long long>> Matrix;

Matrix multiply(Matrix& A, Matrix& B, int k) {
    Matrix C(k, vector<long long>(k, 0));
    for (int i = 0; i < k; i++)
        for (int j = 0; j < k; j++)
            for (int p = 0; p < k; p++)
                C[i][j] = (C[i][j] + A[i][p] * B[p][j]) % MOD;
    return C;
}

Matrix matpow(Matrix M, long long n, int k) {
    // Identity matrix
    Matrix result(k, vector<long long>(k, 0));
    for (int i = 0; i < k; i++) result[i][i] = 1;

    while (n > 0) {
        if (n & 1) result = multiply(result, M, k);
        M = multiply(M, M, k);
        n >>= 1;
    }
    return result;
}

// Fibonacci in O(log n)
long long fibonacci(long long n) {
    if (n <= 1) return n;
    Matrix M = {{1, 1}, {1, 0}};
    Matrix res = matpow(M, n - 1, 2);
    return res[0][0]; // f(n) = M^(n-1)[0][0] * f(1) + M^(n-1)[0][1] * f(0)
}

// General recurrence: f(n) = c[0]*f(n-1) + c[1]*f(n-2) + ... + c[k-1]*f(n-k)
long long solveRecurrence(vector<long long>& base,
                           vector<long long>& coeff, long long n) {
    int k = coeff.size();
    if (n < k) return base[n];

    // Build transition matrix
    Matrix M(k, vector<long long>(k, 0));
    for (int j = 0; j < k; j++) M[0][j] = coeff[j];
    for (int i = 1; i < k; i++) M[i][i-1] = 1;

    Matrix res = matpow(M, n - k + 1, k);

    // Multiply result matrix by initial state [f(k-1), ..., f(0)]
    long long ans = 0;
    for (int j = 0; j < k; j++)
        ans = (ans + res[0][j] * base[k - 1 - j]) % MOD;
    return ans;
}

# Time: O(k^3 * log n), Space: O(k^2)
MOD = 10**9 + 7

def multiply(A, B, k):
    C = [[0] * k for _ in range(k)]
    for i in range(k):
        for j in range(k):
            for p in range(k):
                C[i][j] = (C[i][j] + A[i][p] * B[p][j]) % MOD
    return C

def matpow(M, n, k):
    # Identity matrix
    result = [[0] * k for _ in range(k)]
    for i in range(k):
        result[i][i] = 1

    while n > 0:
        if n & 1:
            result = multiply(result, M, k)
        M = multiply(M, M, k)
        n >>= 1
    return result

# Fibonacci in O(log n)
def fibonacci(n):

        return n

# General recurrence: f(n) = c[0]*f(n-1) + ... + c[k-1]*f(n-k)
def solve_recurrence(base, coeff, n):
    k = len(coeff)
    if n < k:
        return base[n]

    # Build transition matrix
    M = [[0] * k for _ in range(k)]
    for j in range(k):
        M[0][j] = coeff[j]
    for i in range(1, k):
        M[i][i - 1] = 1

    res = matpow(M, n - k + 1, k)

    # Multiply result row by initial state
    ans = 0
    for j in range(k):
        ans = (ans + res[0][j] * base[k - 1 - j]) % MOD
    return ans

Application: Count Paths of Length k in a Graph

Problem: Given adjacency matrix A of a graph with n nodes, count the number of paths of exactly length k from node u to node v.

Solution: Ak[u][v] gives the number of walks of length k from u to v.
Compute Ak using matrix exponentiation in O(n3 log k). 
Graph with 4 nodes:         Adjacency Matrix A:
  0 --- 1                   ⌊ 0  1  1  0 ⌋
  |     |                   ⌊ 1  0  0  1 ⌋
  2 --- 3                   ⌊ 1  0  0  1 ⌋
                            ⌊ 0  1  1  0 ⌋

A^2 = ⌊ 2  0  0  2 ⌋    Paths of length 2 from 0 to 3: 2
      ⌊ 0  2  2  0 ⌋      (0-1-3 and 0-2-3)

      ⌊ 2  0  0  2 ⌋

Application: Tiling Problems

Problem: Count ways to tile a 2 x n grid with 1x2 dominoes.

Recurrence: f(n) = f(n-1) + f(n-2) (same as Fibonacci with f(1)=1, f(2)=2)

For larger grids (m x n): Build a transfer matrix between column states (bitmask of m bits). Matrix size is 2m x 2m. Compute Mn for result.

Complexity

  • Time: O(k3 · log n) where k = matrix dimension, n = exponent
  • Space: O(k2) for storing matrices
  • Comparison: Naive iteration is O(k · n). Matrix method wins when n is very large (109+) and k is small
  • Practical limit: k ≤ 100 or so (k3 must be reasonable); n can be up to 1018

Key Insights

  • When to use: Any DP where transition is linear and n is very large (typically n > 107)
  • Identify the recurrence: dp[i] depends only on dp[i-1], dp[i-2], ..., dp[i-k] with constant coefficients
  • Non-homogeneous recurrences: f(n) = a*f(n-1) + b can be handled by augmenting the state: [f(n), 1]
  • Modular arithmetic: All operations mod p keep numbers small; matrix exponentiation works perfectly with modular arithmetic
