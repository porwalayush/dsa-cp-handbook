Strongly Connected Components (SCC)

A Strongly Connected Component is a maximal set of vertices in a directed graph such that there is a path from every vertex to every other vertex in the set. Think of it as a "group of vertices that can all reach each other."

Key Definition: In an SCC, for any two vertices u and v, there exists a path u -> ... -> v AND a path v -> ... -> u. The component is "maximal" meaning you cannot add any more vertices and keep this property.

Visual Example

    Directed Graph:

        0 -------> 1 -------> 3
        ^         /            |
        |        /             |
        |       v              v
        +------ 2              4 ------> 5
                                         ^
                                         |
                                    6 ---+
                                    ^    |
                                    |    v
                                    +--- 7

    Strongly Connected Components:

    SCC 1: {0, 1, 2}     SCC 2: {3}     SCC 3: {4}     SCC 4: {5, 6, 7}

    Verify SCC 1:
      0 -> 1 -> 2 -> 0  (cycle! everyone reaches everyone)

    Verify SCC 4:
      5 -> ... hmm, let me fix:
      6 -> 5, 6 -> 7, 7 -> 6  => {6, 7} form SCC
      But 5 cannot reach 6... so 5 is its own SCC!

    Corrected SCCs:
    SCC 1: {0, 1, 2}   (0->1->2->0)
    SCC 2: {3}          (no cycle involving 3)
    SCC 3: {4}          (no cycle involving 4)
    SCC 4: {5}          (no outgoing edges back)
    SCC 5: {6, 7}       (6->7->6)


    Condensation (DAG of SCCs):

    [0,1,2] ---> [3] ---> [4] ---> [5]
                                     ^
                                     |
                                   [6,7]

Why SCCs Matter

  • Condensation: Replace each SCC with a single node to get a DAG. Now you can do topological sort on the DAG!
  • Reachability: If u and v are in the same SCC, they can definitely reach each other
  • 2-SAT: SCC is the core algorithm for solving 2-SAT problems

Kosaraju's Algorithm

Kosaraju's algorithm finds all SCCs in two DFS passes. It is beautifully simple and relies on a key insight about the transposed graph.

The Algorithm (3 Steps):
  1. Pass 1: Do DFS on the original graph. Push vertices to a stack in order of finish time (post-order).
  2. Transpose: Reverse all edges in the graph (flip every edge direction).
  3. Pass 2: Pop vertices from the stack. For each unvisited vertex, do DFS on the TRANSPOSED graph. Each DFS tree = one SCC.
    Kosaraju's Walkthrough:

    Original Graph:          Transposed Graph (all edges reversed):

    0 --> 1                  0 <-- 1
    ^    /                   |    ^
    |   v                    v   /
    +-- 2 --> 3              2 <-- 3

    STEP 1: DFS on original, record finish order
      Start at 0: visit 0->1->2->0(back edge)->3
      Finish order (stack): [3, 2, 1, 0]  (3 finishes first, 0 last)

    STEP 2: Transpose the graph (reverse all arrows)

    STEP 3: Process stack order [0, 1, 2, 3] on transposed graph
      Pop 0: DFS on transposed -> visits 0, 2, 1 => SCC = {0, 1, 2}
      Pop 1: already visited
      Pop 2: already visited
      Pop 3: DFS on transposed -> visits 3 => SCC = {3}

    Result: SCCs are {0, 1, 2} and {3}
Why does it work? The vertex that finishes last in DFS belongs to a "source SCC" (no incoming edges from other SCCs in the condensation DAG). When we DFS on the transposed graph starting from it, we can only reach vertices in its OWN SCC (because cross-SCC edges are now reversed and point away).

Implementation: Kosaraju's Algorithm

import java.util.*;

public class KosarajuSCC {
    private int V;
    private List<List<Integer>> adj;      // original graph
    private List<List<Integer>> radj;     // transposed graph

    public KosarajuSCC(int v) {
        V = v;
        adj = new ArrayList<>();
        radj = new ArrayList<>();
        for (int i = 0; i < V; i++) {
            adj.add(new ArrayList<>());
            radj.add(new ArrayList<>());
        }
    }

    public void addEdge(int u, int v) {
        adj.get(u).add(v);
        radj.get(v).add(u); // reversed edge for transpose
    }

    // Pass 1: DFS on original graph, fill stack by finish time
    private void dfs1(int node, boolean[] visited, Stack<Integer> stack) {
        visited[node] = true;
        for (int neighbor : adj.get(node)) {
            if (!visited[neighbor]) {
                dfs1(neighbor, visited, stack);
            }
        }
        stack.push(node); // Push AFTER all descendants finish
    }

    // Pass 2: DFS on transposed graph, collect SCC
    private void dfs2(int node, boolean[] visited, List<Integer> component) {
        visited[node] = true;
        component.add(node);
        for (int neighbor : radj.get(node)) {
            if (!visited[neighbor]) {
                dfs2(neighbor, visited, component);
            }
        }
    }

    public List<List<Integer>> findSCCs() {
        boolean[] visited = new boolean[V];
        Stack<Integer> stack = new Stack<>();

        // PASS 1: Fill stack with finish-time order
        for (int i = 0; i < V; i++) {
            if (!visited[i]) {
                dfs1(i, visited, stack);
            }
        }

        // PASS 2: Process in reverse finish-time on transposed graph
        Arrays.fill(visited, false);
        List<List<Integer>> sccs = new ArrayList<>();

        while (!stack.isEmpty()) {
            int node = stack.pop();
            if (!visited[node]) {
                List<Integer> component = new ArrayList<>();
                dfs2(node, visited, component);
                sccs.add(component);
            }
        }
        return sccs;
    }

    public static void main(String[] args) {
        KosarajuSCC g = new KosarajuSCC(5);
        g.addEdge(0, 1);
        g.addEdge(1, 2);
        g.addEdge(2, 0); // SCC: {0, 1, 2}
        g.addEdge(1, 3);
        g.addEdge(3, 4);
        g.addEdge(4, 3); // SCC: {3, 4}

        List<List<Integer>> sccs = g.findSCCs();
        for (List<Integer> scc : sccs) {
            System.out.println(scc);
        }
        // Output: [0, 2, 1] and [3, 4]
    }
}
                
#include <bits/stdc++.h>
using namespace std;

class KosarajuSCC {
    int V;
    vector<vector<int>> adj;   // original graph
    vector<vector<int>> radj;  // transposed graph

    // Pass 1: DFS on original, fill stack by finish time
    void dfs1(int node, vector<bool>& visited, stack<int>& stk) {
        visited[node] = true;
        for (int neighbor : adj[node]) {
            if (!visited[neighbor]) {
                dfs1(neighbor, visited, stk);
            }
        }
        stk.push(node); // Push AFTER all descendants finish
    }

    // Pass 2: DFS on transposed graph, collect SCC
    void dfs2(int node, vector<bool>& visited, vector<int>& component) {
        visited[node] = true;
        component.push_back(node);
        for (int neighbor : radj[node]) {
            if (!visited[neighbor]) {
                dfs2(neighbor, visited, component);
            }
        }
    }

public:
    KosarajuSCC(int v) : V(v), adj(v), radj(v) {}

    void addEdge(int u, int v) {
        adj[u].push_back(v);
        radj[v].push_back(u); // reversed for transpose
    }

    vector<vector<int>> findSCCs() {
        vector<bool> visited(V, false);
        stack<int> stk;

        // PASS 1: Fill stack with finish-time order
        for (int i = 0; i < V; i++) {
            if (!visited[i]) {
                dfs1(i, visited, stk);
            }
        }

        // PASS 2: Process in reverse finish-time on transposed graph
        fill(visited.begin(), visited.end(), false);
        vector<vector<int>> sccs;

        while (!stk.empty()) {
            int node = stk.top();
            stk.pop();
            if (!visited[node]) {
                vector<int> component;
                dfs2(node, visited, component);
                sccs.push_back(component);
            }
        }
        return sccs;
    }
};

int main() {
    KosarajuSCC g(5);
    g.addEdge(0, 1);
    g.addEdge(1, 2);
    g.addEdge(2, 0); // SCC: {0, 1, 2}
    g.addEdge(1, 3);
    g.addEdge(3, 4);
    g.addEdge(4, 3); // SCC: {3, 4}

    vector<vector<int>> sccs = g.findSCCs();
    for (auto& scc : sccs) {
        for (int v : scc) cout << v << " ";
        cout << endl;
    }
    // Output: 0 2 1 \n 3 4
    return 0;
}
                
import sys
sys.setrecursionlimit(10**6)

def find_sccs(V, adj):
    # Pass 1: DFS on original, fill stack by finish time
    visited = [False] * V
    stack = []

    def dfs1(node):
        visited[node] = True
        for nb in adj[node]:
            if not visited[nb]:
                dfs1(nb)
        stack.append(node)

    for i in range(V):
        if not visited[i]:
            dfs1(i)

    # Build transposed graph
    transpose = [[] for _ in range(V)]
    for u in range(V):
        for v in adj[u]:
            transpose[v].append(u)

    # Pass 2: DFS on transposed in stack order
    visited = [False] * V
    sccs = []

    def dfs2(node, comp):
        visited[node] = True
        comp.append(node)
        for nb in transpose[node]:
            if not visited[nb]:
                dfs2(nb, comp)

    while stack:
        node = stack.pop()
        if not visited[node]:
            comp = []
            dfs2(node, comp)
            sccs.append(comp)

    return sccs

# Example
adj = [[1], [2], [0, 3], [], []]  # 0->1->2->0, 2->3
adj[3].append(4)
print(find_sccs(5, adj))

Tarjan's Algorithm (Brief Overview)

Tarjan's algorithm also finds SCCs but in a single DFS pass. It uses:

  • Discovery time (disc[]): When a vertex is first visited
  • Low-link value (low[]): The smallest discovery time reachable from the subtree of the vertex
  • Stack: Tracks vertices in the current DFS path
Tarjan's Key Rule: A vertex u is the "root" of an SCC if disc[u] == low[u]. When this happens, pop everything from the stack up to and including u — that is one SCC.
Kosaraju vs Tarjan:
- Kosaraju: 2 DFS passes, easier to understand and implement
- Tarjan: 1 DFS pass, slightly faster in practice, more complex
Both are O(V + E). Choose Kosaraju for interviews (simpler to explain).

Complexity

  • Time: O(V + E) for both Kosaraju's and Tarjan's
  • Space: O(V + E) for graph storage and auxiliary arrays

Applications

  • 2-SAT: Build implication graph, find SCCs to determine satisfiability
  • Condensation DAG: Simplify directed graphs for reachability queries
  • Deadlock detection: Cycles in wait-for graphs
  • Social networks: Find tightly-connected communities
  • Compiler optimization: Finding loops in control-flow graphs
