Tabulation (Bottom-Up)

The iterative approach to DP. Build the solution from the smallest subproblems up to the answer, filling a table step by step.

How Tabulation Works

  1. Define the table: dp[i] represents the answer for subproblem of size i
  2. Fill base cases: Initialize dp[0], dp[1], etc.
  3. Fill iteratively: Use the recurrence to fill from small → large
  4. Answer: dp[n] contains the final answer

Interactive: Watch Table Filling

Tabulation vs Memoization

  Memoization (Top-Down)         Tabulation (Bottom-Up)
  ─────────────────────         ──────────────────────
  fib(5) → needs fib(4)        dp[0] = 0
  fib(4) → needs fib(3)        dp[1] = 1
  fib(3) → needs fib(2)        dp[2] = dp[0] + dp[1] = 1
  fib(2) → needs fib(1)        dp[3] = dp[1] + dp[2] = 2
  fib(1) = 1 (base)            dp[4] = dp[2] + dp[3] = 3
  fib(0) = 0 (base)            dp[5] = dp[3] + dp[4] = 5
  Then unwinds back up...       Answer: dp[5] = 5 ✓

  Recursive (calls stack up)    Iterative (fills table left→right)

Code: Fibonacci Tabulation

// Tabulation: O(n) time, O(n) space
int fibTab(int n) {
    if (n <= 1) return n;
    vector<int> dp(n + 1);
    dp[0] = 0;
    dp[1] = 1;
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i-1] + dp[i-2];
    }
    return dp[n];
}

// Space optimized: O(1) space
int fibOpt(int n) {
    if (n <= 1) return n;
    int prev2 = 0, prev1 = 1;
    for (int i = 2; i <= n; i++) {
        int curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

def fib_tab(n):
    if n <= 1:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]

# Space optimized: O(1) space
def fib_opt(n):
    if n <= 1:
        return n
    prev2, prev1 = 0, 1
    for i in range(2, n + 1):
        curr = prev1 + prev2
        prev2 = prev1
        prev1 = curr
    return prev1

Space Optimization Technique

   Full table:     dp[0]  dp[1]  dp[2]  dp[3]  dp[4]  dp[5]
                     0      1      1      2      3      5

   Observation: dp[i] only depends on dp[i-1] and dp[i-2]
   → We only need 2 variables!

   Before:  int dp[n+1]     → O(n) space
   After:   int prev2, prev1 → O(1) space

   Step 1: prev2=0, prev1=1
   Step 2: curr = 0+1 = 1,  prev2=1, prev1=1
   Step 3: curr = 1+1 = 2,  prev2=1, prev1=2
   Step 4: curr = 1+2 = 3,  prev2=2, prev1=3
   Step 5: curr = 2+3 = 5,  prev2=3, prev1=5 ✓

When to Use Tabulation vs Memoization

Use Tabulation When

  • All subproblems need to be solved
  • n is large (avoid stack overflow)
  • Space optimization is needed
  • Iterative is more natural

Use Memoization When

  • Not all states are needed
  • Recursion is more intuitive
  • State space is sparse
  • Hard to determine fill order

Complexity

  • Time: O(n) - iterate once through all states
  • Space: O(n) for full table, O(1) with rolling variables
  • No recursion overhead: No function call stack, no stack overflow risk
