Floyd-Warshall Algorithm

Floyd-Warshall computes the shortest path between ALL pairs of vertices in a weighted graph. Unlike Dijkstra and Bellman-Ford (which find shortest paths from a single source), Floyd-Warshall answers: "What is the shortest distance from EVERY vertex to EVERY other vertex?"

Core Idea (Dynamic Programming)

The key insight is beautifully simple:

For each intermediate vertex k, check if going from i to j through k is shorter than the current best path from i to j.

    The DP recurrence:

    dist[i][j] = min( dist[i][j],  dist[i][k] + dist[k][j] )
                      ----------    -------------------------
                      direct path    path through vertex k


    In words:
    "Is it shorter to go i -> j directly,
     or to go i -> k -> j?"

    We try EVERY possible intermediate vertex k (0 to V-1).

Visual Walkthrough

    Graph (directed, weighted):

        (0)---3--->(1)
         |          |
         7          1
         |          |
         v          v
        (2)<--2---(3)

    Edge list: 0->1(3), 0->2(7), 1->3(1), 3->2(2)

    Initial distance matrix (INF = no direct edge):

         To:  0     1     2     3
    From:
      0  [   0     3     7    INF  ]
      1  [  INF    0    INF    1   ]
      2  [  INF   INF    0    INF  ]
      3  [  INF   INF    2     0   ]

    --- k = 0 (consider paths through vertex 0) ---
    Check all (i,j) pairs: can going through 0 help?
    dist[1][2] = min(INF, dist[1][0] + dist[0][2]) = min(INF, INF+7) = INF
    dist[3][2] = min(2, dist[3][0] + dist[0][2]) = min(2, INF+7) = 2
    No improvements through vertex 0.

         To:  0     1     2     3
    From:
      0  [   0     3     7    INF  ]
      1  [  INF    0    INF    1   ]
      2  [  INF   INF    0    INF  ]
      3  [  INF   INF    2     0   ]

    --- k = 1 (consider paths through vertex 1) ---
    dist[0][3] = min(INF, dist[0][1] + dist[1][3]) = min(INF, 3+1) = 4  UPDATED!

         To:  0     1     2     3
    From:
      0  [   0     3     7     4   ]   <-- 0->3 updated!
      1  [  INF    0    INF    1   ]
      2  [  INF   INF    0    INF  ]
      3  [  INF   INF    2     0   ]

    --- k = 2 (consider paths through vertex 2) ---
    No improvements (vertex 2 has no outgoing edges in our matrix).

    --- k = 3 (consider paths through vertex 3) ---
    dist[0][2] = min(7, dist[0][3] + dist[3][2]) = min(7, 4+2) = 6  UPDATED!
    dist[1][2] = min(INF, dist[1][3] + dist[3][2]) = min(INF, 1+2) = 3  UPDATED!

    FINAL distance matrix:

         To:  0     1     2     3
    From:
      0  [   0     3     6     4   ]
      1  [  INF    0     3     1   ]
      2  [  INF   INF    0    INF  ]
      3  [  INF   INF    2     0   ]

    Now we know the shortest path between EVERY pair!
    e.g., 0 -> 2: shortest = 6 (path: 0->1->3->2)

Algorithm Steps

  1. Create a V x V matrix. Initialize dist[i][j] = weight of edge i->j (or INF if no edge). Set dist[i][i] = 0.
  2. For each intermediate vertex k (0 to V-1):
  3. For each pair (i, j): dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
  4. After all k values, dist[i][j] holds the shortest path from i to j.
  5. (Optional) Detect negative cycles: if any dist[i][i] < 0, there is a negative cycle.
Why does k go in the outer loop? The order matters! When we consider intermediate vertex k, we need all paths using vertices {0, 1, ..., k-1} to already be computed. Putting k in the outermost loop guarantees this property.

Implementation

import java.util.*;

class FloydWarshall {
    static final int INF = 100000000; // Use large number (not MAX_VALUE to avoid overflow)

    static void floydWarshall(int[][] dist, int V) {
        // k = intermediate vertex
        for (int k = 0; k < V; k++) {
            // i = source vertex
            for (int i = 0; i < V; i++) {
                // j = destination vertex
                for (int j = 0; j < V; j++) {
                    // Is path through k shorter?
                    if (dist[i][k] + dist[k][j] < dist[i][j]) {
                        dist[i][j] = dist[i][k] + dist[k][j];
                    }
                }
            }
        }

        // Check for negative cycles
        for (int i = 0; i < V; i++) {
            if (dist[i][i] < 0) {
                System.out.println("Negative cycle detected!");
                return;
            }
        }

        // Print the distance matrix
        for (int i = 0; i < V; i++) {
            for (int j = 0; j < V; j++) {
                if (dist[i][j] == INF) System.out.print("INF ");
                else System.out.print(dist[i][j] + " ");
            }
            System.out.println();
        }
    }

    public static void main(String[] args) {
        int V = 4;
        int[][] dist = {
            {0,   3,   7,   INF},
            {INF, 0,   INF, 1},
            {INF, INF, 0,   INF},
            {INF, INF, 2,   0}
        };

        floydWarshall(dist, V);
        // Output:
        // 0 3 6 4
        // INF 0 3 1
        // INF INF 0 INF
        // INF INF 2 0
    }
}

    for (int k = 0; k < V; k++) {
        // i = source vertex
        for (int i = 0; i < V; i++) {
            // j = destination vertex

def floyd_warshall(V, graph):
    # graph[i][j] = weight of edge i->j, or float('inf') if no edge
    INF = float('inf')
    # Copy input to dist matrix
    dist = [row[:] for row in graph]

    # dp[i][j] = min(dp[i][j], dp[i][k] + dp[k][j])
    for k in range(V):
        for i in range(V):
            for j in range(V):
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]
    return dist

# Example
INF = float('inf')
graph = [
    [0,   3,   INF, 5],
    [2,   0,   INF, 4],
    [INF, 1,   0,   INF],
    [INF, INF, 2,   0]
]
result = floyd_warshall(4, graph)
for row in result:
    print(row)

Time and Space Complexity

  • Time: O(V^3) -- three nested loops, each running V times.
  • Space: O(V^2) -- for the distance matrix.

When to Use Floyd-Warshall

  • Small graphs (V up to ~400-500) where V^3 is manageable.
  • When you need all-pairs shortest paths (not just from one source).
  • Dense graphs (many edges) -- for sparse graphs, running Dijkstra from each vertex might be faster.
  • Graphs with negative edges (but no negative cycles).
  • Problems like: "Can city A reach city B within budget X?"

Comparison: All Three Algorithms

    +-------------------+------------+---------------+----------------+
    | Algorithm         | Dijkstra   | Bellman-Ford  | Floyd-Warshall |
    +-------------------+------------+---------------+----------------+
    | Problem type      | Single-src | Single-src    | All-pairs      |
    | Negative weights  | NO         | YES           | YES            |
    | Negative cycles   | NO         | Detects       | Detects        |
    | Time complexity   | O((V+E)lgV)| O(V*E)        | O(V^3)         |
    | Space             | O(V+E)    | O(V+E)        | O(V^2)         |
    | Best for          | Sparse,    | Neg weights,  | Small graphs,  |
    |                   | non-neg    | cycle detect  | all-pairs      |
    +-------------------+------------+---------------+----------------+
Path Reconstruction: To find the actual path (not just distances), maintain a "next" matrix. Set next[i][j] = j initially, and when dist[i][j] is updated through k, set next[i][j] = next[i][k]. Then reconstruct the path by following next pointers.

Key Takeaways

  1. Three nested loops -- simple to implement, easy to remember.
  2. The outer loop MUST be the intermediate vertex k.
  3. Works for both directed and undirected graphs.
  4. O(V^3) limits it to smaller graphs (V <= ~500).
  5. Use a large constant for INF (not INT_MAX) to avoid integer overflow when adding.
  6. Negative cycle exists if dist[i][i] < 0 for any vertex i.
