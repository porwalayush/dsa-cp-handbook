Bellman-Ford Algorithm

Bellman-Ford finds the shortest path from a single source to all other vertices, just like Dijkstra. But unlike Dijkstra, it can handle negative edge weights and can detect negative-weight cycles.

Core Idea: Relaxation

"Relaxation" means: if we find a shorter path to a node, we update its distance. Bellman-Ford simply relaxes ALL edges, and repeats this V-1 times.

    What is "relaxing" an edge (u, v, weight)?

    Before:  dist[u] = 3,  dist[v] = 10,  weight = 2

    Check:   dist[u] + weight < dist[v]?
             3 + 2 = 5 < 10?  YES!

    After:   dist[v] = 5   (updated/relaxed!)

    Intuition: "Can I reach v faster by going through u?"

Why V-1 Iterations?

In a graph with V vertices, the shortest path between any two vertices can have at most V-1 edges (otherwise we'd revisit a vertex, creating a cycle). Each iteration guarantees that paths using one more edge are correctly computed.

    Example: Longest shortest path has V-1 = 3 edges

    S ---> A ---> B ---> C
    (4 vertices, path uses 3 edges)

    Iteration 1: Correct distances for paths with 1 edge
    Iteration 2: Correct distances for paths with 2 edges
    Iteration 3: Correct distances for paths with 3 edges
                  (all shortest paths now finalized)

Visual Walkthrough

    Graph (directed, with a negative edge):

    (S)---5--->(A)---(-2)--->(B)
     |                         ^
     |                         |
     +----8---->(C)----3------+

    Vertices: S, A, B, C    (V = 4, so V-1 = 3 iterations)
    Edges: S->A(5), A->B(-2), S->C(8), C->B(3)

    Initial: dist = [S:0, A:INF, B:INF, C:INF]

    --- Iteration 1 (relax all edges) ---
    Edge S->A: dist[S]+5 = 5 < INF   -> dist[A] = 5
    Edge A->B: dist[A]+(-2) = 3 < INF -> dist[B] = 3
    Edge S->C: dist[S]+8 = 8 < INF   -> dist[C] = 8
    Edge C->B: dist[C]+3 = 11 > 3    -> no change

    dist = [S:0, A:5, B:3, C:8]

    --- Iteration 2 (relax all edges again) ---
    Edge S->A: 0+5 = 5, not < 5      -> no change
    Edge A->B: 5+(-2) = 3, not < 3   -> no change
    Edge S->C: 0+8 = 8, not < 8      -> no change
    Edge C->B: 8+3 = 11, not < 3     -> no change

    No changes! Algorithm converged early.

    Final: dist = [S:0, A:5, B:3, C:8]

Detecting Negative Cycles

After V-1 iterations, all shortest paths are finalized (if no negative cycle exists). So we do one MORE pass over all edges. If any distance can still be reduced, it means there's a negative-weight cycle reachable from the source.

    Negative cycle example:

    (A)---(-1)--->(B)
     ^             |
     |             |
     +---(-1)------+
          and
    (B)---(-1)--->(A)

    Cycle: A -> B -> A,  total weight = -1 + (-1) = -2
    Each trip around the cycle reduces distance by 2!

    After V-1 iterations, we can STILL relax edges
    = negative cycle detected!

Algorithm Steps

  1. Initialize dist[source] = 0, dist[all others] = INF.
  2. Repeat V-1 times: for every edge (u, v, w), if dist[u] + w < dist[v], update dist[v].
  3. Do one more pass: if any edge can still be relaxed, a negative cycle exists.

Implementation

import java.util.*;

class BellmanFord {
    // Edge represented as [from, to, weight]
    static int[] bellmanFord(int V, int[][] edges, int src) {
        int[] dist = new int[V];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[src] = 0;

        // Step 1: Relax all edges V-1 times
        for (int i = 0; i < V - 1; i++) {
            boolean updated = false;
            for (int[] edge : edges) {
                int u = edge[0], v = edge[1], w = edge[2];
                if (dist[u] != Integer.MAX_VALUE && dist[u] + w < dist[v]) {
                    dist[v] = dist[u] + w;
                    updated = true;
                }
            }
            // Early termination: no updates means we're done
            if (!updated) break;
        }

        // Step 2: Check for negative-weight cycles
        for (int[] edge : edges) {
            int u = edge[0], v = edge[1], w = edge[2];
            if (dist[u] != Integer.MAX_VALUE && dist[u] + w < dist[v]) {
                System.out.println("Negative cycle detected!");
                return null;
            }
        }

        return dist;
    }

    public static void main(String[] args) {
        int V = 4;
        int[][] edges = {
            {0, 1, 5},   // S -> A, weight 5
            {1, 2, -2},  // A -> B, weight -2
            {0, 3, 8},   // S -> C, weight 8
            {3, 2, 3}    // C -> B, weight 3
        };

        int[] dist = bellmanFord(V, edges, 0);
        if (dist != null) {
            System.out.println(Arrays.toString(dist));
            // Output: [0, 5, 3, 8]
        }
    }
}
#include <bits/stdc++.h>
using namespace std;

struct Edge {
    int u, v, w;
};

vector<int> bellmanFord(int V, vector<Edge>& edges, int src) {
    vector<int> dist(V, INT_MAX);
    dist[src] = 0;

    // Step 1: Relax all edges V-1 times
    for (int i = 0; i < V - 1; i++) {
        bool updated = false;
        for (auto& e : edges) {
            if (dist[e.u] != INT_MAX && dist[e.u] + e.w < dist[e.v]) {
                dist[e.v] = dist[e.u] + e.w;
                updated = true;
            }
        }
        // Early termination
        if (!updated) break;
    }

    // Step 2: Check for negative-weight cycles
    for (auto& e : edges) {
        if (dist[e.u] != INT_MAX && dist[e.u] + e.w < dist[e.v]) {
            cout << "Negative cycle detected!" << endl;
            return {};
        }
    }

    return dist;
}

int main() {
    int V = 4;
    vector<Edge> edges = {
        {0, 1, 5},   // S -> A, weight 5
        {1, 2, -2},  // A -> B, weight -2
        {0, 3, 8},   // S -> C, weight 8
        {3, 2, 3}    // C -> B, weight 3
    };

    vector<int> dist = bellmanFord(V, edges, 0);
    if (!dist.empty()) {
        for (int d : dist) cout << d << " ";
        // Output: 0 5 3 8
    }
    return 0;
}
def bellman_ford(V, edges, src):
    # edges = [(u, v, weight), ...]
    dist = [float('inf')] * V
    dist[src] = 0

    # Relax all edges V-1 times
    for _ in range(V - 1):
        for u, v, w in edges:
            if dist[u] != float('inf') and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w

    # Check for negative cycle (V-th iteration)
    for u, v, w in edges:
        if dist[u] != float('inf') and dist[u] + w < dist[v]:
            print("Negative cycle detected!")
            return None

    return dist

# Example
edges = [(0,1,4), (0,2,5), (1,2,-3), (2,3,4)]
print(bellman_ford(4, edges, 0))  # [0, 4, 1, 5]

Time and Space Complexity

  • Time: O(V * E) - we iterate V-1 times over all E edges.
  • Space: O(V) for the distance array (edges stored as input).

Dijkstra vs. Bellman-Ford

    +--------------------+-----------------+------------------+
    | Feature            | Dijkstra        | Bellman-Ford     |
    +--------------------+-----------------+------------------+
    | Negative weights   | NO              | YES              |
    | Negative cycle     | Cannot detect   | CAN detect       |
    | Time complexity    | O((V+E) log V)  | O(V * E)         |
    | Approach           | Greedy          | Dynamic Prog.    |
    | Data structure     | Priority Queue  | Edge list        |
    | Best for           | Sparse, no neg  | Negative weights |
    +--------------------+-----------------+------------------+

    Rule of thumb:
    - All weights >= 0?  Use Dijkstra (faster)
    - Negative weights?  Use Bellman-Ford
    - Need negative cycle detection?  Use Bellman-Ford
Tip: Bellman-Ford can be optimized with early termination. If no distance is updated in an entire iteration, the algorithm has converged and you can stop early. This is already included in the code above.

Key Takeaways

  1. Relax all edges V-1 times (handles paths up to V-1 edges long).
  2. One extra pass detects negative cycles.
  3. Slower than Dijkstra but more versatile (handles negative weights).
  4. Uses a simple edge list -- no fancy data structures needed.
  5. Common in problems involving currency exchange, arbitrage detection.
