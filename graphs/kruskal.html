Kruskal's Algorithm

Kruskal's algorithm finds the Minimum Spanning Tree (MST) of a weighted, connected, undirected graph. It works by sorting all edges by weight and greedily adding them one by one, skipping any edge that would create a cycle.

How It Works (Intuition)

Imagine you have a bunch of islands (vertices) and a list of bridges (edges) you could build, each with a cost. You want to connect all islands with minimum total cost. Kruskal's strategy:

  1. Sort all edges by weight (cheapest first).
  2. Go through edges one by one. For each edge:
    • If it connects two different components (no cycle) -- add it to MST.
    • If it connects vertices already in the same component (would form cycle) -- skip it.
  3. Stop when you have V-1 edges (a tree with V vertices always has V-1 edges).
Cycle Detection: We use Disjoint Set Union (Union-Find / DSU) to efficiently check if two vertices are in the same component. See the DSU page for details.

Implementation with DSU

import java.util.*;

class KruskalsAlgorithm {

    static int[] parent, rank;

    // Find with path compression
    static int find(int x) {
        if (parent[x] != x)
            parent[x] = find(parent[x]);
        return parent[x];
    }

    // Union by rank
    static boolean union(int x, int y) {
        int px = find(x), py = find(y);
        if (px == py) return false; // Already same set

        if (rank[px] < rank[py]) { parent[px] = py; }
        else if (rank[px] > rank[py]) { parent[py] = px; }
        else { parent[py] = px; rank[px]++; }
        return true;
    }

    static int kruskalMST(int V, int[][] edges) {
        // Initialize DSU
        parent = new int[V];
        rank = new int[V];
        for (int i = 0; i < V; i++) parent[i] = i;

        // Sort edges by weight
        Arrays.sort(edges, (a, b) -> a[2] - b[2]);

        int totalWeight = 0;
        int edgesUsed = 0;

        for (int[] edge : edges) {
            int u = edge[0], v = edge[1], w = edge[2];

            // If u and v are in different components, add edge
            if (union(u, v)) {
                totalWeight += w;
                edgesUsed++;
                System.out.println("Added edge: " + u + " - " + v + " (weight " + w + ")");
                if (edgesUsed == V - 1) break; // MST complete
            }
        }
        return totalWeight;
    }

    public static void main(String[] args) {
        int V = 5;
        // edges[i] = {u, v, weight}
        int[][] edges = {
            {0, 1, 2}, {1, 3, 3}, {2, 3, 4},
            {1, 2, 6}, {3, 4, 7}, {0, 3, 8}
        };

        System.out.println("MST weight: " + kruskalMST(V, edges));
        // Output: MST weight: 16
    }
}
#include <bits/stdc++.h>
using namespace std;

int parent[100005], rnk[100005];

// Find with path compression
int find(int x) {
    if (parent[x] != x)
        parent[x] = find(parent[x]);
    return parent[x];
}

// Union by rank - returns true if merged
bool unite(int x, int y) {
    int px = find(x), py = find(y);
    if (px == py) return false;

    if (rnk[px] < rnk[py]) swap(px, py);
    parent[py] = px;
    if (rnk[px] == rnk[py]) rnk[px]++;
    return true;
}

int main() {
    int V = 5, E = 6;

    // edges: {weight, u, v}
    vector<tuple<int,int,int>> edges = {
        {2, 0, 1}, {3, 1, 3}, {4, 2, 3},
        {6, 1, 2}, {7, 3, 4}, {8, 0, 3}
    };

    // Initialize DSU
    for (int i = 0; i < V; i++) { parent[i] = i; rnk[i] = 0; }

    // Sort edges by weight
    sort(edges.begin(), edges.end());

    int totalWeight = 0, edgesUsed = 0;

    for (auto& [w, u, v] : edges) {
        if (unite(u, v)) {
            totalWeight += w;
            edgesUsed++;
            cout << "Added edge: " << u << " - " << v
                 << " (weight " << w << ")" << endl;
            if (edgesUsed == V - 1) break;
        }
    }

    cout << "MST weight: " << totalWeight << endl;
    // Output: MST weight: 16
    return 0;
}
def find(parent, x):
    if parent[x] != x:
        parent[x] = find(parent, parent[x])
    return parent[x]

def union(parent, rank, x, y):
    px, py = find(parent, x), find(parent, y)
    if px == py:
        return False
    if rank[px] < rank[py]:
        parent[px] = py
    elif rank[px] > rank[py]:
        parent[py] = px
    else:
        parent[py] = px
        rank[px] += 1
    return True

def kruskal(V, edges):
    # edges = [(u, v, weight), ...]
    edges.sort(key=lambda x: x[2])  # sort by weight
    parent = list(range(V))
    rank = [0] * V
    mst_cost = 0
    mst_edges = []

    for u, v, w in edges:
        if union(parent, rank, u, v):
            mst_cost += w
            mst_edges.append((u, v, w))

    return mst_cost, mst_edges

Time and Space Complexity

  • Time: O(E log E) dominated by sorting. The DSU operations (find + union) are nearly O(1) each with path compression and union by rank. Processing all edges after sorting is O(E * alpha(V)) which is essentially O(E).
  • Space: O(V + E). O(V) for the DSU arrays, O(E) for storing the edge list.

Kruskal's vs Prim's

  • Kruskal's works with an edge list, sorts globally -- best for sparse graphs where E is much less than V^2.
  • Prim's works with adjacency list/matrix, grows locally -- best for dense graphs.
  • Kruskal's is simpler to implement if you already have DSU as a template.
  • Kruskal's can handle disconnected graphs naturally (it just produces a minimum spanning forest).

Key Insight: Why It Works

Kruskal's correctness follows from the Cut Property: for any cut (partition of vertices into two sets), the minimum weight edge crossing the cut must be in some MST. Since we process edges in sorted order, whenever we add an edge connecting two different components, it is the lightest edge crossing that particular cut.
