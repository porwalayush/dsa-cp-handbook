Sum over Subsets DP (SOS DP)

For each bitmask, efficiently compute the sum (or count, min, max) over all its submasks. SOS DP reduces the naive O(3n) approach to O(n · 2n) by iterating over bit positions and applying inclusion/exclusion dimension by dimension.

Problem Statement

Given an array f[0..2^n - 1], compute for every mask:

dp[mask] = ∑ f[sub]  for all sub where (sub & mask) == sub
                        (i.e., sub is a submask of mask)

Example: n = 3, mask = 110 (binary)
  Submasks of 110: {110, 100, 010, 000}
  dp[110] = f[110] + f[100] + f[010] + f[000]

Naive Approach: O(3n)

Brute Force: For each mask, enumerate all submasks using the standard submask enumeration trick. Total work across all masks is O(3n) because each element is either: (1) in mask and in sub, (2) in mask but not in sub, or (3) not in mask. Three choices per bit gives 3n total.

SOS DP: O(n · 2n)

Key Idea: Process one bit at a time. At step i, dp[mask] accounts for all submasks that differ from mask only in the first i+1 bit positions.

State: dp[mask] = sum of f[sub] over all submasks of mask

Transition:
for i = 0 to n-1:
  for each mask with bit i set:
    dp[mask] += dp[mask ^ (1 << i)]

Base Case: dp[mask] = f[mask] for all mask initially

Intuition: At each step i, we "relax" bit i: dp[mask] absorbs contributions from the submask where bit i is turned off.

Visual: How Bits Propagate

n = 3, processing bits 0, 1, 2 in order:

Initial:  dp[mask] = f[mask]

After bit 0 (relax position 0):
  dp[001] += dp[000]   (001 absorbs 000)
  dp[011] += dp[010]   (011 absorbs 010)
  dp[101] += dp[100]   (101 absorbs 100)
  dp[111] += dp[110]   (111 absorbs 110)

After bit 1 (relax position 1):
  dp[010] += dp[000]   (010 absorbs 000)
  dp[011] += dp[001]   (011 absorbs 001, which already has 000)
  dp[110] += dp[100]   (110 absorbs 100)
  dp[111] += dp[101]   (111 absorbs 101, which already has 100)

After bit 2 (relax position 2):
  dp[100] += dp[000]   (100 absorbs 000)
  dp[101] += dp[001]   (101 absorbs 001+000)
  dp[110] += dp[010]   (110 absorbs 010+000)
  dp[111] += dp[011]   (111 absorbs 011+010+001+000)

Result: dp[111] = f[111]+f[110]+f[101]+f[100]+f[011]+f[010]+f[001]+f[000]
        dp[110] = f[110]+f[100]+f[010]+f[000]
        dp[101] = f[101]+f[100]+f[001]+f[000]
        etc.

Step-by-Step Example

Example: n=3, f = [1, 2, 3, 4, 5, 6, 7, 8] (indices 000 to 111)
Step 1 (bit 0):
dp[001]=f[001]+f[000]=2+1=3, dp[011]=f[011]+f[010]=4+3=7
dp[101]=f[101]+f[100]=6+5=11, dp[111]=f[111]+f[110]=8+7=15
Step 2 (bit 1):
dp[010]=f[010]+dp[000]=3+1=4, dp[011]=7+dp[001]=7+3=10
dp[110]=f[110]+dp[100]=7+5=12, dp[111]=15+dp[101]=15+11=26
Step 3 (bit 2):
dp[100]=f[100]+dp[000]=5+1=6, dp[101]=11+dp[001]=11+3=14
dp[110]=12+dp[010]=12+4=16, dp[111]=26+dp[011]=26+10=36
Verify: dp[111] = 1+2+3+4+5+6+7+8 = 36 (sum of all f values)

Code: Sum over Subsets DP

// Sum over Subsets DP
// Time: O(n * 2^n), Space: O(2^n)
void sosDp(int n, vector<long long>& dp) {
    // dp[mask] initially holds f[mask]
    // After SOS, dp[mask] = sum of f[sub] for all submasks sub of mask
    for (int i = 0; i < n; i++) {
        for (int mask = 0; mask < (1 << n); mask++) {
            if (mask & (1 << i)) {
                dp[mask] += dp[mask ^ (1 << i)];
            }
        }
    }
}

// Example usage: count pairs (i,j) where a[i] & a[j] == a[j]
// (i.e., a[j] is a submask of a[i])
int main() {
    int n = 20; // bit width
    vector<long long> cnt(1 << n, 0);

    // Count frequency of each value
    for (int x : arr) cnt[x]++;

    // SOS: cnt[mask] = number of elements that are submasks of mask
    sosDp(n, cnt);

    // For each element a[i], cnt[a[i]] gives count of a[j]
    // such that a[j] is a submask of a[i]
    long long answer = 0;
    for (int x : arr) answer += cnt[x] - 1;
    return 0;
}

// Sum over Subsets DP
// Time: O(n * 2^n), Space: O(2^n)
static void sosDp(int n, long[] dp) {
    // dp[mask] initially holds f[mask]
    // After SOS, dp[mask] = sum of f[sub] for all submasks sub of mask
    for (int i = 0; i < n; i++) {
        for (int mask = 0; mask < (1 << n); mask++) {
            if ((mask & (1 << i)) != 0) {
                dp[mask] += dp[mask ^ (1 << i)];
            }
        }
    }
}

Inverse SOS: Sum over Supermasks

Variant: Compute dp[mask] = sum of f[sup] for all supermasks sup of mask (i.e., masks that contain mask as a submask).

Transition: Flip the condition — aggregate when bit i is NOT set:
if !(mask & (1 << i)): dp[mask] += dp[mask | (1 << i)] 
// Inverse SOS: sum over supermasks
// dp[mask] = sum of f[sup] where sup is a supermask of mask
void inverseSosDp(int n, vector<long long>& dp) {
    for (int i = 0; i < n; i++) {
        for (int mask = 0; mask < (1 << n); mask++) {
            if (!(mask & (1 << i))) {
                dp[mask] += dp[mask | (1 << i)];
            }
        }
    }
}

// Inverse SOS: sum over supermasks
static void inverseSosDp(int n, long[] dp) {
    for (int i = 0; i < n; i++) {
        for (int mask = 0; mask < (1 << n); mask++) {
            if ((mask & (1 << i)) == 0) {
                dp[mask] += dp[mask | (1 << i)];
            }
        }
    }
}

# Inverse SOS: sum over supermasks
def inverse_sos_dp(n, dp):
    for i in range(n):
        for mask in range(1 << n):
            if not (mask & (1 << i)):
                dp[mask] += dp[mask | (1 << i)]

Application: AND Convolution

Problem: Given arrays a[] and b[], compute c[mask] = sum of a[x]*b[y] for all (x,y) where x & y == mask.

Solution via SOS:
1. Compute A[mask] = SOS(a) and B[mask] = SOS(b)
2. Multiply: C[mask] = A[mask] * B[mask]
3. Apply inverse (Mobius inversion) to get c[mask]

Similarly, OR convolution uses the supermask variant.

Complexity

  • Time: O(n · 2n) — n passes, each visiting 2n masks
  • Space: O(2n) — single dp array, computed in-place
  • Comparison: Naive submask enumeration is O(3n). For n=20: SOS = 20M ops vs naive = 3.5B ops
  • Practical limit: n ≤ 20–22 (memory: 222 = 4M entries)

Key Insights

  • Core principle: SOS DP is the bitmask analogue of prefix sums — it "accumulates" over the subset lattice dimension by dimension
  • Direction matters: Processing submasks vs supermasks gives different transforms (Zeta vs Mobius)
  • Composable: SOS can be combined with other bitmask DPs; first compute SOS, then use results in further DP
  • Works for any associative operation: Replace + with max, min, XOR, etc. as needed
