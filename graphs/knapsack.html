0/1 Knapsack

Given n items with weights and values, find the maximum value you can carry in a knapsack of capacity W. Each item can be taken at most once.

Problem Formulation

   Items:  weight = [1, 3, 4, 5]
           value  = [1, 4, 5, 7]
   Capacity W = 7

   Choices for each item: TAKE or SKIP

   Take items 2,3 (0-indexed 1,2): weight=3+4=7, value=4+5=9
   Take items 2,4 (0-indexed 1,3): weight=3+5=8 > 7 ✗
   Take items 1,2,3: weight=1+3+4=8 > 7 ✗
   Take items 1,2: weight=1+3=4, value=1+4=5
   Take items 3,4 (0-indexed 2,3): weight=4+5=9 > 7 ✗

   Best: Take items with weight 3 and 4 → value = 4+5 = 9 ✓

State & Transition

State: dp[i][w] = max value using first i items with capacity w

Transition:
If weight[i] > w: dp[i][w] = dp[i-1][w] (can't take item i)
Else: dp[i][w] = max(dp[i-1][w], dp[i-1][w-weight[i]] + value[i])
      (skip item i) vs (take item i)

Base: dp[0][w] = 0 for all w (no items = no value)

Code: 0/1 Knapsack

// 2D DP: O(n*W) time, O(n*W) space
int knapsack(vector<int>& wt, vector<int>& val, int W) {
    int n = wt.size();
    vector<vector<int>> dp(n + 1, vector<int>(W + 1, 0));

    for (int i = 1; i <= n; i++) {
        for (int w = 0; w <= W; w++) {
            dp[i][w] = dp[i-1][w]; // skip item
            if (wt[i-1] <= w) {
                dp[i][w] = max(dp[i][w], dp[i-1][w - wt[i-1]] + val[i-1]);
            }
        }
    }
    return dp[n][W];
}

// Space optimized: O(W) space (iterate W backwards!)
int knapsackOpt(vector<int>& wt, vector<int>& val, int W) {
    int n = wt.size();
    vector<int> dp(W + 1, 0);

    for (int i = 0; i < n; i++) {
        for (int w = W; w >= wt[i]; w--) { // RIGHT to LEFT!
            dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);
        }
    }
    return dp[W];
}

// 2D DP: O(n*W) time, O(n*W) space
static int knapsack(int[] wt, int[] val, int W) {
    int n = wt.length;
    int[][] dp = new int[n + 1][W + 1];

    for (int i = 1; i <= n; i++) {
        for (int w = 0; w <= W; w++) {
            dp[i][w] = dp[i-1][w]; // skip
            if (wt[i-1] <= w) {
                dp[i][w] = Math.max(dp[i][w],
                    dp[i-1][w - wt[i-1]] + val[i-1]);
            }
        }
    }
    return dp[n][W];
}

# 2D DP: O(n*W) time, O(n*W) space
def knapsack(wt, val, W):
    n = len(wt)
    dp = [[0] * (W + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        for w in range(W + 1):
            dp[i][w] = dp[i-1][w]  # skip
            if wt[i-1] <= w:
                dp[i][w] = max(dp[i][w],
                    dp[i-1][w - wt[i-1]] + val[i-1])
    return dp[n][W]

# Space optimized: O(W) space
def knapsack_opt(wt, val, W):
    dp = [0] * (W + 1)
    for i in range(len(wt)):
        for w in range(W, wt[i] - 1, -1):  # RIGHT to LEFT!
            dp[w] = max(dp[w], dp[w - wt[i]] + val[i])
    return dp[W]

Why Iterate Backwards in 1D?

   If we go LEFT to RIGHT:
   dp[w] = max(dp[w], dp[w - wt[i]] + val[i])

   Problem: dp[w - wt[i]] might already be updated for item i!
   → Same item used multiple times (this is UNBOUNDED knapsack)

   Going RIGHT to LEFT:
   When we compute dp[w], dp[w - wt[i]] still has the value
   from the previous row (item i-1) → item i used at most once ✓

   0/1 Knapsack: iterate w from W → 0
   Unbounded:    iterate w from 0 → W

Complexity

  • Time: O(n * W) — pseudo-polynomial (depends on value of W)
  • Space: O(n * W) for 2D, O(W) with space optimization
  • Note: NP-hard problem! No known polynomial time algorithm
