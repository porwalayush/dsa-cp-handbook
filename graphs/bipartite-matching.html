Bipartite Matching

Given a bipartite graph with two disjoint sets of vertices L and R, a matching is a set of edges with no shared vertices. The maximum matching is the largest such set. This has wide applications from job assignment to resource allocation.

Problem Statement

Given a bipartite graph G = (L + R, E) where edges only go between L and R, find the maximum number of edges such that no vertex appears more than once.

Visual Walkthrough

    Bipartite Graph:          Maximum Matching:

    Workers    Jobs           Workers    Jobs
    -------    ----           -------    ----
    Alice  --- Web Dev        Alice  === Web Dev
    Alice  --- Backend
    Bob    --- Backend        Bob    === Backend
    Bob    --- Database
    Carol  --- Database       Carol  === Database
    Carol  --- Web Dev

    Maximum matching size = 3

    Augmenting Path Example:
    ─────────────────────────
    Current matching: Alice-WebDev, Bob-Backend

    Carol wants Database... free! Match Carol-Database.
    Done! Matching size = 3.

    But what if Carol only connects to WebDev?
    Carol -> WebDev (matched to Alice)
          -> Alice -> Backend (matched to Bob)
                   -> Bob -> Database (FREE!)

    Augmenting path found! Rearrange:
    Carol-WebDev, Alice-Backend, Bob-Database
    Matching size = 3

Kuhn's Algorithm (Hungarian Method for Unweighted)

Kuhn's algorithm finds maximum matching by trying to find augmenting paths from each unmatched left vertex using DFS.

Step 1: For each unmatched vertex u in L, try to find an augmenting path via DFS.
Step 2: In DFS from u, try each neighbor v in R. If v is unmatched, we found an augmenting path. If v is matched to some w, recursively try to find an augmenting path from w.
Step 3: If augmenting path found, flip all edges along it (matched becomes unmatched and vice versa).

Hopcroft-Karp Algorithm

Optimizes Kuhn's by finding multiple augmenting paths simultaneously using BFS to find shortest augmenting paths, then DFS to augment along them.

  • BFS builds a layered graph of shortest augmenting paths.
  • DFS finds vertex-disjoint augmenting paths in the layered graph.
  • Each phase increases matching by at least 1, and there are at most O(sqrt(V)) phases.
  • Total time: O(E * sqrt(V)).

Implementation (Kuhn's Algorithm)

#include <bits/stdc++.h>
using namespace std;

int n, m; // n = left vertices, m = right vertices
vector<vector<int>> adj; // adj[u] = list of right vertices u connects to
vector<int> match_l, match_r; // match_l[u] = matched right vertex (-1 if unmatched)
vector<bool> visited;

bool dfs(int u) {
    for (int v : adj[u]) {
        if (visited[v]) continue;
        visited[v] = true;

        // If v is free OR we can reroute v's current match
        if (match_r[v] == -1 || dfs(match_r[v])) {
            match_l[u] = v;
            match_r[v] = u;
            return true;
        }
    }
    return false;
}

int maxMatching() {
    match_l.assign(n, -1);
    match_r.assign(m, -1);
    int result = 0;

    for (int u = 0; u < n; u++) {
        visited.assign(m, false);
        if (dfs(u)) result++;
    }
    return result;
}

int main() {
    n = 3; m = 3; // 3 workers, 3 jobs
    adj.resize(n);

    // Alice(0) -> WebDev(0), Backend(1)
    adj[0] = {0, 1};
    // Bob(1) -> Backend(1), Database(2)
    adj[1] = {1, 2};
    // Carol(2) -> Database(2), WebDev(0)
    adj[2] = {2, 0};

    cout << maxMatching() << endl; // Output: 3
    return 0;
}
import java.util.*;

class BipartiteMatching {
    static int n, m;
    static List<List<Integer>> adj;
    static int[] matchL, matchR;
    static boolean[] visited;

    static boolean dfs(int u) {
        for (int v : adj.get(u)) {
            if (visited[v]) continue;
            visited[v] = true;

            // If v is free OR we can reroute v's match
            if (matchR[v] == -1 || dfs(matchR[v])) {
                matchL[u] = v;
                matchR[v] = u;
                return true;
            }
        }
        return false;
    }

    static int maxMatching() {
        matchL = new int[n];
        matchR = new int[m];
        Arrays.fill(matchL, -1);
        Arrays.fill(matchR, -1);
        int result = 0;

        for (int u = 0; u < n; u++) {
            visited = new boolean[m];
            if (dfs(u)) result++;
        }
        return result;
    }

    public static void main(String[] args) {
        n = 3; m = 3;
        adj = new ArrayList<>();
        for (int i = 0; i < n; i++) adj.add(new ArrayList<>());

        // Alice(0) -> WebDev(0), Backend(1)
        adj.get(0).addAll(Arrays.asList(0, 1));
        // Bob(1) -> Backend(1), Database(2)
        adj.get(1).addAll(Arrays.asList(1, 2));
        // Carol(2) -> Database(2), WebDev(0)
        adj.get(2).addAll(Arrays.asList(2, 0));

        System.out.println(maxMatching()); // Output: 3
    }
}
def max_bipartite_matching(n, m, adj):
    """
    n = number of left vertices (0..n-1)
    m = number of right vertices (0..m-1)
    adj[u] = list of right vertices u connects to
    Returns: maximum matching size
    """
    match_l = [-1] * n
    match_r = [-1] * m

    def dfs(u, visited):
        for v in adj[u]:
            if visited[v]:
                continue
            visited[v] = True

            # If v is free OR we can reroute v's match
            if match_r[v] == -1 or dfs(match_r[v], visited):
                match_l[u] = v
                match_r[v] = u
                return True
        return False

    result = 0
    for u in range(n):
        visited = [False] * m
        if dfs(u, visited):
            result += 1
    return result

# Example: 3 workers, 3 jobs
n, m = 3, 3
adj = [
    [0, 1],  # Alice -> WebDev, Backend
    [1, 2],  # Bob -> Backend, Database
    [2, 0],  # Carol -> Database, WebDev
]

print(max_bipartite_matching(n, m, adj))  # Output: 3

Hungarian Algorithm (Weighted Matching)

When edges have weights and we want to find the maximum weight perfect matching (or minimum cost assignment), we use the Hungarian algorithm.

  • Works on a cost matrix where cost[i][j] = cost of assigning worker i to job j.
  • Finds the assignment that minimizes (or maximizes) total cost.
  • Time complexity: O(n^3) for n x n matrix.

Konig's Theorem

Konig's Theorem: In any bipartite graph, the size of the maximum matching equals the size of the minimum vertex cover.
  • Minimum Vertex Cover: Smallest set of vertices that touches every edge.
  • Maximum Independent Set = Total vertices - Minimum Vertex Cover.
  • This does NOT hold for general (non-bipartite) graphs!

Complexity Analysis

Algorithm Time Space Use Case
Kuhn's (DFS) O(V * E) O(V + E) Simple unweighted matching
Hopcroft-Karp O(E * sqrt(V)) O(V + E) Large unweighted graphs
Hungarian O(n^3) O(n^2) Weighted perfect matching
Max Flow reduction O(E * sqrt(V)) O(V + E) Alternative via Dinic's

Applications

  • Job assignment: Assign n workers to n jobs optimally.
  • Stable matching: Foundation for Gale-Shapley algorithm.
  • Vertex cover: Via Konig's theorem, find minimum vertex cover.
  • Edge coloring: Bipartite edge coloring uses matching decomposition.
  • Network design: Pairing servers to clients.
  • Scheduling: Assign time slots to tasks with constraints.

Key Takeaways

  1. Kuhn's algorithm is simple and sufficient for most problems.
  2. Hopcroft-Karp is better for large graphs (competitive programming).
  3. Bipartite matching can be reduced to max flow (source -> L -> R -> sink).
  4. Konig's theorem links matching to vertex cover in bipartite graphs.
  5. For weighted matching, use Hungarian algorithm O(n^3).
