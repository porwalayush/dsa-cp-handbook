Arrays & Hashing

The bread and butter of problem solving. Prefix sums answer range queries in O(1), and hash maps turn "have I seen this before?" into an O(1) lookup — together they crack a huge share of array problems.

Prefix Sums

   a      :  3   1   4   1   5   9
   prefix : 0  3   4   8   9  14  23     (prefix[i] = a[0..i-1])

   sum of a[l..r]  =  prefix[r+1] - prefix[l]
   e.g. a[1..3] = prefix[4] - prefix[1] = 9 - 3 = 6   (1+4+1)

   Build once in O(n) → every range sum is O(1).

Interactive: Subarray Sum = Target (with a hash map)

Classic trick: as we sweep, keep prefix sums in a hash map. At each index, if prefix − target was seen before, we found a subarray summing to the target. The blue window is the matched subarray.

Code: Prefix Sums & Subarray-Sum Hash

// Prefix sums: O(n) build, O(1) range sum
vector<long long> pre(n + 1, 0);
for (int i = 0; i < n; i++) pre[i+1] = pre[i] + a[i];
auto rangeSum = [&](int l, int r) { return pre[r+1] - pre[l]; };

// Count subarrays summing to k, using a hash map
int subarraySum(vector<int>& a, int k) {
    unordered_map<long long,int> seen{{0, 1}};
    long long sum = 0; int count = 0;
    for (int x : a) {
        sum += x;
        count += seen[sum - k];   // prefix(j) = sum - k seen?
        seen[sum]++;
    }
    return count;
}

// Two-sum: value -> index
vector<int> twoSum(vector<int>& a, int target) {
    unordered_map<int,int> idx;
    for (int i = 0; i < a.size(); i++) {
        if (idx.count(target - a[i])) return {idx[target - a[i]], i};
        idx[a[i]] = i;
    }
    return {};
}
// Prefix sums
long[] pre = new long[n + 1];
for (int i = 0; i < n; i++) pre[i+1] = pre[i] + a[i];
// range sum [l,r] = pre[r+1] - pre[l]

// Count subarrays summing to k
int subarraySum(int[] a, int k) {
    Map<Long,Integer> seen = new HashMap<>();
    seen.put(0L, 1);
    long sum = 0; int count = 0;
    for (int x : a) {
        sum += x;
        count += seen.getOrDefault(sum - k, 0);
        seen.merge(sum, 1, Integer::sum);
    }
    return count;
}

# Count subarrays summing to k
from collections import defaultdict
def subarray_sum(a, k):
    seen = defaultdict(int); seen[0] = 1
    total = count = 0
    for x in a:
        total += x
        count += seen[total - k]   # prefix == total - k seen?
        seen[total] += 1
    return count

# Two-sum
def two_sum(a, target):
    idx = {}
    for i, x in enumerate(a):
        if target - x in idx:
            return [idx[target - x], i]
        idx[x] = i

The Core Patterns

Prefix / Difference

  • Range sum in O(1)
  • Difference array for range updates
  • 2D prefix sums for submatrices

Hashing

  • Seen-before checks in O(1)
  • Frequency / counting maps
  • Grouping (anagrams, buckets)

The Difference Array (range updates)

  • To add v to all of a[l..r] many times, record diff[l] += v; diff[r+1] -= v.
  • A single prefix sum of diff at the end applies every update — O(1) per update, O(n) to finalize.
  • The mirror image of prefix sums: prefix answers queries, difference applies updates.

Go-To Problems

Two Sum / group anagrams — hash map lookups.

Subarray sum = k — prefix sums in a map.

Range sum queries (immutable) — prefix array.

Range add, query later — difference array.

First duplicate / missing number — set or in-place hashing.

Common Pitfalls

1. Prefix off-by-one: with pre[0]=0, range [l,r] is pre[r+1]−pre[l] — get the +1 right.

2. Seed the map with {0:1}: for subarray-sum-k, the empty prefix must count.

3. Overflow: prefix sums of large arrays need 64-bit.

4. Hash worst case: adversarial keys can make hashing O(n) — use a randomized hash for anti-hack safety.
