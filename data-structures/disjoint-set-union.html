Disjoint Set Union (DSU)

Also called Union-Find. It tracks a collection of disjoint groups and answers two questions blazingly fast: "are x and y in the same group?" and "merge the groups of x and y." With two optimizations, each op is effectively O(α(n)) ≈ O(1).

The Forest Idea

   Each element points to a PARENT. The root of a tree is the
   group's representative. Two elements are in the same group
   iff they share the same root.

   find(x)      — follow parents to the root
   union(x, y)  — attach one root under the other

   Two optimizations make it near-constant:
   • PATH COMPRESSION — find() flattens the path to the root
   • UNION BY RANK/SIZE — attach the smaller tree under the bigger

Code: DSU with Both Optimizations

struct DSU {
    vector<int> parent, sz;
    DSU(int n) : parent(n), sz(n, 1) {
        iota(parent.begin(), parent.end(), 0);  // each its own root
    }
    int find(int x) {
        if (parent[x] != x)
            parent[x] = find(parent[x]);   // PATH COMPRESSION
        return parent[x];
    }
    bool unite(int a, int b) {
        a = find(a); b = find(b);
        if (a == b) return false;          // already together
        if (sz[a] < sz[b]) swap(a, b);      // UNION BY SIZE
        parent[b] = a; sz[a] += sz[b];
        return true;
    }
    bool connected(int a, int b) { return find(a) == find(b); }
};
class DSU {
    int[] parent, size;
    DSU(int n) {
        parent = new int[n]; size = new int[n];
        for (int i = 0; i < n; i++) { parent[i] = i; size[i] = 1; }
    }
    int find(int x) {
        if (parent[x] != x) parent[x] = find(parent[x]); // compress
        return parent[x];
    }
    boolean unite(int a, int b) {
        a = find(a); b = find(b);
        if (a == b) return false;
        if (size[a] < size[b]) { int t = a; a = b; b = t; }
        parent[b] = a; size[a] += size[b];
        return true;
    }
    boolean connected(int a, int b) { return find(a) == find(b); }
}
class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size = [1] * n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]  # path halving
            x = self.parent[x]
        return x

    def union(self, a, b):
        a, b = self.find(a), self.find(b)
        if a == b:
            return False
        if self.size[a] < self.size[b]:
            a, b = b, a            # union by size
        self.parent[b] = a
        self.size[a] += self.size[b]
        return True

    def connected(self, a, b):
        return self.find(a) == self.find(b)

Why It's Nearly O(1)

  • Path compression flattens trees during find, so future finds are short.
  • Union by size/rank keeps trees shallow by attaching the smaller under the larger.
  • Together: O(α(n)) amortized, where α is the inverse Ackermann function — ≤ 4 for any n you'll ever see.

Where DSU Shines

Kruskal's MST — union edges in increasing weight, skip cycles.

Connected components — count groups, check connectivity.

Cycle detection in an undirected graph.

Offline queries — process unions in a chosen order.

DSU on tree, DSU with rollback — advanced variants.

Common Pitfalls

1. Comparing raw indices: always compare find(a) == find(b), not a == b or parents directly.

2. Skipping an optimization: path compression alone is O(log n) amortized; pair it with union-by-size for near-O(1).

3. Rollback needs no compression: "DSU with rollback" must use union-by-rank only (compression can't be undone cheaply).

4. Extra data per set: store aggregates (size, count) on the root and update during union.
