Sparse Table

For a static array, a sparse table answers range minimum/maximum (or GCD, OR, AND) in O(1) after an O(n log n) build. The catch: it only works for idempotent operations and the array can't change.

Powers-of-Two Coverage

   Precompute st[k][i] = min of a[i .. i + 2^k - 1]
   (the min of the 2^k elements starting at i).

   Build by doubling:
     st[0][i] = a[i]
     st[k][i] = min( st[k-1][i] , st[k-1][i + 2^(k-1)] )

   Any range [l, r] is covered by TWO overlapping power-of-2
   blocks — overlap is fine because min(x, x) = x (idempotent).

The O(1) Query Trick

   For range [l, r]:  let k = floor(log2(r - l + 1))

   answer = min( st[k][l] , st[k][r - 2^k + 1] )
                  └ left block ┘  └ right block, ending at r ┘

   The two blocks of length 2^k may OVERLAP in the middle —
   that's OK for min/max/gcd (idempotent), NOT for sum.

Code: Sparse Table (range minimum)

int st[LOG][MAXN], lg[MAXN];

void build(vector<int>& a) {
    int n = a.size();
    for (int i = 2; i <= n; i++) lg[i] = lg[i/2] + 1;
    for (int i = 0; i < n; i++) st[0][i] = a[i];
    for (int k = 1; (1 << k) <= n; k++)
        for (int i = 0; i + (1 << k) <= n; i++)
            st[k][i] = min(st[k-1][i], st[k-1][i + (1 << (k-1))]);
}
// O(1) range minimum on [l, r]
int query(int l, int r) {
    int k = lg[r - l + 1];
    return min(st[k][l], st[k][r - (1 << k) + 1]);
}
int[][] st; int[] lg;

void build(int[] a) {
    int n = a.length;
    lg = new int[n + 1];
    for (int i = 2; i <= n; i++) lg[i] = lg[i/2] + 1;
    int K = lg[n] + 1;
    st = new int[K][n];
    for (int i = 0; i < n; i++) st[0][i] = a[i];
    for (int k = 1; (1 << k) <= n; k++)
        for (int i = 0; i + (1 << k) <= n; i++)
            st[k][i] = Math.min(st[k-1][i], st[k-1][i + (1 << (k-1))]);
}
int query(int l, int r) {
    int k = lg[r - l + 1];
    return Math.min(st[k][l], st[k][r - (1 << k) + 1]);
}
import math

def build(a):
    n = len(a)
    K = n.bit_length()
    st = [a[:]] + [[0]*n for _ in range(K)]
    k = 1
    while (1 << k) <= n:
        for i in range(n - (1 << k) + 1):
            st[k][i] = min(st[k-1][i], st[k-1][i + (1 << (k-1))])
        k += 1
    return st

def query(st, l, r):
    k = (r - l + 1).bit_length() - 1
    return min(st[k][l], st[k][r - (1 << k) + 1])

Sparse Table vs Segment Tree

Sparse Table

  • O(1) query, O(n log n) build/space
  • Idempotent ops only (min/max/gcd)
  • No updates — static array

Segment Tree

  • O(log n) query & update
  • Any associative op (incl. sum)
  • Supports point/range updates
  • Why not sum? Sum needs disjoint blocks; the two power-of-two blocks overlap, so you'd double-count. Min/max/gcd are immune because the overlap is harmless.
  • For sum on a static array, use a prefix-sum array (O(1)) instead.

Common Pitfalls

1. Non-idempotent ops: never use a sparse table for sum/product/XOR — the overlap breaks them.

2. Precompute logs: a lg[] table makes queries truly O(1); calling log2 each time is slower & float-risky.

3. No updates: if the array changes, rebuild (O(n log n)) or switch to a segment tree.

4. Index bounds: the right block starts at r − 2^k + 1 — get the +1 right.
