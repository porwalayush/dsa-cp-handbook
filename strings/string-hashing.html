String Hashing

Turn any string into a single number (its hash) so you can compare substrings in O(1) instead of character-by-character. The backbone of Rabin-Karp pattern matching and countless substring problems.

Polynomial Hashing

   Treat the string like a number in base p:

   hash(s) = s[0]·p⁰ + s[1]·p¹ + s[2]·p² + … (mod M)

   • p = a prime bigger than the alphabet (e.g. 31 or 131)
   • M = a big prime modulus (e.g. 1e9+7 or 2⁶¹−1)

   Two strings with the same hash are *almost certainly* equal.

The Magic: Substring Hash in O(1)

Precompute prefix hashes once. Then the hash of any substring s[l..r] comes out with one subtraction and one multiply — so two substrings can be compared instantly.

  • prefix[i] = hash of the first i characters.
  • hash(l, r) = prefix[r+1] − prefix[l]·p^(r−l+1) (all mod M).
  • Precompute powers of p so the formula is truly O(1).

Code: Polynomial Hash + Rabin-Karp

const long long P = 131, MOD = 1000000007;

struct Hash {
    vector<long long> h, pw;
    Hash(const string& s) {
        int n = s.size();
        h.assign(n + 1, 0); pw.assign(n + 1, 1);
        for (int i = 0; i < n; i++) {
            h[i+1] = (h[i] * P + s[i]) % MOD;
            pw[i+1] = pw[i] * P % MOD;
        }
    }
    // hash of s[l..r] (0-indexed, inclusive)
    long long get(int l, int r) {
        return ((h[r+1] - h[l] * pw[r-l+1]) % MOD + MOD) % MOD;
    }
};

// find all occurrences of pat in txt
vector<int> rabinKarp(const string& txt, const string& pat) {
    Hash H(txt); Hash P2(pat);
    long long target = P2.get(0, pat.size()-1);
    vector<int> res;
    for (int i = 0; i + pat.size() <= txt.size(); i++)
        if (H.get(i, i + pat.size() - 1) == target) res.push_back(i);
    return res;
}
static final long P = 131, MOD = 1_000_000_007L;

static long[] prefixHash(String s) {
    int n = s.length();
    long[] h = new long[n + 1];
    for (int i = 0; i < n; i++)
        h[i+1] = (h[i] * P + s.charAt(i)) % MOD;
    return h;
}
static long[] powers(int n) {
    long[] pw = new long[n + 1]; pw[0] = 1;
    for (int i = 1; i <= n; i++) pw[i] = pw[i-1] * P % MOD;
    return pw;
}
static long sub(long[] h, long[] pw, int l, int r) {
    return ((h[r+1] - h[l] * pw[r-l+1]) % MOD + MOD) % MOD;
}
P, MOD = 131, 10**9 + 7

def build(s):
    n = len(s)
    h = [0] * (n + 1)
    pw = [1] * (n + 1)
    for i, ch in enumerate(s):
        h[i+1] = (h[i] * P + ord(ch)) % MOD
        pw[i+1] = pw[i] * P % MOD
    return h, pw

def sub_hash(h, pw, l, r):       # s[l..r] inclusive
    return (h[r+1] - h[l] * pw[r-l+1]) % MOD

def rabin_karp(txt, pat):
    h, pw = build(txt)
    hp, _ = build(pat)
    target = hp[len(pat)]
    m = len(pat)
    return [i for i in range(len(txt) - m + 1)
            if sub_hash(h, pw, i, i + m - 1) == target]

Avoiding Collisions

Double hashing: use two different (p, M) pairs; collision only if BOTH match — odds become astronomically small.

Big modulus: 2⁶¹−1 with 128-bit multiply resists anti-hash test cases.

Random base: pick p randomly at runtime so adversaries can't craft collisions.

Verify on match: when correctness is critical, do a real character compare after a hash hit.

What It Unlocks

  • Pattern matching (Rabin-Karp) in O(n + m) average.
  • Compare any two substrings in O(1) → longest common prefix via binary search.
  • Count distinct substrings, find repeats, detect palindromes.
  • 2D hashing for grid/pattern matching.

Common Pitfalls

1. Single hash on hard problems: a known modulus invites anti-hash tests — use double or random.

2. Overflow: h[i]*P can overflow 64 bits near large M — reduce carefully or use __int128.

3. Negative after subtraction: add M before the final mod.

4. Mapping 'a' to 0: use s[i] - 'a' + 1 so leading characters don't vanish ("a" vs "aa").
