Z-Algorithm

The Z-array of a string gives, for each position i, the length of the longest substring starting at i that matches a prefix of the string. Built in O(n) using a clever sliding "Z-box".

What Z[i] Means

   s    :  a  a  b  a  a  b  a  a  a
   index:  0  1  2  3  4  5  6  7  8
   Z    :  -  1  0  4  1  0  2  1  0

   Z[3]=4 → s[3..6]="aaba" matches prefix "aaba" (length 4)
   Z[0] is left undefined (the whole string).

The Z-Box Trick

  • Keep a window [l, r] — the rightmost prefix-match found so far.
  • If i is inside the box, copy a previously-computed Z value as a head start (instead of comparing from scratch).
  • Only extend by direct character comparison past r — that's what keeps it linear.

Code: Z-Function & Pattern Matching

vector<int> zFunction(const string& s) {
    int n = s.size();
    vector<int> z(n, 0);
    z[0] = n;
    for (int i = 1, l = 0, r = 0; i < n; i++) {
        if (i < r) z[i] = min(r - i, z[i - l]); // inside box: copy
        while (i + z[i] < n && s[z[i]] == s[i + z[i]]) z[i]++; // extend
        if (i + z[i] > r) { l = i; r = i + z[i]; }  // move box
    }
    return z;
}

// Pattern matching: run Z on  pat + '#' + txt
vector<int> search(const string& txt, const string& pat) {
    string s = pat + "#" + txt;
    vector<int> z = zFunction(s), res;
    int m = pat.size();
    for (int i = m + 1; i < (int)s.size(); i++)
        if (z[i] == m) res.push_back(i - m - 1);
    return res;
}
static int[] zFunction(String s) {
    int n = s.length();
    int[] z = new int[n];
    z[0] = n;
    for (int i = 1, l = 0, r = 0; i < n; i++) {
        if (i < r) z[i] = Math.min(r - i, z[i - l]);
        while (i + z[i] < n && s.charAt(z[i]) == s.charAt(i + z[i])) z[i]++;
        if (i + z[i] > r) { l = i; r = i + z[i]; }
    }
    return z;
}
// match: zFunction(pat + "#" + txt), look for z[i] == pat.length()
def z_function(s):
    n = len(s)
    z = [0] * n
    z[0] = n
    l = r = 0
    for i in range(1, n):
        if i < r:
            z[i] = min(r - i, z[i - l])        # copy inside box
        while i + z[i] < n and s[z[i]] == s[i + z[i]]:
            z[i] += 1                          # extend
        if i + z[i] > r:
            l, r = i, i + z[i]                 # move box
    return z

def search(txt, pat):
    s = pat + "#" + txt
    z = z_function(s)
    m = len(pat)
    return [i - m - 1 for i in range(m + 1, len(s)) if z[i] == m]

Z vs KMP

  • Both are O(n) prefix-based string tools — often interchangeable.
  • Z is usually simpler to reason about for "match against the prefix" questions.
  • Pattern matching: run Z on pattern + separator + text; any Z[i] == len(pattern) is a hit.
  • The separator must be a character that appears in neither string.

What It Solves

Pattern matching in O(n + m).

Count distinct substrings, find all borders/periods.

String periodicity and compression.

Longest common prefix of a suffix with the whole string (directly Z[i]).

Common Pitfalls

1. The cap min(r−i, z[i−l]): forgetting it lets Z run past the box and breaks linearity/correctness.

2. Separator collision: pick a sentinel not in the alphabet, else false matches.

3. Z[0] convention: some set it to 0, some to n — be consistent with how you use it.

4. Index arithmetic: the match offset is i − m − 1 with one separator char.
