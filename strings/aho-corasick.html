Aho-Corasick

Search for many patterns at once in a single pass over the text. Build a trie of all patterns, add KMP-style failure links, and you get an automaton that finds every occurrence of every pattern in O(text + matches).

Three Ingredients

   1. TRIE         — insert all patterns; shared prefixes merge.
   2. FAIL LINKS   — like KMP's LPS, but on the trie: from a node,
                     point to the longest proper suffix that is
                     also a node (BFS to build).
   3. OUTPUT LINKS — chain of patterns that END at this state
                     (so one position can report several matches).

Interactive: Multi-Pattern Matcher

Patterns he, she, his, hers built into one automaton. Step through the text — the amber node is the current state; dashed red are failure links. Nodes light green when a pattern is found.

Code: Build & Run Aho-Corasick

const int K = 26;
struct Aho {
    vector<array<int,K>> go = {{}};   // transitions (auto)
    vector<int> fail = {0}, cnt = {0};
    void add(const string& s) {
        int v = 0;
        for (char ch : s) {
            int c = ch - 'a';
            if (!go[v][c]) { go.push_back({}); fail.push_back(0); cnt.push_back(0); go[v][c] = go.size()-1; }
            v = go[v][c];
        }
        cnt[v]++;
    }
    void build() {                  // BFS to set fail links
        queue<int> q;
        for (int c = 0; c < K; c++) if (go[0][c]) q.push(go[0][c]);
        while (!q.empty()) {
            int v = q.front(); q.pop();
            cnt[v] += cnt[fail[v]];     // inherit output
            for (int c = 0; c < K; c++) {
                int u = go[v][c];
                if (u) { fail[u] = go[fail[v]][c]; q.push(u); }
                else go[v][c] = go[fail[v]][c];   // automaton edge
            }
        }
    }
    long long match(const string& t) {
        int v = 0; long long total = 0;
        for (char ch : t) { v = go[v][ch - 'a']; total += cnt[v]; }
        return total;
    }
};
class Aho {
    int K = 26;
    List<int[]> go = new ArrayList<>();
    List<Integer> fail = new ArrayList<>(), cnt = new ArrayList<>();
    Aho() { go.add(new int[K]); fail.add(0); cnt.add(0); }
    void add(String s) {
        int v = 0;
        for (char ch : s.toCharArray()) {
            int c = ch - 'a';
            if (go.get(v)[c] == 0) { go.add(new int[K]); fail.add(0); cnt.add(0); go.get(v)[c] = go.size()-1; }
            v = go.get(v)[c];
        }
        cnt.set(v, cnt.get(v) + 1);
    }
    void build() {
        Queue<Integer> q = new LinkedList<>();
        for (int c = 0; c < K; c++) if (go.get(0)[c] != 0) q.add(go.get(0)[c]);
        while (!q.isEmpty()) {
            int v = q.poll();
            cnt.set(v, cnt.get(v) + cnt.get(fail.get(v)));
            for (int c = 0; c < K; c++) {
                int u = go.get(v)[c];
                if (u != 0) { fail.set(u, go.get(fail.get(v))[c]); q.add(u); }
                else go.get(v)[c] = go.get(fail.get(v))[c];
            }
        }
    }
}
from collections import deque

class Aho:
    def __init__(self):
        self.go = [{}]; self.fail = [0]; self.cnt = [0]
    def add(self, s):
        v = 0
        for ch in s:
            if ch not in self.go[v]:
                self.go.append({}); self.fail.append(0); self.cnt.append(0)
                self.go[v][ch] = len(self.go) - 1
            v = self.go[v][ch]
        self.cnt[v] += 1
    def build(self):
        q = deque()
        for ch, u in self.go[0].items():
            q.append(u)
        while q:
            v = q.popleft()
            self.cnt[v] += self.cnt[self.fail[v]]
            for ch in 'abcdefghijklmnopqrstuvwxyz':
                if ch in self.go[v]:
                    u = self.go[v][ch]
                    self.fail[u] = self.go[self.fail[v]].get(ch, 0)
                    q.append(u)
    def match(self, t):
        v = total = 0
        for ch in t:
            while v and ch not in self.go[v]:
                v = self.fail[v]
            v = self.go[v].get(ch, 0)
            total += self.cnt[v]
        return total

Why It's Fast

  • Build trie: O(Σ pattern lengths). Build fail links: one BFS, same cost.
  • Match: each text char makes O(1) amortized transitions (fail links never net-increase depth) → O(text length + total matches).
  • Precomputing the full automaton (go[v][c] for all c) removes the inner fail-loop at match time.

Where It's Used

Dictionary matching — virus signatures, spam filters, censorship lists.

Count occurrences of many patterns in one text.

DP on the automaton — "count strings avoiding all forbidden patterns".

Bioinformatics — search many DNA motifs at once.

Common Pitfalls

1. Forgetting output links: "she" contains "he" — without inheriting cnt[fail[v]] you miss the nested match.

2. Root fail links: depth-1 nodes must fail to the root, not loop.

3. Building the automaton edges (go[v][c] = go[fail[v]][c]) is what makes matching O(1) per char.

4. Memory: a dense 26-way table per node can be large — use a map for sparse alphabets.
