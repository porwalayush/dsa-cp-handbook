Suffix Automaton (SAM)

The smallest automaton that recognizes every substring of a string. Built online in O(n) (for a fixed alphabet), it has at most 2n−1 states yet encodes all substrings — a remarkably compact powerhouse.

The Core Idea

  • Each state represents a set of substrings that share the same set of end positions (an "endpos" class).
  • Transitions on characters let you walk any substring from the start state.
  • Suffix links form a tree (the "link tree") connecting each state to a shorter suffix-class — the backbone for most queries.

Code: Suffix Automaton

struct SAM {
    struct State { int len, link; map<char,int> next; };
    vector<State> st;
    int last;
    SAM() { st.push_back({0, -1, {}}); last = 0; }

    void extend(char c) {
        int cur = st.size();
        st.push_back({st[last].len + 1, -1, {}});
        int p = last;
        while (p != -1 && !st[p].next.count(c)) {
            st[p].next[c] = cur; p = st[p].link;
        }
        if (p == -1) st[cur].link = 0;
        else {
            int q = st[p].next[c];
            if (st[p].len + 1 == st[q].len) st[cur].link = q;
            else {
                int clone = st.size();
                st.push_back({st[p].len + 1, st[q].link, st[q].next});
                while (p != -1 && st[p].next[c] == q) {
                    st[p].next[c] = clone; p = st[p].link;
                }
                st[q].link = st[cur].link = clone;
            }
        }
        last = cur;
    }
};
// distinct substrings = sum over states of (len[v] - len[link[v]])
class SAM {
    int[] len, link;
    Map<Character,Integer>[] next;
    int sz, last;
    SAM(int maxLen) {
        int cap = 2 * maxLen + 5;
        len = new int[cap]; link = new int[cap];
        next = new HashMap[cap];
        next[0] = new HashMap<>(); len[0] = 0; link[0] = -1;
        sz = 1; last = 0;
    }
    void extend(char c) {
        int cur = sz++; next[cur] = new HashMap<>();
        len[cur] = len[last] + 1; link[cur] = -1;
        int p = last;
        while (p != -1 && !next[p].containsKey(c)) { next[p].put(c, cur); p = link[p]; }
        if (p == -1) link[cur] = 0;
        else {
            int q = next[p].get(c);
            if (len[p] + 1 == len[q]) link[cur] = q;
            else {
                int clone = sz++;
                next[clone] = new HashMap<>(next[q]);
                len[clone] = len[p] + 1; link[clone] = link[q];
                while (p != -1 && next[p].get(c) == q) { next[p].put(c, clone); p = link[p]; }
                link[q] = link[cur] = clone;
            }
        }
        last = cur;
    }
}
class SAM:
    def __init__(self):
        self.length = [0]; self.link = [-1]; self.next = [{}]
        self.last = 0
    def extend(self, c):
        cur = len(self.length)
        self.length.append(self.length[self.last] + 1)
        self.link.append(-1); self.next.append({})
        p = self.last
        while p != -1 and c not in self.next[p]:
            self.next[p][c] = cur; p = self.link[p]
        if p == -1:
            self.link[cur] = 0
        else:
            q = self.next[p][c]
            if self.length[p] + 1 == self.length[q]:
                self.link[cur] = q
            else:
                clone = len(self.length)
                self.length.append(self.length[p] + 1)
                self.link.append(self.link[q])
                self.next.append(dict(self.next[q]))
                while p != -1 and self.next[p].get(c) == q:
                    self.next[p][c] = clone; p = self.link[p]
                self.link[q] = self.link[cur] = clone
        self.last = cur

# distinct substrings = sum(length[v] - length[link[v]]) for v > 0

What SAM Computes (fast)

  • Count distinct substrings: Σ (len[v] − len[link[v]]) over all states.
  • Check if a string is a substring: just walk the transitions — O(query length).
  • Number of occurrences of each substring (via endpos sizes on the link tree).
  • Longest common substring of two strings; k-th smallest substring; lexicographic substring queries.

SAM vs Suffix Array

SAM — online, transition-based, great for "is it a substring / count occurrences / DP over substrings". O(n) build.

Suffix Array — offline, sorted-order based, great for "k-th suffix / LCP / lexicographic" queries.

They're often interchangeable; pick whichever matches the query shape more naturally.

Common Pitfalls

1. The clone step: the trickiest part — clone copies q's transitions and link, then redirects edges. Get the len[p]+1 == len[q] check right.

2. Allocate 2n states: SAM has up to 2n−1 states and 3n−4 transitions — size arrays accordingly.

3. endpos sizes: to count occurrences, propagate counts up the suffix-link tree (clones start at 0).

4. Map vs array transitions: arrays are faster for small alphabets; maps save memory for large ones.
