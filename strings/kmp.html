KMP Algorithm

Knuth-Morris-Pratt finds a pattern in text in O(n + m) — never re-examining a text character. The trick is a precomputed failure function that says how far to jump back on a mismatch, instead of restarting.

The Failure Function (LPS array)

lps[i] = length of the longest proper prefix of pattern[0..i] that is also a suffix of it. When a mismatch happens, this tells us the longest already-matched prefix we can reuse.

   pattern:  a  b  a  b  a  c
   index:    0  1  2  3  4  5
   lps:      0  0  1  2  3  0

   lps[3]=2 → "abab" has prefix "ab" = suffix "ab" (length 2)
   On a mismatch after matching k chars, jump to lps[k-1].

Code: Build LPS + Search

// Build the LPS / prefix-function for the pattern
vector<int> buildLPS(const string& p) {
    int m = p.size();
    vector<int> lps(m, 0);
    for (int i = 1, len = 0; i < m; ) {
        if (p[i] == p[len]) lps[i++] = ++len;
        else if (len) len = lps[len - 1];   // fall back
        else lps[i++] = 0;
    }
    return lps;
}

// Find every start index of pat in txt
vector<int> kmp(const string& txt, const string& pat) {
    vector<int> lps = buildLPS(pat), res;
    for (int i = 0, j = 0; i < (int)txt.size(); ) {
        if (txt[i] == pat[j]) { i++; j++;
            if (j == (int)pat.size()) { res.push_back(i - j); j = lps[j - 1]; }
        } else if (j) j = lps[j - 1];     // reuse prefix
        else i++;
    }
    return res;
}
static int[] buildLPS(String p) {
    int m = p.length();
    int[] lps = new int[m];
    for (int i = 1, len = 0; i < m; ) {
        if (p.charAt(i) == p.charAt(len)) lps[i++] = ++len;
        else if (len != 0) len = lps[len - 1];
        else lps[i++] = 0;
    }
    return lps;
}
static List<Integer> kmp(String txt, String pat) {
    int[] lps = buildLPS(pat);
    List<Integer> res = new ArrayList<>();
    for (int i = 0, j = 0; i < txt.length(); ) {
        if (txt.charAt(i) == pat.charAt(j)) {
            i++; j++;
            if (j == pat.length()) { res.add(i - j); j = lps[j - 1]; }
        } else if (j != 0) j = lps[j - 1];
        else i++;
    }
    return res;
}
def build_lps(p):
    m = len(p)
    lps = [0] * m
    length = 0
    i = 1
    while i < m:
        if p[i] == p[length]:
            length += 1; lps[i] = length; i += 1
        elif length:
            length = lps[length - 1]      # fall back
        else:
            lps[i] = 0; i += 1
    return lps

def kmp(txt, pat):
    lps = build_lps(pat)
    res, j = [], 0
    for i, ch in enumerate(txt):
        while j and ch != pat[j]:
            j = lps[j - 1]            # reuse prefix
        if ch == pat[j]:
            j += 1
            if j == len(pat):
                res.append(i - j + 1); j = lps[j - 1]
    return res

Why O(n + m)?

  • The text pointer i only moves forward — never backtracks.
  • Each mismatch jump strictly decreases j, and j rises by at most 1 per step, so total work is linear.
  • Building the LPS is itself O(m) by the same argument.

What Else KMP Gives You

Count occurrences of a pattern (even overlapping).

Smallest period of a string: n − lps[n−1] (if it divides n, it's periodic).

Prefix-function tricks: count distinct substrings, string compression.

Build a KMP automaton for matching against many texts / DP on strings.

Common Pitfalls

1. Off-by-one in LPS: lps[i] uses lps[len−1] on fallback — get the indices exactly right.

2. Resetting on a full match: after a match set j = lps[j−1] to catch overlapping occurrences.

3. Confusing prefix vs proper prefix: LPS uses a proper prefix (not the whole string).

4. Empty pattern: handle m = 0 as a special case.
