Manacher's Algorithm

Finds the longest palindromic substring in O(n) — and in fact the palindrome radius at every center. It reuses symmetry the same way Z-algorithm reuses a prefix box, avoiding the O(n²) of expand-around-center.

The Odd/Even Trick

Palindromes come in odd ("aba") and even ("abba") lengths. Manacher unifies them by inserting separators so every palindrome becomes odd-length.

   original:  a  b  a  b  a
   transform: ^ # a # b # a # b # a # $

   Now we only handle ODD palindromes around each position.
   p[i] = radius of the palindrome centered at transformed index i.
   The real longest = max(p[i]); its length = p[i].

Reusing Symmetry

  • Track the rightmost palindrome [center − p, center + p] seen so far.
  • If i is inside it, mirror across the center to get a free starting radius p[mirror].
  • Only expand by character comparison past the right edge — keeping it linear.

Code: Manacher's Algorithm

// Longest palindromic substring in O(n)
string longestPalindrome(const string& s) {
    string t = "^";
    for (char c : s) { t += "#"; t += c; }
    t += "#$";
    int n = t.size();
    vector<int> p(n, 0);
    int center = 0, right = 0;
    for (int i = 1; i < n - 1; i++) {
        if (i < right) p[i] = min(right - i, p[2*center - i]);
        while (t[i + p[i] + 1] == t[i - p[i] - 1]) p[i]++;
        if (i + p[i] > right) { center = i; right = i + p[i]; }
    }
    int maxLen = 0, centerIdx = 0;
    for (int i = 1; i < n - 1; i++)
        if (p[i] > maxLen) { maxLen = p[i]; centerIdx = i; }
    int start = (centerIdx - maxLen) / 2;
    return s.substr(start, maxLen);
}
static String longestPalindrome(String s) {
    StringBuilder sb = new StringBuilder("^");
    for (char c : s.toCharArray()) sb.append('#').append(c);
    sb.append("#$");
    String t = sb.toString();
    int n = t.length();
    int[] p = new int[n];
    int center = 0, right = 0;
    for (int i = 1; i < n - 1; i++) {
        if (i < right) p[i] = Math.min(right - i, p[2*center - i]);
        while (t.charAt(i + p[i] + 1) == t.charAt(i - p[i] - 1)) p[i]++;
        if (i + p[i] > right) { center = i; right = i + p[i]; }
    }
    int maxLen = 0, c = 0;
    for (int i = 1; i < n - 1; i++) if (p[i] > maxLen) { maxLen = p[i]; c = i; }
    return s.substring((c - maxLen) / 2, (c - maxLen) / 2 + maxLen);
}
def longest_palindrome(s):
    t = '^#' + '#'.join(s) + '#$'
    n = len(t)
    p = [0] * n
    center = right = 0
    for i in range(1, n - 1):
        if i < right:
            p[i] = min(right - i, p[2*center - i])
        while t[i + p[i] + 1] == t[i - p[i] - 1]:
            p[i] += 1
        if i + p[i] > right:
            center, right = i, i + p[i]
    max_len = max(p)
    c = p.index(max_len)
    start = (c - max_len) // 2
    return s[start:start + max_len]

What It Computes

Longest palindromic substring in O(n) (vs O(n²) brute / O(n log n) hashing).

Count all palindromic substrings: sum of ⌈p[i]/2⌉ over centers.

Is s[l..r] a palindrome? answerable in O(1) after preprocessing.

Palindromic prefixes/suffixes for string-building problems.

Common Pitfalls

1. Sentinels matter: the ^ and $ guards stop the inner while from running off the ends — don't omit them.

2. The mirror cap: min(right − i, p[mirror]) is essential for both correctness and O(n).

3. Index mapping back: transformed length p[i] equals the real palindrome length; start = (i − p[i]) / 2.

4. Even palindromes: only the #-padding makes them appear — without it you miss them.
