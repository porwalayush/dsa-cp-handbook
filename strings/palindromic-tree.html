Palindromic Tree (Eertree)

A data structure that stores every distinct palindromic substring of a string — and there are at most n of them! Built online in O(n), it makes palindrome counting and analysis almost trivial.

The Key Fact

  • A string of length n has at most n distinct palindromic substrings (each new character adds at most one new palindrome).
  • The Eertree has two roots: an imaginary node of length −1 and an empty node of length 0. Every other node is one distinct palindrome.
  • Each node stores a suffix link to the longest proper palindromic suffix of itself.

How a New Character Extends It

   To add character c at position i:
   1. From the last palindrome, follow suffix links until
      s[i − len − 1] == c  (so c·...·c forms a palindrome).
   2. If that palindrome+c already exists → just move there.
   3. Otherwise create a NEW node (a brand-new palindrome),
      and set its suffix link by continuing to follow links.

   Each step adds ≤ 1 new node → total O(n) nodes.

Code: Palindromic Tree

struct Eertree {
    struct Node { int len, link, cnt; map<char,int> next; };
    vector<Node> t;
    string s;
    int last;
    Eertree() {
        t.push_back({-1, 0, 0, {}}); // imaginary root (len -1)
        t.push_back({ 0, 0, 0, {}}); // empty root (len 0)
        last = 1;
    }
    int getLink(int v, int i) {
        while (i - t[v].len - 1 < 0 || s[i - t[v].len - 1] != s[i])
            v = t[v].link;
        return v;
    }
    void add(char c) {
        s += c;
        int i = s.size() - 1;
        int cur = getLink(last, i);
        if (!t[cur].next.count(c)) {
            int now = t.size();
            t.push_back({t[cur].len + 2, 0, 0, {}});
            if (t[now].len == 1) t[now].link = 1;
            else t[now].link = t[getLink(t[cur].link, i)].next[c];
            t[cur].next[c] = now;
        }
        last = t[cur].next[c];
        t[last].cnt++;
    }
};
// distinct palindromes = t.size() - 2 (minus the two roots)
class Eertree {
    int[] len, link, cnt;
    Map<Character,Integer>[] next;
    StringBuilder s = new StringBuilder();
    int sz, last;
    Eertree(int n) {
        len = new int[n+3]; link = new int[n+3]; cnt = new int[n+3];
        next = new HashMap[n+3];
        len[0] = -1; link[0] = 0; next[0] = new HashMap<>();
        len[1] = 0;  link[1] = 0; next[1] = new HashMap<>();
        sz = 2; last = 1;
    }
    int getLink(int v, int i) {
        while (i - len[v] - 1 < 0 || s.charAt(i - len[v] - 1) != s.charAt(i)) v = link[v];
        return v;
    }
    void add(char c) {
        s.append(c);
        int i = s.length() - 1;
        int cur = getLink(last, i);
        if (!next[cur].containsKey(c)) {
            int now = sz++; next[now] = new HashMap<>();
            len[now] = len[cur] + 2;
            link[now] = (len[now] == 1) ? 1 : next[getLink(link[cur], i)].get(c);
            next[cur].put(c, now);
        }
        last = next[cur].get(c);
        cnt[last]++;
    }
}
class Eertree:
    def __init__(self):
        self.len = [-1, 0]; self.link = [0, 0]
        self.cnt = [0, 0]; self.next = [{}, {}]
        self.s = []; self.last = 1

    def get_link(self, v, i):
        while i - self.len[v] - 1 < 0 or self.s[i - self.len[v] - 1] != self.s[i]:
            v = self.link[v]
        return v

    def add(self, c):
        self.s.append(c)
        i = len(self.s) - 1
        cur = self.get_link(self.last, i)
        if c not in self.next[cur]:
            now = len(self.len)
            self.len.append(self.len[cur] + 2)
            self.next.append({}); self.cnt.append(0)
            if self.len[now] == 1:
                self.link.append(1)
            else:
                self.link.append(self.next[self.get_link(self.link[cur], i)][c])
            self.next[cur][c] = now
        self.last = self.next[cur][c]
        self.cnt[self.last] += 1

What It Computes

  • Number of distinct palindromic substrings = node count − 2.
  • Count occurrences of each palindrome (propagate cnt down suffix links at the end).
  • Palindromes ending at each position — directly from the suffix-link chain of last.
  • Longest palindromic suffix/prefix, palindromic factorization DP.

Eertree vs Manacher

Manacher — finds the longest palindrome / radius at every center, O(n), but doesn't enumerate distinct palindromes.

Eertree — stores all distinct palindromes as a tree, enabling counting, occurrence stats, and palindrome DP.

Use Manacher for "longest / is-palindrome" questions; reach for Eertree when you need the set of palindromes and their counts.

Common Pitfalls

1. Two roots: the imaginary root has length −1; it's what makes adding single characters work. Don't drop it.

2. getLink condition: s[i − len − 1] == s[i] is the exact test for "can extend into a palindrome".

3. Length-1 suffix link: a new palindrome of length 1 links to the empty root (node 1), not the imaginary one.

4. Occurrence counts: the raw cnt only counts the longest palindrome ending there — propagate along suffix links to get true totals.
