Binary Search

The fastest way to find something in sorted data. Each guess throws away half the remaining candidates — so a million items take only ~20 steps.

The Core Idea

   Look at the MIDDLE element:
     • equal to target?  → found it!
     • target smaller?   → throw away the right half
     • target bigger?    → throw away the left half
   Repeat on what's left.

   n → n/2 → n/4 → … → 1   means  O(log₂ n) steps.
   1,000,000 items? Just ~20 guesses.
  • Precondition: the data must be sorted (or have some monotonic property).
  • No sorted order, no binary search.

Interactive: Find the Target

Pick a target, then step through. Greyed cells are discarded, blue is the active range, amber is the middle we're checking.

Code: Classic Binary Search

int binarySearch(vector<int>& a, int target) {
    int lo = 0, hi = a.size() - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2; // avoids overflow
        if (a[mid] == target) return mid;
        else if (a[mid] < target) lo = mid + 1; // go right
        else hi = mid - 1;                  // go left
    }
    return -1; // not found
}
int binarySearch(int[] a, int target) {
    int lo = 0, hi = a.length - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2; // avoids overflow
        if (a[mid] == target) return mid;
        else if (a[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}
def binary_search(a, target):
    lo, hi = 0, len(a) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if a[mid] == target:
            return mid
        elif a[mid] < target:
            lo = mid + 1      # go right
        else:
            hi = mid - 1      # go left
    return -1             # not found

# or just: import bisect; bisect.bisect_left(a, target)

The Real Superpower: "Binary Search on the Answer"

Binary search isn't only for arrays. If a question has a monotonic yes/no boundary — once it becomes true, it stays true — you can binary search the answer itself.

   Predicate p(x): "is x a feasible answer?"
   If p looks like:   NO NO NO NO YES YES YES
                                  ↑ first YES = the answer

   Binary search for that boundary in O(log range).

   Example: "minimum capacity to ship all packages in D days"
     • too small capacity → NO (can't finish in time)
     • big enough         → YES
     • find the smallest YES.

Code: Search on the Answer (template)

// find the smallest x in [lo, hi] with feasible(x) == true
int lo = minPossible, hi = maxPossible, ans = hi;
while (lo <= hi) {
    int mid = lo + (hi - lo) / 2;
    if (feasible(mid)) { ans = mid; hi = mid - 1; } // try smaller
    else lo = mid + 1;                            // need bigger
}
// ans = smallest feasible value

lo, hi, ans = min_possible, max_possible, max_possible

    if feasible(mid):
        ans = mid

    else:
        lo = mid + 1   # need bigger

lower_bound vs upper_bound

lower_bound(x)

  • First index with value ≥ x
  • Where x would be inserted (left)
  • Python: bisect_left

upper_bound(x)

  • First index with value > x
  • Insert position (right)
  • Python: bisect_right
  • Count of x in a sorted array = upper_bound(x) − lower_bound(x).

Common Pitfalls

1. Overflow: use mid = lo + (hi - lo) / 2, not (lo + hi) / 2 in C++/Java.

2. Infinite loop: make sure lo or hi actually moves each iteration.

3. Wrong boundary: decide <= vs < and mid±1 carefully — pick one template and reuse it.

4. Forgetting sorted-ness: binary search on unsorted data silently returns garbage.
