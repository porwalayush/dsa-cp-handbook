Meet in the Middle

A clever trick for problems where brute force is O(2ⁿ) — too slow for n ≈ 40. Split the items into two halves, brute-force each half separately, then combine. The exponent gets cut in half: O(2^(n/2)).

Why Halving the Exponent is Huge

   n = 40

   Brute force:   2^40  ≈ 1,000,000,000,000   (a trillion — too slow)

   Meet in middle:
       2^20 + 2^20 ≈ 2,000,000   (two million — instant!)

   Splitting in half doesn't halve the work… it SQUARE-ROOTs it.
   2^40 = (2^20)²  →  we replace a product with a sum.
  • Sweet spot: n roughly 30–45 — too big for 2ⁿ, too small/unstructured for DP.
  • Works when the problem splits cleanly and partial results can be combined (often via sorting + binary search or a hash map).

The Recipe (Subset-Sum example)

   Question: is there a subset of the array summing to TARGET?

   1. SPLIT    array into left half L and right half R
   2. ENUMERATE all 2^(n/2) subset sums of L  → list A
                all 2^(n/2) subset sums of R  → list B
   3. SORT     list B
   4. COMBINE  for each sum a in A, binary-search B
               for (TARGET − a). Found a pair? → YES.

// all subset sums of a[lo..hi)
vector<long long> subsetSums(vector<int>& a, int lo, int hi) {
    int k = hi - lo;
    vector<long long> sums;
    for (int mask = 0; mask < (1 << k); mask++) {
        long long s = 0;
        for (int i = 0; i < k; i++)
            if (mask & (1 << i)) s += a[lo + i];
        sums.push_back(s);
    }
    return sums;
}
bool hasSubsetSum(vector<int>& a, long long target) {
    int n = a.size(), mid = n / 2;
    auto A = subsetSums(a, 0, mid);
    auto B = subsetSums(a, mid, n);
    sort(B.begin(), B.end());
    for (long long x : A)              // COMBINE
        if (binary_search(B.begin(), B.end(), target - x))
            return true;
    return false;
}
long[] subsetSums(int[] a, int lo, int hi) {
    int k = hi - lo;
    long[] sums = new long[1 << k];
    for (int mask = 0; mask < (1 << k); mask++) {
        long s = 0;
        for (int i = 0; i < k; i++)
            if ((mask & (1 << i)) != 0) s += a[lo + i];
        sums[mask] = s;
    }
    return sums;
}
boolean hasSubsetSum(int[] a, long target) {
    int n = a.length, mid = n / 2;
    long[] A = subsetSums(a, 0, mid);
    long[] B = subsetSums(a, mid, n);
    Arrays.sort(B);
    for (long x : A)
        if (Arrays.binarySearch(B, target - x) >= 0) return true;
    return false;
}
from itertools import combinations
import bisect

def subset_sums(part):
    sums = []
    for r in range(len(part) + 1):
        for combo in combinations(part, r):
            sums.append(sum(combo))
    return sums

def has_subset_sum(a, target):
    mid = len(a) // 2
    A = subset_sums(a[:mid])
    B = sorted(subset_sums(a[mid:]))
    for x in A:                         # COMBINE
        need = target - x
        i = bisect.bisect_left(B, need)
        if i < len(B) and B[i] == need:
            return True
    return False

Brute Force vs Meet in the Middle

Brute Force

  • Try all 2ⁿ subsets
  • Dies around n > 25
  • O(2ⁿ)

Meet in the Middle

  • 2 × 2^(n/2), then combine
  • Comfortable up to n ≈ 40
  • O(2^(n/2) · n)

Where It Shows Up

Subset Sum / Count subsets with sum = target (shown above).

Closest subset sum to a target — combine with two pointers.

4-SUM — split pairs, hash all pair-sums of each half.

Discrete log / baby-step giant-step — the number-theory cousin.

Knapsack with huge capacity but small n (~40).

Common Pitfalls

1. Memory blowup: 2^(n/2) lists can still be millions of entries — watch RAM.

2. Forgetting to sort the second half before binary searching it.

3. Overflow: subset sums add up fast — use 64-bit integers.

4. Using it when n is small: if n ≤ 20, plain brute force is simpler and fast enough.
