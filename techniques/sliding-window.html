Sliding Window

A window is a contiguous range [left, right] that slides across the array. Instead of recomputing each subarray from scratch (O(n²)), you add one element on the right, drop some from the left, and keep a running answer — O(n).

Two Kinds of Window

Fixed size (k)

  • Window length never changes
  • Slide: add right, remove left
  • "Max sum of k consecutive elements"

Variable size

  • Grow right; shrink left when a rule breaks
  • "Longest substring without repeats"
  • "Smallest subarray with sum ≥ S"

Interactive: Longest Subarray with Sum ≤ Target

Grow the window by moving right. The moment the window sum exceeds the target, shrink from the left until it's valid again. Track the longest valid window seen.

The Universal Template

   left = 0
   for right in 0 .. n-1:
       add a[right] to the window        ← always grow right

       while (window is INVALID):
           remove a[left] from the window ← shrink from left
           left++

       update answer with current window  ← window is now valid
  • Each element is added once and removed once → total work is O(n), even with the inner while.
  • The hard part is defining "invalid" — a sum cap, a duplicate char, too many distinct values, etc.

Code: Variable & Fixed Windows

// Longest subarray with sum <= target (variable window)
int longestSubarray(vector<int>& a, int target) {
    int left = 0, sum = 0, best = 0;
    for (int right = 0; right < a.size(); right++) {
        sum += a[right];                   // grow
        while (sum > target) sum -= a[left++]; // shrink
        best = max(best, right - left + 1);
    }
    return best;
}

// Max sum of any k consecutive elements (fixed window)
int maxSumK(vector<int>& a, int k) {
    int sum = 0;
    for (int i = 0; i < k; i++) sum += a[i];
    int best = sum;
    for (int i = k; i < a.size(); i++) {
        sum += a[i] - a[i - k];            // slide
        best = max(best, sum);
    }
    return best;
}
// Longest subarray with sum <= target (variable window)
int longestSubarray(int[] a, int target) {
    int left = 0, sum = 0, best = 0;
    for (int right = 0; right < a.length; right++) {
        sum += a[right];
        while (sum > target) sum -= a[left++];
        best = Math.max(best, right - left + 1);
    }
    return best;
}

// Longest substring without repeating characters
int lengthOfLongest(String s) {
    Set<Character> win = new HashSet<>();
    int left = 0, best = 0;
    for (int right = 0; right < s.length(); right++) {
        while (win.contains(s.charAt(right))) win.remove(s.charAt(left++));
        win.add(s.charAt(right));
        best = Math.max(best, right - left + 1);
    }
    return best;
}
# Longest subarray with sum <= target (variable window)
def longest_subarray(a, target):
    left = total = best = 0
    for right in range(len(a)):
        total += a[right]                  # grow
        while total > target:
            total -= a[left]; left += 1     # shrink
        best = max(best, right - left + 1)
    return best

# Longest substring without repeating characters
def length_of_longest(s):
    seen, left, best = set(), 0, 0
    for right, ch in enumerate(s):
        while ch in seen:
            seen.remove(s[left]); left += 1
        seen.add(ch)
        best = max(best, right - left + 1)
    return best

When to Reach for a Window

Signal words: "contiguous", "subarray", "substring", "consecutive".

Asking for: longest / shortest / max-sum / count of ranges meeting a condition.

The condition is monotonic: growing the window only makes it "more invalid", shrinking only makes it "more valid". That monotonicity is what lets the window work.

Common Pitfalls

1. Negative numbers: the "shrink when too big" trick assumes non-negative values. With negatives, use prefix sums instead.

2. Updating the answer at the wrong time: update after the window is made valid, not during shrinking.

3. Forgetting to remove from the left bookkeeping (counts, sets, sums) when shrinking.

4. Fixed vs variable confusion: if k is given, it's fixed; if you're maximizing length, it's variable.
