Two Pointers

Instead of a nested loop checking every pair (O(n²)), you keep two indices and move them cleverly — usually turning an O(n²) brute force into a single O(n) pass.

The Three Flavors

   1. OPPOSITE ENDS   →  ← left starts at 0, right at n-1, move inward
                          (two-sum on sorted, palindrome check, reverse)

   2. SAME DIRECTION  →→   both move forward at different speeds
                          (remove duplicates, sliding window, merge)

   3. FAST / SLOW     →→→  one moves 2×, one moves 1×
                          (cycle detection, find middle of list)

Interactive: Two-Sum on a Sorted Array

Find two numbers that add to the target. Start left at the smallest, right at the largest. Sum too small? Move left up. Too big? Move right down. This is the opposite-ends pattern.

Why It Works

  • Because the array is sorted, moving left right can only increase the sum, and moving right left can only decrease it.
  • So at every step we can safely discard one candidate — no need to recheck it. That's the O(n) magic.

Code: Common Two-Pointer Patterns

// 1. Opposite ends: two-sum on sorted array
pair<int,int> twoSum(vector<int>& a, int target) {
    int l = 0, r = a.size() - 1;
    while (l < r) {
        int s = a[l] + a[r];
        if (s == target) return {l, r};
        else if (s < target) l++;   // need bigger
        else r--;                    // need smaller
    }
    return {-1, -1};
}

// 2. Same direction: remove duplicates in-place
int dedup(vector<int>& a) {
    int slow = 0;
    for (int fast = 1; fast < a.size(); fast++)
        if (a[fast] != a[slow]) a[++slow] = a[fast];
    return slow + 1; // new length
}
// 1. Opposite ends: two-sum on sorted array
int[] twoSum(int[] a, int target) {

        int s = a[l] + a[r];
        if (s == target) return new int[]{l, r};
        else if (s < target) l++;
        else r--;
    }
    return new int[]{-1, -1};
}

int dedup(int[] a) {

# 1. Opposite ends: two-sum on sorted array
def two_sum(a, target):
    l, r = 0, len(a) - 1
    while l < r:
        s = a[l] + a[r]
        if s == target:
            return (l, r)
        elif s < target:
            l += 1          # need bigger
        else:
            r -= 1          # need smaller
    return (-1, -1)

# 2. Same direction: remove duplicates in-place
def dedup(a):
    slow = 0
    for fast in range(1, len(a)):
        if a[fast] != a[slow]:
            slow += 1
            a[slow] = a[fast]
    return slow + 1

Brute Force vs Two Pointers

Nested loops

  • Check every pair (i, j)
  • O(n²) time
  • Works on unsorted data

Two pointers

  • Single sweep, smart moves
  • O(n) time (after sorting)
  • Needs sorted / monotonic structure

Classic Problems

Two Sum / 3Sum / 4Sum — sort, then opposite-end sweeps.

Container With Most Water — opposite ends, move the shorter wall.

Valid Palindrome — compare from both ends inward.

Remove Duplicates / Move Zeroes — slow/fast same-direction.

Merge Two Sorted Arrays — one pointer per array.

Linked List Cycle — fast/slow (Floyd's tortoise & hare).

Common Pitfalls

1. Forgetting to sort: opposite-end logic only works on sorted data.

2. Pointers crossing: use while (l < r) and stop the moment they meet.

3. Duplicates in 3Sum: skip equal neighbors to avoid duplicate triples.

4. Off-by-one on the slow pointer: decide whether it points at the last kept item or the next free slot — and stay consistent.
