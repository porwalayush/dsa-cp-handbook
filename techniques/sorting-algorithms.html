Sorting Algorithms

Sorting is the most-used building block in all of programming. Master a handful of algorithms, know their trade-offs, and learn to sort by custom rules — that's 90% of what you'll ever need.

The Big Picture

There are two families of sorting algorithms:

Comparison sorts

  • Compare pairs of elements
  • Best possible: O(n log n)
  • Merge sort, quick sort, heap sort

Non-comparison sorts

  • Use the values as indices/digits
  • Can hit O(n) — beats the log barrier!
  • Counting sort, radix sort, bucket sort

Quick Sort: the partition trick

   Pick a PIVOT. Rearrange so:
       [ everything ≤ pivot ] [ pivot ] [ everything > pivot ]
   Now the pivot is in its final spot. Recurse on both sides.

   pivot = 5
   [3 7 1 5 2 8 4]  →  [3 1 2 4] 5 [7 8]
                         ↑ recurse    ↑ recurse

Code: Quick Sort & Counting Sort

// Quick sort (Lomuto partition)
int partition(vector<int>& a, int lo, int hi) {
    int pivot = a[hi], i = lo;
    for (int j = lo; j < hi; j++)
        if (a[j] < pivot) swap(a[i++], a[j]);
    swap(a[i], a[hi]);
    return i;
}
void quickSort(vector<int>& a, int lo, int hi) {
    if (lo >= hi) return;
    int p = partition(a, lo, hi);
    quickSort(a, lo, p - 1);
    quickSort(a, p + 1, hi);
}

// Counting sort: values in [0, k]
void countingSort(vector<int>& a, int k) {
    vector<int> cnt(k + 1, 0);
    for (int x : a) cnt[x]++;
    int idx = 0;
    for (int v = 0; v <= k; v++)
        while (cnt[v]--) a[idx++] = v;
}

    if (a.size() != b.size()) return a.size() < b.size();
    return a < b;
});
// Quick sort (Lomuto partition)
int partition(int[] a, int lo, int hi) {
    int pivot = a[hi], i = lo;
    for (int j = lo; j < hi; j++)
        if (a[j] < pivot) { int t=a[i]; a[i]=a[j]; a[j]=t; i++; }
    int t=a[i]; a[i]=a[hi]; a[hi]=t;
    return i;
}
void quickSort(int[] a, int lo, int hi) {
    if (lo >= hi) return;
    int p = partition(a, lo, hi);
    quickSort(a, lo, p - 1);
    quickSort(a, p + 1, hi);
}

# Quick sort (concise)
def quick_sort(a):
    if len(a) <= 1:
        return a
    pivot = a[len(a) // 2]
    left  = [x for x in a if x < pivot]
    mid   = [x for x in a if x == pivot]
    right = [x for x in a if x > pivot]
    return quick_sort(left) + mid + quick_sort(right)

# Counting sort
def counting_sort(a, k):
    cnt = [0] * (k + 1)
    for x in a: cnt[x] += 1
    return [v for v in range(k + 1) for _ in range(cnt[v])]

How to Choose

Just sort numbers/strings? → library sort. Done.

Need stability (multi-key)? → merge sort / Timsort (Python & Java objects are stable).

Small integer range (0..k)? → counting sort, O(n).

Fixed-width integers / strings? → radix sort.

Memory-tight, guaranteed O(n log n)? → heap sort.

Common Pitfalls

1. Quick sort worst case: already-sorted input + bad pivot → O(n²). Use random/median pivot.

2. Counting sort memory: if k is huge (10⁹), the count array won't fit — use comparison sort.

3. Unstable when you needed stable: breaks multi-key sorts silently.

4. Comparator must be consistent: a buggy comparator (not a strict weak ordering) can crash C++ sort.
