Bit Manipulation

Numbers are stored as binary. By poking individual bits directly, you get blazing-fast tricks: test/set/flip flags, treat an integer as a tiny set, and count things in O(1) hardware instructions.

Essential One-Liners

   check bit i:        (n >> i) & 1
   set bit i:          n | (1 << i)
   clear bit i:        n & ~(1 << i)
   toggle bit i:       n ^ (1 << i)

   is power of two:    n > 0 && (n & (n - 1)) == 0
   lowest set bit:     n & (-n)
   remove lowest set:  n & (n - 1)
   is odd:             n & 1
   count set bits:     __builtin_popcount(n)

XOR: the magic eraser

  • x ^ x = 0 and x ^ 0 = x. So XOR-ing a value twice cancels it out.
  • Find the single number where all others appear twice: XOR everything → pairs cancel, the loner survives. O(n) time, O(1) space.
  • Swap without temp: a ^= b; b ^= a; a ^= b;

Where It Shows Up

Bitmask DP — represent visited sets in TSP / assignment problems.

Single Number / Missing Number — XOR tricks.

Subset generation — iterate 1<<n masks.

Fast flags — pack many booleans into one integer.

Power-of-two checks — hashing, memory alignment, sizing.

Common Pitfalls

1. Operator precedence: & is lower than == in C/Java — wrap in parens: (n & (1<<i)) != 0.

2. Signed shifts: shifting into the sign bit is undefined/surprising — use unsigned or 1L<<i for big shifts.

3. Overflow: 1 << 31 overflows a 32-bit int; use 1LL << k in C++, 1L in Java.

4. Python ints are unbounded: ~n won't behave like a fixed-width complement — mask with & 0xFF if you need 8-bit wrap.
