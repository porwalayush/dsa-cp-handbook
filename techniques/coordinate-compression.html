Coordinate Compression

Sometimes the values are huge (up to 10⁹) but there are only a few of them. Coordinate compression renames those values to small dense indices 0, 1, 2, … while keeping their order — so they can index a Fenwick tree, segment tree, or simple array.

The Problem It Solves

  • You want a frequency array or BIT indexed by value, but values go up to 10⁹ → an array that big won't fit in memory.
  • But you only have, say, n ≤ 10⁵ distinct values. Map each to its rank (sorted position) → indices fit in [0, n).
  • Order is preserved, so all comparisons / range queries still work.

Interactive: Compress These Values

The 3-Step Recipe

   original:  [1000000000, 5, 5, 80, 1000000000, 12]

   1. COPY + SORT + UNIQUE:
        sorted distinct = [5, 12, 80, 1000000000]
                  ranks =  0   1    2       3

   2. For each original value, BINARY SEARCH its rank:
        1000000000 → 3,  5 → 0,  80 → 2,  12 → 1

   3. compressed = [3, 0, 0, 2, 3, 1]   ← now indexes a size-4 array

Code: Coordinate Compression

// Compress values to ranks [0, m)
vector<int> compress(vector<int> a) {
    vector<int> sorted = a;
    sort(sorted.begin(), sorted.end());
    sorted.erase(unique(sorted.begin(), sorted.end()), sorted.end());
    // rank of value x = lower_bound index
    for (int& x : a)
        x = lower_bound(sorted.begin(), sorted.end(), x) - sorted.begin();
    return a; // now every entry is in [0, sorted.size())
}

// Typical use: count inversions / range queries with a BIT
// indexed by compressed rank instead of raw value.
// Compress values to ranks [0, m)
int[] compress(int[] a) {
    int[] sorted = a.clone();
    Arrays.sort(sorted);
    // build distinct list
    int m = 0;
    for (int i = 0; i < sorted.length; i++)
        if (i == 0 || sorted[i] != sorted[i-1]) sorted[m++] = sorted[i];
    int[] uniq = Arrays.copyOf(sorted, m);
    int[] res = new int[a.length];
    for (int i = 0; i < a.length; i++) {
        int r = Arrays.binarySearch(uniq, a[i]);
        res[i] = r;  // rank in [0, m)
    }
    return res;
}
import bisect

# Compress values to ranks [0, m)
def compress(a):
    uniq = sorted(set(a))
    return [bisect.bisect_left(uniq, x) for x in a]

# Even shorter with a dict lookup
def compress_dict(a):
    rank = {v: i for i, v in enumerate(sorted(set(a)))}
    return [rank[x] for x in a]

# compress([10**9, 5, 5, 80, 10**9, 12]) -> [3, 0, 0, 2, 3, 1]

Where You'll Need It

Count inversions — BIT indexed by compressed value.

Sweep line — compress x-coordinates of events / segments.

Segment tree on values — when values are sparse but you query ranges of them.

Offline range queries — compress both coordinates and query bounds together.

2D problems — compress each axis independently.

Common Pitfalls

1. Compress queries too: if a query asks about value v, map it with the SAME sorted list (use lower_bound / upper_bound carefully if v isn't present).

2. Forgetting duplicates: use unique / set or your ranks will have gaps.

3. lower vs upper bound: for "values < v" vs "values ≤ v" the boundary index differs by one.

4. Losing original values: keep the sorted list around if you need to decompress (rank → value) later.
