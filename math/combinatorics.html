Combinatorics

The art of counting without listing. How many ways can you arrange, choose, or distribute things? Master a few formulas and you can count astronomically large sets instantly.

Permutations vs Combinations

Permutation (order matters)

  • Arrange r of n in a sequence
  • P(n,r) = n! / (n−r)!
  • "podium: gold, silver, bronze"

Combination (order doesn't)

  • Choose r of n, no order
  • C(n,r) = n! / (r!·(n−r)!)
  • "pick a 3-person team"

The Counting Formulas

   factorial        n!            = n·(n−1)·…·1          (arrangements of n)
   permutations     P(n,r)        = n!/(n−r)!
   combinations     C(n,r)        = n!/(r!(n−r)!)
   with repetition  (stars&bars)  = C(n+k−1, k−1)
   Catalan          Cₙ            = C(2n,n)/(n+1)

nCr under a Modulus

Competitive problems ask for C(n, r) mod p with huge n. Precompute factorials and their modular inverses once, then every query is O(1).

const int MOD = 1e9 + 7, MAXN = 1000005;
long long fact[MAXN], inv_fact[MAXN];

long long power(long long a, long long n) {
    long long r = 1; a %= MOD;
    while (n) { if (n & 1) r = r * a % MOD; a = a * a % MOD; n >>= 1; }
    return r;
}
void precompute() {
    fact[0] = 1;
    for (int i = 1; i < MAXN; i++) fact[i] = fact[i-1] * i % MOD;
    inv_fact[MAXN-1] = power(fact[MAXN-1], MOD-2);
    for (int i = MAXN-2; i >= 0; i--)
        inv_fact[i] = inv_fact[i+1] * (i+1) % MOD;
}
long long C(int n, int r) {
    if (r < 0 || r > n) return 0;
    return fact[n] * inv_fact[r] % MOD * inv_fact[n-r] % MOD;
}

# Python: math.comb does exact big-int nCr directly
from math import comb, perm, factorial
comb(10, 3)      # 120
perm(10, 3)      # 720

# Modular nCr with precomputed factorials
MOD = 10**9 + 7
MAXN = 10**6 + 5
fact = [1] * MAXN
for i in range(1, MAXN):
    fact[i] = fact[i-1] * i % MOD
inv_fact = [1] * MAXN
inv_fact[-1] = pow(fact[-1], MOD-2, MOD)
for i in range(MAXN-2, -1, -1):
    inv_fact[i] = inv_fact[i+1] * (i+1) % MOD

def C(n, r):
    if r < 0 or r > n: return 0
    return fact[n] * inv_fact[r] % MOD * inv_fact[n-r] % MOD

Named Sequences & Tools

  • Catalan numbers: count balanced parentheses, BSTs with n nodes, triangulations. Cₙ = C(2n,n)/(n+1).
  • Stars and bars: distribute k identical items into n bins → C(n+k−1, k).
  • Inclusion-Exclusion: count unions by adding singles, subtracting pairs, adding triples…
  • Burnside's lemma: count distinct arrangements under symmetry (necklaces, colorings).

Common Pitfalls

1. Overflow: n! explodes fast — work mod p, or use Python big ints.

2. Off-by-one in stars & bars: "non-negative" vs "positive" bins changes the formula.

3. Double counting: when order doesn't matter, divide out the overcount (that's why C divides by r!).

4. Inverse needs prime modulus: the factorial-inverse trick assumes p is prime (use Lucas' theorem otherwise).
