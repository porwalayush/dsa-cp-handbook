Probability & Expected Value

Expected value is the long-run average outcome — what you'd get per trial if you repeated an experiment forever. It's the single most useful idea for "average case" and randomized problems.

The Core Formula

   E[X] = Σ  (value) × (probability of that value)

   Fair 6-sided die:
   E[X] = 1·⅙ + 2·⅙ + 3·⅙ + 4·⅙ + 5·⅙ + 6·⅙
        = (1+2+3+4+5+6)/6 = 21/6 = 3.5

Interactive: Dice Roll Simulator

Roll a fair die many times. The empirical average wobbles at first, then settles toward the theoretical E[X] = 3.5. This is the law of large numbers in action.

Linearity of Expectation (the superpower)

  • E[X + Y] = E[X] + E[Y] — always, even if X and Y are dependent!
  • This lets you break a messy random quantity into simple indicator pieces and just add them up.
  • Example: expected number of fixed points in a random permutation = n × (1/n) = 1, regardless of n.

Indicator Variables Trick

   Want E[number of events that happen]?

   1. Define Xᵢ = 1 if event i happens, else 0.
   2. E[Xᵢ] = P(event i happens).
   3. Total = X₁ + X₂ + … so by linearity:
        E[Total] = Σ P(event i)

   No need for the joint distribution — just per-event chances!

Expected Value DP (handling cycles)

Many problems ask "expected number of steps to reach a goal". Set up E[state] equations; if there are cycles (you can return to a state), solve the resulting linear system.

// Expected rolls of a fair die to first see a 6.
// E = 1 + (5/6)E  =>  E = 6.  (geometric distribution)

// Expected dice rolls to go from square 0 to n on a
// simple board (no cycles): DP from the end backwards.
double expectedRolls(int n) {
    vector<double> E(n + 1, 0);
    for (int i = n - 1; i >= 0; i--) {
        double e = 1.0;             // this roll
        for (int f = 1; f <= 6; f++)
            e += (1.0 / 6) * E[min(n, i + f)];
        E[i] = e;
    }
    return E[0];
}
// Expected dice rolls from 0 to n (no cycles)
double expectedRolls(int n) {
    double[] E = new double[n + 1];
    for (int i = n - 1; i >= 0; i--) {
        double e = 1.0;
        for (int f = 1; f <= 6; f++)
            e += (1.0 / 6) * E[Math.min(n, i + f)];
        E[i] = e;
    }
    return E[0];
}
// E[rolls to first 6] = 6.0 (geometric: E = 1/p)
# Expected dice rolls from 0 to n (no cycles)
def expected_rolls(n):
    E = [0.0] * (n + 1)
    for i in range(n - 1, -1, -1):
        e = 1.0                       # this roll counts
        for f in range(1, 7):
            e += (1 / 6) * E[min(n, i + f)]
        E[i] = e
    return E[0]

# Geometric: expected trials until first success = 1/p
# Fair die first 6: p = 1/6, so E = 6 rolls.

Quick Reference

Geometric: expected trials until first success = 1/p.

Coupon collector: expected draws to collect all n coupons = n·(1 + 1/2 + … + 1/n) ≈ n ln n.

Conditional: P(A | B) = P(A ∩ B) / P(B).

Bayes: P(A|B) = P(B|A)·P(A) / P(B).

Variance: Var(X) = E[X²] − (E[X])².

Common Pitfalls

1. Assuming independence: linearity of expectation does NOT need it, but multiplying probabilities does.

2. Cycles in expected-value DP: plain DP fails if a state reaches itself — set up & solve linear equations.

3. Modular probabilities: in CP, answers are often p/q mod M — output p · q⁻¹ mod M.

4. Confusing average with most likely: the expected value need not be an achievable outcome (3.5 on a die).
