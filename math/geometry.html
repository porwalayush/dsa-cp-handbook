Computational Geometry

Geometry problems hinge on one tiny primitive: the cross product, which tells you whether you turn left, right, or go straight. Get that right and convex hulls, polygon area, and intersection tests all fall into place.

The Cross Product = Orientation

   cross(O, A, B) = (A−O) × (B−O)
                  = (Ax−Ox)(By−Oy) − (Ay−Oy)(Bx−Ox)

   > 0  → counter-clockwise (left turn)
   < 0  → clockwise         (right turn)
   = 0  → collinear         (straight)

   This one sign powers almost every 2D geometry algorithm.

Code: Geometry Primitives + Convex Hull

struct P { long long x, y; };

// cross product of OA and OB
long long cross(P O, P A, P B) {
    return (A.x-O.x)*(B.y-O.y) - (A.y-O.y)*(B.x-O.x);
}

// Andrew's monotone chain convex hull
vector<P> convexHull(vector<P> pts) {
    sort(pts.begin(), pts.end(), [](P a, P b){
        return a.x < b.x || (a.x == b.x && a.y < b.y); });
    int n = pts.size(), k = 0;
    vector<P> h(2*n);
    for (int i = 0; i < n; i++) {            // lower hull
        while (k >= 2 && cross(h[k-2], h[k-1], pts[i]) <= 0) k--;
        h[k++] = pts[i];
    }
    for (int i = n-2, t = k+1; i >= 0; i--) { // upper hull
        while (k >= t && cross(h[k-2], h[k-1], pts[i]) <= 0) k--;
        h[k++] = pts[i];
    }
    h.resize(k-1);
    return h;
}

// Polygon area (shoelace) x2
long long area2(vector<P>& p) {
    long long s = 0;
    for (int i = 0, n = p.size(); i < n; i++)
        s += p[i].x*p[(i+1)%n].y - p[(i+1)%n].x*p[i].y;
    return abs(s); // real area = s/2
}
static long cross(long[] O, long[] A, long[] B) {
    return (A[0]-O[0])*(B[1]-O[1]) - (A[1]-O[1])*(B[0]-O[0]);
}

// Shoelace polygon area x2
static long area2(long[][] p) {
    long s = 0; int n = p.length;
    for (int i = 0; i < n; i++)
        s += p[i][0]*p[(i+1)%n][1] - p[(i+1)%n][0]*p[i][1];
    return Math.abs(s);
}

// orientation: +1 ccw, -1 cw, 0 collinear
static int orient(long[] a, long[] b, long[] c) {
    long v = cross(a, b, c);
    return Long.signum(v);
}
def cross(o, a, b):
    return (a[0]-o[0])*(b[1]-o[1]) - (a[1]-o[1])*(b[0]-o[0])

# Andrew's monotone chain convex hull
def convex_hull(pts):
    pts = sorted(set(map(tuple, pts)))
    if len(pts) <= 2: return pts
    lower = []
    for p in pts:
        while len(lower) >= 2 and cross(lower[-2], lower[-1], p) <= 0:
            lower.pop()
        lower.append(p)
    upper = []
    for p in reversed(pts):
        while len(upper) >= 2 and cross(upper[-2], upper[-1], p) <= 0:
            upper.pop()
        upper.append(p)
    return lower[:-1] + upper[:-1]

# Shoelace area
def area(p):
    n = len(p)
    s = sum(p[i][0]*p[(i+1)%n][1] - p[(i+1)%n][0]*p[i][1] for i in range(n))
    return abs(s) / 2

The Geometry Toolkit

  • Orientation test (cross sign): left / right / collinear — the workhorse.
  • Segment intersection: two segments cross if their endpoints straddle each other (opposite orientations).
  • Point in polygon: ray casting (count crossings) or winding number.
  • Shoelace formula: polygon area from vertex coordinates.
  • Convex hull: O(n log n) wrap; basis for diameter, width, rotating calipers.

Why Integers Beat Floats

Keep coordinates as integers whenever possible — the cross product stays exact, no rounding.

Avoid division & sqrt: compare squared distances instead of distances; compare cross products instead of angles.

Watch overflow: cross products multiply coordinates — use 64-bit (or 128-bit) if values are large.

Common Pitfalls

1. Float precision: equality and orientation with doubles need an epsilon; integers avoid the problem entirely.

2. Collinear points on the hull: decide whether to keep them (<= 0 vs < 0 in the cross test).

3. Degenerate cases: duplicate points, all-collinear input, fewer than 3 points.

4. Angle sorting bugs: use atan2 carefully, or better, sort by cross product to dodge precision issues.
