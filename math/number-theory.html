Number Theory

The math of integers: divisibility, primes, and remainders. These tools power cryptography, hashing, and a huge share of competitive problems — usually under a modulus like 10⁹+7.

GCD & LCM

The greatest common divisor is the largest number dividing both. Euclid's trick: gcd(a, b) = gcd(b, a mod b) — keep replacing the bigger by the remainder until one hits 0.

   gcd(48, 18):
     48 = 2·18 + 12   →  gcd(18, 12)
     18 = 1·12 +  6   →  gcd(12,  6)
     12 = 2· 6 +  0   →  gcd( 6,  0) = 6  ✓

   lcm(a, b) = a / gcd(a, b) · b      (divide first to avoid overflow)

Modular Arithmetic ("clock math")

  • Add/multiply, then take remainder: (a + b) % m, (a * b) % m.
  • Keep every intermediate result reduced mod m to avoid overflow.
  • Subtraction: ((a - b) % m + m) % m to stay non-negative.
  • Division is special — you can't just divide. You multiply by the modular inverse.

Fast Exponentiation & Modular Inverse

Compute aⁿ mod m in O(log n) by squaring. When m is prime, Fermat's little theorem gives the inverse for free: a⁻¹ ≡ a^(m-2) (mod m).

   Binary exponentiation: 3^13 mod m,  13 = 1101₂

   result = 1, base = 3
   bit 1 → result *= base;  base = base²   (3)
   bit 0 →                  base = base²   (9 → 81)
   bit 1 → result *= base;  base = base²
   bit 1 → result *= base;
   → 13 multiplications collapse into ~log₂(13) ≈ 4 squarings.

Code: Core Number-Theory Toolkit

long long gcd(long long a, long long b) {
    return b == 0 ? a : gcd(b, a % b);
}
long long lcm(long long a, long long b) {
    return a / gcd(a, b) * b;   // divide first
}

const long long MOD = 1000000007;
long long power(long long a, long long n, long long m) {
    long long r = 1; a %= m;
    while (n > 0) {
        if (n & 1) r = r * a % m;
        a = a * a % m;
        n >>= 1;
    }
    return r;
}
// modular inverse (m prime) via Fermat
long long inv(long long a) { return power(a, MOD - 2, MOD); }

// Euler's totient: count of k in [1,n] with gcd(k,n)=1
long long phi(long long n) {
    long long result = n;
    for (long long p = 2; p * p <= n; p++)
        if (n % p == 0) {
            while (n % p == 0) n /= p;
            result -= result / p;
        }
    if (n > 1) result -= result / n;
    return result;
}

Key Theorems at a Glance

Fermat's Little

  • If p prime, a^(p-1) ≡ 1 (mod p)
  • Gives inverse: a^(p-2)

Euler's Theorem

  • a^φ(n) ≡ 1 (mod n) if gcd(a,n)=1
  • Generalizes Fermat to any modulus
  • CRT (Chinese Remainder Theorem): a system of remainders mod pairwise-coprime moduli has a unique solution mod their product. Lets you split a big modulus into small ones.

Common Pitfalls

1. Overflow: a * b can exceed 64 bits — reduce mod m at every multiply, use 128-bit or __int128 if needed.

2. Negative mod: in C++/Java -3 % 5 == -3; fix with ((x % m) + m) % m.

3. Inverse needs coprime: the modular inverse of a exists only when gcd(a, m) = 1.

4. lcm overflow: always a / gcd * b, never a * b / gcd.
