Primes & Sieve

A prime has exactly two divisors: 1 and itself. The Sieve of Eratosthenes finds every prime up to n in almost-linear time by crossing out multiples — far faster than testing each number alone.

The Sieve Idea

   Write 2, 3, 4, …, n.
   Take the smallest unmarked number — it's PRIME.
   Cross out ALL its multiples (they can't be prime).
   Repeat with the next unmarked number.

   Start crossing at p·p (smaller multiples already crossed).
   Total work: n/2 + n/3 + n/5 + …  ≈  O(n log log n).

Code: Sieve, Factorization & Primality

// Sieve of Eratosthenes: primes up to n
vector<bool> sieve(int n) {
    vector<bool> isPrime(n + 1, true);
    isPrime[0] = isPrime[1] = false;
    for (int p = 2; (long long)p * p <= n; p++)
        if (isPrime[p])
            for (int m = p * p; m <= n; m += p)
                isPrime[m] = false;
    return isPrime;
}

// Trial-division primality: O(sqrt n)
bool isPrime(long long n) {
    if (n < 2) return false;
    for (long long i = 2; i * i <= n; i++)
        if (n % i == 0) return false;
    return true;
}

// Prime factorization: O(sqrt n)
vector<pair<long long,int>> factorize(long long n) {
    vector<pair<long long,int>> f;
    for (long long p = 2; p * p <= n; p++) {
        int c = 0;
        while (n % p == 0) { n /= p; c++; }
        if (c) f.push_back({p, c});
    }
    if (n > 1) f.push_back({n, 1});
    return f;
}

# Sieve of Eratosthenes
def sieve(n):
    is_prime = [True] * (n + 1)
    is_prime[0] = is_prime[1] = False
    for p in range(2, int(n**0.5) + 1):
        if is_prime[p]:
            for m in range(p*p, n + 1, p):
                is_prime[m] = False
    return is_prime

# O(sqrt n) primality
def is_prime(n):
    if n < 2: return False
    i = 2
    while i * i <= n:
        if n % i == 0: return False
        i += 1
    return True

# Prime factorization
def factorize(n):
    f, p = {}, 2
    while p * p <= n:
        while n % p == 0:
            f[p] = f.get(p, 0) + 1; n //= p
        p += 1
    if n > 1: f[n] = f.get(n, 0) + 1
    return f

Beyond the Basic Sieve

  • Linear sieve: O(n) — also gives smallest prime factor of every number, enabling O(log n) factorization.
  • Segmented sieve: find primes in a range [L, R] with R up to 10¹² using O(√R) memory.
  • Miller-Rabin: fast probabilistic primality for huge numbers (deterministic with the right witnesses for 64-bit).

Handy Facts

Divisor count: if n = p₁^a · p₂^b · … then number of divisors = (a+1)(b+1)…

Divisor sum: multiply (p^(a+1) − 1)/(p − 1) over each prime factor.

Primes thin out: roughly n / ln n primes below n (Prime Number Theorem).

Only check up to √n: a composite n always has a factor ≤ √n.

Common Pitfalls

1. p*p overflow: cast to 64-bit in the loop condition for large n.

2. Forgetting 0 and 1: neither is prime — mark them false.

3. Starting multiples too early: begin at p*p, not 2*p, for efficiency.

4. Sieving when n is huge: a boolean array up to 10⁹ won't fit — use a segmented sieve or per-number test.
