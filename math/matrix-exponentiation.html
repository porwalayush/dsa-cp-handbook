Matrix Exponentiation

Any linear recurrence (like Fibonacci) can be written as repeated multiplication by a fixed matrix. Raise that matrix to the n-th power with fast exponentiation, and you compute the n-th term in O(log n) instead of O(n).

Fibonacci as a Matrix

   [ F(n+1) ]   [ 1  1 ] [ F(n)   ]
   [ F(n)   ] = [ 1  0 ] [ F(n-1) ]

   Apply it n times:

   [ F(n+1) ]   [ 1  1 ]^n  [ F(1) ]   [ 1  1 ]^n [ 1 ]
   [ F(n)   ] = [ 1  0 ]    [ F(0) ] = [ 1  0 ]   [ 0 ]

   So F(n) = (M^n)[1][0].  Compute M^n in O(log n) by squaring!

Code: Matrix Power (generic n×n)

const long long MOD = 1e9 + 7;
using Mat = vector<vector<long long>>;

Mat mul(const Mat& A, const Mat& B) {
    int n = A.size();
    Mat C(n, vector<long long>(n, 0));
    for (int i = 0; i < n; i++)
        for (int k = 0; k < n; k++) if (A[i][k])
            for (int j = 0; j < n; j++)
                C[i][j] = (C[i][j] + A[i][k] * B[k][j]) % MOD;
    return C;
}
Mat matpow(Mat M, long long p) {
    int n = M.size();
    Mat R(n, vector<long long>(n, 0));
    for (int i = 0; i < n; i++) R[i][i] = 1; // identity
    while (p) {
        if (p & 1) R = mul(R, M);
        M = mul(M, M);
        p >>= 1;
    }
    return R;
}
long long fib(long long n) {
    if (n == 0) return 0;
    Mat M = {{1,1},{1,0}};
    return matpow(M, n)[0][1];
}

static long[][] mul(long[][] A, long[][] B) {
    int n = A.length;
    long[][] C = new long[n][n];

        for (int k = 0; k < n; k++) if (A[i][k] != 0)
            for (int j = 0; j < n; j++)
                C[i][j] = (C[i][j] + A[i][k] * B[k][j]) % MOD;
    return C;
}
static long[][] matpow(long[][] M, long p) {
    int n = M.length;
    long[][] R = new long[n][n];
    for (int i = 0; i < n; i++) R[i][i] = 1;
    while (p > 0) {
        if ((p & 1) == 1) R = mul(R, M);
        M = mul(M, M);
        p >>= 1;
    }
    return R;
}
MOD = 10**9 + 7

def mul(A, B):
    n = len(A)
    C = [[0]*n for _ in range(n)]
    for i in range(n):
        for k in range(n):
            if A[i][k]:
                for j in range(n):
                    C[i][j] = (C[i][j] + A[i][k]*B[k][j]) % MOD
    return C

def matpow(M, p):
    n = len(M)
    R = [[1 if i==j else 0 for j in range(n)] for i in range(n)]
    while p:
        if p & 1: R = mul(R, M)
        M = mul(M, M)
        p >>= 1
    return R

def fib(n):
    if n == 0: return 0
    return matpow([[1,1],[1,0]], n)[0][1]

How to Build the Matrix

1. Write the recurrence: e.g. f(n) = a·f(n−1) + b·f(n−2) + c·f(n−3).

2. State vector: stack the last k terms: [f(n), f(n−1), f(n−2)]ᵀ.

3. Transition matrix: top row = coefficients [a, b, c]; below it, an identity shift that copies values down.

4. Answer: Mⁿ × initial_vector, read the entry you need.

When to Use It

  • Linear recurrence with huge n (n up to 10¹⁸) — the headline use.
  • Count walks of length k in a graph: the (i,j) entry of (adjacency)ᵏ is the number of i→j walks.
  • DP with a fixed small state that repeats every step — fold the transition into a matrix.
  • Cost: O(k³ log n) where k is the matrix size. Great when k is small (≤ ~100).

Common Pitfalls

1. Identity for the base: the result starts as the identity matrix, not zeros.

2. Multiplication order: matrices don't commute — keep R = R·M consistent.

3. Overflow: reduce mod p inside the multiply, every addition.

4. k too large: O(k³ log n) blows up if the matrix is big — keep the state minimal.
