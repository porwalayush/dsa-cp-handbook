Game Theory

For two-player, perfect-information games (no luck, both play optimally), we can label every position as a win or loss for whoever moves next — and often find a clean formula instead of searching.

Winning & Losing Positions

   A position is a LOSS (for the mover) if EVERY move
   leads to a WIN for the opponent.

   A position is a WIN if AT LEAST ONE move
   leads to a LOSS for the opponent.

   Base case: no moves available = LOSS (you're stuck).
   Work backwards from there.

Interactive: Play Nim vs the Computer

Nim & the XOR Rule

  • Nim: several piles; remove any positive count from one pile; last to move wins.
  • Theorem: the position is losing for the mover ⟺ p₁ XOR p₂ XOR … = 0.
  • To win from a non-zero XOR, move to make the XOR zero — there's always exactly such a move.

Sprague-Grundy: every game is "a Nim pile"

Any impartial game position has a Grundy number (nimber). The Grundy value of a position is the mex (minimum excludant) of the Grundy values of all positions you can move to. A sum of independent games = XOR of their Grundy values.

   mex(S) = smallest non-negative integer NOT in S

   mex({0,1,2}) = 3      mex({1,2}) = 0      mex({0,2,3}) = 1

   grundy(pos) = mex{ grundy(next) for every legal move }
   grundy == 0  ⟺  losing position
   Combined game value = XOR of each subgame's grundy.

Code: Grundy Numbers & Nim

// Nim: first player wins iff XOR of piles != 0
bool nimFirstWins(vector<int>& piles) {
    int x = 0;
    for (int p : piles) x ^= p;
    return x != 0;
}

// Grundy numbers for a game where from value i you
// may move to i-1, i-2, or i-3 (example).
vector<int> grundy(int n) {
    vector<int> g(n + 1, 0);
    for (int i = 1; i <= n; i++) {
        set<int> reachable;
        for (int take = 1; take <= 3 && take <= i; take++)
            reachable.insert(g[i - take]);
        int mex = 0;
        while (reachable.count(mex)) mex++;  // minimum excludant
        g[i] = mex;
    }
    return g;
}

        Set<Integer> reach = new HashSet<>();
        for (int t = 1; t <= 3 && t <= i; t++) reach.add(g[i - t]);
        int mex = 0;

        g[i] = mex;
    }
    return g;
}

def grundy(n):

        mex = 0
        while mex in reach:
            mex += 1
        g[i] = mex
    return g

Recognizing Game Problems

Signals: "two players", "optimal play", "who wins", "last move wins/loses".

Impartial game (same moves for both) → Sprague-Grundy / Nim.

Scoring game (maximize your score) → minimax / DP, not Grundy.

Independent subgames → compute each Grundy, XOR them together.

Common Pitfalls

1. Misère vs normal play: "last move loses" (misère) flips the analysis — Nim's misère rule differs.

2. Partisan games: if players have different move sets, Sprague-Grundy doesn't apply.

3. Forgetting to XOR subgames: a combined game's value is the XOR of Grundy numbers, not their sum.

4. mex bugs: mex is the smallest missing non-negative integer — start counting at 0.
