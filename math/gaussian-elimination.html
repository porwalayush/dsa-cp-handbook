Gaussian Elimination

A systematic way to solve a system of linear equations: use row operations to carve the matrix into a staircase (row echelon form), then back-substitute to read off the answers.

The Three Legal Row Operations

   1. SWAP two rows
   2. SCALE a row by a non-zero constant
   3. ADD a multiple of one row to another

   Goal: make zeros below each pivot, forming a triangle:

     [ 2  1 -1 |  8 ]        [ 2  1 -1 |  8 ]
     [-3 -1  2 | -11] ──►    [ 0 ½  ½ |  1 ]
     [-2  1  2 | -3 ]        [ 0  0 -1 |  1 ]

Code: Gaussian Elimination (with partial pivoting)

// Solve A x = b. aug = [A | b], size n x (n+1)
vector<double> solve(vector<vector<double>> a) {
    int n = a.size();
    for (int col = 0, row = 0; col < n && row < n; col++, row++) {
        // partial pivot: pick largest |value| for stability
        int sel = row;
        for (int i = row + 1; i < n; i++)
            if (abs(a[i][col]) > abs(a[sel][col])) sel = i;
        swap(a[sel], a[row]);
        // eliminate below
        for (int i = row + 1; i < n; i++) {
            double f = a[i][col] / a[row][col];
            for (int j = col; j <= n; j++)
                a[i][j] -= f * a[row][j];
        }
    }
    // back-substitution
    vector<double> x(n);
    for (int i = n - 1; i >= 0; i--) {
        x[i] = a[i][n];
        for (int j = i + 1; j < n; j++) x[i] -= a[i][j] * x[j];
        x[i] /= a[i][i];
    }
    return x;
}
// Solve A x = b. a = [A | b], size n x (n+1)
static double[] solve(double[][] a) {
    int n = a.length;
    for (int col = 0, row = 0; col < n && row < n; col++, row++) {
        int sel = row;
        for (int i = row + 1; i < n; i++)
            if (Math.abs(a[i][col]) > Math.abs(a[sel][col])) sel = i;
        double[] tmp = a[sel]; a[sel] = a[row]; a[row] = tmp;
        for (int i = row + 1; i < n; i++) {
            double f = a[i][col] / a[row][col];
            for (int j = col; j <= n; j++) a[i][j] -= f * a[row][j];
        }
    }
    double[] x = new double[n];
    for (int i = n - 1; i >= 0; i--) {
        x[i] = a[i][n];
        for (int j = i + 1; j < n; j++) x[i] -= a[i][j] * x[j];
        x[i] /= a[i][i];
    }
    return x;
}
# Solve A x = b.  a = [A | b], size n x (n+1)
def solve(a):
    n = len(a)
    for col in range(n):
        # partial pivot
        sel = max(range(col, n), key=lambda i: abs(a[i][col]))
        a[col], a[sel] = a[sel], a[col]
        for i in range(col + 1, n):
            f = a[i][col] / a[col][col]
            for j in range(col, n + 1):
                a[i][j] -= f * a[col][j]
    # back-substitution
    x = [0.0] * n
    for i in range(n - 1, -1, -1):
        x[i] = a[i][n] - sum(a[i][j]*x[j] for j in range(i+1, n))
        x[i] /= a[i][i]
    return x

Beyond Solving Systems

  • Matrix rank: count non-zero pivot rows after elimination.
  • Determinant: product of pivots (track sign flips from row swaps).
  • XOR basis / linear basis over GF(2): Gaussian elimination on bits — max XOR subset, count distinct XORs.
  • Inverse matrix: reduce [A | I] to [I | A⁻¹].

XOR Basis (the CP favorite)

   Insert numbers; keep a basis of independent bit-vectors.
   Then you can: find the MAX XOR subset, test if a value is
   reachable, or count how many distinct XORs exist (2^rank).

   for each number x:
       for b from high bit to low:
           if x has bit b:
               if basis[b]==0: basis[b]=x; break
               else x ^= basis[b]

Common Pitfalls

1. Dividing by a tiny pivot: floating-point blows up — always partial-pivot (pick the largest magnitude).

2. No unique solution: a zero pivot row means infinitely many or no solutions — detect it.

3. Float precision: compare to an epsilon, not == 0. Use rationals/modular arithmetic when exactness matters.

4. Over GF(2) use XOR: don't run float elimination on binary systems — use the XOR basis.
